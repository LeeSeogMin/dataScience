### 9.1 시계열 데이터란 무엇인가

시계열 데이터는 **시간 순서로 정렬된 관측값**이다. 횡단면 데이터가 "한 시점에 여러 대상을 촬영한 사진"이라면, 시계열 데이터는 "한 대상을 여러 시점에 걸쳐 촬영한 동영상"이다.

| 구분 | 횡단면 데이터 | 시계열 데이터 |
| ---- | ------------- | ------------- |
| 비유 | 단체 사진 | 타임랩스 영상 |
| 예시 | 오늘 100개 매장의 매출 | 한 매장의 3년간 월별 매출 |
| 핵심 가정 | 개체 간 독립 | 시간 순서에 정보가 있다 |

시계열 분석의 핵심 가정은 **과거 패턴이 미래에도 지속된다**는 것이다. 이 가정이 성립하면 과거에서 추세, 계절성, 자기상관 구조를 학습하여 미래를 예측할 수 있다. 다만 코로나19 같은 구조적 변화(structural break)가 발생하면 과거 패턴이 무효화될 수 있으므로, 시계열 예측은 항상 "과거 패턴이 지속된다면"이라는 전제 하의 예측임을 인식해야 한다.

**비즈니스 활용 예시**:

| 분야 | 활용 | 핵심 |
| ---- | ---- | ---- |
| 수요 예측 | 다음 달 제품별 판매량 예측 | 재고 최적화 |
| 이상 탐지 | 설비 센서 데이터 모니터링 | 고장 조기 발견 |
| 금융 예측 | 주가, 환율, 금리 예측 | 투자 의사결정 |
| 자원 계획 | 전력 수요, 콜센터 통화량 예측 | 운영 효율화 |

### 9.2 시계열의 구성 요소: 추세, 계절성, 잔차

시계열을 분석하는 첫 단계는 구조를 분해하는 것이다. 대부분의 시계열은 세 가지 기본 요소의 조합으로 이해할 수 있다.

**추세(Trend)**: 장기적인 방향성. GDP의 꾸준한 상승, 출산율의 장기 하락 등. 비유하면 엘리베이터가 올라가는 방향이다.

**계절성(Seasonality)**: 일정 주기로 반복되는 패턴. 아이스크림 판매의 여름 증가, 난방비의 겨울 급증 등. 비유하면 엘리베이터 안에서 사람이 반복적으로 점프하는 것이다.

**잔차(Residual)**: 추세와 계절성으로 설명되지 않는 불규칙 변동. 잘 만든 모형의 잔차는 예측 불가능한 "백색잡음"에 가까워야 한다.

#### 가법 분해와 승법 분해

| 분해 방식 | 수식 | 적합한 경우 | 비유 |
| --------- | ---- | ----------- | ---- |
| 가법 | Y = T + S + R | 계절 변동 크기가 일정 | 월급에 매달 고정 보너스 ±10만원 |
| 승법 | Y = T × S × R | 계절 변동이 수준에 비례 | 월급의 매달 ±10% 보너스 |

아이스크림 매출이 100만원일 때 여름 효과가 +20만원이고, 매출이 200만원으로 성장해도 여름 효과가 여전히 +20만원이면 **가법**. 매출에 비례하여 +40만원으로 커지면 **승법**이다.

### 9.3 정상성과 차분: 예측을 위한 전처리

**정상성(Stationarity)**이란 시계열의 평균과 분산이 시간에 따라 일정한 상태를 말한다. 대부분의 시계열 모형은 정상성을 가정하므로, 비정상 시계열은 **차분(Differencing)**으로 정상화해야 한다.

| 차분 종류 | 계산 | 제거하는 것 | 예시 |
| --------- | ---- | ----------- | ---- |
| 1차 차분 | Yt - Yt-1 | 추세 | 매달 변화량 계산 |
| 계절 차분 | Yt - Yt-12 | 계절성 | 전년 동월 대비 변화 |
| 1차 + 계절 차분 | 둘 다 적용 | 추세 + 계절성 | 완전한 정상화 |

**ADF(Augmented Dickey-Fuller) 검정**으로 정상성을 테스트한다:
- 귀무가설: "시계열이 비정상이다"
- p-value < 0.05이면 귀무가설 기각 → 정상 시계열
- p-value ≥ 0.05이면 비정상 → 차분 필요

### 9.4 ARIMA: 과거 값과 오차로 미래를 예측

ARIMA(p,d,q)는 세 가지 구성 요소의 조합이다:

| 구성 요소 | 의미 | 비유 |
| --------- | ---- | ---- |
| AR(p) — 자기회귀 | 과거 p개 시점의 값에 의존 | "어제 기온이 높았으니 오늘도 높겠지" |
| I(d) — 차분 | d번 차분하여 정상화 | 추세 제거 |
| MA(q) — 이동평균 | 과거 q개 시점의 오차에 의존 | "어제 예측이 빗나갔으니 오늘 보정하자" |

**SARIMA(p,d,q)(P,D,Q)m**: 계절성이 있는 시계열에는 계절 항을 추가한다. 예를 들어 SARIMA(0,1,1)(0,1,1)12는 월별 데이터에 1차 차분 + 계절 차분을 적용하고, MA(1)과 계절 MA(1)로 패턴을 포착한다.

**모형 선택 — ACF/PACF 패턴**:

| ACF 패턴 | PACF 패턴 | 권장 모형 |
| -------- | --------- | --------- |
| 지수적 감소 | lag p 이후 절단 | AR(p) |
| lag q 이후 절단 | 지수적 감소 | MA(q) |
| 지수적 감소 | 지수적 감소 | ARMA(p,q) |

**정보 기준으로 최종 선택**: AIC와 BIC 값이 낮을수록 좋은 모형. 적합도(우도)와 복잡도(파라미터 수) 사이의 균형을 추구한다.

### 9.5 Prophet: 추세와 계절성의 가법 모형

Prophet은 Meta(Facebook)에서 개발한 비즈니스 시계열 예측 도구다. 시계열을 세 가지 구성 요소로 분해한다:

**y(t) = g(t) + s(t) + h(t) + e(t)**

| 구성 요소 | 의미 | 특징 |
| --------- | ---- | ---- |
| g(t) — 추세 | 장기적 성장/감소 | 변화점(changepoint) 자동 탐지 |
| s(t) — 계절성 | 주기적 패턴 | 푸리에 급수로 표현, 연간/주간 등 |
| h(t) — 휴일 효과 | 특별 이벤트 | 추석, 설날, 블랙프라이데이 등 |
| e(t) — 잔차 | 불규칙 변동 | 모형이 포착하지 못한 나머지 |

**ARIMA vs Prophet 핵심 차이**:

| 항목 | ARIMA | Prophet |
| ---- | ----- | ------- |
| 접근 방식 | 자기상관 구조 모형화 | 구성 요소 분리 후 합산 |
| 계절성 처리 | 계절 차분 + 계절 AR/MA | 푸리에 급수 |
| 추세 변화 | 차분으로 암묵적 처리 | 변화점 명시적 탐지 |
| 휴일 효과 | 외생 변수로 수동 추가 | 내장 기능 |
| 해석 가능성 | 계수 해석 | 구성 요소 시각화 |

**가법 vs 승법 계절성**: Prophet은 `seasonality_mode` 파라미터로 선택한다.
- `'additive'`: 계절 변동 크기가 일정할 때
- `'multiplicative'`: 계절 변동이 추세에 비례할 때

### 9.6 LSTM: 순서를 기억하는 신경망

**RNN(순환 신경망)**은 시퀀스 데이터를 처리하기 위한 신경망이다. 이전 시점의 은닉 상태를 다음 시점으로 전달하여 순서 정보를 활용한다. 그러나 기본 RNN은 **장기 의존성 문제**가 있다 — 시퀀스가 길어지면 초기 정보가 점차 희석된다.

**LSTM(Long Short-Term Memory)**은 **게이트 메커니즘**으로 이 문제를 해결한다:

| 게이트 | 역할 | 비유 |
| ------ | ---- | ---- |
| 망각 게이트 (Forget) | 불필요한 정보 삭제 | 노트에서 필요 없는 내용 지우기 |
| 입력 게이트 (Input) | 새 정보 중 저장할 것 선택 | 새로 배운 내용 중 중요한 것만 기록 |
| 출력 게이트 (Output) | 현재 출력에 필요한 정보 선택 | 시험에 필요한 내용만 꺼내기 |

핵심은 **셀 상태(Cell State)** — 컨베이어 벨트처럼 시퀀스 전체를 관통하며 정보를 거의 손실 없이 전달한다. 3개월 전의 계절 패턴이 현재 예측에 중요하면, 게이트가 그 정보를 유지하도록 학습한다.

**시계열 예측에서 LSTM 적용**: 과거 T개 시점(lookback 윈도우)을 입력으로 받아 다음 시점을 예측한다. 예를 들어 과거 24개월의 CO2 농도를 입력으로 받아 25번째 달의 농도를 예측하는 구조다.

**LSTM이 유리한 조건**:

| 조건 | 이유 |
| ---- | ---- |
| 데이터가 충분히 많음 (수천 개 이상) | 수만 개의 파라미터를 학습하려면 데이터가 필요 |
| 비선형 패턴 존재 | LSTM의 복잡한 표현력이 빛을 발함 |
| 다변량 입력이 중요 | 여러 외생 변수를 동시에 처리 가능 |

### 9.7 모형 비교: 어떤 모형을 선택할 것인가

시계열 예측에서 "최고의 모형"은 없다. 데이터 특성과 문제 맥락에 따라 적합한 모형이 달라진다.

| 모형 | 데이터 요구량 | 장기 의존성 | 다변량 지원 | 해석 가능성 | 계산 비용 |
| ---- | ------------- | ----------- | ----------- | ----------- | --------- |
| ARIMA/SARIMA | 낮음 | 제한적 | 제한적 | 높음 | 낮음 |
| Prophet | 중간 (1-2년) | 중간 | 제한적 | 높음 | 낮음 |
| LSTM | 높음 (수천+) | 중간 | 우수 | 낮음 | 중간 |
| Transformer | 매우 높음 | 우수 | 우수 | 낮음 | 높음 |

**모형 선택 가이드**:
- **데이터가 적고 해석이 중요**: ARIMA 또는 Prophet
- **계절성 + 휴일 효과가 핵심**: Prophet
- **데이터가 많고 비선형/다변량**: LSTM/딥러닝
- **빠른 베이스라인 구축**: Prophet 또는 TimeGPT(Foundation Model)

### 핵심 정리

```text
1. 시계열은 추세, 계절성, 잔차로 분해된다. 분해 방식(가법/승법)은 데이터 특성에 따라 선택한다.
2. 정상성은 대부분의 시계열 모형의 기본 가정이다. ADF 검정으로 확인하고, 차분으로 확보한다.
3. ARIMA는 자기상관 구조를 모형화한다. ACF/PACF로 차수를 결정하고, AIC/BIC로 최적 모형을 선택한다.
4. Prophet은 추세+계절성+휴일을 분리하여 모형화한다. 해석이 쉽고 실무 친화적이다.
5. LSTM은 게이트 메커니즘으로 장기 의존성을 학습한다. 데이터가 충분하고 비선형/다변량일 때 강하다.
6. "복잡한 모형이 항상 좋은 것은 아니다." 데이터가 단순하면 단순한 모형이, 복잡하면 복잡한 모형이 낫다.
```
