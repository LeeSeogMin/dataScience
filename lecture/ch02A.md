### 2.1 전처리는 모델의 시야를 만드는 과정이다

머신러닝에서 가장 흔한 오해는 "좋은 모델만 고르면 성능은 알아서 나온다"는 생각이다.

쉽게 비유하면 전처리는 안경을 닦는 일이다.

- 안경이 더러우면 세상이 흐리게 보인다
- 아무리 똑똑한 사람이 써도 흐린 렌즈로는 정확히 보기 어렵다

데이터도 마찬가지다. 결측치가 섞여 있거나, 단위가 뒤죽박죽이거나, 필요 없는 변수가 너무 많으면 모델은 제대로 학습하기 어렵다.

| 전처리 문제 | 그대로 둘 때 생기는 일 | 전처리의 역할 |
| ----------- | ---------------------- | ------------- |
| 결측치 | 학습 불가 또는 편향 | 정보 손실과 왜곡 최소화 |
| 스케일 차이 | 특정 변수가 거리 계산을 지배 | 변수 간 영향 균형 조정 |
| 고차원 입력 | 계산 증가, 과적합 위험 | 핵심 구조만 남기기 |
| 표현 부족 | 의미를 잘 못 배움 | 모델이 배우기 쉬운 입력 설계 |

전처리는 모델을 "꾸미는 과정"이 아니라, **모델이 무엇을 보고 무엇을 중요하게 여길지 결정하는 과정**이다.

### 2.2 결측치: 비어 있는 칸이 아니라 데이터 생성 과정의 흔적

현실 데이터에는 거의 항상 결측치가 있다. 설문 응답 거부, 센서 오류, 시스템 누락, 데이터 입력 실수 등 원인이 다양하다.

#### 모델은 왜 결측치를 싫어할까

대부분의 머신러닝 알고리즘은 결측치를 직접 처리할 수 없다.

- 선형 회귀는 Xβ를 계산하려면 모든 X 값이 있어야 한다
- KNN은 거리를 계산하려면 모든 좌표가 필요하다
- 신경망은 가중합을 계산하려면 모든 입력이 정의되어야 한다

결측치를 만나면 두 가지 중 하나를 해야 한다.

1. 결측이 있는 행을 제거한다 → 정보 손실
2. 어떤 값으로 대체한다 → 인위적 패턴 도입

둘 다 비용이 있다. 결측치 처리에는 항상 트레이드오프가 존재한다.

참고: XGBoost, LightGBM 같은 그래디언트 부스팅 모델은 결측치를 내재적으로 처리한다. 분할 시 결측을 별도 경로로 보내며, 결측 여부 자체를 분할 기준으로 활용할 수 있다. 그러나 이 경우에도 결측 패턴이 학습/테스트 데이터에서 일관적이어야 한다.

#### 결측의 세 가지 유형

결측은 **왜 비었는가**에 따라 처리 전략이 달라진다. 시험 답안지로 비유하면:

- **MCAR**(Missing Completely At Random): 답안지가 우연히 찢어져 일부가 사라짐
- **MAR**(Missing At Random): 특정 반 학생들만 더 많이 빠뜨림
- **MNAR**(Missing Not At Random): 어려운 문항이라 일부러 비워 둠

| 유형 | 뜻 | 직관적 예시 | 처리 방향 |
| ---- | --- | ------------ | --------- |
| MCAR | 아무 변수와도 무관하게 빠짐 | 파일 손상, 시스템 오류 | 행 삭제 가능 (편향 없음) |
| MAR | 다른 관측 변수와 관련되어 빠짐 | 고소득층이 소득 공개 꺼림 | 다른 변수 활용 대체 (다중 대체법) |
| MNAR | 빠진 값 자체와 관련되어 빠짐 | 소득 낮을수록 소득 숨김 | 결측 메커니즘 모형화 필요, 가장 어려움 |

실무에서는 결측 유형을 정확히 판단하기 어려운 경우가 많다. 최소한 결측 패턴을 탐색하고, 결측이 특정 그룹에 집중되어 있는지 확인해야 한다.

#### 결측치 처리 전략 비교

| 방법 | 장점 | 단점 | 적합한 상황 |
| ---- | ---- | ---- | ----------- |
| 행 삭제 | 단순, 편향 없음(MCAR 시) | 정보 손실, 표본 감소 | 결측 5% 이하, MCAR |
| 평균/중앙값 대체 | 빠르고 쉬움 | 분산 과소추정, 상관관계 약화 | 결측 10% 이하, 빠른 프로토타입 |
| 그룹별 대체 | MAR에서 편향 감소 | 그룹 정의 필요 | 성별·지역 등 그룹별 차이가 클 때 |
| KNN 대체 | 변수 간 관계 활용 | 계산 비용, K 선택 필요 | 변수 간 상관 있을 때 |
| 반복 대체(MICE) | 불확실성 반영, 상관 보존 | 계산 복잡, 수렴 불안정 | 결측 많을 때, 정교한 분석 |
| 결측 지시 변수 | 결측 정보 활용 | 차원 증가 | 결측 자체가 의미 있을 때 |

핵심: "결측치는 무조건 채우면 된다"가 아니다. 결측 자체가 중요한 신호라면, 결측 여부를 따로 변수로 남기는 것이 더 나을 수 있다. 예를 들어 신용평가에서 특정 정보의 미제공이 위험 신호일 수 있다.

### 2.3 스케일링: 숫자의 크기를 맞춰 공정하게 비교하기

변수의 측정 단위와 범위가 다르면 일부 알고리즘의 성능이 저하된다.

예를 들어 집값 예측에서 방 개수(0~10)와 면적(0~10,000)을 함께 쓴다고 하자. 두 집의 방 개수가 1 차이 나고 면적이 100 차이 나면, 유클리드 거리는 √(1² + 100²) ≈ 100이 되어 방 개수의 영향은 거의 없다. 스케일링 없이는 KNN이 거의 면적만으로 이웃을 결정한다.

#### 어떤 알고리즘이 스케일링에 민감한가

| 모델 계열 | 스케일링 필요성 | 이유 |
| --------- | --------------- | ---- |
| KNN, SVM, K-Means, DBSCAN | 높음 | 거리 계산에 직접 영향 |
| 신경망 | 높음 | 학습 안정성과 수렴 속도에 영향. 활성화 함수(sigmoid, tanh)의 포화 구간 문제 |
| 선형/로지스틱 회귀 | 보통~높음 | 계수 크기와 최적화 안정성에 영향 |
| 트리 기반 모델 (DT, RF, XGBoost) | 낮음 | 분할점 기준 분류라 값의 순서만 중요, 스케일 불변 |
| 나이브 베이즈 | 낮음 | 각 변수의 조건부 확률을 독립 추정 |

#### 대표적인 스케일링 방법

| 방법 | 수식 | 결과 범위 | 장점 | 주의점 |
| ---- | ---- | --------- | ---- | ------ |
| StandardScaler | z = (x - μ) / σ | 제한 없음 | 가장 범용적, 정규분포 가정에 적합 | 이상치에 민감 |
| MinMaxScaler | (x - x_min) / (x_max - x_min) | [0, 1] | 신경망 입력에 유용 | 이상치에 매우 민감 (대부분 값이 좁은 범위로 압축) |
| RobustScaler | (x - median) / IQR | 제한 없음 | 이상치에 강건 | 해석이 덜 직관적 |
| MaxAbsScaler | x / \|x_max\| | [-1, 1] | 희소 행렬에서 0 보존 | 이상치 영향 가능 |

#### 데이터 누수 방지

스케일링에서 가장 중요한 규칙: 파라미터(평균, 표준편차, 최소값 등)는 **반드시 훈련 데이터에서만** 계산해야 한다. 테스트 데이터의 통계량을 사용하면 미래 정보가 누수된다.

```python
# 올바른 방법
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # 훈련 데이터로 fit + transform
X_test_scaled = scaler.transform(X_test)          # 훈련 기준으로 transform만

# 파이프라인으로 누수 자동 방지
pipeline = Pipeline([
    ('scaler', StandardScaler()),
    ('model', KNeighborsRegressor(n_neighbors=5))
])
```

파이프라인은 교차검증 시 매 폴드마다 훈련 데이터로만 fit → transform을 수행하므로 누수를 자동으로 방지한다.

### 2.4 차원 축소: 너무 많은 축에서 핵심만 남기기

변수가 많아지면 계산량 증가, 과적합 위험, 시각화 어려움, 거리 개념 왜곡이 생긴다.

#### 차원의 저주

고차원 공간에서 데이터 포인트들은 점점 멀어진다. 차원이 높아질수록 모든 점이 서로 비슷하게 멀어지므로, "가까운 이웃"의 개념이 무의미해진다.

직관적으로 보면:

- 2차원 지도에서는 동네 구조가 잘 보인다
- 100차원 공간에서는 동시에 너무 많은 방향을 봐야 해서 감각이 무너진다

또 다른 예: 2차원에서 단위 원은 단위 정사각형 면적의 약 78.5%를 차지하지만, 10차원에서 단위 초구는 단위 초입방체의 약 0.25%만 차지한다. 대부분의 데이터가 "가장자리"에 몰린다.

차원 축소는 "정보를 버리는 작업"이 아니라, **핵심 구조를 남기는 압축 작업**이다.

#### PCA: 분산이 큰 방향을 찾는 방법

PCA는 가장 널리 쓰이는 선형 차원 축소 방법이다. 핵심 생각은 단순하다.

> "데이터가 가장 많이 퍼져 있는 방향을 먼저 잡자."

쉽게 말하면, PCA는 **데이터 구름을 가장 잘 펼쳐 보이는 방향으로 좌표축을 새로 돌려 잡는 방법**이다.

- PC1: 데이터를 한 줄로 요약해도 가장 많은 차이를 담는 방향
- PC2: PC1과 직교(90도)하면서 남은 퍼짐을 가장 많이 담는 방향
- 이런 식으로 서로 직각인 새 축들을 순서대로 찾는다

여기서 "직교"란 "두 방향이 서로 겹치지 않는다"는 뜻이다. 종이 위의 가로축(x)과 세로축(y)처럼, 한쪽 방향의 변화가 다른 쪽에 영향을 주지 않는다.

수학적으로는 공분산 행렬의 고유값 분해다. 고유벡터가 주성분 방향이고, 고유값이 해당 주성분이 설명하는 분산의 크기다.

PCA의 장점:
- 계산이 빠르고 결과가 일관적 (재현성 보장)
- 다중공선성 문제 자동 해결
- 설명 분산 비율로 적절한 차원 수 선택 가능
- 새로운 데이터에 `transform` 적용 가능

PCA의 한계:
- 선형 변환이므로 비선형 관계를 포착하지 못함
- 분산이 크다고 예측에 반드시 중요한 것은 아님 (비지도 학습)
- 주성분의 의미 해석이 어려움

#### t-SNE: 가까운 이웃 관계를 보존하는 비선형 축소

t-SNE는 고차원 데이터를 2~3차원으로 시각화할 때, "가까운 이웃 관계"가 최대한 유지되도록 점들을 배치한다.

작동 원리:
1. 고차원에서 각 점 쌍의 유사도를 가우시안 분포로 확률 계산
2. 저차원에서도 유사한 확률을 t-분포(꼬리가 두꺼운 분포)로 정의
3. 두 확률 분포의 차이(KL-발산)를 최소화하도록 저차원 좌표를 최적화

t-분포를 쓰는 이유: 고차원의 중간 거리들이 2차원으로 압축되면 서로 뭉치기 쉬운데(밀림 현상), t-분포는 멀리 떨어진 점에 상대적으로 더 큰 확률을 부여해 군집 사이를 벌릴 여유를 제공한다.

주의점:
- 결과가 확률적이어서 매번 다른 결과 (random_state 고정 필요)
- 전역 구조보다 지역 구조 보존에 최적화 → 군집 간 거리는 신뢰하기 어려움
- perplexity 파라미터가 결과에 큰 영향 (보통 5~50 범위)
- 새로운 데이터에 대한 변환(out-of-sample) 제한적
- 축 간 거리나 군집 크기는 절대적 해석 불가

#### UMAP: t-SNE의 대안

UMAP은 t-SNE와 유사한 목적이지만 더 빠르고 전역 구조를 더 잘 보존한다.

t-SNE와의 차이:
- t-SNE는 모든 점 쌍을 계산 → O(n²). UMAP은 근사 최근접이웃 탐색 → O(n) 수준
- UMAP은 "연결되어야 할 점이 떨어진 경우"와 "연결되지 않아야 할 점이 붙어 있는 경우" 모두에 패널티 → 전역 구조 보존이 더 좋음
- 새로운 데이터에 `transform` 적용 가능 (out-of-sample 확장)

주요 파라미터:
- `n_neighbors`: 지역 이웃 수 (보통 5~50, 기본값 15)
- `min_dist`: 점들이 얼마나 가깝게 뭉칠 수 있는지 (시각화용 0.1 정도)

#### 차원 축소 방법 선택 기준

| 목적 | 적합한 방법 | 이유 |
| ---- | ----------- | ---- |
| 빠른 전처리, 입력 압축 | PCA | 속도, 안정성, out-of-sample 적용 |
| 소규모 데이터 시각화 | t-SNE | 지역 구조 표현에 강함 |
| 대규모 시각화 | UMAP | 속도와 품질 균형 |
| 후속 모델 입력용 | PCA 우선 | 새로운 데이터에 적용 쉬움, 다중공선성 해소 |
| 비선형 구조 시각화 | UMAP | 전역+지역 구조 균형 |

직관적으로 보면:
- PCA는 전체 지형을 크게 보는 지도
- t-SNE는 가까운 골목 구조를 자세히 보는 지도
- UMAP은 둘의 균형을 잡은 지도

### 2.5 특성 공학: 모델이 배우기 쉬운 표현 만들기

특성 공학은 원시 데이터를 모델이 더 잘 배울 수 있는 형태로 바꾸는 작업이다. 단순히 열을 더 만드는 것이 아니라, **데이터를 표현하는 방식을 설계하는 작업**이다.

왜 필요한가:
- 선형 모델은 y = βx 형태만 학습 가능. 타깃이 x²에 비례하면 z = x²를 만들어야 포착 가능
- 트리 모델은 축에 평행한 분할만 수행. x₁ + x₂ > 10이라는 경계는 여러 분할로 근사해야 하지만, x₃ = x₁ + x₂를 만들면 한 번에 포착
- 신경망도 적절한 특성이 주어지면 더 적은 층과 더 적은 데이터로 학습 가능

#### 수치형 변수 변환

| 방법 | 적용 상황 | 효과 |
| ---- | --------- | ---- |
| 로그 변환 | 소득, 가격 등 오른쪽 치우친 분포 | 곱셈 관계를 덧셈으로 변환, 선형 모델에 유리 |
| 제곱근, 박스-콕스 | 분포 정규화 필요 | 최적 변환 파라미터를 데이터에서 추정 |
| 비닝(Binning) | 나이를 10대/20대/30대로 구분 | 비선형 관계 포착, 단 정보 손실 주의 |

예시: `log(income) = β₀ + β₁·education + β₂·experience` — education이 1 증가하면 income이 exp(β₁)배 증가한다는 해석이 가능하다.

#### 범주형 변수 인코딩

| 방법 | 원리 | 장점 | 주의점 |
| ---- | ---- | ---- | ------ |
| 원-핫 인코딩 | 각 범주를 별도 이진 변수로 변환 | 범주 간 순서 가정 없음, 명목형에 적합 | 범주 많으면 고차원, 관계 정보 손실 |
| 레이블 인코딩 | 범주에 정수 할당 (A=0, B=1, C=2) | 트리 모델에 사용 가능 | 순서형 아닌 범주에 거짓 순서 도입 |
| 타겟 인코딩 | 범주의 타깃 평균으로 대체 | 차원 안 늘림, 타깃 관계 직접 반영 | 데이터 누수 위험 → 교차검증 내 폴드별 계산 필수 |

#### 날짜/시간 변수 처리

날짜/시간 데이터는 다양한 특성으로 분해할 수 있다.

- **순환적 특성**: 23시와 0시가 가깝다는 것을 반영하려면 사인/코사인 변환
  - `hour_sin = sin(2π × hour / 24)`, `hour_cos = cos(2π × hour / 24)`
- **이진 특성**: 주말 여부, 공휴일 여부, 업무 시간 여부
- **상대적 특성**: 월초까지 남은 일수, 급여일까지 남은 일수
- **누적 특성**: 서비스 가입 후 경과일, 마지막 구매 후 경과일

#### 상호작용 특성

두 변수의 조합이 중요한 경우 상호작용 특성을 생성한다.

| 원시 데이터 | 파생 특성 예시 | 이유 |
| ----------- | -------------- | ---- |
| 방 개수 + 면적 | 방당 면적 | 주거 밀도 표현 |
| 클릭 수 + 노출 수 | 클릭률 | 광고 효과 직접 표현 |
| 채널 + 시간대 | 채널×시간대 조합 | 광고 효과 분석 |
| 위치 + 이전 위치 | 이동 거리 | 이상 탐지 |

주의: 모든 변수 쌍의 상호작용을 만들면 차원이 p²로 폭발한다. 도메인 지식에 기반해 선택적으로 생성해야 한다.

#### LLM을 활용한 특성 아이디어 생성

LLM에게 "신용카드 사기 탐지 모델에 유용한 파생 특성을 제안해달라"고 요청하면, 최근 N시간 거래 횟수, 평소 금액 대비 비율, 야간 거래 여부 같은 아이디어를 빠르게 얻을 수 있다. 하지만 이런 아이디어가 실제로 유효한지는 반드시 검증해야 한다. LLM의 역할은 **대체자**가 아니라 **보조 아이디어 생성기**다.

### 2.6 텍스트 임베딩: 의미를 잃지 않고 숫자로 바꾸기

텍스트는 머신러닝 모델에 바로 들어갈 수 없다. 결국 숫자로 바꿔야 한다.

#### TF-IDF: 전통적인 텍스트 수치화

TF-IDF는 문서에서 단어의 중요도를 측정한다. 문서 내 출현 빈도(TF)가 높고, 전체 문서에서 희귀한(IDF) 단어에 높은 가중치를 부여한다.

```
TF-IDF(t, d, D) = TF(t, d) × log(N / (1 + DF(t)))
```

장점: 빠르고 구현 쉽고 기준선으로 좋다.

한계:
- 의미 유사성 무시: "좋다"와 "훌륭하다"가 완전히 다른 단어로 취급
- 문맥 무시: "은행에 돈을 맡겼다"와 "강 은행에 앉았다"의 '은행'이 동일 처리
- 어순 무시: "개가 사람을 물었다"와 "사람이 개를 물었다"가 같은 벡터
- 고차원 희소 행렬 생성

#### 임베딩: 의미를 보존하는 벡터 표현

임베딩은 문장이나 단어를 밀집 벡터(384~1536차원)로 바꾼다. 의미가 비슷한 텍스트끼리 벡터 공간에서 가깝게 놓이도록 학습된다.

쉽게 말하면, TF-IDF는 **단어 개수표**이고, 임베딩은 **의미 지도**다.

실무에서 많이 쓰는 임베딩 모델:
- **Sentence-BERT (SBERT)**: 문장 수준 임베딩에 최적화. `all-MiniLM-L6-v2`(384차원, 빠름), `all-mpnet-base-v2`(768차원, 성능 좋음)
- **API 임베딩**: OpenAI `text-embedding-3-small`, HyperCLOVA 등. 최신 모델 품질, GPU 불필요. 단, 토큰당 과금, 민감 데이터 외부 전송 주의

| 항목 | TF-IDF | 임베딩 (SBERT 등) |
| ---- | ------ | ----------------- |
| 표현 방식 | 단어 빈도 기반, 희소 벡터 | 의미 기반, 밀집 벡터 |
| 차원 | 수천~수만 | 384~1536 |
| 장점 | 빠르고 구현 쉬움 | 의미 유사성 반영, 낮은 차원 |
| 단점 | 문맥/의미 무시 | 계산 비용 큼 |
| 적합한 상황 | 빠른 기준선 | 유사 문서 검색, 의미 기반 군집화, RAG |

- TF-IDF 차원(5,000) vs SBERT 차원(384): 차원이 약 13배 높아도 TF-IDF가 정확도가 낮다. 왜? → 대부분 0인 희소 벡터라 차원 수 ≠ 정보량
- SBERT는 "야구"와 "홈런"이 가깝다는 것을 표현 가능, TF-IDF는 불가능

### 2.7 범주형 임베딩: 범주 사이의 관계를 학습하기

임베딩 개념은 텍스트에만 쓰이지 않는다. 범주형 변수도 임베딩으로 표현할 수 있다.

원-핫 인코딩에서는 서울, 부산, 인천이 완전히 별개다. 서울과 인천의 거리가 서울과 뉴욕의 거리와 같다. Entity Embedding은 각 범주를 학습 가능한 저차원 벡터로 표현하여, 신경망 학습 과정에서 타깃 예측에 유용하도록 벡터를 최적화한다.

#### Rossmann Store Sales 사례

2015년 Kaggle 경진대회에서 3위 팀이 Entity Embedding을 도입해 주목받았다. 1,115개 가게 ID, 요일, 주(state) 등을 임베딩으로 변환해 신경망에 입력한 결과:

- 지리적으로 인접한 주들이 벡터 공간에서 가깝게 배치됨
- 매출 패턴이 유사한 가게들이 자연스럽게 군집 형성
- 요일 임베딩에서 주중/주말이 명확히 구분됨

#### Entity Embedding vs 원-핫 인코딩 선택 기준

| 상황 | 권장 방법 |
| ---- | --------- |
| 범주 < 10개 | 원-핫 (또는 레이블) |
| 범주 10~100개 | 원-핫 또는 Entity Embedding |
| 범주 > 100개 | Entity Embedding |
| 트리 기반 모델 | 레이블 인코딩 |
| 신경망 모델 | Entity Embedding |
| 범주 간 관계 활용 필요 | Entity Embedding |

임베딩 차원은 경험적 규칙으로 설정: `emb_dim = min(50, (cat_dim + 1) // 2)`

이 관점은 이후 딥러닝, 추천 시스템, 그래프 학습으로 자연스럽게 이어진다.

### 핵심 정리

```text
1. 전처리는 모델이 세상을 보는 입력을 설계하는 과정이다.
2. 결측치는 유형(MCAR/MAR/MNAR)에 따라 처리 전략이 달라진다.
3. 스케일링은 거리 기반 모델과 신경망에서 특히 중요하며, 파이프라인으로 누수를 방지한다.
4. 차원 축소는 보는 목적(시각화)과 쓰는 목적(전처리)을 구분해 PCA/t-SNE/UMAP을 선택한다.
5. 특성 공학은 수치 변환, 범주 인코딩, 날짜 처리, 상호작용 특성 등으로 모델 입력을 설계한다.
6. TF-IDF는 빈도 기반 표현, 임베딩은 의미 기반 표현이다.
7. Entity Embedding으로 범주형 변수의 관계를 학습할 수 있다.
```
