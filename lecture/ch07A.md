### 7.1 신경망의 학습 원리

#### 퍼셉트론에서 다층 신경망으로

퍼셉트론은 가장 단순한 신경망이다. 입력에 가중치를 곱해 더한 뒤 활성화 함수를 적용한다: y = σ(Σwᵢxᵢ + b). 하지만 단층 퍼셉트론은 직선 하나로 분리 가능한 문제만 풀 수 있다. XOR 문제(두 입력이 다를 때만 1)처럼 직선으로 나눌 수 없는 문제는 풀지 못한다.

해결책은 **은닉층(hidden layer)**을 추가하는 것이다. 은닉층의 뉴런들이 입력 공간을 비선형으로 변환하면, 출력층은 변환된 공간에서 선형 분류를 수행한다. 비유하자면, 구겨진 종이를 펴서 직선을 긋는 것과 같다. **보편 근사 정리**에 따르면 뉴런을 충분히 많이 쓰면 어떤 복잡한 패턴이든 학습할 수 있다. 레고 블록을 충분히 모으면 어떤 형태든 만들 수 있는 것과 같다.

![신경망과 은닉층](../diagram/7-2.png)

#### 활성화 함수: 비선형성의 핵심

활성화 함수가 없으면 여러 층을 쌓아도 결국 하나의 선형 변환에 불과하다. 활성화 함수가 비선형성을 도입하여 복잡한 함수를 표현할 수 있게 한다.

| 함수 | 수식 | 장점 | 단점 |
| ---- | ---- | ---- | ---- |
| Sigmoid | 1/(1+e⁻ˣ) | 확률 해석 가능 | 기울기 소실, 중심이 0이 아님 |
| Tanh | (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ) | 출력 중심이 0 | 기울기 소실 |
| ReLU | max(0, x) | 계산 빠름, 기울기 소실 완화 | Dead ReLU (뉴런이 영원히 꺼짐) |
| Leaky ReLU | max(0.01x, x) | Dead ReLU 해결 | 추가 파라미터 |
| GELU | x·Φ(x) | Transformer에서 우수 | 계산 복잡 |

**ReLU**가 현대 딥러닝의 기본인 이유: 양수 영역에서 기울기가 1로 일정하여 깊은 네트워크에서도 기울기가 잘 전파되고, 계산이 단순하다. 단, 음수 영역에서 기울기가 0이 되면 뉴런이 영원히 "죽는" 문제가 있어 Leaky ReLU로 보완한다.

#### 역전파와 경사 하강법

신경망 학습은 4단계로 요약된다:

1. **순전파**: 입력을 넣어 예측값 계산
2. **손실 계산**: 예측값과 정답의 차이(손실) 계산
3. **역전파**: 손실이 각 가중치에 얼마나 민감한지(기울기)를 연쇄법칙으로 계산
4. **가중치 갱신**: 기울기 반대 방향으로 가중치 조정 — w ← w − α · (∂L/∂w)

비유하자면 경사 하강법은 안개 낀 산에서 발밑 경사만 느끼며 가장 낮은 곳으로 내려가는 것이다. **학습률(α)**은 한 걸음의 크기로, 너무 크면 발산하고 너무 작으면 수렴이 느리다.

| 옵티마이저 | 특징 | 적합한 상황 |
| ---------- | ---- | ----------- |
| SGD | 기본적, 안정적 | 대규모 데이터, 일반화 중시 |
| SGD + Momentum | 관성으로 수렴 가속 | 손실 곡면이 복잡할 때 |
| Adam | 적응적 학습률 | 빠른 수렴이 필요할 때 |
| AdamW | Adam + Weight Decay 개선 | Transformer 학습 |

#### 배치 학습과 미니배치

| 방식 | 기울기 계산 대상 | 장점 | 단점 |
| ---- | --------------- | ---- | ---- |
| 배치 GD | 전체 데이터 | 정확한 기울기 | 메모리 부족 |
| 확률적 GD (SGD) | 샘플 1개 | 메모리 효율적 | 노이즈가 큼 |
| 미니배치 GD | 32~512개 | 효율과 안정의 균형 | 배치 크기 선택 필요 |

실무에서는 GPU 메모리가 허용하는 최대 배치 크기에서 시작하여, 학습이 불안정하면 줄이는 방식을 권장한다.

### 7.2 신경망이 데이터 분석에 기여하는 방식

#### 표현 학습: 특성 공학의 자동화

전통 머신러닝에서는 어떤 변수가 예측에 유용한지를 사람이 직접 판단해 특성을 설계했다. 이미지의 SIFT/HOG, 텍스트의 TF-IDF 같은 수작업 특성 공학은 도메인 전문지식에 크게 의존한다.

신경망은 **표현 학습(representation learning)**으로 이 문제를 해결한다. 각 층이 이전 층의 출력을 입력으로 받아 점진적으로 추상화된 표현을 자동으로 구성한다. 좋은 표현이란, 같은 클래스의 샘플이 벡터 공간에서 가깝게 모이고 다른 클래스와는 멀리 떨어지는 것이다.

#### 딥러닝이 필요한 상황 vs 불필요한 상황

| 상황 | 권장 접근법 | 근거 |
| ---- | ----------- | ---- |
| 비정형 데이터 (이미지, 텍스트) | 딥러닝 | 자동 표현 학습 |
| 정형 데이터 + 빠른 베이스라인 | XGBoost/LightGBM | 튜닝 시간 짧음, 안정적 성능 |
| 정형 데이터 + 해석 필요 | XGBoost + SHAP | 특성 중요도 명확 |
| 멀티모달 파이프라인 | 딥러닝 | End-to-end 학습 가능 |
| 사전학습 활용 가능 | 딥러닝 (전이학습) | 레이블 부족 상황 완화 |

핵심 메시지: 정형 데이터에서는 여전히 XGBoost가 강력한 베이스라인이지만, 고차원 데이터에서 복잡한 상호작용이 있을 때는 딥러닝이 우위를 보일 수 있다.

#### 데이터 분석에서 신경망의 4가지 활용

| 전통 방식 | AI 시대 진화 |
| --------- | ------------ |
| 수동 특성 공학 | 사전학습 임베딩 + 표현 학습으로 자동 특성 추출 |
| 선형 차원 축소 (PCA) | 오토인코더/VAE로 비선형 잠재 표현 학습 |
| 통계적 임계치 기반 이상 탐지 | 재구성 오류 기반 이상 탐지 |
| 규칙적 데이터 증강 | GAN/VAE/Diffusion 기반 합성 데이터 생성 |

### 7.3 정형 데이터 딥러닝: 트리 모델과의 비교

정형 데이터는 이미지나 텍스트처럼 공간적/순차적 구조가 없어 딥러닝 적용이 어렵다고 여겨져 왔다. 실제로 Grinsztajn et al.(2022)의 벤치마크에서 XGBoost/LightGBM이 대부분의 정형 데이터셋에서 최고 수준의 성능을 보였다.

#### 정형 데이터에서 딥러닝이 불리한 이유

| 원인 | 설명 |
| ---- | ---- |
| 불규칙한 목표 함수 | 신경망은 매끄러운 함수에 편향 → 불규칙 패턴 학습 어려움 |
| 무관련 특성 민감성 | 노이즈 특성에 불안정 |
| 적은 데이터 | 1만 샘플 미만에서 과적합 위험 증가 |

#### TabNet: Attention 기반 특성 선택

TabNet은 "단계별로 중요한 특성만 골라 쓰는 신경망"이다. 각 의사결정 단계마다 Sparsemax로 중요한 특성만 선택하고, 나머지에는 정확히 0을 부여한다.

- Softmax: "A가 50%, B가 30%, C가 20% 중요"
- Sparsemax: "A가 62.5%, B가 37.5%, C는 무시"

이 명시적 선택 덕분에 어떤 특성이 예측에 사용되었는지 해석이 가능하다.

| 파라미터 | 설명 | 권장값 |
| -------- | ---- | ------ |
| n_d, n_a | 의사결정/어텐션 임베딩 차원 | 8-64 |
| n_steps | 의사결정 단계 수 | 3-10 |
| γ (relaxation) | 특성 재사용 계수 | 1.0-2.0 |

#### 모델 선택 가이드

```
데이터 유형?
    ├─ 비정형 (이미지, 텍스트) ───► 딥러닝
    └─ 정형 (테이블) ─► XGBoost로 베이스라인 구축
                        ├─ 해석 필요? ─► XGBoost + SHAP
                        ├─ 복잡한 상호작용? ─► TabNet 시도
                        └─ 사전학습/멀티모달? ─► 딥러닝 검토
```

### 7.4 오토인코더와 표현 학습

오토인코더는 입력을 저차원 잠재 공간으로 압축(인코더)했다가 다시 복원(디코더)하는 신경망이다. 이 과정에서 잠재 표현은 원본의 핵심 정보를 담게 된다. PCA가 선형 압축만 가능한 반면, 오토인코더는 비선형 활성화 함수를 통해 더 복잡한 관계를 포착한다.

```
입력 (고차원) → [인코더] → 잠재 벡터 (저차원) → [디코더] → 복원 (고차원)
                             ↓
                  핵심 정보만 보존하는 "병목"
```

#### 이상 탐지: 재구성 오류 활용

정상 데이터만으로 학습시키면 오토인코더는 정상 패턴을 "외우게" 된다. 이상 데이터는 제대로 복원하지 못해 **재구성 오류가 크다**. 이 차이를 이용해 이상 여부를 판단한다.

```
[학습 단계] 정상 데이터 → 오토인코더 → 정상 패턴 학습
[추론 단계] 정상 데이터 → 재구성 오류 작음 → 정상 판정
           이상 데이터 → 재구성 오류 큼   → 이상 판정
```

#### 하이브리드 파이프라인: 오토인코더 + XGBoost

오토인코더의 잠재 표현을 XGBoost의 입력으로 활용하면, 원본 특성만 사용할 때보다 성능이 향상될 수 있다. 특히 중복 특성이 많은 고차원 데이터에서 효과적이다.

```
원본 데이터 (1000차원) → 오토인코더 → 잠재 표현 (64차원)
                                        ↓
                         [원본 + 잠재 표현] → XGBoost → 예측
```

#### 정규화 기법: 과적합 방지의 3가지 도구

| 기법 | 원리 | 비유 |
| ---- | ---- | ---- |
| Weight Decay (L2) | 가중치가 커지지 않도록 페널티 | 전 과목 고르게 공부하라는 규칙 |
| Dropout | 학습 중 뉴런을 무작위로 끄기 | 팀원이 빠져도 나머지가 해내도록 역량 분산 |
| 배치 정규화 | 각 층 입력을 평균 0, 분산 1로 정규화 | 데이터 분포를 일정하게 유지 |

#### 조기 중단 (Early Stopping)

검증 손실이 일정 횟수(patience) 동안 개선되지 않으면 학습을 중단한다. 가장 간단하면서 효과적인 정규화 기법이다.

```python
class EarlyStopping:
    def __init__(self, patience=10):
        self.patience = patience
        self.counter = 0
        self.best_loss = float('inf')
        self.best_state = None

    def update(self, val_loss, model):
        if val_loss < self.best_loss:
            self.best_loss = val_loss
            self.best_state = model.state_dict().copy()
            self.counter = 0
        else:
            self.counter += 1

    @property
    def should_stop(self):
        return self.counter >= self.patience
```

#### 검증 전략

| 전략 | 방법 | 적합한 상황 |
| ---- | ---- | ----------- |
| 홀드아웃 | 학습/검증/테스트 분할 | 대규모 데이터 |
| K-Fold CV | K개 폴드로 교차 검증 | 중소규모 데이터 |
| 층화 K-Fold | 클래스 비율 유지 | 불균형 분류 |
| 시간 기반 분할 | 과거→미래 순서 | 시계열 데이터 |
