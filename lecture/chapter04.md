## 4.1 군집화는 "정답을 맞히는 일"이 아니라 "구조를 발견하는 일"이다

3장에서 분류와 회귀는 정답 `Y`가 있는 문제라고 설명했다. 군집화는 여기서 출발점이 다르다. 군집화에서는 정답이 없다. 대신 데이터 안에 비슷한 것끼리 자연스럽게 모이는 구조가 있다고 가정하고, 그 구조를 찾아낸다.

같은 고객 데이터를 보더라도 질문이 다르면 방법이 달라진다.

- "이 고객이 이탈할까?"는 분류다.
- "우리 고객은 어떤 유형으로 나뉘는가?"는 군집화다.

쉽게 말하면 분류는 이미 이름표가 붙어 있는 상자를 보고 새 물건을 어느 상자에 넣을지 판단하는 문제다. 반면 군집화는 이름표가 전혀 없는 상태에서 비슷한 물건끼리 먼저 모아 보는 문제다.

<표 4-1: 분류와 군집화의 차이>

| 구분 | 분류 | 군집화 |
| ---- | ---- | ------ |
| 학습 유형 | 지도학습 | 비지도학습 |
| 정답 레이블 | 미리 있음 | 없음 |
| 핵심 질문 | 어느 범주에 속하는가 | 어떤 그룹 구조가 숨어 있는가 |
| 결과 해석 | 예측 성능 중심 | 구조 해석 중심 |
| 대표 예시 | 스팸 필터, 대출 승인 | 고객 세분화, 리뷰 묶기 |

군집화에서 가장 중요한 질문은 하나다.

**무엇을 비슷하다고 볼 것인가?**

이 질문에 따라 알고리즘 선택이 달라진다.

- 거리가 가까우면 비슷하다고 볼 것인가
- 밀도가 높게 연결되면 같은 집단이라고 볼 것인가
- 문장 의미가 비슷하면 같은 주제라고 볼 것인가

결국 군집화는 데이터를 묶는 기술이기 전에, **유사성을 정의하는 기술**이다.

### 4.1.1 군집화가 실제로 쓰이는 장면

군집화는 단순히 "비슷한 것끼리 묶기"에서 끝나지 않는다. 실무에서는 다음과 같이 해석 가능한 구조를 만드는 출발점이 된다.

- 고객 세분화: VIP, 충성 고객, 가격 민감 고객, 이탈 위험 고객 구분
- 이상 패턴 탐지: 정상 집단에서 벗어난 거래나 센서 값을 찾기
- 텍스트 정리: 수천 건의 VOC를 배송, 품질, 가격, 서비스 이슈로 묶기
- 이미지 정리: 비슷한 상품 사진이나 콘텐츠 묶기

여기서 중요한 점은 군집 라벨이 자동으로 "의미 있는 이름"이 되지 않는다는 것이다. 알고리즘은 그룹을 나눠 줄 뿐이고, 그 그룹에 어떤 이름을 붙일지는 사람이 해석해야 한다.

## 4.2 K-Means: 중심점에 가까운 것끼리 묶는 가장 기본적인 방법

군집화를 처음 배울 때 K-Means를 먼저 다루는 이유는 구조가 단순하고 직관이 분명하기 때문이다.

K-Means의 아이디어는 간단하다.

1. 군집 중심점 몇 개를 임의로 놓는다.
2. 각 데이터를 가장 가까운 중심점에 배정한다.
3. 배정된 데이터들의 평균으로 중심점을 다시 계산한다.
4. 더 이상 중심점이 크게 바뀌지 않을 때까지 반복한다.

학생들을 조별로 배치한다고 생각해 보자. 처음에는 임의로 조장을 세운다. 각 학생은 가장 가까운 조장에게 간다. 그러고 나서 조원들의 평균 위치를 기준으로 조장 위치를 다시 정한다. 이 과정을 반복하면 어느 정도 안정된 조 구성이 생긴다. K-Means는 바로 그 과정을 수치적으로 수행한다.

### 4.2.1 K-Means가 만드는 경계는 왜 단순한가

K-Means는 각 데이터가 중심점에서 얼마나 가까운지를 본다. 즉, "중심에서의 거리"가 유사성의 기준이다. 그래서 K-Means가 잘 작동하는 경우는 대체로 다음과 같다.

- 군집이 둥글거나 타원에 가까울 때
- 군집 간 거리가 충분히 떨어져 있을 때
- 이상치가 많지 않을 때

반대로 다음과 같은 경우에는 약해진다.

- 반달 모양처럼 구부러진 군집
- 한쪽은 빽빽하고 한쪽은 느슨한 군집
- 극단값이 중심을 흔드는 경우

<표 4-2: K-Means의 장점과 한계>

| 항목 | 내용 |
| ---- | ---- |
| 장점 1 | 구현이 쉽고 계산이 빠르다 |
| 장점 2 | 결과를 직관적으로 설명하기 좋다 |
| 장점 3 | 대규모 수치형 데이터의 빠른 베이스라인으로 유용하다 |
| 한계 1 | 군집 수 `K`를 미리 정해야 한다 |
| 한계 2 | 비구형 군집을 잘 나누지 못한다 |
| 한계 3 | 이상치에 민감하다 |

### 4.2.2 K를 정하는 문제

K-Means는 이름 그대로 군집 수 `K`를 먼저 정해야 한다. 그런데 실제 데이터에서는 "군집이 몇 개인가"를 미리 아는 경우가 드물다. 그래서 보통 엘보우 방법이나 실루엣 점수를 참고한다.

- 엘보우 방법: `K`를 늘릴수록 군집 내 분산은 줄어든다. 그 감소 폭이 급격히 꺾이는 지점을 찾는다.
- 실루엣 점수: 같은 군집 안에서는 가깝고 다른 군집과는 멀수록 높은 점수를 준다.

하지만 이 방법들도 정답을 보장하지는 않는다. 결국 군집 수 선택은 수학적 지표와 해석 가능성을 함께 봐야 한다.

즉, "점수가 가장 좋은 K"가 항상 "가장 의미 있는 K"는 아니다.

## 4.3 HDBSCAN: 촘촘하게 연결된 집단과 노이즈를 함께 보는 방법

현실 데이터는 K-Means가 좋아하는 형태로만 주어지지 않는다. 고객 행동 데이터도 그렇고, 센서 데이터도 그렇고, 텍스트 임베딩도 그렇다. 어떤 집단은 매우 촘촘하고, 어떤 집단은 퍼져 있으며, 일부는 아예 잡음처럼 흩어져 있다.

이때 밀도 기반 접근이 필요하다.

HDBSCAN은 간단히 말해 **촘촘한 영역을 군집으로 보고, 충분히 촘촘하지 않으면 노이즈로 남기는 방법**이다.

광장에 사람이 모여 있다고 생각해 보자.

- 사람이 빽빽하게 모인 곳은 하나의 집단처럼 보인다.
- 군데군데 듬성듬성 서 있는 사람은 특정 집단이라고 부르기 어렵다.

HDBSCAN은 이런 직관을 수치화한다. 그래서 다음 상황에서 특히 강하다.

- 군집 모양이 복잡할 때
- 군집 수를 사전에 모를 때
- 노이즈가 섞여 있을 때
- 한 데이터셋 안에 밀도가 서로 다른 군집이 함께 있을 때

![K-Means와 HDBSCAN 비교](../diagram/4-3-kmeans-vs-hdbscan.png)

위 그림에서 왼쪽처럼 중심점 기준으로 나누면 애매한 부분이 생기지만, 오른쪽처럼 밀도를 기준으로 보면 자연스러운 군집과 노이즈를 동시에 구분할 수 있다.

### 4.3.1 실습: HDBSCAN으로 군집과 노이즈를 함께 읽기

이 장의 첫 실습은 [4-3-hdbscan-clustering.py](/Users/callii/Documents/dataScience/practice/chapter04/code/4-3-hdbscan-clustering.py)이다. 이 실습의 핵심은 "몇 개로 나눌 것인가"보다 "어디를 군집으로 인정할 것인가"를 읽는 데 있다.

```python
import hdbscan

clusterer = hdbscan.HDBSCAN(min_cluster_size=15, min_samples=5)
labels = clusterer.fit_predict(X)
```

이 코드에서 눈여겨볼 부분은 두 가지다.

- `min_cluster_size`: 최소 몇 개 이상 모여야 군집으로 보겠는가
- `min_samples`: 얼마나 보수적으로 노이즈를 판정할 것인가

즉, HDBSCAN은 단순히 결과를 내는 것이 아니라 "어느 정도 모여 있어야 진짜 집단이라고 인정할 것인가"라는 기준을 설정하게 만든다.

![HDBSCAN 개념도](../diagram/4-hdbscan.png)

실습 결과를 볼 때는 다음 순서로 읽으면 된다.

1. 노이즈로 분류된 점이 얼마나 되는가
2. 군집 수가 몇 개인가
3. 각 군집의 밀도와 크기가 얼마나 다른가
4. 노이즈로 보낸 점이 실제로는 중요한 이상 패턴은 아닌가

군집화에서는 모든 점을 반드시 군집에 넣는 것이 좋은 결과가 아니다. 애매한 점을 억지로 넣는 것보다, 노이즈로 남겨 두는 것이 더 정직한 결과일 수 있다.

### 4.3.2 HDBSCAN과 K-Means를 어떤 기준으로 구분할까

<표 4-3: K-Means와 HDBSCAN 선택 기준>

| 상황 | 더 적합한 방법 | 이유 |
| ---- | -------------- | ---- |
| 군집 수를 미리 알고 있음 | K-Means | 빠르고 단순하다 |
| 대규모 수치형 데이터의 빠른 분할 | K-Means | 계산 효율이 좋다 |
| 노이즈가 많음 | HDBSCAN | 노이즈를 별도로 처리한다 |
| 군집 모양이 복잡함 | HDBSCAN | 비구형 군집에 강하다 |
| 밀도가 서로 다른 군집이 섞여 있음 | HDBSCAN | 밀도 기반 분리가 가능하다 |

## 4.4 텍스트 군집화에서는 "단어가 같은가"보다 "의미가 가까운가"가 중요하다

텍스트 데이터는 수치형 데이터와 다르다. 고객 리뷰 두 문장이 있다고 해 보자.

- "배송이 빨랐어요"
- "배달이 엄청 빨리 왔어요"

표면적으로는 단어가 다르다. 그러나 의미는 매우 가깝다. 이런 경우 단순한 단어 빈도 기반 거리만으로는 두 문장을 충분히 가깝게 보기 어렵다.

그래서 텍스트 군집화에서는 임베딩이 중요해진다. 임베딩은 문장을 밀집 벡터로 바꾸어, 비슷한 의미를 가진 문장이 벡터 공간에서도 가까운 위치에 놓이도록 한다.

![임베딩의 직관](../diagram/4-4-embedding-principle.png)

이 그림이 보여 주는 핵심은 하나다.  
문장은 원래 단어의 나열이지만, 임베딩을 거치면 "의미 좌표"로 바뀐다.

### 4.4.1 실습: 임베딩 기반 리뷰 군집화

[4-4-embedding-clustering.py](/Users/callii/Documents/dataScience/practice/chapter04/code/4-4-embedding-clustering.py)는 Sentence-BERT 계열 임베딩을 만든 뒤 차원 축소와 HDBSCAN을 결합해 리뷰를 의미 단위로 묶는다.

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")
embeddings = model.encode(texts, normalize_embeddings=True)
```

이 실습에서 중요한 점은 "임베딩을 만들었다"가 아니라, **유사성의 기준이 단어 일치에서 의미 유사성으로 바뀌었다**는 것이다.

![임베딩 군집화 결과](../diagram/4-4-embedding-clusters.png)

실습 결과를 읽을 때는 다음을 확인해야 한다.

- 같은 단어를 안 써도 같은 주제로 묶였는가
- 반대로 단어가 일부 겹쳐도 의미가 다르면 분리되는가
- 노이즈 문서가 어떤 특성을 가지는가

이 단계에서 학생들이 가장 자주 하는 오해는 "임베딩을 쓰면 언제나 자동으로 좋은 군집이 나온다"는 생각이다. 실제로는 그렇지 않다. 임베딩이 포착하는 의미와 우리가 군집화로 구분하고 싶은 기준이 다르면 결과가 기대와 다를 수 있다.

예를 들어 "배송 불만"과 "품질 불만"이 모두 부정적 감정이라는 점에서는 비슷하게 보일 수 있다. 임베딩은 감정의 유사성을 더 강하게 포착하고, 우리는 주제의 차이를 더 보고 싶을 수도 있다. 이 차이를 이해해야 군집 결과를 과신하지 않는다.

## 4.5 딥 클러스터링: 표현 자체를 군집에 맞게 다시 배우는 접근

앞 절까지의 군집화는 대체로 두 단계였다.

1. 입력을 적당한 벡터로 바꾼다.
2. 그 벡터에 군집 알고리즘을 적용한다.

하지만 이 접근에는 한계가 있다. 벡터 표현이 군집 목적에 최적화되어 있지 않을 수 있기 때문이다. 이 문제를 해결하려는 접근이 딥 클러스터링이다.

딥 클러스터링은 "군집이 잘 보이도록 표현을 다시 학습하자"는 생각에서 출발한다.

쉽게 말하면 그냥 안경을 쓰고 묶는 것이 아니라, **묶기 좋은 렌즈를 먼저 학습한 다음 군집화를 수행하는 방식**이다.

### 4.5.1 DEC의 기본 아이디어

대표적인 방법이 DEC(Deep Embedded Clustering)다. 전체 구조를 직관적으로 정리하면 다음과 같다.

1. 오토인코더로 데이터를 압축한다.
2. 잠재 공간에서 초기 군집 중심을 만든다.
3. 각 샘플이 어느 군집에 가까운지 소프트하게 계산한다.
4. 더 확실한 군집 구조가 나타나도록 표현과 군집 중심을 함께 조정한다.

여기서 핵심은 표현 학습과 군집화가 분리되어 있지 않다는 점이다.

### 4.5.2 실습: K-Means, DEC, IDEC 비교

[4-5-deep-clustering-comparison.py](/Users/callii/Documents/dataScience/practice/chapter04/code/4-5-deep-clustering-comparison.py)는 단순 군집화와 딥 클러스터링을 비교한다. 이 실습은 "딥러닝을 쓰면 무조건 좋아지는가"라는 질문에 답하는 데 중요하다.

실습의 핵심 비교 대상은 다음과 같다.

- K-Means: 고정된 벡터에서 바로 군집화
- DEC: 군집 목적에 맞게 잠재 표현을 조정
- IDEC: 재구성 손실을 함께 유지해 표현 붕괴를 완화

실습 결과는 보통 한 가지를 분명하게 보여 준다.

**딥 클러스터링은 항상 이기는 방법이 아니다.**

데이터가 이미 잘 분리되어 있다면 단순한 방법도 충분하다. 반대로 임베딩이 군집 목적과 어긋나 있다면 딥 클러스터링이 일부 개선을 줄 수 있다. 그러나 표현 자체가 문제 구조를 거의 담지 못하면, 딥 클러스터링도 근본적인 한계를 크게 넘어서지 못한다.

이 점이 중요하다. 학생들은 딥러닝을 "자동으로 더 좋은 방법"으로 이해하기 쉽지만, 실제로는 **표현의 질, 데이터의 구조, 목적 함수의 일치 여부**가 더 중요하다.

## 4.6 군집 결과는 숫자 하나로 끝나지 않는다

분류에서는 정확도나 AUC처럼 비교적 익숙한 지표가 있다. 군집화에서는 평가가 훨씬 더 어렵다. 왜냐하면 정답이 없는 경우가 많기 때문이다.

그래서 군집 평가는 보통 세 층위에서 본다.

1. **내부 지표**: 같은 군집 안은 가깝고 다른 군집은 먼가
2. **외부 지표**: 정답 라벨이 있을 때 실제 그룹과 얼마나 맞는가
3. **해석 가능성**: 결과가 실제 문제를 설명하는 데 도움이 되는가

예를 들어 같은 군집 안 문서들이 정말 같은 이슈를 말하고 있는지, 노이즈로 빠진 문서가 어떤 성격인지, 군집 수가 너무 잘게 나뉘지는 않았는지 등을 함께 봐야 한다.

<표 4-4: 군집 결과를 읽을 때 확인할 질문>

| 확인 질문 | 의미 |
| --------- | ---- |
| 같은 군집 안 샘플이 실제로 비슷한가 | 내부 응집도 확인 |
| 다른 군집과 충분히 구분되는가 | 군집 분리도 확인 |
| 노이즈 포인트는 무엇인가 | 이상치 또는 애매한 경계 확인 |
| 군집에 비즈니스 의미를 붙일 수 있는가 | 해석 가능성 확인 |
| 다른 알고리즘과 비교해도 구조가 유지되는가 | 결과의 안정성 확인 |

## 4.7 정리

이 장의 핵심은 알고리즘 이름을 외우는 것이 아니라, 각 알고리즘이 어떤 방식으로 유사성을 정의하는지 이해하는 것이다.

```text
1. 군집화는 정답 라벨 없이 데이터 안의 구조를 발견하는 문제다.
2. K-Means는 중심점과의 거리를 기준으로 군집을 만든다.
3. HDBSCAN은 밀도와 노이즈를 함께 고려해 더 유연하게 군집을 찾는다.
4. 텍스트에서는 단어 일치보다 의미 유사성이 중요하므로 임베딩 기반 군집화가 강력하다.
5. 딥 클러스터링은 표현 자체를 군집 목적에 맞게 다시 학습하는 접근이다.
6. 군집 결과는 군집 수보다 해석 가능성과 목적 적합성까지 함께 봐야 한다.
```
