## 1주차: 데이터 사이언스의 진화 — 통계에서 AI까지

> **미션**: 데이터 사이언스의 발전 흐름을 이해하고, 머신러닝 유형을 구분하며, AI 시대 워크플로우의 원칙과 전통 ML vs 딥러닝의 선택 기준을 익힌다

### 학습목표

이 수업을 마치면 다음을 수행할 수 있다:

1. 통계학 → 데이터마이닝 → 머신러닝 → 딥러닝 → LLM으로 이어지는 패러다임 변화를 설명할 수 있다
2. 지도학습, 비지도학습, 강화학습, 자기지도학습을 구분하고 적합한 상황을 판단할 수 있다
3. CRISP-DM과 AI 증강 워크플로우의 차이를 설명하고, AI 도구 활용의 원칙을 이해한다
4. LLM 기반 EDA의 가능성과 한계를 파악할 수 있다
5. 데이터 유형과 규모에 따라 전통 ML과 딥러닝 중 적절한 접근을 선택할 수 있다

### 1.1 데이터 사이언스의 역사와 패러다임 변화

데이터 사이언스라는 용어는 최근에 등장했지만, 데이터에서 지식을 추출하려는 시도는 오랜 역사를 가지고 있다.

#### 시장의 성장

글로벌 머신러닝 시장은 2022년 214억 달러에서 2027년 약 1,000억 달러로 연평균 36.2% 성장이 전망된다(Grand View Research, 2023). 이 성장을 주도하는 세 가지 요인:

- **데이터 폭증**: 전 세계 데이터 생성량의 기하급수적 증가
- **컴퓨팅 자원의 민주화**: 클라우드(AWS, GCP, Azure)와 오픈소스(scikit-learn, PyTorch, Hugging Face)
- **생성형 AI의 등장**: ChatGPT 이후 AI 투자 급증, 연간 2.6조~4.4조 달러의 경제적 가치 잠재력(McKinsey, 2023)

#### 역사적 발전

| 시대 | 핵심 접근 | 대표 기법 | 특성 학습 | 주요 한계 |
| --- | --- | --- | --- | --- |
| 통계학 (1900s~1970s) | 가설 검증 | 회귀, t-test, ANOVA | 수동 설계 | 대규모 데이터 처리 한계 |
| 데이터마이닝 (1980s~1990s) | 패턴 발견 | 연관규칙, 의사결정나무 | 수동 설계 | 비정형 데이터 처리 한계 |
| 머신러닝 (2000s~2010s) | 예측 모델링 | SVM, Random Forest, XGBoost | 특성 공학 | 특성 설계에 전문성 필요 |
| 딥러닝 (2012~2020) | 표현 학습 | CNN, RNN, Transformer | 자동 학습 | 대량 데이터/컴퓨팅 필요 |
| LLM (2020s~) | 범용 AI | GPT-4, Claude, Gemini | 사전학습 전이 | 비용, Hallucination |

각 시대는 이전 시대의 한계를 극복하며 진화했다. 통계학은 가설 주도(hypothesis-driven), 데이터마이닝은 데이터 주도(data-driven), 머신러닝은 특성 공학, 딥러닝은 표현 학습(representation learning)이 핵심이다. LLM 시대에는 사전학습-미세조정, 프롬프트 기반, 멀티모달 확장이 새로운 패러다임을 열었다.

#### AI가 데이터 분석가를 대체하는가

LLM이 **대체할 수 있는** 영역: 기초 EDA, 코드 생성, 문서화, 데이터 정제 등 반복적 작업.

LLM이 **대체하기 어려운** 영역: 비즈니스 맥락을 고려한 문제 정의, 이해관계자 커뮤니케이션, 윤리적 판단, 창의적 문제 해결.

가장 현실적인 관점은 LLM을 **"대체재"가 아닌 "증강 도구"**로 보는 것이다.

### 1.2 머신러닝의 주요 유형

머신러닝은 학습 방식에 따라 네 가지 유형으로 분류된다.

#### 지도학습: 정답이 있는 학습

입력 X와 정답 Y가 쌍으로 주어질 때, X → Y를 예측하는 함수를 학습한다. 출력이 범주형이면 **분류**, 연속형이면 **회귀**다. 명확한 평가 기준이 있다는 것이 강점이다.

활용: 신용 평가, 고객 이탈 예측, 사기 탐지, 품질 불량 예측 등.

#### 비지도학습: 정답 없이 구조 발견

레이블 없이 데이터 X만으로 내재된 구조를 발견한다.

- **군집 분석**: 유사한 데이터를 그룹으로 묶음 (K-Means, DBSCAN)
- **차원 축소**: 핵심 정보를 보존하며 저차원으로 변환 (PCA, t-SNE, UMAP)
- **밀도 추정**: 데이터의 확률 분포 추정

평가 기준이 명확하지 않다는 어려움이 있다.

#### 강화학습: 시행착오를 통한 학습

에이전트가 환경과 상호작용하며 보상을 최대화하는 행동을 학습한다. 상태(state), 행동(action), 보상(reward), 정책(policy)이 핵심 개념이다. 의사결정이 시간에 걸쳐 누적되고 행동의 효과가 지연되는 문제(동적 가격 책정, 재고 관리, 추천 시스템)에서 유용하다.

#### 자기지도학습: LLM의 핵심 원리

레이블 없이 데이터 자체에서 학습 신호를 생성한다. GPT는 **다음 토큰 예측**, BERT는 **마스크 언어 모델링**으로 학습한다. 레이블링 비용 없이 대규모 데이터에서 학습할 수 있다는 것이 혁신이다.

| 유형 | 학습 데이터 | 학습 신호 | 대표 태스크 | 대표 알고리즘 |
| --- | --- | --- | --- | --- |
| 지도학습 | X, Y 쌍 | 정답 레이블 | 분류, 회귀 | XGBoost, RF, 신경망 |
| 비지도학습 | X만 | 없음 | 군집, 차원축소 | K-Means, PCA, UMAP |
| 강화학습 | 상태-행동 | 보상 신호 | 게임, 로봇 제어 | Q-Learning, PPO |
| 자기지도학습 | X만 | 자기생성 | 사전학습 | GPT, BERT |

### 1.3 AI 시대 데이터 분석 워크플로우

#### CRISP-DM: 전통적 프로세스

CRISP-DM은 비즈니스 이해 → 데이터 이해 → 데이터 준비 → 모델링 → 평가 → 배포의 6단계로 구성된다. 어느 단계에서든 이전 단계로 돌아갈 수 있는 반복적(iterative) 특성이 강점이다.

#### AI 증강 워크플로우

LLM과 AutoML의 등장으로 각 단계에 AI 도구를 결합한다.

| 단계 | CRISP-DM (전통) | AI 증강 워크플로우 |
| --- | --- | --- |
| 비즈니스 이해 | 수동 자료 조사 | LLM으로 도메인 지식 수집 가속화 |
| 데이터 이해 | 수동 EDA 코드 작성 | ChatGPT/Claude로 자동 EDA |
| 데이터 준비 | 60~80% 시간 소요 | 자동화 도구로 시간 단축 |
| 모델링 | 수동 알고리즘 선택/튜닝 | AutoML로 베이스라인 자동 구축 |
| 평가 | 수동 성능 측정 | 자동 실험 추적, 비교 대시보드 |
| 배포 | 수동 배포 스크립트 | MLOps 파이프라인 자동화 |

핵심 변화는 **시간 배분**이다. 데이터 준비에 60~80% → 30~40%로 줄어들고, 분석가는 문제 정의와 모델 해석에 더 집중할 수 있다.

#### AI 시대 워크플로우의 원칙

- AI 도구를 적극 활용하되, 분석의 책임과 통제권은 사람에게 남긴다
- LLM 인사이트는 데이터와 도메인 지식으로 반드시 검증한다
- 핵심 의사결정에 사용된 프롬프트와 결과를 문서화해 재현성을 확보한다
- API 비용은 개발 단계와 최종 산출물 단계를 분리해 관리한다

### 1.4 LLM 기반 탐색적 데이터 분석

EDA는 데이터의 구조, 패턴, 이상치를 파악하고 후속 분석 방향을 결정하는 첫 단계다.

#### 두 가지 접근

- **대화형**: ChatGPT Code Interpreter, Claude Artifacts 등에 데이터를 업로드하고 자연어로 분석 요청. 빠르지만 재현성 제약.
- **API 기반**: 데이터 프로파일을 생성해 LLM API에 전달. 재현성과 운영 자동화에 유리.

실무에서는 아이디어 발굴·가설 설정은 대화형으로, 산출물 생성·반복 실행은 API 기반으로 혼합 사용한다.

#### 주요 도구 비교

| 도구 | 특징 | 장점 | 단점 |
| --- | --- | --- | --- |
| ChatGPT Code Interpreter | 파일 업로드 후 대화형 분석 | 직관적, 코드 자동 생성 | 파일 크기 제한, 세션 만료 |
| Claude Artifacts | 실시간 시각화, 코드 실행 | 빠른 반복, 시각화 우수 | 대용량 데이터 제한 |
| GitHub Copilot | IDE 통합 코드 제안 | 개발 환경 통합 | 맥락 이해 제한적 |
| Pandas AI | Python 라이브러리 | 기존 워크플로우 통합 | 복잡한 분석은 제한적 |

#### 한계와 주의점

- **환각**: 데이터에 없는 패턴을 "발견"했다고 할 수 있다. 통계적 검증 필수.
- **보안**: 민감 데이터를 외부 API로 전송하는 위험. 데이터 마스킹이나 온프레미스 모델 검토.
- **비용**: 토큰 단위 과금. 대규모 데이터는 프로파일 요약이나 샘플링 활용.

#### 전통적 EDA vs LLM 기반 EDA

| 항목 | 전통적 EDA | LLM 기반 EDA |
| --- | --- | --- |
| 소요 시간 | 수 시간~수일 | 수 분 |
| 초기 인사이트 | 분석가 역량 의존 | 자동 생성 |
| 도메인 지식 | 분석가가 제공 | 프롬프트로 주입 |
| 깊이 있는 분석 | 우수 | 제한적 |
| 커스터마이징 | 완전 가능 | 프롬프트 의존 |

LLM은 "첫 탐색 단계의 가속기"로 두고, 핵심 의사결정은 전통적 EDA와 검증 절차로 마무리한다.

### 1.5 전통 ML vs 딥러닝: 선택 기준

#### 정형 데이터에서 트리 기반 모델의 우위

정형 데이터(테이블 형태)에서는 XGBoost, LightGBM, CatBoost 같은 그래디언트 부스팅 모델이 딥러닝보다 일반적으로 우수하다(Grinsztajn et al., NeurIPS 2022). 이유:

- 이질적인 입력 변수를 그대로 다루기 쉬움
- 비선형 관계와 상호작용을 분할 규칙으로 자연스럽게 포착
- 극단값과 결측에 대한 민감도가 낮음
- SHAP 등으로 예측 근거 설명 용이

#### 딥러닝이 유리한 상황

비정형 데이터(이미지, 텍스트, 음성)가 핵심 신호일 때, 샘플이 충분히 클 때, 장기 의존성이 중요할 때, 멀티모달 처리가 필요할 때, 전이학습 활용이 가능할 때.

#### 선택 가이드

| 고려 요소 | 전통 ML 유리 | 딥러닝 유리 |
| --- | --- | --- |
| 데이터 유형 | 정형 데이터 | 이미지, 텍스트, 시계열 |
| 데이터 규모 | 중소규모 (< 100만) | 대규모 (> 100만) |
| 해석 필요성 | 높음 | 낮음 |
| 개발 속도 | 빠름 | 상대적으로 느림 |
| 컴퓨팅 자원 | CPU 충분 | GPU 권장/필수 |
| 특성 공학 | 수동 또는 경량 | 자동 학습 |
| 전이학습 | 제한적 | 활용 가능 |

실무에서는 트리 모델로 빠르게 베이스라인을 구축한 뒤, 추가 성능이 필요하거나 비정형 데이터를 활용해야 할 때 딥러닝으로 확장하는 순서가 효율적이다.

### 핵심 정리

```text
1. 데이터 사이언스는 통계학 → 데이터마이닝 → 머신러닝 → 딥러닝 → LLM으로 진화했다.
2. 지도/비지도/강화/자기지도학습은 학습 신호와 문제 구조에 따라 선택한다.
3. CRISP-DM은 여전히 유효한 뼈대이며, AI 도구로 반복 작업을 줄일 수 있다.
4. LLM 기반 EDA는 가속기로 유용하지만, 검증 없이 신뢰해서는 안 된다.
5. 정형 데이터에서는 트리 기반 모델이, 비정형·대규모 데이터에서는 딥러닝이 유리하다.
6. 선택은 일회성이 아니라 데이터와 요구사항 변화에 따라 재평가해야 한다.
```
