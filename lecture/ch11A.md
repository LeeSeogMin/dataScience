### 11.1 그래프 기초: 관계를 데이터로 표현하기

전통적인 데이터 분석은 개별 관측치의 속성(나이, 구매 금액 등)에 집중하지만, **관계 정보**에서만 드러나는 패턴이 있다. 그래프 분석은 "누가 누구와 연결되어 있는가"를 활용하여 개별 속성만으로는 보이지 않는 구조를 발견한다.

#### 노드와 엣지

그래프(Graph)는 **노드(Node)**와 **엣지(Edge)**로 구성된다. 비유하자면, 노드는 지하철역이고 엣지는 역을 잇는 노선이다.

| 구성 요소 | 의미 | 예시 |
| --------- | ---- | ---- |
| 노드(Node) | 분석 대상 개체 | 사용자, 제품, 웹페이지, 도시 |
| 엣지(Edge) | 개체 간 관계 | 친구, 구매, 링크, 도로 |

![노드와 엣지의 기본 구조](../diagram/11-1-node.png)

#### 방향성과 가중치

| 속성 | 설명 | 예시 |
| ---- | ---- | ---- |
| 무방향 그래프 | 엣지에 방향 없음 (양방향) | 페이스북 친구 관계 |
| 방향 그래프 | 엣지에 방향 있음 (A→B) | 트위터 팔로우, 웹 링크 |
| 비가중치 그래프 | 엣지가 단순 연결만 표현 | 친구 여부 (있다/없다) |
| 가중치 그래프 | 엣지에 강도/빈도 정보 포함 | 통화 횟수, 거래 금액 |

![속성이 포함된 그래프 예시](../diagram/11-2-meta.png)

![방향 그래프와 무방향 그래프의 비교](../diagram/11-3-direction.png)

![가중치 그래프와 비가중치 그래프의 비교](../diagram/11-4-weight.png)

#### 인접 행렬

그래프를 컴퓨터가 다루기 위해 **인접 행렬(Adjacency Matrix)**로 표현한다. n개 노드가 있으면 n×n 행렬이고, A[i][j] = 1이면 노드 i와 j가 연결된 것이다. 무방향 그래프의 인접 행렬은 대칭이다.

#### 그래프 분석의 5대 질문

| 질문 | 분석 목표 | 핵심 기법 |
| ---- | --------- | --------- |
| 누가 가장 중요한가? | 영향력 있는 노드 식별 | 중심성 분석 (11.2절) |
| 어떤 그룹이 존재하는가? | 커뮤니티 구조 발견 | 커뮤니티 탐지 (11.3절) |
| 누가 연결될 것인가? | 미래 관계 예측 | 링크 예측 (11.4절) |
| 노드를 어떻게 표현할까? | 구조적 유사성 포착 | 그래프 임베딩 (11.5절) |
| 무엇을 추천할까? | 개인화 추천 | 협업 필터링 (11.6절) |

### 11.2 중심성 분석: 네트워크에서 가장 중요한 노드는?

"중요하다"는 개념은 맥락에 따라 다르다. 친구가 많은 사람이 중요한가, 서로 다른 그룹을 연결하는 사람이 중요한가? 중심성 지표는 이러한 다양한 관점에서 노드의 중요도를 측정한다.

#### 4가지 중심성 지표

| 지표 | 핵심 질문 | 비유 | 적합한 시나리오 |
| ---- | --------- | ---- | --------------- |
| 연결 중심성 | 몇 명과 직접 연결? | 명함이 많은 사람 | 인기도, 인플루언서 식별 |
| 매개 중심성 | 다른 사람들의 소통 경로에 얼마나 자주 등장? | 부서 간 연결 담당자 | 병목점 탐지, 핵심 인력 |
| 근접 중심성 | 다른 모든 노드에 얼마나 빨리 도달? | 지하철 환승역 | 정보 전파 시작점 |
| PageRank | 중요한 노드들로부터 얼마나 연결 받음? | 네이버 메인 링크 | 웹 순위, 인용 영향력 |

**연결 중심성**: C_D(v) = deg(v) / (n-1). 연결 수를 전체 노드 수로 나누어 정규화한다.

**매개 중심성**: 모든 노드 쌍의 최단 경로 중 해당 노드를 거치는 비율. 이 값이 높은 노드가 이탈하면 네트워크 소통이 크게 위축된다.

**근접 중심성**: 다른 모든 노드까지 거리의 합의 역수. 네트워크의 "지리적 중심"에 위치한 노드가 높다.

**PageRank**: Google 검색 순위의 원리. "중요한 페이지가 링크한 페이지도 중요하다"는 재귀적 아이디어. 스팸 링크에 강건하다.

### 11.3 커뮤니티 탐지: 숨겨진 그룹 찾기

커뮤니티(Community)는 내부적으로는 밀접하게 연결되어 있으나, 외부와는 드물게 연결된 노드 집합이다. 비유하자면, 학교에서 같은 반 친구들끼리는 자주 만나지만 다른 반과는 교류가 적은 것과 같다.

#### 모듈성(Modularity): 커뮤니티 품질 척도

모듈성(Q)은 커뮤니티 분할의 품질을 측정한다. "커뮤니티 내부 연결 밀도가 무작위로 기대되는 수준보다 얼마나 높은가"를 나타낸다.

| Q 값 범위 | 해석 |
| --------- | ---- |
| Q ≈ 0 | 무작위 그래프 수준 (커뮤니티 구조 없음) |
| Q ≥ 0.3 | 유의미한 커뮤니티 구조 존재 |
| Q → 1 | 매우 강한 커뮤니티 구조 |

#### Louvain 알고리즘

대규모 네트워크에서 효율적으로 커뮤니티를 탐지하는 탐욕적 최적화 방법이다. 두 단계를 반복한다:

1. **지역 최적화**: 각 노드를 모듈성 증가가 가장 큰 이웃 커뮤니티로 이동
2. **네트워크 압축**: 각 커뮤니티를 하나의 "슈퍼 노드"로 축소

시간 복잡도 O(n log n)으로, 수백만 노드 규모에도 적용 가능하다.

| 특성 | Louvain | 스펙트럴 군집화 |
| ---- | ------- | --------------- |
| 커뮤니티 수 | 자동 결정 | 사전 지정 필요 (k) |
| 확장성 | 대규모 가능 O(n log n) | 중소규모 O(n³) |
| 결과 안정성 | 시드에 따라 변동 가능 | 안정적 |

### 11.4 링크 예측: 앞으로 누가 연결될까?

링크 예측은 현재 존재하지 않는 엣지가 미래에 형성될 가능성을 추정하는 기법이다. 소셜 네트워크의 "알 수도 있는 사람" 추천이 대표적 응용이다.

#### 구조적 유사도 지표

핵심 아이디어: 공통 친구가 많은 두 사람은 서로 알게 될 가능성이 높다.

| 지표 | 핵심 아이디어 | 장점 | 단점 |
| ---- | ------------- | ---- | ---- |
| 공통 이웃 | 공유 친구 수 | 단순, 직관적 | 허브 노드에 편향 |
| Jaccard | 정규화된 공통 이웃 | 허브 편향 보정 | 희소 네트워크에서 0 다수 |
| Adamic-Adar | 희귀 공통 이웃 강조 | 정보성 있는 연결 포착 | 계산 비용 약간 높음 |
| Preferential Attachment | 연결 수의 곱 | 성장 네트워크에 적합 | 구조 정보 미활용 |

**공통 이웃(CN)**: CN(u,v) = |N(u) ∩ N(v)|. 두 노드의 공통 친구 수.

**Jaccard**: 공통 이웃을 전체 이웃 합집합으로 나누어 정규화. 연결 수가 많은 허브 노드의 편향을 보정한다.

**Adamic-Adar**: 친구가 적은(희귀한) 공통 이웃에 더 높은 가중치. 유명인과의 공통 친구보다 일반인 친구를 공유하는 것이 더 강한 신호다.

**Preferential Attachment**: |N(u)| × |N(v)|. "부익부" 가설 — 연결이 많은 노드끼리 더 연결되기 쉽다.

### 11.5 그래프 임베딩: 노드를 벡터로 표현하기

그래프 임베딩은 노드를 저차원 벡터 공간에 매핑하여, 그래프 구조를 벡터 연산으로 다룰 수 있게 하는 기법이다. 비유하자면, 지도 위의 도시 위치처럼 "가까운 노드는 비슷한 관계"를 갖도록 배치하는 것이다.

#### Node2Vec: 랜덤 워크 + Skip-gram

Node2Vec은 그래프에서 랜덤 워크(무작위 이동)를 수행하여 노드 시퀀스를 만들고, 이를 Word2Vec처럼 학습한다. 자주 함께 등장하는 노드들은 유사한 벡터를 갖게 된다.

**p, q 파라미터로 탐색 전략 조절**:

| 설정 | 탐색 특성 | 포착하는 유사성 |
| ---- | --------- | --------------- |
| p=1, q=1 | 균형 (DeepWalk) | 구조적 + 커뮤니티 |
| p < 1, q > 1 | BFS-like (지역 탐색) | 지역 이웃 구조 |
| p > 1, q < 1 | DFS-like (원거리 탐색) | 전역 커뮤니티 소속 |

#### 활용 분야

| 활용 | 설명 |
| ---- | ---- |
| 유사 노드 검색 | 코사인 유사도로 가장 가까운 노드 찾기 |
| 노드 분류 | 임베딩을 특성으로 사용하여 레이블 예측 |
| 그래프 시각화 | t-SNE/UMAP으로 2차원 투영하여 커뮤니티 확인 |

### 11.6 추천 시스템: 사용자-아이템 관계 분석

추천 시스템은 사용자의 과거 행동 데이터로부터 선호할 만한 아이템을 예측한다. 그래프 관점에서 보면 사용자-아이템 이분 그래프(Bipartite Graph)에서 빠진 엣지를 예측하는 링크 예측 문제다.

#### 협업 필터링

| 방식 | 원리 | 비유 |
| ---- | ---- | ---- |
| 사용자 기반 (User-KNN) | 나와 비슷한 사용자가 좋아한 것을 추천 | "너랑 취향 비슷한 친구가 이 영화 좋아하더라" |
| 아이템 기반 (Item-KNN) | 내가 좋아한 것과 비슷한 아이템 추천 | "이 상품을 본 고객이 함께 본 상품" |

#### 행렬 분해 (SVD)

사용자-아이템 평점 행렬 R을 사용자 행렬 P와 아이템 행렬 Q로 분해한다: R ≈ P × Q^T

비유: 영화 추천에서 각 차원이 "액션 선호도", "로맨스 선호도" 같은 잠재 취향을 나타낸다. 사용자와 영화를 같은 "취향 공간"에 배치하여, 가까운 영화를 추천한다.

| 특성 | 협업 필터링 (KNN) | 행렬 분해 (SVD) |
| ---- | ----------------- | --------------- |
| 학습 방식 | 유사도 직접 계산 | 잠재 요인 학습 |
| 확장성 | 대규모 어려움 | 대규모 가능 |
| 설명 가능성 | "비슷한 사용자가..." | 잠재 요인은 추상적 |
| Cold-start | 취약 | 동일하게 취약 |

![추천 시스템 성능 비교](../diagram/ch11_recommendation_comparison.png)

#### 평가 지표

| 지표 | 측정 내용 |
| ---- | --------- |
| RMSE | 예측 평점과 실제 평점의 차이 (작을수록 좋음) |
| MAE | 평균 절대 오차 (이상치에 덜 민감) |
| Precision@K | 상위 K개 추천 중 실제 선호 비율 |
| NDCG | 추천 순위까지 고려한 품질 |
