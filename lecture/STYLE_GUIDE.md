# Lecture Style Guide

이 문서는 `/Users/callii/Documents/causal/lecture`의 문서 작성 방식을 기준으로, 현재 프로젝트의 `lecture/` 원고 작성 규칙을 고정한다.

## 1. 문서 목적

`lecture/` 문서는 원문 보관용이 아니라 강의용 교재 원고다. 따라서 설명은 쉬워야 하고, 장 구조는 반복 가능해야 하며, 실습과 직접 연결되어야 한다.

## 2. 기본 문체

- 대상 독자: 학부생 또는 비전공 초중급 학습자
- 문체: 설명형, 직관적, 과도하게 학술적이지 않음
- 원칙: 정의 -> 쉬운 설명 -> 비유/사례 -> 표/코드 -> 핵심 요약

## 3. 필수 구조

각 장은 아래 순서를 기본으로 한다.

1. 장 제목
2. `학습 목표`
3. `이 장에서 배울 내용 미리보기`
4. 본문 절 (`1.1`, `1.2` 등)
5. 표, 예시, 필요 시 짧은 코드 블록
6. `핵심 요약`
7. 실습 연결 문구

## 4. 본문 규칙

- 절 제목은 계층형 번호를 유지한다.
- 한 절은 하나의 핵심 개념만 설명한다.
- 어려운 개념은 반드시 비유나 사례를 붙인다.
- 표는 비교 기준을 명확히 적는다.
- 코드는 길게 붙이지 말고, 핵심 설명 후 실습 파일로 연결한다.

## 5. 금지 사항

- `docs/` 문장을 거의 그대로 옮기지 않는다.
- 논문식 압축 문체로만 밀어붙이지 않는다.
- 실습과 연결되지 않는 장황한 설명을 늘리지 않는다.
- 장마다 형식이 달라지지 않도록 한다.

## 6. 실습 연결 규칙

- 실습 경로는 `practice/chapterXX/code/...` 형식으로 쓴다.
- "전체 코드는 ..." 문구로 마무리할 수 있다.
- 장 내용과 실습의 연결 이유를 한두 문장으로 설명한다.

## 7. 작성 순서

1. `docs/chXX.md`에서 핵심 개념 추출
2. 강의용 개요 재구성
3. 쉬운 설명과 사례 추가
4. 표/실습 연결 삽입
5. 장말 요약 작성
