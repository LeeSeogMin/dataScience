## 1.1 왜 먼저 "큰 지도"를 봐야 할까

데이터 사이언스를 처음 배우면 알고리즘 이름이 너무 많아서 길을 잃기 쉽다. 이 장은 그 문제를 풀기 위한 장이다. 비유하면, 여행을 떠나기 전에 세부 골목길부터 외우는 것이 아니라 먼저 지도를 펴서 "산이 어디에 있고, 강이 어디를 지나고, 도시가 어디에 모여 있는지"를 보는 단계라고 생각하면 된다.

이 장의 핵심 질문은 세 가지다.

1. 데이터 분석은 어떻게 통계학에서 AI까지 발전했는가?
2. 문제에 따라 어떤 학습 방식을 골라야 하는가?
3. AI 도구를 써도 결국 사람이 꼭 해야 하는 일은 무엇인가?

---

## 1.2 데이터 사이언스는 어떻게 진화했는가

가장 단순하게 요약하면 흐름은 아래와 같다.

- 통계학: "이 차이가 우연인지 아닌지"를 따지는 시대
- 머신러닝: "잘 맞히는 모델"을 만드는 시대
- 딥러닝: 사람이 특성을 일일이 만들지 않고 표현을 자동으로 학습하는 시대
- LLM: 언어와 지식을 폭넓게 다루는 범용 AI 도구의 시대

쉽게 비유하면 다음과 같다.

- 통계학은 정밀한 자와 저울
- 머신러닝은 숙련된 분류기
- 딥러닝은 스스로 특징을 찾아내는 현미경
- LLM은 설명서, 요약 도구, 초안 작성기까지 겸하는 조수

중요한 점은 새 방법이 옛 방법을 완전히 없애지 않는다는 것이다. 실무에서는 여전히 통계적 검증, 해석 가능한 모델, 전통적 전처리가 매우 중요하다.

<표 1-1: 데이터 사이언스 패러다임을 한눈에 보기>

| 시대 | 핵심 질문 | 대표 도구 | 강점 | 주의점 |
| ---- | --------- | --------- | ---- | ------ |
| 통계학 | 차이가 유의한가? | 회귀, 가설검정 | 해석이 명확함 | 복잡한 비선형 패턴에 약함 |
| 머신러닝 | 얼마나 잘 예측하는가? | 랜덤 포레스트, XGBoost | 정형 데이터에 강함 | 특성 설계가 중요함 |
| 딥러닝 | 데이터를 어떻게 표현할까? | CNN, Transformer | 비정형 데이터에 강함 | 데이터와 자원 요구가 큼 |
| LLM | 지식과 언어를 어떻게 활용할까? | GPT 계열, Claude 계열 | 요약, 생성, 보조 분석에 강함 | 환각과 검증 문제가 있음 |

## 1.3 머신러닝의 네 가지 학습 방식

이 부분은 학생들이 가장 많이 헷갈리는 영역이다. 쉬운 비유로 정리하면 다음과 같다.

- 지도학습: 정답지를 보고 푸는 공부
- 비지도학습: 정답 없이 비슷한 것끼리 스스로 묶는 공부
- 강화학습: 해 보면서 보상을 많이 받는 행동을 익히는 공부
- 자기지도학습: 데이터 안에 숨어 있는 문제를 스스로 만들어 푸는 공부

예를 들어 고객 데이터를 다룬다고 하자.

- 고객 이탈 여부를 예측하면 지도학습
- 비슷한 고객군을 나누면 비지도학습
- 어떤 쿠폰을 언제 보낼지 실험하며 학습하면 강화학습
- 대량 텍스트에서 문장 의미를 미리 학습하면 자기지도학습

핵심은 "알고리즘 이름"보다 "문제 구조"를 먼저 보는 것이다.

## 1.4 AI 시대의 분석 흐름

전통적 분석은 대체로 `문제 정의 -> 데이터 수집 -> 분석 -> 보고` 흐름으로 진행된다. AI가 들어오면 여기에 자동화와 보조 기능이 붙는다. 하지만 문제를 잘못 정의하면 AI가 있어도 결과는 크게 좋아지지 않는다.

비유하면 이렇다.

- 전통적 분석은 사람이 수작업으로 운전하는 자동차
- AI 증강 분석은 내비게이션과 보조 운전 기능이 붙은 자동차

차가 더 똑똑해졌다고 해서 목적지를 대신 정해 주는 것은 아니다. 목적지는 여전히 사람이 정해야 한다.

특히 LLM 기반 EDA는 빠르게 아이디어를 주지만, 그 결과를 바로 믿으면 안 된다. LLM은 "그럴듯한 설명"을 잘 만들지만 "항상 사실인 설명"을 보장하지는 않는다.

## 1.5 전통 ML과 딥러닝은 어떻게 고를까

가장 실용적인 기준은 네 가지다.

1. 데이터 유형이 정형인가 비정형인가
2. 데이터 양이 충분한가
3. 해석 가능성이 중요한가
4. 계산 자원과 시간이 충분한가

보통은 아래 원칙이 안전하다.

- 정형 데이터: 트리 기반 모델부터 시작
- 이미지/텍스트/음성: 딥러닝 우선 검토
- 설명 책임이 큰 업무: 해석 가능한 모델 또는 해석 도구 병행
- 빠른 베이스라인이 필요할 때: 단순한 모델부터 시작
