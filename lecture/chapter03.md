## 3.1 분류와 회귀: 예측 문제의 두 가지 기본 형태

지도학습은 입력 변수 `X`로부터 정답 `Y`를 예측하는 규칙을 학습하는 방법이다. 겉으로 보면 "입력으로부터 답을 맞힌다"는 한 문장으로 정리되지만, 실제로는 **무엇을 맞히는가**에 따라 접근이 크게 달라진다.

가장 기본적인 구분은 다음 두 가지다.

- **분류(Classification)**: 정답이 범주일 때
- **회귀(Regression)**: 정답이 숫자일 때

이 구분은 단순해 보이지만, 모델 선택과 평가 방식, 결과 해석까지 모두 바꿔 놓는다.

### 3.1.1 분류는 "어느 칸에 속하는가"를 묻는다

분류는 대상을 몇 개의 범주 중 하나로 나누는 문제다.

예를 들면 다음과 같다.

- 메일이 스팸인가 정상인가
- 고객이 이탈할 것인가 유지할 것인가
- 환자가 질환 양성인가 음성인가
- 사진 속 동물이 고양이인가 개인가

쉽게 말하면, 분류는 "이 대상을 어느 서랍에 넣을 것인가"를 묻는 문제다.

분류는 다시 세 가지 정도로 나눠서 생각할 수 있다.

1. **이진 분류**: 두 범주 중 하나를 예측
2. **다중 분류**: 세 범주 이상 중 하나를 예측
3. **다중 레이블 분류**: 하나의 대상이 여러 범주에 동시에 속할 수 있음

영화 추천을 예로 들면,

- "좋아요/싫어요"는 이진 분류
- "액션/로맨스/스릴러 중 하나"는 다중 분류
- "액션이면서 코미디"는 다중 레이블 분류

### 3.1.2 회귀는 "숫자가 얼마나 되는가"를 묻는다

회귀는 결과값이 연속적인 숫자일 때 사용한다.

예를 들면 다음과 같다.

- 집값 예측
- 월별 매출 예측
- 고객 생애 가치(LTV) 추정
- 제품 수요량 예측

회귀는 "합격/불합격"처럼 칸을 나누는 것이 아니라, "몇 점쯤 될까"처럼 값을 추정하는 문제다.

분류가 서랍에 넣는 문제라면, 회귀는 자와 저울로 길이와 무게를 재는 문제에 가깝다.

### 3.1.3 같은 현상도 질문에 따라 문제 유형이 달라진다

실무에서 특히 중요한 점은 **문제 자체가 고정되어 있지 않다**는 것이다. 같은 현상도 질문을 어떻게 던지느냐에 따라 분류가 되기도 하고 회귀가 되기도 한다.

예를 들어 고객 이탈 문제를 생각해 보자.

- "이 고객이 이탈할까?"라고 물으면 분류
- "이 고객은 몇 개월 뒤 이탈할까?"라고 물으면 회귀 또는 생존 분석

학생 성적 문제도 마찬가지다.

- "합격할까?"라고 물으면 분류
- "몇 점을 받을까?"라고 물으면 회귀

즉, 모델보다 먼저 정해야 하는 것은 **질문의 형태**다.

<표 3-1: 분류와 회귀를 구분하는 가장 쉬운 기준>

| 질문 | 문제 유형 | 예시 |
| ---- | --------- | ---- |
| 어느 범주에 속하는가? | 분류 | 스팸/정상, 생존/사망, 이탈/유지 |
| 숫자가 얼마나 되는가? | 회귀 | 집값, 점수, 수요량, 매출 |

### 3.1.4 자주 헷갈리는 사례

아래 사례들은 강의에서 한 번 짚고 넘어가면 좋다.

<표 3-2: 자주 헷갈리는 문제 정의>

| 현상 | 질문 | 문제 유형 |
| ---- | ---- | --------- |
| 고객 이탈 | 이탈 여부를 예측 | 분류 |
| 고객 이탈 | 이탈 시점을 예측 | 생존 분석 |
| 대출 심사 | 승인/거절 여부 | 분류 |
| 대출 심사 | 예상 연체 금액 | 회귀 |
| 의료 진단 | 양성/음성 여부 | 분류 |
| 의료 진단 | 검사 수치 추정 | 회귀 |

이 표가 중요한 이유는, 학생이 모델 이름을 외우기 전에 **문제를 어떤 언어로 쓰는가**부터 정리해야 하기 때문이다.

## 3.2 평가 지표: 정확도 하나로는 부족하다

예측 문제를 정했다면, 다음에는 모델이 잘 작동하는지 평가해야 한다. 그런데 "잘 맞는다"는 말은 생각보다 복잡하다.

같은 모델이라도,

- 정확도가 높다고 좋은 모델일 수도 있고
- 전혀 쓸모없는 모델일 수도 있다

그 차이를 만드는 것이 **평가 지표**다.

### 3.2.1 혼동행렬: 분류 평가의 출발점

분류 문제에서는 먼저 혼동행렬을 읽을 수 있어야 한다. 혼동행렬은 예측 결과를 네 칸으로 나눈 표다.

![혼동행렬](../diagram/3-1.png)

<표 3-3: 혼동행렬을 읽는 방법>

| 구분 | 뜻 | 해석 |
| ---- | --- | ---- |
| TP | 양성을 양성으로 맞힘 | 찾아야 할 것을 제대로 찾음 |
| TN | 음성을 음성으로 맞힘 | 정상은 정상으로 유지 |
| FP | 음성을 양성으로 잘못 판단 | 거짓 경보 |
| FN | 양성을 음성으로 놓침 | 놓침 |

이 네 칸만 이해하면 주요 지표 대부분을 읽을 수 있다.

### 3.2.2 정확도는 왜 자주 사람을 속일까

정확도는 `(TP + TN) / 전체` 이다. 가장 직관적이지만, 클래스 불균형이 심할 때 큰 함정이 있다.

예를 들어 1000건의 거래 중 사기 거래가 5건뿐이라고 하자. 모든 거래를 정상이라고만 예측해도 정확도는 99.5%가 된다. 숫자만 보면 매우 뛰어난 모델처럼 보이지만, 실제로는 사기를 하나도 잡지 못했다.

즉, 정확도는 전체 평균 점수일 뿐이다.  
전체 평균은 좋아 보여도, 정작 중요한 소수 사례를 놓치고 있을 수 있다.

### 3.2.3 정밀도와 재현율은 서로 다른 실패를 보여 준다

정확도의 한계를 보완하는 대표 지표가 정밀도와 재현율이다.

- **정밀도(Precision)**: 양성이라고 예측한 것 중 실제 양성 비율
- **재현율(Recall)**: 실제 양성 중 양성으로 찾아낸 비율

이 둘은 비슷해 보이지만 역할이 다르다.

정밀도는 "양성이라고 말한 판단이 얼마나 믿을 만한가"를 보여 준다.  
재현율은 "놓치지 않고 얼마나 잘 찾아냈는가"를 보여 준다.

비유하면,

- 정밀도는 체포한 사람이 실제 범인일 확률
- 재현율은 전체 범인 중 실제로 잡아낸 비율

### 3.2.4 짧은 숫자 예시로 읽기

아래 가상의 사기 탐지 결과를 보자.

<표 3-4: 가상의 사기 탐지 결과>

| 항목 | 값 |
| ---- | -- |
| TP | 8 |
| FP | 12 |
| TN | 975 |
| FN | 5 |

이 경우:

- 정확도는 높다
- 하지만 사기를 5건 놓쳤다
- 정상 거래 12건도 괜히 막았다

따라서 이 모델을 좋다고 말하려면 단순히 "정확도가 높다"가 아니라,  
"사기 13건 중 8건을 잡았고, 대신 정상 거래 12건을 불편하게 만들었다"라고 읽어야 한다.

이런 식의 해석이 실제 업무와 연결된다.

### 3.2.5 F1 점수와 ROC-AUC

정밀도와 재현율을 동시에 보고 싶을 때 F1 점수를 쓴다. F1은 둘의 조화 평균이므로, 어느 한쪽만 높고 다른 쪽이 낮으면 값이 크게 좋아지지 않는다.

ROC-AUC는 임계값 전체에서 모델이 양성과 음성을 얼마나 잘 구분하는지를 보는 지표다. 모델의 "순위 매기기 능력"을 평가할 때 유용하다.

![ROC 곡선과 PR 곡선 비교](../diagram/3-2.png)

ROC-AUC가 중요한 이유는 임계값 하나에 묶이지 않기 때문이다. 다만 불균형 데이터에서는 PR-AUC나 정밀도/재현율 해석이 더 직접적일 수 있다.

### 3.2.6 어떤 지표를 우선할 것인가

정답은 하나가 아니다. 도메인에 따라 달라진다.

<표 3-5: 상황별로 더 중요하게 보는 지표>

| 상황 | 더 중요한 것 | 이유 |
| ---- | ------------ | ---- |
| 암 진단 | 재현율 | 놓치면 치명적이기 때문 |
| 스팸 필터 | 정밀도 | 정상 메일을 막으면 불편이 크기 때문 |
| 사기 탐지 | 균형 판단 | 거짓 경보와 놓침 모두 비용이 큼 |
| 추천 시스템 | 정밀도 | 엉뚱한 추천이 반복되면 경험이 나빠짐 |
| 설비 고장 예측 | 재현율 | 고장 놓침이 큰 손실로 이어짐 |

이 표의 핵심은 지표가 수학 공식이 아니라 **운영 비용의 언어**라는 점이다.

## 3.3 의사결정나무: 질문을 이어 가며 판단하는 모델

의사결정나무는 예측 모델 중에서도 가장 설명하기 쉬운 편에 속한다. 이유는 모델이 판단하는 흐름을 눈으로 따라갈 수 있기 때문이다.

예를 들어 대출 심사에서 아래와 같은 질문을 순서대로 한다고 생각해 보자.

- 소득이 일정 수준 이상인가?
- 신용점수가 기준 이상인가?
- 연체 이력이 있는가?

이런 질문을 차례대로 던져 마지막 판단에 도달하는 구조가 바로 의사결정나무다.

### 3.3.1 분할의 직관

트리는 데이터를 반복적으로 나누어 더 비슷한 집단을 만든다.

1. 가장 구분력이 큰 질문을 먼저 고른다
2. 나뉜 집단 안에서 다시 중요한 질문을 고른다
3. 더 이상 나눌 필요가 없을 때 멈춘다

이 과정을 통해 처음에는 섞여 있던 집단이 점점 더 순수한 집단으로 바뀐다.

![의사결정나무 대출 승인 예시](../diagram/3-3.png)

### 3.3.2 불순도와 정보 이득

트리가 어떤 질문을 먼저 할지 결정할 때 사용하는 개념이 불순도다. 불순도는 한 노드 안에 클래스가 얼마나 섞여 있는지를 뜻한다.

- 모두 같은 클래스면 불순도 0에 가까움
- 여러 클래스가 뒤섞여 있으면 불순도가 큼

트리는 분할 뒤 불순도가 가장 많이 줄어드는 질문을 선택한다. 이 감소량을 보통 정보 이득이라고 설명한다.

쉽게 말하면,  
"이 질문을 던졌을 때 집단이 얼마나 더 깔끔하게 갈라지는가?"를 계산하는 것이다.

### 3.3.3 의사결정나무의 장점과 한계

<표 3-6: 의사결정나무의 장점과 한계>

| 항목 | 내용 |
| ---- | ---- |
| 장점 1 | 질문 흐름을 시각화할 수 있어 해석이 쉽다 |
| 장점 2 | 비선형 관계를 어느 정도 직관적으로 처리한다 |
| 장점 3 | 수치형, 범주형 문제에 비교적 유연하다 |
| 한계 1 | 데이터가 조금만 바뀌어도 구조가 크게 달라질 수 있다 |
| 한계 2 | 깊어지면 과적합되기 쉽다 |
| 한계 3 | 복잡한 경계를 효율적으로 표현하지 못할 수 있다 |

따라서 의사결정나무는 "항상 최고의 모델"이라기보다, **예측 구조를 설명하는 출발점**으로 특히 강하다.

### 3.3.4 타이타닉 생존 예측 예시

[practice/chapter03/code/3-3-titanic-decision-tree.py](/Users/callii/Documents/dataScience/practice/chapter03/code/3-3-titanic-decision-tree.py) 는 타이타닉 승객의 생존 여부를 의사결정나무로 예측하는 예시다.

사용되는 핵심 특성은 다음 네 가지다.

- `Sex`
- `3rd_class`
- `Age`
- `1st_class`

실습 코드의 핵심은 아래와 같다.

```python
X = df[['Sex', '3rd_class', 'Age', '1st_class']]
y = df['Survived']

dt_model = DecisionTreeClassifier(max_depth=4, random_state=42)
dt_model.fit(X_train, y_train)
```

여기서 중요한 것은 세 가지다.

1. 입력 특성이 많지 않다
2. `max_depth=4`로 트리 깊이를 제한했다
3. 학습 뒤 정확도뿐 아니라 혼동행렬과 특성 중요도를 함께 본다

### 3.3.5 결과를 어떻게 읽을 것인가

이 실습에서 먼저 봐야 할 것은 **첫 번째 분할 기준**이다. 만약 성별이 첫 분할 기준으로 선택되었다면, 이는 생존 여부를 구분하는 데 성별이 가장 큰 설명력을 가졌다는 뜻이다.

그다음에는 같은 성별 안에서 객실 등급과 나이가 어떤 추가 구분을 만드는지 본다.

즉, 트리 해석은 아래 순서로 진행하면 좋다.

1. 첫 분할 기준이 무엇인가
2. 그 기준이 역사적/도메인 맥락과 맞는가
3. 하위 분기에서 어떤 조건이 추가되는가
4. 트리 깊이를 더 늘렸을 때 과적합 위험이 커질까

타이타닉 사례에서는 흔히 다음과 같은 해석이 가능하다.

- 성별이 큰 영향을 미침
- 객실 등급이 구조 가능성과 연결됨
- 나이가 세부적인 구분을 형성함

이처럼 의사결정나무는 정답을 말하는 기계라기보다, **판단 구조를 지도처럼 보여 주는 모델**이다.

## 3.4 로지스틱 회귀와 SVM: 오래된 모델이 여전히 중요한 이유

최신 딥러닝 모델이 많다고 해서 오래된 모델이 가치 없어진 것은 아니다. 오히려 분류 문제의 기초를 이해하려면 로지스틱 회귀와 SVM을 함께 보는 것이 좋다.

### 3.4.1 로지스틱 회귀

로지스틱 회귀는 입력 특성을 선형 결합한 뒤, 그 결과를 0과 1 사이 확률로 변환하는 모델이다.

![로지스틱 회귀 흐름도](../diagram/3-4-logistic.png)

이 모델의 장점은 명확하다.

- 빠르다
- 해석이 쉽다
- 적은 데이터에서도 비교적 안정적이다

그래서 규제가 강한 금융, 의료, 정책 분석 같은 영역에서는 여전히 자주 사용된다.

직관적으로 보면 로지스틱 회귀는 "점수를 계산한 뒤 합격 확률을 내는 모델"에 가깝다.

### 3.4.2 SVM

SVM은 두 집단을 가장 여유 있게 가르는 경계를 찾는 모델이다.

![SVM 마진 최대화](../diagram/3-4-svm.png)

비유하면, 두 집단이 지나가는 통로 사이에 가능한 한 넓은 안전거리를 확보하는 방식이다. 이 안전거리를 마진이라고 부른다.

커널을 사용하면 비선형 경계도 다룰 수 있다.

![커널 트릭](../diagram/3-4-kernel.png)

다만 SVM은 데이터가 아주 커질수록 계산 부담이 커질 수 있고, 파라미터 설정에 민감한 편이다.

### 3.4.3 세 모델의 역할 비교

<표 3-7: 대표 분류 모델의 직관적 비교>

| 모델 | 직관 | 강점 | 주의점 |
| ---- | ---- | ---- | ------ |
| 로지스틱 회귀 | 점수를 계산해 확률로 변환 | 빠름, 해석 쉬움 | 복잡한 비선형성에 약함 |
| 의사결정나무 | 질문을 순서대로 던짐 | 설명 가능성 높음 | 과적합 가능성 큼 |
| SVM | 두 집단 사이 마진 최대화 | 고차원 분류에 강함 | 큰 데이터에서 부담 |

<표 3-8: 간단한 선택 기준>

| 상황 | 먼저 떠올릴 모델 |
| ---- | ---------------- |
| 설명 가능성이 중요함 | 로지스틱 회귀, 의사결정나무 |
| 적은 데이터로 빠른 기준선 필요 | 로지스틱 회귀 |
| 규칙 형태의 설명이 중요함 | 의사결정나무 |
| 복잡한 경계가 필요하지만 데이터는 아주 크지 않음 | SVM |

핵심은 하나다.  
좋은 모델은 최신 모델이 아니라 **현재 문제를 가장 잘 설명하고 가장 안정적으로 해결하는 모델**이다.

## 3.5 이상 탐지: 정상과 다른 신호를 어떻게 찾을 것인가

앞에서는 정답이 있는 분류 문제를 다루었다. 이제는 훨씬 더 까다로운 문제로 넘어간다. 바로 이상 탐지다.

이상 탐지는 "정상과 다른 것"을 찾는 문제인데, 일반 분류와는 중요한 차이가 있다.

### 3.5.1 왜 이상 탐지는 어려운가

이상 탐지는 대체로 다음 특징을 가진다.

1. 이상 사례가 매우 드물다
2. 라벨이 없거나 불완전한 경우가 많다
3. FP와 FN의 비용이 크게 다를 수 있다

예를 들어 신용카드 사기를 생각해 보자.

- 정상 거래를 사기로 잘못 잡으면 고객이 불편하다
- 사기 거래를 놓치면 직접 손실이 생긴다

즉, 이상 탐지는 단순 정확도보다 **운영상 허용 가능한 균형**이 더 중요하다.

### 3.5.2 대표 알고리즘의 직관

이 장에서는 대표적으로 네 가지 접근을 비교한다.

- Isolation Forest
- One-class SVM
- Local Outlier Factor
- Autoencoder

<표 3-9: 이상 탐지 알고리즘 비교>

| 방법 | 한 줄 설명 | 직관 |
| ---- | --------- | ---- |
| Isolation Forest | 빨리 고립되는 점 탐지 | 혼자 튀어 있는 점 찾기 |
| One-class SVM | 정상 영역 밖 점 탐지 | 안전 구역 바깥 점 찾기 |
| LOF | 주변보다 밀도가 낮은 점 탐지 | 주변보다 듬성한 점 찾기 |
| Autoencoder | 복원이 잘 안 되는 점 탐지 | 익숙하지 않은 패턴 찾기 |

이 표에서 보듯이, 네 방법은 모두 "이상"을 찾지만 이상을 정의하는 방식은 서로 다르다.

### 3.5.3 신용카드 사기 탐지 비교 예시

[practice/chapter03/code/3-5-anomaly-detection.py](/Users/callii/Documents/dataScience/practice/chapter03/code/3-5-anomaly-detection.py) 는 네 가지 방법을 한 번에 비교하는 예시다.

실습 코드에서 Autoencoder 부분을 보면, 정상 데이터만 따로 학습해 정상 패턴을 익힌 뒤 복원 오차가 큰 거래를 이상치로 판단한다.

```python
X_normal_scaled = X_scaled[y == 1]
ae_model = train_autoencoder(X_normal_scaled, epochs=50, lr=0.01)
y_pred_ae = predict_autoencoder(ae_model, X_scaled, contamination)
```

이 짧은 코드만으로도 각 알고리즘의 철학 차이를 읽을 수 있다.

- Isolation Forest는 분할을 반복해 고립 속도를 본다
- One-class SVM은 정상 경계를 학습한다
- LOF는 주변 밀도를 본다
- Autoencoder는 복원 실패를 본다

### 3.5.4 결과를 해석하는 순서

이 실습 결과는 다음 순서로 읽는 것이 좋다.

1. 사기 비율이 얼마나 낮은가
2. 정확도보다 정밀도, 재현율, F1이 어떤가
3. 속도 차이가 얼마나 나는가
4. 어떤 알고리즘이 어떤 유형의 이상을 잘 잡는가

예를 들어 F1이 가장 높더라도 실행 시간이 지나치게 길거나 운영이 복잡하면 실제 현장에서는 덜 적합할 수 있다.

따라서 이상 탐지는 "가장 정확한 모델을 고르는 문제"가 아니라, **놓침과 거짓 경보의 비용을 감안해 가장 적절한 운영 모델을 고르는 문제**에 가깝다.

### 3.5.5 확인 질문

이 장의 이상 탐지 부분을 읽은 뒤에는 다음 질문에 답할 수 있어야 한다.

1. 라벨이 거의 없는 환경에서는 어떤 접근이 더 현실적인가
2. FP를 줄이는 것이 중요한가, FN을 줄이는 것이 중요한가
3. 속도가 빠른 방법이 실제 운영에서 더 유리한가
4. 딥러닝 기반 방법이 항상 전통 방법보다 좋은가

## 3.6 AutoML: 기준선을 빠르게 만드는 도구

AutoML은 여러 모델과 하이퍼파라미터를 자동으로 탐색해 빠르게 기준선을 만드는 도구다. 여기서 중요한 것은 AutoML이 분석을 대신해 주는 것이 아니라, **초기 탐색 시간을 줄여 준다**는 점이다.

### 3.6.1 AutoML이 잘하는 일

- 여러 후보 모델을 빠르게 비교
- 기본 전처리와 모델 탐색 자동화
- 기준선 성능을 짧은 시간 안에 확보

### 3.6.2 AutoML이 대신하지 못하는 일

- 문제 정의
- 데이터 품질 점검
- 데이터 누수 확인
- 어떤 지표를 우선할지 결정
- 결과를 도메인 맥락에서 해석

즉, AutoML은 "대신 생각해 주는 도구"가 아니라 "더 중요한 생각에 시간을 쓰게 해 주는 도구"다.

현재 저장소에는 [3-6-automl-pipeline.html](/Users/callii/Documents/dataScience/content/graphics/ch03/3-6-automl-pipeline.html) 자료가 있어, AutoML이 전체 ML 파이프라인 안에서 어디에 위치하는지 시각적으로 참고할 수 있다.
