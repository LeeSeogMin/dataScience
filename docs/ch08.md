# 제8장: 생성 모델 기반 데이터 분석: 합성 데이터, 불확실성

본 장에서는 생성 모델(Generative Model)이 데이터 분석에서 어떤 역할을 하는지 살펴본다. 생성 모델은 데이터의 분포 자체를 학습하여 개인정보 보호를 위한 합성 데이터 생성, 데이터 증강을 통한 불균형 해소, 예측의 불확실성 정량화 등 다양한 실무 문제를 해결하는 도구로 활용된다. 구체적으로 CTGAN을 통한 정형 데이터 합성, VAE를 이용한 데이터 증강과 이상치 탐지, Diffusion Model을 통한 예측 불확실성 정량화 방법을 다룬다.

## 8.1 생성 모델이 데이터 분석에 기여하는 방식

생성 모델(Generative Model)은 데이터의 분포 자체를 학습하여, 기존 데이터로부터 새로운 데이터를 만들어내는 능력을 갖춘 모델이다. 전통적인 판별 모델(Discriminative Model)이 입력 X가 주어졌을 때 출력 Y를 예측하는 조건부 확률 p(Y|X)를 학습하는 데 비해, 생성 모델은 데이터의 결합 확률 p(X, Y) 또는 주변 확률 p(X)를 학습한다. 이러한 특성 덕분에 생성 모델은 관측되지 않은 데이터를 생성하거나, 데이터의 구조를 탐색하거나, 미래의 불확실성을 정량화하는 데 활용될 수 있다.

본 장에서는 생성 모델이 데이터 분석에서 실질적으로 기여하는 세 가지 방식을 다룬다. 첫째, **개인정보 보호를 위한 합성 데이터 생성**이다. 의료, 금융, 사회복지 데이터는 민감한 개인정보를 포함하고 있어 연구나 분석 목적으로 공유하기 어렵다. 생성 모델은 원본 데이터의 통계적 특성을 보존하면서도 개별 레코드는 완전히 새로 생성하는 합성 데이터를 만들어낸다. 8.2절에서는 정형 데이터에 특화된 CTGAN의 원리와 활용법을 다룬다.

둘째, **데이터 증강과 이상치 탐지를 통한 모델 성능 개선**이다. 제조업의 불량 데이터나 금융권의 사기 거래처럼, 중요한 사건은 매우 희귀하게 발생한다. VAE는 소수 클래스의 분포를 학습하여 다양한 변형 데이터를 생성함으로써 불균형 문제를 완화한다. 또한 VAE의 재구성 오차(Reconstruction Error)는 정상 데이터와 이상 데이터를 구별하는 지표로 활용될 수 있다. 8.3절에서는 VAE를 이용한 데이터 증강과 이상치 탐지 방법을 다룬다.

셋째, **예측의 불확실성 정량화**다. 전통적인 회귀 모델은 점 추정(Point Estimate)만을 제공하며, 불확실성은 표준오차로 간접적으로 표현된다. 그러나 현실의 불확실성은 정규분포를 따르지 않는 경우가 많다. Diffusion Model은 순수 노이즈에서 시작하여 점진적으로 데이터를 복원하는 과정에서 무한히 많은 가능한 경로를 샘플링할 수 있다. 이를 통해 비대칭적이거나 다봉(multimodal)인 불확실성을 자연스럽게 표현할 수 있다. 8.4절에서는 Diffusion Model을 이용한 예측 불확실성 정량화 방법을 다룬다.

대표적인 생성 모델로는 VAE, GAN, Diffusion Model이 있다. VAE는 잠재공간(Latent Space)을 활용한 구조화된 표현 학습에 강점이 있으며, 데이터 증강과 이상치 탐지에 적합하다. GAN은 고품질 합성 데이터 생성에 탁월하나 학습 불안정성(Mode Collapse)이 단점이다. CTGAN은 GAN의 한계를 극복하고 정형 데이터 합성에 최적화된 모델이다. Diffusion Model은 2020년대 초반부터 급부상한 방법으로, 점진적 노이즈 제거 과정을 통해 다양성과 안정성을 확보한다.


## 8.2 합성 데이터 생성: CTGAN과 정형 데이터

### 8.2.1 GAN의 기본 원리와 CTGAN

생성적 적대 신경망(Generative Adversarial Network, GAN)은 Goodfellow et al. (2014)가 제안한 혁신적인 생성 모델 프레임워크다. GAN의 핵심 아이디어는 두 개의 신경망이 서로 경쟁하며 학습하는 적대적(adversarial) 구조에 있다. Generator(생성자)는 무작위 노이즈로부터 가짜 데이터를 생성하고, Discriminator(판별자)는 실제 데이터와 생성된 가짜 데이터를 구별하려 한다. 이 과정을 게임 이론의 관점에서 보면, Generator는 Discriminator를 속이려 하고, Discriminator는 속지 않으려 하는 미니맥스 게임(minimax game)으로 해석된다.

수학적으로 GAN의 목적 함수는 다음과 같다.

min_G max_D V(D, G) = E[log D(x)] + E[log(1 - D(G(z)))]

여기서 첫 번째 항은 Discriminator가 실제 데이터를 실제로 판별하는 능력을, 두 번째 항은 가짜 데이터를 가짜로 판별하는 능력을 나타낸다. 학습이 진행됨에 따라 Generator는 점점 더 실제 같은 데이터를 생성하게 되고, 이상적인 경우 Discriminator는 실제와 가짜를 구별하지 못해 50%의 정확도에 수렴하게 된다. 이 시점에서 Generator가 생성하는 데이터 분포는 실제 데이터 분포와 일치하게 된다.

그러나 기본 GAN은 테이블 형태의 정형 데이터에 적용하기 어려운 세 가지 한계가 있다. 첫째, 연속형 변수와 범주형 변수가 혼재된 데이터를 처리하기 어렵다. 둘째, 변수별로 분포 형태가 다양하여(정규, 다봉, 편포 등) 단일 정규화 방식이 적합하지 않다. 셋째, 범주형 변수의 클래스 불균형 문제가 심각하다. CTGAN은 이러한 한계를 극복하기 위해 설계되었다.

**CTGAN의 핵심 기술**: CTGAN은 Xu et al. (2019)이 제안한 정형 데이터 전용 생성 모델이다. 기존 GAN이 이미지 생성에서 뛰어난 성능을 보인 반면 테이블 데이터에는 적합하지 않았던 문제를 해결하기 위해 세 가지 핵심 기술을 도입한 것이다.

첫째, Mode-specific Normalization(모드별 정규화)이다. 연속형 변수의 분포가 다봉(multimodal)인 경우, 단순 정규화는 각 모드의 특성을 왜곡한다. CTGAN은 각 연속형 변수에 대해 변분 가우시안 혼합 모델(Variational Gaussian Mixture Model)을 적합하여 여러 모드를 식별하고, 각 모드별로 정규화를 수행한다. 이를 통해 복잡한 분포 구조를 보존하면서 학습이 가능해진다.

둘째, Conditional Vector(조건 벡터)를 활용한 조건부 생성이다. 범주형 변수의 빈도가 불균형한 경우(예: 사기 거래 1%, 정상 거래 99%), 단순 GAN은 다수 클래스에 편향된 샘플을 생성하는 경향이 있다. CTGAN은 학습 시 각 범주를 균등하게 샘플링하고, Generator에 해당 범주 조건을 명시적으로 입력하여 소수 클래스도 충분히 학습되도록 한다.

셋째, Training-by-Sampling(샘플링 기반 훈련) 전략이다. 각 학습 반복에서 특정 범주 조건을 균등하게 샘플링하여 Generator를 학습시킴으로써, 희귀한 조합의 데이터도 생성할 수 있게 된다. 이는 정책 분석에서 소수 취약계층의 데이터를 생성하는 데 특히 유용하다.

**Wasserstein Loss(바서슈타인 손실)와 훈련 안정성**: 기본 GAN의 학습은 불안정하기로 악명 높다. Generator와 Discriminator의 균형이 무너지면 mode collapse(생성 다양성 붕괴)나 훈련 발산이 발생한다. CTGAN은 이 문제를 해결하기 위해 Wasserstein Loss(또는 Earth Mover's Distance)를 채택한다. Wasserstein Distance는 두 확률 분포 사이의 거리를 측정하는 지표로, 직관적으로 "한 분포를 다른 분포로 변환하는 데 필요한 최소 비용"으로 이해할 수 있다.

비유하자면 "흙 옮기기" 문제와 같다. 산 모양의 흙더미(실제 분포)를 골짜기 모양(생성 분포)으로 바꾸려면 얼마나 많은 흙을 얼마나 멀리 옮겨야 할까? 이 총 이동 비용이 바로 Wasserstein Distance다. 두 분포가 비슷할수록 흙을 적게, 가깝게 옮기면 되므로 거리가 작아진다. 수학적으로는 다음과 같이 정의된다.

W(Pᵣ, Pᵍ) = inf_{γ∈Π(Pᵣ,Pᵍ)} E_{(x,y)~γ}[||x - y||]

여기서 Pᵣ은 실제 데이터 분포, Pᵍ는 생성 데이터 분포이며, Π(Pᵣ,Pᵍ)는 두 분포를 주변 분포로 갖는 모든 결합분포(커플링) γ의 집합이다.

왜 이것이 중요한가? 기본 GAN은 JS Divergence를 직접 최소화하는 형태로 이해할 수 있는데, 실제 학습에서는 두 분포가 충분히 겹치지 않는 초기 구간에서 Discriminator가 너무 쉽게 구분해 버려 Generator 쪽으로 전달되는 기울기(gradient)가 약해지는 문제가 자주 발생한다. 마치 산과 골짜기가 멀리 떨어져 있을 때 "어느 방향으로 흙을 옮겨야 하는지" 신호가 흐려지는 것과 같다. 반면 Wasserstein Distance는 분포 간 거리를 연속적으로 측정하므로, "저쪽으로 조금만 이동하면 더 가까워진다"는 식의 더 안정적인 학습 신호를 제공한다. 이것이 WGAN 계열이 학습 안정성에서 장점이 있는 이유다.

WGAN-GP(Wasserstein GAN with Gradient Penalty)는 Lipschitz 제약을 gradient penalty로 구현하여 추가적인 안정성을 확보한다.

L = E[D(G(z))] - E[D(x)] + λ · E[(||∇D(x̂)||₂ - 1)²]

첫 두 항은 Critic(=Discriminator)가 "가짜 점수 - 진짜 점수"를 최소화하도록 만드는 Wasserstein 목적이며, 세 번째 항은 1-Lipschitz 조건을 근사하기 위한 gradient penalty다. Generator는 별도로 -E[D(G(z))]를 최소화하여, Critic이 높게 점수 매기는 샘플을 생성하도록 학습한다. λ는 보통 10으로 설정하며, gradient penalty는 Critic의 출력이 입력에 대해 과도하게 민감해지지 않도록 제약해 학습을 안정화한다.

### 8.2.2 CTGAN 하이퍼파라미터와 품질 평가

**하이퍼파라미터 튜닝 가이드**: CTGAN의 성능은 하이퍼파라미터 설정에 민감하다. 다음은 실무에서 권장하는 튜닝 가이드다.

<표 8-1> CTGAN 주요 하이퍼파라미터

| 파라미터            | 기본값    | 권장 범위           | 설명                                    |
| ------------------- | --------- | ------------------- | --------------------------------------- |
| epochs              | 300       | 100-500             | 학습 반복 횟수, 데이터 크기에 비례      |
| batch_size          | 500       | 256-1000            | 큰 배치가 안정적, GPU 메모리 고려       |
| generator_dim       | (256,256) | (128,128)-(512,512) | Generator 은닉층 크기                   |
| discriminator_dim   | (256,256) | (128,128)-(512,512) | Discriminator 은닉층 크기               |
| generator_lr        | 2e-4      | 1e-4 - 5e-4         | Generator 학습률                        |
| discriminator_lr    | 2e-4      | 1e-4 - 5e-4         | Discriminator 학습률                    |
| discriminator_steps | 1         | 1-5                 | Generator 1회당 Discriminator 학습 횟수 |
| pac                 | 10        | 1-20                | PacGAN 패킹 수                          |

데이터 크기별 권장 설정은 다음과 같다. 1,000건 미만일 때는 epochs=500, batch_size=100, generator_dim=(128,128)을 사용한다. 1,000-10,000건 규모에서는 epochs=300, batch_size=500, generator_dim=(256,256)이 적합하다. 10,000건 이상의 대규모 데이터에서는 epochs=150, batch_size=1000, generator_dim=(512,512)로 설정한다.

변수 특성별로는 범주형 변수가 전체의 50%를 초과하는 경우 pac를 20으로 증가시키고, 연속형 변수가 다봉분포를 보이면 epochs를 50% 증가시킨다. 클래스 불균형이 1:99 이상으로 심한 경우에는 epochs를 2배로 늘려야 소수 클래스의 패턴을 충분히 학습할 수 있다.

**합성 데이터 품질 평가 프레임워크**: 합성 데이터의 품질은 크게 세 가지 차원에서 평가해야 한다.

첫째, 통계적 유사도(Statistical Fidelity)이다. 합성 데이터가 원본 데이터의 통계적 특성을 얼마나 잘 재현하는지를 측정한다. 평균, 분산, 상관계수 같은 1차·2차 통계량의 일치 여부, 변수 간 상관구조의 보존, 분포 형태의 유사성 등이 평가 대상이다. Wasserstein Distance나 Kolmogorov-Smirnov 검정을 통해 정량화할 수 있다.

둘째, 머신러닝 효용성(ML Utility)이다. 합성 데이터가 실제 분석 업무에서 원본 데이터를 대체할 수 있는지를 검증한다. TSTR(Train on Synthetic, Test on Real) 프로토콜이 대표적이다. 합성 데이터로 머신러닝 모델을 학습한 뒤 실제 데이터로 테스트하여, 실제 데이터로 학습한 모델(TRTR)과 성능을 비교한다. TSTR 비율은 (TSTR 정확도 / TRTR 정확도) × 100으로 계산하며, 이 값이 90% 이상이면 실무에서 대체 사용이 가능하다고 판단한다.

셋째, 개인정보 보호 수준(Privacy Preservation)이다. 합성 데이터로부터 원본 개인을 역추적할 수 없어야 한다 (Jordon et al., 2018). DCR(Distance to Closest Record)은 각 합성 레코드와 가장 가까운 원본 레코드 간의 거리를 측정한다. 이 거리가 충분히 크면 역추적이 어렵다. MIA(Membership Inference Attack)는 공격자가 특정 레코드가 학습 데이터에 포함되었는지 추론할 수 있는지를 테스트한다. 성공률이 무작위 추측(50%)에 가까우면 안전하다고 평가한다.

| ML 효용성     | TSTR 비율                 | ≥ 90%     |
| 개인정보 보호 | DCR > 5th percentile 비율 | ≥ 90%     |
| 개인정보 보호 | MIA 성공률                | ≤ 55%     |

**정형 데이터 생성 모델 비교**: CTGAN 외에도 정형 데이터 합성을 위한 다양한 모델이 제안되어 있다. ARF(Adversarial Random Forest)는 랜덤 포레스트 기반의 생성 모델로, 트리 구조를 활용하여 범주형 변수를 자연스럽게 처리한다. TT-GAN(Tabular Transformer GAN)은 트랜스포머 아키텍처를 도입하여 변수 간 장거리 의존성을 포착한다.

<표 8-3> 정형 데이터 생성 모델 비교

| 모델   | 아키텍처      | 범주형 처리   | 훈련 난이도 | 적합한 상황            |
| ------ | ------------- | ------------- | ----------- | ---------------------- |
| CTGAN  | GAN + VGM     | Mode-specific | 중간        | 다봉 분포, 불균형 범주 |
| ARF    | Random Forest | 자연스러움    | 낮음        | 트리 기반 모델 호환    |
| TT-GAN | Transformer   | 임베딩        | 높음        | 장거리 의존성          |
| TVAE   | VAE           | 유사 CTGAN    | 중간        | 잠재공간 활용 필요 시  |

실무에서는 데이터 특성과 목적에 따라 적합한 모델을 선택해야 한다. 빠른 프로토타이핑이 필요하면 ARF가, 복잡한 분포 구조가 있으면 CTGAN이, 변수 간 복잡한 상호작용이 있으면 TT-GAN이 적합하다.

### 8.2.3 실습: GAN으로 합성 데이터 생성

**CTGAN 선택 실습**: CTGAN 실습은 외부 패키지(`ctgan`) 설치가 필요한 선택 실습이다. 이 장의 필수 실습은 본 항 후반의 "간단한 GAN 실습"으로 구성되어 있으며, CTGAN은 원리와 사용 관점 중심으로 다룬다. 다만 CTGAN을 실제로 실행해 보고 싶다면, 아래처럼 학습과 샘플링 흐름을 구성할 수 있다.

```python
from ctgan import CTGAN
ctgan = CTGAN(epochs=300, verbose=True)
ctgan.fit(df, discrete_columns=discrete_columns)
synthetic = ctgan.sample(len(df))
```

CTGAN을 사용할 때는 생성 자체보다 평가 설계를 먼저 정하는 편이 안전하다. 통계적 유사도(분포·상관 구조), ML 효용성(TSTR), 프라이버시(원본 근접도, 멤버십 추론 가능성)를 함께 보고, 목적에 맞게 기준을 세운 뒤 모델을 튜닝해야 한다.

이때 "지표를 계산했다"는 사실보다 더 중요한 것은, 어떤 지표를 어떤 의사결정에 연결할지 미리 정해 두는 일이다. 합성 데이터는 목적이 바뀌면 기준도 바뀐다. 예를 들어 외부 공유를 위한 합성이라면, downstream 성능이 다소 떨어지더라도 프라이버시 위험을 더 엄격히 통제해야 한다. 반대로 내부 모델 개발의 데이터 증강이 목적이라면, 프라이버시보다 성능과 일반화에 초점을 맞추는 편이 합리적일 수 있다.

또한 합성 데이터 평가는 "전체 평균"만으로 끝나기 어렵다. 정책 대상, 취약 계층, 극단값 영역처럼 의사결정에 중요한 구간이 따로 있는 경우가 많기 때문이다. 따라서 전체 분포가 비슷해 보여도, 중요한 구간에서의 왜곡이 있으면 실제 업무에서는 사용이 어려워진다. 이 문제는 정형 데이터에서 특히 심각하며, 상관 구조와 조합의 현실성을 별도로 점검해야 한다.

## 8.3 VAE: 데이터 증강과 탐색 도구로서의 활용

VAE(Variational Autoencoder) (Kingma & Welling, 2014)는 고차원 데이터를 저차원 잠재공간(Latent Space)으로 압축하여 학습하는 생성 모델이다. 본 절에서는 VAE가 데이터 분석에서 실질적으로 기여하는 세 가지 영역인 **데이터 증강, 이상치 탐지, 초기 탐색**을 다룬다.

### 8.3.1 데이터 증강: 소수 클래스 생성

실무에서 클래스 불균형은 매우 흔한 문제다. 금융 사기 거래는 전체의 0.1% 미만이고, 희귀질환 환자는 전체의 1% 미만인 경우가 많다. 이런 경우 머신러닝 모델은 다수 클래스만 학습하고 소수 클래스를 무시하는 경향이 있다. SMOTE(Synthetic Minority Over-sampling Technique)는 소수 클래스 샘플 간 선형 보간으로 합성 샘플을 생성하는 전통적인 오버샘플링 기법이다. 그러나 단순 보간으로 샘플을 생성하므로, 고차원 데이터에서는 비현실적인 조합을 만들 위험이 있다. VAE는 잠재공간에서 데이터의 구조를 학습하기 때문에, 현실적인 합성 샘플을 생성할 수 있다. β-VAE (Higgins et al., 2017)는 잠재 표현의 해석 가능성을 향상시킨 VAE의 변형이다.

**시나리오 1 - 의료 영상 진단**: 병원이 폐질환 진단 모델을 훈련하는데, 정상 흉부 X-ray는 10,000장이지만 희귀질환(예: 폐섬유증)은 100장뿐이다. 이대로 학습하면 모델은 모든 입력을 "정상"으로 분류하며 99% 정확도를 달성하지만, 실제 희귀질환은 전혀 탐지하지 못한다. VAE를 활용한 데이터 증강은 다음과 같이 진행된다.

먼저 VAE를 전체 10,100장의 X-ray로 학습시킨다. 학습된 VAE는 정상과 질환의 잠재공간 분포를 학습하며, 희귀질환 샘플들은 잠재공간의 특정 영역에 모여 있게 된다. 다음으로 100장의 희귀질환 샘플을 인코딩하여 잠재벡터 z₁, z₂, ..., z₁₀₀을 얻는다. 이 잠재벡터들의 평균(μ)과 분산(σ)을 계산한 뒤, N(μ, σ)에서 1,000개의 새로운 잠재벡터를 샘플링한다. 마지막으로 이들을 디코딩하여 1,000장의 합성 희귀질환 X-ray를 생성한다.

이렇게 생성된 합성 데이터는 원본과 시각적으로 유사하면서도, 원본에 없던 변이를 포함한다. 예를 들어 원본에 "좌측 상엽 섬유화" 패턴만 있었다면, 합성 데이터는 "우측 중엽 섬유화"나 "양측 섬유화" 패턴도 포함할 수 있다. 이는 모델의 일반화 성능을 높이는 데 기여한다. 실무에서는 합성 데이터의 품질을 의료 전문가가 검토하여, 비현실적인 이미지(예: 해부학적으로 불가능한 구조)를 제거한 뒤 학습에 사용한다.

**시나리오 2 - 금융 사기 탐지**: 은행이 신용카드 사기 탐지 모델을 구축하는데, 정상 거래는 100만 건이지만 사기 거래는 1,000건뿐이다. SMOTE로 증강하면 "결제 금액 100만원, 거래 시각 03:00, 거래처 편의점"처럼 특징 조합이 현실적이지 않을 수 있다. VAE는 사기 거래의 잠재 패턴(심야 고액 결제, 해외 거래 직후 국내 거래 등)을 학습하여, 현실적인 합성 사기 거래를 생성한다.

1,000건의 사기 거래를 VAE로 학습시킨 뒤, 잠재공간에서 10,000건의 합성 사기 거래를 생성한다. 이 합성 데이터를 원본 정상 거래와 결합하여 학습하면, 모델은 소수 클래스를 무시하지 않고 사기 패턴을 학습한다. 실무에서는 합성 거래의 현실성을 전문가가 검증하며, FDS(Fraud Detection System)의 오탐률과 탐지율을 모니터링하여 합성 데이터의 비율을 조정한다.

핵심은 VAE 데이터 증강이 분류 모델 학습용으로 활용되며, 생성된 데이터가 실제 관측 데이터를 대체하는 것은 아니라는 점이다. 또한 잠재공간의 품질에 따라 합성 데이터의 현실성이 결정되므로, 최소 수백 샘플 이상의 충분한 원본 데이터가 필요하다.

### 8.3.2 이상치 탐지: 재구성 오차 활용

이상치 탐지는 레이블이 없거나 매우 적은 경우가 많다. 제조 공정에서 불량품은 극히 드물고, 네트워크 침입은 새로운 패턴이 계속 등장하여 미리 정의하기 어렵다. VAE는 정상 데이터의 분포를 학습한 뒤, 재구성이 잘 되지 않는 샘플을 이상치로 판단한다. 이는 "정상 데이터는 잠재공간에서 고밀도 영역에 위치하고, 이상치는 저밀도 영역에 위치한다"는 가정에 기반한다.

**시나리오 3 - 제조 공정 불량품 탐지**: 반도체 제조 공정에서 웨이퍼 이미지를 실시간으로 촬영하여 불량품을 탐지한다. 정상 웨이퍼는 하루 10,000장이지만, 불량품은 10장 미만이고 불량 유형도 매우 다양하다(스크래치, 오염, 균열 등). 전통적인 분류 모델은 불량 샘플이 부족하여 학습이 어렵다. VAE 이상치 탐지는 정상 샘플만 학습하여 이 문제를 해결한다.

먼저 정상 웨이퍼 10,000장으로 VAE를 학습시킨다. VAE는 정상 패턴(균일한 표면, 일정한 색상 분포 등)을 잠재공간에 인코딩한다. 다음으로 새로운 웨이퍼 이미지 x를 입력받아 인코딩-디코딩하여 재구성 이미지 x̂를 생성한다. 재구성 오차 ||x - x̂||²를 계산하며, 오차가 임계값(예: 95% 백분위수)을 초과하면 이상치로 판단한다.

재구성 오차가 왜 이상치를 탐지하는가? VAE는 정상 데이터의 패턴을 압축하여 잠재공간에 저장한다. 정상 웨이퍼는 잠재공간의 고밀도 영역에 위치하며, 재구성이 잘 된다. 그러나 스크래치가 있는 불량 웨이퍼는 학습 데이터에 없던 패턴이므로, 잠재공간의 저밀도 영역에 위치하고 재구성이 부정확하다. 예를 들어 스크래치를 제거하거나, 오염을 평활화하여 정상처럼 보이도록 복원하는데, 이 과정에서 재구성 오차가 크게 증가한다.

실무에서는 재구성 오차 분포를 시각화하여 임계값을 설정한다. 예를 들어 정상 웨이퍼의 재구성 오차 평균이 0.05, 표준편차가 0.01이라면, 평균+3σ = 0.08을 임계값으로 설정한다. 오차가 0.08을 초과하면 경고를 발생시켜 작업자가 육안으로 재검토한다.

**시나리오 4 - 네트워크 침입 탐지**: 기업 네트워크에서 정상 트래픽 패턴(로그인 시간, 접속 IP, 데이터 전송량 등)을 VAE로 학습한 뒤, 비정상적인 접속(심야 로그인, 비정상 국가 IP, 급격한 데이터 다운로드 등)을 탐지한다. 전통적인 룰 기반 시스템은 미리 정의된 공격만 탐지하지만, VAE는 학습하지 않은 새로운 공격 패턴도 재구성 오차로 감지할 수 있다.

예를 들어 직원 A의 평소 접속 패턴은 "09:00-18:00, 국내 IP, 일평균 전송량 100MB"인데, 어느 날 "03:00, 러시아 IP, 10GB 다운로드"가 발생했다면, VAE는 이 패턴을 재구성하지 못하고 높은 오차를 반환한다. 보안팀은 이를 침입 의심 신호로 간주하여 계정을 일시 차단하고 조사를 시작한다.

핵심은 VAE 이상치 탐지가 정상 데이터만 학습하여 이상치를 탐지하므로, 라벨이 없는 상황에 적합하다는 점이다. 그러나 재구성 오차는 모든 종류의 이상치를 동일하게 취급하므로, 이상치 유형을 분류하려면 후속 분석이 필요하다.

### 8.3.3 초기 탐색: 잠재공간 시각화를 통한 가설 생성

실무 데이터는 수백 개의 변수를 가지며, 변수 간 관계가 복잡하여 직관적으로 파악하기 어렵다. 차원 축소(PCA, t-SNE, UMAP)는 시각화에 유용하지만, 새로운 데이터를 생성할 수 없다. VAE는 차원 축소와 생성 모델을 결합하여, 잠재공간에서 패턴을 탐색하고 "만약 이 변수를 바꾸면?"이라는 What-if 시나리오를 탐색할 수 있다.

**시나리오 5 - 고객 세분화**: 전자상거래 기업이 100만 명의 고객 데이터(나이, 성별, 구매 이력, 방문 빈도, 평균 구매액 등 50개 변수)를 분석하여 마케팅 전략을 수립하려 한다. 50차원 데이터를 직접 시각화할 수 없으므로, VAE로 2차원 잠재공간에 투영한다.

VAE를 학습시킨 뒤, 모든 고객을 2차원 잠재공간에 플로팅하면 클러스터가 자연스럽게 형성된다. 예를 들어 클러스터 A는 "20-30대, 저가 제품 위주, 빈번한 방문"이고, 클러스터 B는 "40-50대, 고가 제품, 드문 방문"이며, 클러스터 C는 "30-40대, 중간 가격대, 정기 구매"로 해석된다. 이는 k-means로 미리 세그먼트 수를 정하지 않고도, 데이터가 자연스럽게 형성하는 패턴을 발견할 수 있다.

마케팅 팀은 이 클러스터를 기반으로 가설을 생성한다. "클러스터 A는 가격에 민감하므로 할인 쿠폰 제공", "클러스터 B는 품질을 중시하므로 프리미엄 제품 추천", "클러스터 C는 충성도가 높으므로 멤버십 혜택 강화" 등이다. 이후 A/B 테스트로 각 전략의 효과를 검증한다.

**시나리오 6 - 신약 개발**: 제약사가 수백만 개의 화합물 데이터(분자 구조, 용해도, 독성, 약효 등)를 분석하여 신약 후보를 찾는다. VAE를 학습시켜 화합물을 잠재공간에 인코딩한 뒤, "높은 약효 + 낮은 독성" 영역에 위치한 화합물을 탐색한다. 또한 기존 약물의 잠재벡터를 수정하여 "독성은 낮추고 약효는 유지"하는 신규 화합물 구조를 생성한다.

예를 들어 기존 약물 X의 잠재벡터가 z_X = [0.5, -0.3, 0.8]이고, 독성이 높다면, 독성과 상관이 높은 잠재 차원(예: 두 번째 차원)을 변경하여 z_new = [0.5, 0.0, 0.8]로 수정한다. 이를 디코딩하여 새로운 분자 구조를 생성하고, 실험실에서 합성하여 독성을 측정한다. 이는 무작위 탐색보다 효율적으로 신약 후보를 발견하는 데 기여한다.

핵심은 VAE 초기 탐색이 가설 생성용으로 활용되며, 발견된 패턴은 반드시 추가 분석으로 검증해야 한다는 점이다. 잠재공간에서 발견한 클러스터나 패턴은 상관관계를 나타낼 뿐이므로, A/B 테스트나 실험 등으로 실제 효과를 확인해야 한다.

<표 8-5> VAE 사용 권장 영역

| 용도 | 조건 | 검증 방법 | 주의사항 |
|------|------|-----------|----------|
| **데이터 증강** | 소수 클래스 샘플 부족 (최소 수백 개) | 합성 데이터의 현실성을 전문가가 검토 | 합성 데이터는 학습용이며 실제 관측 데이터를 대체하지 않음 |
| **이상치 탐지** | 정상 데이터만 있고 이상치 라벨 없음 | 재구성 오차 분포를 시각화하여 임계값 설정 | 모든 이상치를 동일하게 취급하므로 유형 분류는 추가 분석 필요 |
| **초기 탐색** | 고차원 데이터의 패턴 발견 | 발견된 패턴을 A/B 테스트나 실험으로 검증 | 잠재공간의 패턴은 상관관계이며 추가 검증 필요 |

### 8.3.4 실습: VAE 데이터 증강과 이상치 탐지

신용카드 사기 거래 탐지 문제에서 VAE의 효과를 검증한다. 실무의 극단적 불균형 상황을 시뮬레이션하기 위해, 학습에 사용하는 사기 샘플 수를 의도적으로 제한한다.

**데이터 구성**:
- 전체 데이터: 정상 900개, 사기 100개
- 학습 데이터: 정상 630개, **사기 30개** (21:1 비율)
- 테스트 데이터: 정상 270개, 사기 30개
- 특징: 10차원 (거래 금액, 시간, 위치 등)

실험 절차는 다음과 같다. VAE를 사용하여 30개의 사기 거래를 100 에포크 동안 학습시키며, 잠재 차원은 2로 설정한다. 학습된 VAE로 원본 사기 샘플을 인코딩하여 잠재 분포를 파악하고, 해당 분포에서 200개의 합성 사기 거래를 샘플링한다. Random Forest로 원본 데이터만 사용한 경우와 원본+합성 데이터를 사용한 경우의 성능을 비교한다.

**표 8.4** VAE 데이터 증강 효과

| 방법 | Accuracy | F1-Score | Recall (사기 탐지율) | 개선도 |
|------|----------|----------|---------------------|--------|
| 원본 데이터만 | 0.9633 | 0.7843 | 0.6667 | - |
| 원본 + VAE 증강 | 0.9767 | 0.8772 | 0.8333 | **+25.0%** |

결과를 해석하면, VAE 데이터 증강으로 사기 탐지율(Recall)이 66.7%에서 83.3%로 25% 개선되었다. F1-Score도 11.8% 향상되었다. 30개의 소수 샘플만으로 VAE를 학습하여 200개의 합성 샘플을 생성했고, 이를 통해 분류기의 소수 클래스 인식 능력이 크게 향상되었다.

**VAE 데이터 증강의 한계와 실행 조건**: VAE 데이터 증강이 효과를 발휘하려면 몇 가지 조건이 충족되어야 한다. 첫째, **최소 수십 개 이상의 원본 샘플**이 필요하다. 10개 미만의 샘플로는 VAE가 데이터의 잠재 구조를 학습하지 못하며, 오히려 노이즈만 생성하여 성능을 저하시킬 수 있다. 둘째, **클래스 간 경계가 모호한 데이터**에서 효과가 크다. 두 클래스가 명확히 분리되어 원본만으로 100% 정확도를 달성하면, 증강의 추가적 이점이 없다. 셋째, 합성 샘플 생성 시 **학습된 잠재 분포에서 샘플링**해야 한다. 표준 정규분포 N(0,1)에서 무작위 샘플링하면 원본 데이터와 무관한 샘플이 생성되어 오히려 분류 성능을 떨어뜨린다.

[그림 8-2] VAE 잠재공간 시각화 (왼쪽)와 재구성 오차 분포 (오른쪽)
![VAE 잠재공간과 재구성 오차](../diagram/8-vae-latent.png)

그림 8-2 왼쪽은 VAE가 학습한 2차원 잠재공간이다. 정상 거래(회색 원)와 사기 거래(검정 X표시)가 일부 영역에서 겹쳐 있지만, 사기 거래가 특정 방향으로 분리되는 경향을 보인다. 오른쪽은 재구성 오차 분포로, 정상 거래(밝은 막대)는 낮은 오차에 집중되고 사기 거래(사선 패턴 막대)는 상대적으로 높은 오차 영역에 분포한다. 95% 백분위수 임계값(검은 점선)을 사용하면 오탐률 5%, 미탐률 49%로, 사기의 절반 이상을 탐지하면서 오탐을 낮게 유지하는 균형 전략이 된다.

실무 시사점은 다음과 같다. VAE 데이터 증강은 불균형 데이터에서 소수 클래스 학습을 개선하는 효과적인 방법이다. 재구성 오차 기반 이상치 탐지는 라벨이 없는 상황에서도 적용 가능하다. 그러나 임계값 설정에 따라 오탐률과 미탐률이 크게 변하므로, 도메인 전문가의 검토가 필수적이다.

**VAE 이상치 탐지의 실무 한계**: 본 실습에서 VAE 이상치 탐지의 미탐률은 49%로, 사기 거래의 절반을 놓친다. 이 수준의 성능으로 **단독 FDS(Fraud Detection System)를 운영하는 것은 적절하지 않다**. VAE 이상치 탐지는 (1) 규칙 기반 탐지, 지도학습 모델과 결합한 **앙상블 시스템의 일부**로 활용하거나, (2) 라벨이 전혀 없는 상황에서 **전문가 검토용 후보군을 추출**하는 초기 필터링 단계에 적합하다. 실무에서는 단일 기법에 의존하기보다, 여러 탐지 기법의 장단점을 보완하는 다층 방어 전략이 필수적이다.

## 8.4 Diffusion Model: 불확실성 정량화와 예측

Diffusion Model (Ho et al., 2020; Nichol & Dhariwal, 2021; Song et al., 2020)은 순수 노이즈에서 점진적으로 데이터를 복원하는 생성 모델로, 예측의 불확실성을 정량화하고 리스크를 관리하는 데 유용하다. 본 절에서는 Diffusion이 데이터 분석에서 실질적으로 기여하는 사용 사례를 다룬다.

### 8.4.1 Diffusion의 세 가지 사용 사례

**순수 예측 - 시계열 미래 분포 정량화**: 과거 패턴을 조건으로 미래의 불확실성을 정량화하여 리스크 관리에 활용한다. 인과 관계 (Pearl, 2009)가 아닌 시간적 의존성만 모델링하며, "이런 패턴 이후 과거에 어떤 일이 일어났는가"를 학습한다. 100개 이상의 미래 경로를 샘플링하여 분포 전체를 시각화한다.

재무 리스크 관리에서 2025년 1분기 매출 예측이 100억원일 때, CFO는 "최악의 5% 시나리오에서는 매출이 얼마까지 떨어질 수 있는가?"를 알아야 비상 자금을 준비할 수 있다. Diffusion은 과거 20개월 매출 패턴을 조건으로 미래 3개월의 분포를 생성하여, 5% 분위수(70억원), 중앙값(100억원), 95% 분위수(130억원)를 제공한다.

수요 예측에서 제조업체가 다음 분기 생산량을 계획할 때, 단일 예측값(1,000개)보다 분포(5% 분위수: 800개, 95% 분위수: 1,200개)를 알면 재고 전략을 수립할 수 있다. 보수적 시나리오(800개)로 기본 생산, 낙관적 시나리오(1,200개)로 추가 원자재 확보 옵션 계약을 체결한다.

의료 예후 예측에서 당뇨병 환자의 6개월 후 HbA1c 분포를 예측하여, "60% 확률로 7.0% 이하", "10% 확률로 8.5% 이상"을 제시한다. 의사는 이를 바탕으로 환자와 치료 계획을 논의한다.

```python
# 조건부 Diffusion: 과거 → 미래 분포 생성
class ConditionalDiffusionForecaster(nn.Module):
    def reverse_diffusion(self, x_past, n_samples=100):
        """과거 데이터 조건으로 100개 미래 경로 생성"""
        x_t = torch.randn(n_samples, future_dim)  # 순수 노이즈
        for t in reversed(range(self.T)):
            noise_pred = self.predict_noise(x_t, t, x_past)
            x_t = denoise_step(x_t, noise_pred, t)
        return x_t  # 100개의 가능한 미래
```

이 방법의 장점은 정규분포 가정 없이 임의 형태의 분포를 학습할 수 있다는 점이다. 비대칭적이거나 다봉 분포도 모델링 가능하며, 극단값(Tail risk)을 정량화하여 Black Swan 사건에 대비할 수 있다. 또한 분위수별 시나리오를 제공하여 리스크 관리를 직접 지원한다. 한계도 존재한다. 과거에 없던 극단적 사건(팬데믹, 금융위기 등)은 예측할 수 없으며, 100번 샘플링에 각각 1000 단계를 거쳐야 하는 높은 계산 비용이 발생한다. 무엇보다 과거 패턴이 반복된다는 가정하에서만 유효하므로, 미래를 보장하지는 않는다.

**시나리오 탐색 - 조건부 패턴 비교**: 다양한 조건 변수를 입력하여 "이런 조건이었을 때 과거에는 어떤 결과 분포가 나타났는가?"를 탐색한다.

재무 계획에서 기업이 마케팅 예산을 현재 대비 50% 증액/유지/감축할 때, 각 시나리오에서 매출 분포가 어떻게 달라지는지 과거 패턴을 탐색한다. 이를 통해 예산 배분의 초기 방향을 설정하고, 세부 전략은 A/B 테스트로 검증한다.

수요 계획에서 제조업체가 신제품 가격을 다양한 수준(저가/중가/고가)으로 설정할 때, 과거 유사 제품의 판매 분포를 탐색한다. 이는 초기 가격대를 설정하는 참고 자료로 활용되며, 실제 시장 반응은 테스트 마케팅으로 확인한다.

```python
def explore_scenarios(model, baseline, variable_levels):
    """조건 변수를 바꿔가며 과거 패턴 탐색"""
    scenarios = {}
    for level in variable_levels:
        context = baseline.copy()
        context['condition_var'] = level
        futures = model.reverse_diffusion(context, n_samples=100)
        scenarios[level] = futures  # 각 조건에서 100개 경로
    return scenarios
```

이 접근법의 장점은 실제 실험 없이도 수초 내에 다양한 조건 조합을 빠르게 탐색할 수 있다는 점이다. 초기 설계 단계에서 방향을 잡는 참고 자료로 활용하기에 적합하다. 그러나 과거 패턴에 기반하므로 새로운 상황에는 적용이 제한되며, 실제 전략의 효과는 별도 실험으로 검증해야 한다.

**불확실성 시각화 - 의사결정 지원 도구**: 예측의 불확실성을 시각적으로 제시하여 의사결정자가 리스크를 직관적으로 이해하도록 돕는다. 점 추정 대신 분포 전체를 시각화하며, 5%/50%/95% 분위수를 팬 차트(Fan Chart)로 표현한다. 왜도(Skewness)와 첨도(Kurtosis) 분석을 통해 비대칭 리스크를 파악할 수 있다.

경영 보고에서 CFO가 이사회에 "2025년 매출은 100억원 예상"이라고 보고할 때, Diffusion 분포 차트를 함께 제시한다. "중앙값은 100억원이지만, 90% 예측구간은 [80억, 120억]이며, 최악의 5% 시나리오에서는 70억원까지 하락 가능"이라고 설명하면, 이사회는 리스크를 명확히 인지하고 보수적 예산 편성 여부를 결정할 수 있다.

투자 심사에서 벤처캐피탈이 스타트업 투자를 검토할 때, "3년 후 매출 100억원 예상"이라는 점 추정보다 "50% 확률로 100억원 이상, 10% 확률로 30억원 이하"라는 분포 정보를 받으면, 하방 리스크를 반영한 투자 조건(전환사채, 청산 우선권 등)을 설계할 수 있다.

```python
# 분위수 계산 및 시각화
futures = model.reverse_diffusion(x_past, n_samples=100)
p5 = np.percentile(futures, 5)   # 최악 시나리오
p50 = np.percentile(futures, 50)  # 중앙값
p95 = np.percentile(futures, 95)  # 낙관 시나리오

# 팬 차트 생성
plt.fill_between(time, p5, p95, alpha=0.3, label='90% CI')
plt.plot(time, p50, label='Median')
```

이 방식의 장점은 불확실성을 정량적이면서도 시각적으로 전달하여, 의사결정자가 리스크와 기회를 균형있게 평가할 수 있도록 한다는 점이다. 단일 예측값에 대한 과신(Overconfidence)을 방지하는 효과도 있다. 그러나 분포 해석에 익숙하지 않은 의사결정자를 위한 교육이 필요하며, 과거 패턴에 기반하므로 구조적 변화를 반영하기 어렵다는 한계가 있다.

### 8.4.2 실습: Diffusion Model 불확실성 정량화

월별 매출 시계열 데이터에서 Diffusion Model을 사용하여 미래 매출의 불확실성을 정량화한다. 과거 12개월 패턴을 조건으로 다음 달 매출의 분포를 생성하고, 리스크 시나리오를 분석한다.

**데이터 구성**:
- 시계열 길이: 60개월 (2020-01 ~ 2024-12)
- 컬럼: date, sales
- 입력 파일: practice/chapter8/data/input/monthly_sales.csv

실험 절차는 다음과 같다. 먼저 과거 12개월 데이터로부터 다음 1개월을 예측하는 시퀀스를 생성한다. Diffusion Model을 100 에포크, 50 Diffusion Steps로 학습시킨 뒤, 마지막 12개월을 조건으로 100개의 미래 경로를 샘플링한다. 마지막으로 분위수(5%, 25%, 50%, 75%, 95%)를 계산하여 시각화한다.

<표 8-7> 실험 결과

| 시나리오 | 예측 매출 | 발생 확률 | 리스크 대응 |
|----------|-----------|-----------|-------------|
| 최악 (5%ile) | 84.66 | 5% | 긴급 비용 절감 |
| 보수적 (25%ile) | 101.66 | 25% | 보수적 예산 |
| 중립 (중앙값) | 110.74 | 50% | 계획대로 진행 |
| 낙관적 (75%ile) | 121.87 | 75% | 공격적 투자 |
| 최선 (95%ile) | 134.24 | 95% | 확장 계획 |

**불확실성 지표**: 예측 매출의 중앙값은 110.74이며, 불확실성 범위는 49.58(95%ile - 5%ile)로 나타났다. 이는 상대적 불확실성 44.8%에 해당하며, 예측값이 중앙값 기준으로 약 ±45% 범위 내에서 변동할 수 있음을 의미한다.

결과를 해석하면, Diffusion Model은 단일 점 추정 대신 분포 전체를 제공하여 불확실성을 명시적으로 정량화한다. 5%ile와 95%ile 사이의 차이(49.58)는 불확실성이 크다는 신호이며, 리스크 관리 강화가 필요함을 시사한다. CFO는 중앙값(110.74)뿐 아니라 최악 시나리오(84.66)를 고려하여 보수적 예산을 편성할 수 있다.

실무 시사점은 다음과 같다. Diffusion은 예측의 불확실성을 분포로 표현하여 의사결정자가 리스크를 직관적으로 이해하도록 돕는다. 분위수 기반 시나리오 분석은 최악/중립/낙관 상황별 대응 전략 수립에 유용하다. 그러나 과거 패턴에 기반하므로 팬데믹 같은 구조적 변화는 반영하지 못한다는 한계가 있다.

### 8.4.3 실무 가이드와 핵심 메시지

**Diffusion이 적합한 경우**:

| 상황 | 사용 목적 | 예시 |
|------|-----------|------|
| **순수 예측** | 과거 패턴 기반 미래 분포 | 매출/수요 예측, 재고 관리, 리스크 시나리오 |
| **시나리오 탐색** | 초기 설계 단계 | "이런 조건일 때 과거 패턴은?" 참고용 |
| **리스크 관리** | 극단값 정량화 | 최악/낙관 시나리오, VaR 계산 |
| **불확실성 시각화** | 의사결정 지원 | 분포 차트, 예측구간 제시 |

**Diffusion이 부적합한 경우**:

| 상황 | 이유 | 대안 |
|------|------|------|
| **실시간 예측** | 높은 계산 비용 (100×1000 단계) | Transformer, ARIMA |
| **구조적 변화** | 과거 패턴만 학습 | 도메인 지식 + 시나리오 분석 |

**핵심 메시지**: Diffusion Model은 예측의 불확실성 정량화와 리스크 관리에서 강력한 도구다. 단일 점 추정 대신 분포 전체를 제공함으로써, 의사결정자가 최악/낙관 시나리오를 고려하여 보다 안전한 전략을 수립할 수 있게 한다. 매출 예측, 수요 관리, 재무 리스크 분석 등에서 Diffusion의 다중 샘플링 능력은 전통적인 점 추정 모델이 제공하지 못하는 통찰을 제공한다.
