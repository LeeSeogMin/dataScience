# 제7장: 신경망 기반 데이터 분석 - 표현 학습과 정형 데이터 모델링

이 장에서는 신경망의 학습 원리를 먼저 살펴본 뒤, 신경망이 데이터 분석에 어떻게 기여하는지 검토한다. 역전파와 경사 하강법의 직관적 이해를 바탕으로 신경망의 학습 메커니즘을 설명하고, 이를 토대로 표현 학습과 특성 공학 자동화의 개념을 다룬다. 이어서 TabNet과 XGBoost의 실제 비교를 통해 "언제 딥러닝이 트리 모델보다 나은 선택인가"라는 질문에 답하고, 오토인코더를 활용한 표현 학습과 이상 탐지, 전이학습의 데이터 사이언스 활용법을 다루며, 마지막으로 PyTorch 기반 실무 파이프라인을 단계별로 실습한다.

각주: 이 장은 데이터 사이언스을 위한 신경망과 딥러닝을 지면 제약상 간략하게만 살펴보도록 한다. 그러나 언어모델에 대한 기본적인 내용, 특히 Attention과 transformer 아키텍처는 다른 자료들을 통해 학습하기를 권장한다.

## 7.1 신경망의 학습 원리

신경망이 복잡한 패턴을 학습할 수 있는 이유는 비선형 변환의 조합에 있다. 이 절에서는 수학적 증명보다는 "왜 작동하는가"에 대한 직관적 이해에 집중한다.

### 퍼셉트론에서 다층 신경망으로

[그림 7-1] 인공신경망
![인공신경망](../diagram/7_bw.png)

퍼셉트론(perceptron)은 가장 단순한 신경망으로, 입력의 가중 합에 활성화 함수를 적용하여 출력을 계산한다. 수학적으로 표현하면 y = σ(Σᵢ wᵢxᵢ + b)이다. 여기서 σ는 활성화 함수, w는 가중치, b는 편향이다.

단층 퍼셉트론은 선형 결정 경계만 학습할 수 있다. Minsky & Papert(1969)가 지적한 유명한 XOR 문제가 이를 보여준다. XOR 함수는 두 입력이 다를 때만 1을 출력하는데, 이 패턴은 직선 하나로 분리할 수 없다. 이 한계를 극복하는 방법은 은닉층(hidden layer)을 추가하는 것이다. 은닉층의 각 뉴런이 입력 공간을 비선형으로 변환하면, 최종 출력층은 이 변환된 공간에서 선형 분류를 수행한다. 결과적으로 원래 입력 공간에서는 비선형 결정 경계가 형성된다.

[그림 7-2] 다층 퍼셉트론의 비선형 문제 해결
![다층 퍼셉트론의 비선형 문제 해결](../diagram/7-1_bw.png)

다층 퍼셉트론은 입력을 고차원 특징 공간으로 매핑하여 2차원 평면에서 선형으로 분리할 수 없는 문제를 선형으로 분리할 수 있게 만든다. 은닉층의 각 뉴런은 입력을 비선형으로 변환하여 공간을 재배열하고, 이 변환된 공간에서는 XOR과 같은 비선형 문제도 선형 분류로 해결할 수 있다. 종이를 휘어 선을 그리는 비유가 이를 잘 설명한다.

[그림 7-3] 신경망과 은닉층
![신경망과 은닉층](../diagram/7-2.png)

보편 근사 정리(Universal Approximation Theorem, Hornik et al., 1989)는 "뉴런을 충분히 많이 쓰면, 신경망은 어떤 복잡한 패턴이든 학습할 수 있다"는 것을 수학적으로 증명한 이론이다. 레고 블록을 충분히 많이 모으면 어떤 형태든 만들 수 있는 것처럼, 뉴런을 많이 쌓으면 어떤 함수든 표현할 수 있다는 의미이다. 다만 "충분히 많이"가 실제로 얼마인지는 문제마다 다르고, 학습이 잘 된다는 보장은 아니라는 점에 유의해야 한다.

### 비선형 활성화 함수의 역할

각 입력값에 가중치를 곱한 뒤 합을 내는 선형 결합은 "원시 점수"를 만든다. 이 점수는 그대로 다음 층으로 넘어가지 않고 활성화 함수에 의해 변환된 값이 다음 층의 입력으로 전달된다. 활성화 함수가 없다면, 여러 층을 쌓아도 전체 네트워크는 하나의 선형 변환과 같아진다. 활성화 함수는 비선형성을 도입하여 신경망이 복잡한 함수를 표현할 수 있게 한다.

[그림 7-4] 활성화 함수
![활성화 함수](../diagram/7-3.png)

```
입력 → [가중 합 Σwᵢxᵢ+b] → [활성화 함수 φ] → 출력
                              ↓
             비선형 변환으로 표현력 확보
```

어떤 활성화 함수를 쓰느냐에 따라 같은 입력과 가중치라도 은닉층에서 만들어지는 표현과 그에 따른 학습(그래디언트 흐름)이 달라지며, 이는 최종 성능과 수렴 특성에 직접적인 영향을 준다. 시그모이드는 값을 0과 1 사이로 압축해 확률처럼 해석되게 하고, tanh는 출력을 중심이 0인 범위로 밀어 음·양 방향 정보를 표현하게 하며, ReLU는 음수 출력을 0으로 만들어 희소한 표현을 유도한다.

<표 7-1> 주요 활성화 함수 비교

| 함수       | 수식                        | 장점                          | 단점                         |
| ---------- | --------------------------- | ----------------------------- | ---------------------------- |
| Sigmoid    | σ(x) = 1/(1+e⁻ˣ)            | 확률 해석 가능                | 기울기 소실, 중심이 0이 아님 |
| Tanh       | tanh(x) = (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ) | 중심이 0                      | 기울기 소실                  |
| ReLU       | max(0, x)                   | 계산 효율적, 기울기 소실 완화 | Dead ReLU 문제               |
| Leaky ReLU | max(0.01x, x)               | Dead ReLU 해결                | α 값 선택 필요               |
| GELU       | x·Φ(x)                      | Transformer에서 좋은 성능     | 계산 복잡                    |

[그림 7-5] Sigmoid & ReLU 함수
![Sigmoid & ReLU 함수](../diagram/7-4.png)

ReLU(Rectified Linear Unit)가 현대 딥러닝의 기본 활성화 함수가 된 이유는 단순함과 효과의 조합이다. 양수 영역에서 기울기가 1로 일정하여 깊은 신경망에서도 기울기가 잘 전파되고, 계산이 단순하여 학습 속도가 빠르다. 다만 ReLU는 입력이 음수일 때 출력과 기울기가 모두 0이 되는데, 한번 0이 되면 업데이트가 일어나지 않아 그 뉴런이 영원히 "죽어버리는" Dead ReLU 문제가 있다. Leaky ReLU는 음수 영역에서도 아주 작은 기울기(예: 0.01)를 유지해서, 뉴런이 완전히 죽지 않고 회복할 기회를 준다.

```python
import torch
import torch.nn.functional as F
x = torch.linspace(-3, 3, 200)
relu, sigmoid = F.relu(x), torch.sigmoid(x)

### 역전파와 경사 하강법

신경망 학습의 핵심은 역전파(backpropagation)와 경사 하강법(gradient descent)이다. 먼저 입력을 신경망에 통과시켜 예측값을 계산하고, 예측값과 실제 값의 차이로 손실을 구한다. 이후 손실이 각 가중치에 얼마나 민감한지(기울기)를 계산해 기울기 반대 방향으로 가중치를 조금씩 조정하며, 이 과정을 반복해 손실이 작아지는 방향으로 모델을 개선한다. 역전파는 연쇄 법칙(chain rule)을 활용한다. 합성 함수의 미분은 각 함수 미분의 곱이므로, 출력층에서 시작하여 각 층의 기울기를 순차적으로 계산하면 전체 기울기를 효율적으로 구할 수 있다.

[순전파 (Forward Pass)]                    [역전파 (Backward Pass)]

  입력층      은닉층       출력층              손실에서 역방향으로
                                              기울기 전파
    ○ ──w₁──→ ○ ──w₃──→ ○
    x₁        h₁         ŷ ──→ L(ŷ,y)              ∂L/∂w₃ ←── ∂L/∂ŷ
                          ↑     손실                   ↑
    ○ ──w₂──→ ○ ──w₄──→  │                     ∂L/∂w₁ ←── ∂L/∂h
    x₂        h₂

              ─────────────→                   ←─────────────
              데이터 흐름                       기울기 흐름


[가중치 업데이트]
┌──────────────────────────────────────┐
│  w ← w - α · ∂L/∂w                   │
│                                      │
│  α: 학습률 (보폭)                    │
│  ∂L/∂w: 손실의 가중치에 대한 기울기  │
└──────────────────────────────────────┘
```
직관적으로 경사 하강법은 손실 함수가 정의하는 다차원 지형에서 최저점으로 내려가는 수치적 탐색 방법이다. 손실 L(w)의 기울기 ∂L/∂w는 현재 지점에서 손실이 커지는 방향을 가리키므로, 그 반대 방향인 −∂L/∂w로 이동하면 손실이 줄어든다. 기본 갱신식은 w ← w − α · (∂L/∂w)이며, α(학습률)는 한 걸음의 크기를 결정한다. 학습률이 너무 크면 발산하거나 진동하고, 너무 작으면 수렴이 매우 느려지므로 학습률 스케줄링·모멘텀·적응적 옵티마이저 등으로 보완한다. 배치 크기를 작게 하면 미니배치로 인한 노이즈가 생겨 지역적 극솟값이나 안장점(∂L/∂w≈0)을 벗어나는 데 유리하지만, 노이즈가 크면 학습 불안정성이 증가한다.

역전파는 출력에서 계산한 손실이 각 파라미터에 미치는 민감도(기울기)를 효율적으로 계산하는 알고리즘으로, 합성 함수의 연쇄법칙을 이용해 출력층에서 입력층 쪽으로 오차 신호를 전달한다. 각 층은 자신으로 들어온 신호와 국소적 미분값(local gradient)을 곱해 상위층으로 전파된 기여도를 분해·계산하므로, 전체 네트워크의 기울기를 한 번의 순전파·역전파로 얻을 수 있다. 연쇄적 곱셈 구조 때문에 깊은 네트워크에서는 기울기가 매우 작아지는(gradient vanishing) 또는 매우 커지는(gradient exploding) 문제가 발생할 수 있으며, 이를 완화하기 위해 ReLU와 같은 활성화, 배치정규화, 잔차 연결(Residual)·적절한 초기화·그래디언트 클리핑 등을 사용한다. 현대 프레임워크의 자동미분(autograd)은 이 전체 과정을 자동화하므로 연구자는 구조·하이퍼파라미터 설계와 옵티마이저 선택에 집중할 수 있다.

<표 7-2> 주요 옵티마이저 비교

| 옵티마이저     | 특징                     | 적합한 상황                |
| -------------- | ------------------------ | -------------------------- |
| SGD            | 기본적, 안정적           | 대규모 데이터, 일반화 중시 |
| SGD + Momentum | 관성 추가로 수렴 가속    | 손실 곡면이 복잡할 때      |
| Adam           | 적응적 학습률            | 빠른 수렴이 필요할 때      |
| AdamW          | Adam + Weight Decay 개선 | Transformer 학습           |

Adam(Kingma & Ba, 2015)은 현재 가장 널리 사용되는 옵티마이저로, 각 파라미터에 대해 적응적 학습률을 적용한다. 1차 모멘트(평균)와 2차 모멘트(분산)를 추적하여 파라미터별로 학습률을 자동 조정한다. 초기 수렴이 빠르다는 장점이 있지만, SGD가 최종 일반화 성능에서 더 나은 경우도 있어 상황에 맞게 선택해야 한다.

### 배치 학습과 미니배치

전체 데이터로 기울기를 계산하는 배치 경사 하강법(Batch GD)은 정확하지만 메모리 제약으로 대규모 데이터에 적용하기 어렵다. 반대로 샘플 하나씩 처리하는 확률적 경사 하강법(SGD)은 메모리 효율적이지만 노이즈가 크다. 미니배치 경사 하강법(Mini-batch GD)은 이 둘의 절충안으로, 적절한 크기의 배치로 기울기를 추정한다.

[그림 7-7] 배치 크기에 따른 경사 하강법 비교
![배치 크기에 따른 경사 하강법 비교](../diagram/7-6-batch-comparison.png)

실무에서는 GPU 메모리가 허용하는 최대 배치 크기에서 시작하여, 학습이 불안정하면 줄이는 방식을 권장한다. 일반적으로 32~512 사이의 배치 크기가 많이 사용된다. 작은 배치는 정규화 효과가 있어 과적합을 완화할 수 있지만, 너무 작으면 학습이 불안정해진다.

### 딥러닝 최적화
앞에서 경사하강법과 배치학습을 살펴보았다. 이는 딥러닝 최적화의 방법들로서 앞에서 살펴본 이 둘의 관계를 정리해보자. 딥러닝 최적화는 (1) 어떤 데이터로 그래디언트를 계산할지 결정하는 샘플링 전략과 (2) 계산된 그래디언트로 가중치를 갱신하는 업데이트 알고리즘이라는 두 축으로 나뉜다. 샘플링은 배치/확률적(SGD)/미니배치가 있으며, 미니배치는 효율성과 안정성의 균형을 제공한다. 업데이트 알고리즘은 기본 경사하강법, 모멘텀, Adam 등으로 수렴 속도와 안정성, 하이퍼파라미터 민감도가 달라지며, 실무에서는 보통 미니배치 샘플링과 Adam 같은 적응적 옵티마이저를 결합해 사용한다. 현대 프레임워크는 데이터 로더(샘플링)와 옵티마이저(업데이트)를 분리해 설정하도록 설계되어 있다.

(학습률: 파라미터 업데이트의 보폭을 결정하는 값; weight decay: 가중치 크기에 페널티를 주는 L2 정규화로 과적합 억제; 모멘텀: 이전 기울기를 반영해 진동을 줄이고 수렴을 가속하는 기법; 그래디언트 클리핑: 기울기 크기를 제한해 폭발을 방지)

## 7.2 신경망이 데이터 분석에 기여하는 방식

### 표현 학습: 특성 공학의 자동화

전통적 머신러닝에서는 중요한 특성 변수를 사람이 직접 선정한다. 즉, 도메인 지식·선행 연구·탐색적 데이터 분석을 바탕으로 어떤 변수가 예측에 유용한지를 판단하고, 이미지의 경우 SIFT·HOG 같은 특성 추출기나 텍스트의 경우 TF‑IDF·n‑gram과 같은 기법으로 수동으로 특성을 설계·전처리한다. 이러한 수작업 특성 공학은 시간과 전문지식에 크게 의존하며, 문제마다 처음부터 반복해야 하는 한계가 있다.

신경망은 이 문제를 표현 학습(representation learning)으로 해결한다. Goodfellow et al.(2016)은 딥러닝의 근본적 장점을 "원시 데이터에서 계층적 표현을 자동으로 학습하는 능력"으로 설명한다. 신경망의 각 층은 이전 층의 출력을 입력으로 받아 점진적으로 추상화된 표현(representation)을 구성한다. 초기 층은 저수준 패턴(선, 모서리)을, 후반 층은 고수준 개념(얼굴, 객체)을 포착한다.

주석: 표현이란 원시 데이터에서 예측에 유용한 정보만을 추출하여 수치적 형태로 변환한 것이다. 한국어로는 보통 "표현" 또는 학술적으론 "표상(表象)"이라 번역하고, 구체적 구현을 지칭할 때는 임베딩(embedding)이나 특징 벡터(feature vector)라 부른다. 좋은 표현은 분류나 회귀 같은 다운스트림 태스크(학습된 표현을 실제로 사용하는 후속(목표) 작업)를 더 쉽게 만든다. 예를 들어 잘 학습된 이미지 표현에서는 같은 클래스의 샘플들이 벡터 공간에서 가깝게 모이고, 다른 클래스와는 멀리 떨어진다.

### 딥러닝이 필요한 상황

딥러닝이 전통 머신러닝보다 우위를 보이는 상황은 비교적 명확하다. 첫째, 비정형 데이터를 다룰 때이다. 이미지, 텍스트, 음성 데이터는 CNN, RNN, Transformer 같은 딥러닝 아키텍처가 수작업 특성 공학 없이도 복잡한 패턴을 학습한다. 둘째, 데이터 규모가 충분히 커서 모델의 표현력을 활용할 여지가 있을 때이다. 일반적으로 수만 건 이상의 데이터가 있어야 딥러닝의 장점이 드러난다. 셋째, 특성 공학이 어렵거나 불가능한 도메인에서 원시 데이터에서 유용한 표현을 자동으로 학습해야 할 때이다.

반면 정형 데이터(테이블 데이터)에서는 상황이 다르다. Grinsztajn et al.(2022)은 45개 정형 데이터셋에 대한 광범위한 벤치마크를 통해 XGBoost, LightGBM 같은 그래디언트 부스팅 모델이 여전히 최고 수준의 성능을 보인다고 입증했다. 트리 기반 모델은 특성 선택을 본질적으로 수행하며, 불규칙한 결정 경계도 효율적으로 학습한다.

<표 7-3> 딥러닝 vs 전통 ML 선택 기준

| 상황                           | 권장 접근법       | 근거                        |
| ------------------------------| ----------------- | ---------------------------|

| 정형 데이터 + 빠른 베이스라인   | XGBoost/LightGBM  | 튜닝 시간 짧음, 안정적 성능  |

주: 빠른 베이스라인은 프로젝트 초기 단계에서 "짧은 시간에 신뢰할 수 있는 성능 기준"을 얻기 위해 최소한의 전처리와 기본 하이퍼파라미터로 빠르게 학습시킨 간단한 모델을 의미한다. 주된 목적은 데이터 품질·레이블 문제를 조기 발견하고, 복잡한 모델을 적용하기 전 비교점(sanity check)과 의사결정의 출발점을 제공하는 것이다. 정형 데이터에서는 학습 속도가 빠르고 기본 설정만으로도 안정적 성능을 내는 XGBoost/LightGBM 계열이 흔히 빠른 베이스라인으로 권장된다.

### 데이터 분석에서 신경망의 활용 영역

첫째, 사전학습된 모델의 중간층 출력을 임베딩으로 쓰면 원시 수치·범주 변수보다 더 추상적이고 의미 있는 특성을 얻을 수 있다. 예컨대 상품 예측에서는 ResNet의 최종 직전층 출력을 이미지 임베딩으로 추출하여 표형 특성(가격·카테고리 등)과 이어붙여 X_concat = [tabular_features, image_embedding] 형태로 XGBoost에 입력하면, 이미지의 시각적 단서가 결합되어 예측 성능이 좋아질 수 있다. 실무적으로는 (1) 임베딩을 고정하고 분류 헤드나 트리 모델만 학습하는 Feature Extraction 방식(라벨이 적을 때 안정적)과 (2) 임베딩 초기값으로 전체 또는 일부 층을 함께 재학습하는 Fine‑tuning 방식(라벨이 충분할 때 더 유연함)을 상황에 따라 선택하면 된다. 임베딩은 이미지의 모양·질감·객체 정보 같은 고수준 신호를 포착하므로, 정형 특성과 결합하면 서로 보완하는 효과를 기대할 수 있다.

둘째, 차원 축소 도구로서 오토인코더나 VAE와 같은 모델은 입력층보다 작은 차원의 은닉층을 통해 입력의 고차원 정보를 압축하여 더 추상적인 표현을 학습하고, 이렇게 얻은 잠재공간(latent space)은 PCA가 잡아내기 힘든 비선형 구조를 포착하므로 시각화, 클러스터링, 노이즈 제거 등 다양한 작업에서 유용한 저차원 표현을 제공한다. 입력의 병목 구조는 핵심 정보만을 보존하도록 강제하여 오토인코더와 VAE가 PCA의 선형적 한계를 극복하게 해 주며, 따라서 시각화·클러스터링·노이즈 제거뿐만 아니라 다운스트림 태스크의 입력으로도 활용할 수 있다. 다만 잠재차원 수와 드롭아웃·정규화 항과 같은 정규화 기법을 적절히 설계·검증하지 않으면 정보 손실 또는 과적합이 발생할 수 있으므로 하이퍼파라미터 선택과 검증을 신중히 수행해야 한다.

셋째, 이상 탐지에서 오토인코더는 "입력을 압축했다가 다시 복원하는 복사기"처럼 작동한다. 정상 데이터만으로 학습시키면, 모델은 정상 패턴을 "외우게" 되어 정상 입력은 잘 복원하지만 한 번도 본 적 없는 이상 패턴은 제대로 복원하지 못한다. 이때 원본과 복원본의 차이를 재구성 오류(Reconstruction Error)라 하며, 이 오류가 크면 이상으로 판단한다. 임계값은 정상 데이터의 오류 분포에서 상위 일정 비율(예: 5%)로 설정하는 것이 일반적이다. 필요하면 잠재공간의 밀도나 생성모델의 확률 점수와 결합해 탐지 성능을 높일 수 있다. 뒤에서 다시 자세히 다룬다. 

넷째, 데이터 증강과 합성이다. 학습 데이터가 부족하거나 특정 클래스가 극소수일 때, GAN·VAE·Diffusion 같은 생성 모델로 새로운 데이터를 만들어낼 수 있다. 예를 들어 사기 거래가 1%뿐인 데이터셋에서 합성 사기 데이터를 추가하면 모델 학습이 개선된다. 또한 개인정보가 포함된 원본 데이터 대신 통계적 특성만 보존한 합성 데이터를 연구자에게 제공할 수도 있다. 다만 합성 데이터가 원본의 편향을 그대로 물려받을 수 있으므로 품질 검증이 필수다. 생성 모델의 원리와 실습은 8장에서 상세히 다룬다.

<표 7-4> AI 시대 데이터 분석 방식의 진화

| 전통 방식                     | AI 시대 진화                                                  |
| ---------------------------- | ------------------------------------------------------------- |
| 수동 특성 공학                | 사전학습 임베딩과 표현 학습을 통한 자동 특성 추출             |
| 선형 차원 축소 (PCA)          | 오토인코더 / VAE 등으로 비선형 잠재 표현 학습                 |

| 단일 모델 중심 워크플로       | 임베딩 생성 → 전통 모델(예: XGBoost) 또는 하이브리드 앙상블 적용 |

이처럼 신경망은 딥러닝의 원래 목적인 "예측을 위한 최종 모델"로서 역할을 수행하고, 데이터 사이언스를 위해 "분석을 위한 도구"로서도 역할을 수행한다. 이 장에서는 후자, 즉 데이터 분석 파이프라인에서 신경망을 효과적으로 활용하는 방법에 집중한다.

## 7.3 정형 데이터 딥러닝: 트리 모델과의 비교

정형 데이터는 딥러닝의 전통적 강점인 공간적(CNN) 또는 순차적(RNN) 패턴이 없어 딥러닝 적용이 어렵다고 여겨져 왔다. 그러나 TabNet을 비롯한 최근 연구들은 정형 데이터에 특화된 딥러닝 아키텍처가 특정 상황에서 경쟁력 있는 성능을 보일 수 있음을 보여주었다. 정형 데이터는 수치형·범주형·순서형 특성이 혼재하는 경우가 많고, 이미지의 픽셀처럼 자연스러운 인접 구조가 없어 특성 간 관계가 불명확하다. 또한 실제 데이터에서는 결측치가 흔하며, 예측에 실질적으로 기여하는 특성이 일부에 집중되는 편차도 자주 관찰된다. CNN은 공간적 지역성을, RNN은 순차적 의존성을 가정하는데, 정형 데이터에는 이러한 구조가 없다. 단순 MLP는 모든 특성을 동등하게 처리하여 무관련 특성에 취약하다.

<표 7-5> 정형 데이터에서 딥러닝이 불리한 이유

| 원인               | 설명                                                                        |
| ------------------ | --------------------------------------------------------------------------- |
| 불규칙한 목표 함수 | 신경망은 매끄러운(smooth) 함수에 편향되어 있어, 불규칙한 패턴 학습에 어려움 |
| 무관련 특성 민감성 | 신경망은 무관련 특성(uninformative features)에 불안정                       |
| 회전 불변성 문제   | MLP 유사 아키텍처의 회전 불변성으로 인해 데이터 방향 정보 손실              |
| 적은 데이터        | 1만 샘플 미만에서 과적합 위험 증가                                          |

트리 기반 모델은 본질적으로 특성 선택을 수행하며, 데이터의 축에 정렬된 분할을 통해 불규칙한 결정 경계도 효율적으로 학습한다. 또한 하이퍼파라미터에 덜 민감하여 기본 설정으로도 좋은 성능을 내는 경우가 많다.

### TabNet: Attention 기반 특성 선택

TabNet(Arik & Pfister, 2021)은 Sequential Attention 메커니즘을 통해 각 의사결정 단계에서 어떤 특성을 사용할지 명시적으로 선택한다. 이는 트리 기반 모델의 특성 선택과 유사한 효과를 딥러닝에서 구현한 것이다. TabNet의 설계 철학은 "트리 모델의 장점을 딥러닝으로 가져오기"이다. 트리 기반 모델은 각 분기에서 하나의 특성만 사용하여 결정을 내리는데, TabNet은 이를 Sparsemax 기반 소프트 마스킹으로 미분 가능하게 구현했다. 이를 통해 해석 가능성과 표현력을 동시에 확보한다.

쉽게 말하면 TabNet은 "단계별로 중요한 특성만 골라 쓰는 신경망"이다. 입력에서 한 번에 모든 특성을 쓰지 않고, 각 결정 단계마다 어텐션을 이용해 지금 중요한 특성들만 선택한 뒤 그 특성들로 표현을 만들고 다음 단계로 넘긴다. 이때 Sparsemax는 선택 가중치를 계산할 때 많은 값들을 정확히 0으로 만들어 실제로 사용된 특성이 무엇인지 명확히 보여주므로 해석성이 좋아진다. 즉, TabNet은 (1) 어떤 특성을 썼는지 알려주는 선택 메커니즘과 (2) 선택된 특성들로부터 유용한 표현을 학습하는 표현 변환기를 결합하여 해석 가능성과 표현력을 동시에 얻는다. 다만, 단점으로는 트리 모델보다 구조가 복잡하고 학습/튜닝에 시간이 더 걸릴 수 있다.

![TabNet 아키텍처 다이어그램](../diagram/7-tab.png)

TabNet의 핵심 구성 요소는 다음과 같다. Attentive Transformer는 어떤 특성을 사용할지 결정하고, Sparsemax는 희소한 마스크를 생성하여 선택을 명시화한다. Feature Transformer는 선택된 특성을 표현으로 변환하고, Split Block은 다음 단계 입력과 최종 출력에 기여할 정보를 분리한다. 이 조합은 정형 데이터에서 중요한 일부 특성에 선택적으로 집중하면서도, 선택 과정이 해석 가능하도록 설계되어 있다.
Sparsemax는 Softmax의 "엄격한 버전"이다. Softmax는 모든 특성에 작더라도 양수의 가중치를 부여하지만, Sparsemax는 중요하지 않은 특성에 정확히 0을 부여하여 "이 특성은 사용하지 않음"을 명확히 한다. 마치 Softmax가 "A가 50%, B가 30%, C가 20% 중요"라고 말한다면, Sparsemax는 "A가 62.5%, B가 37.5%, C는 무시"라고 결정을 내리는 것과 같다.
구체적인 예로, 점수가 [2.0, 1.5, 0.5]일 때 Softmax 출력은 [0.506, 0.314, 0.180]으로 세 번째 특성도 18%를 차지하지만, Sparsemax 출력은 [0.625, 0.375, 0]으로 세 번째 특성을 완전히 제외한다. 이렇게 명시적으로 0이 되는 특성들이 있어서, TabNet이 각 단계에서 어떤 특성을 실제로 사용했는지 쉽게 파악할 수 있다. 

<표 7-6> TabNet 주요 하이퍼파라미터

| 파라미터       | 설명                        | 권장값              |
| -------------- | --------------------------- | ------------------- |
| n_d, n_a       | 의사결정/어텐션 임베딩 차원 | 8-64 (동일 값 권장) |
| n_steps        | 의사결정 단계 수            | 3-10                |
| γ (relaxation) | 특성 재사용 계수            | 1.0-2.0             |
| λ_sparse       | 희소성 정규화 강도          | 1e-3 ~ 1e-1         |

n_steps는 트리 모델의 깊이와 유사한 개념으로, 더 많은 단계는 더 복잡한 패턴을 학습할 수 있지만 과적합 위험도 증가한다. relaxation factor γ는 이전 단계에서 사용된 특성이 다음 단계에서 다시 선택될 확률을 조절한다. γ=1이면 한번 사용된 특성은 다시 선택되기 어렵고, γ 값이 클수록 재사용이 허용된다. TabNet은 입력 특성의 일부를 마스킹하고 복원하는 방식의 자기지도학습도 제안하여, 레이블이 적고 비레이블 데이터가 많은 상황에서 표현 학습을 보조할 수 있다. 이 방식은 BERT의 마스크드 언어 모델링과 유사한 개념을 정형 데이터에 적용한 것이다.

### FT-Transformer: 특성 간 상호작용 학습

FT-Transformer(Gorishniy et al., 2021)는 Transformer 아키텍처를 정형 데이터에 적용한 모델이다. 각 특성을 토큰으로 취급하고 Multi-Head Attention을 통해 특성 간 상호작용을 학습한다. NLP에서 단어 토큰 간 관계를 학습하듯, 정형 데이터에서는 특성 토큰 간 관계를 학습한다.

[그림 7-9] FT-Transformer 아키텍처
![FT-Transformer 아키텍처 다이어그램](../diagram/7-ft.png)

Feature Tokenizer는 수치형 특성에는 별도의 FC 층을, 범주형 특성에는 임베딩 테이블을 적용하여 각 특성을 동일한 차원의 벡터로 변환한다. 특히 수치형 특성의 경우 단순 스칼라가 아닌 d차원 벡터로 변환하여 더 풍부한 표현력을 확보한다.

<표 7-7> MLP vs TabNet vs FT-Transformer 

| 측면          | MLP           | TabNet        | FT-Transformer       |
| ------------- | ------------- | ------------- | -------------------- |
| 특성 상호작용 | 암묵적 (FC층) | 단계별 선택적 | 전역적 (Attention)   |
| 수치형 처리   | 그대로 입력   | 그대로 입력   | 임베딩 변환          |
| 해석 가능성   | 낮음          | 높음          | 중간 (Attention map) |
| 복잡도        | 낮음          | 중간          | 높음                 |

주: MLP(Multi-Layer Perceptron)는 단순 다층 신경망으로 기본적인 베이스라인으로, 특별한 구조 없이 Fully Connected 층만 쌓은 형태이다. TabNet과 FT-Transformer는 이 MLP의 한계(특성 선택 불가, 상호작용 학습 어려움)를 해결하기 위해 설계된 모델이다.  

[CLS] 토큰은 BERT에서 차용한 개념으로, 모든 특성 토큰의 정보를 집약하는 역할을 한다. Transformer의 Self-Attention을 거치면서 [CLS] 토큰은 모든 특성과 상호작용하며, 최종적으로 전체 샘플을 대표하는 벡터가 된다. 이 벡터를 분류나 회귀 헤드에 입력하여 예측을 수행한다. Attention 메커니즘을 통해 모든 특성 쌍의 상호작용을 학습할 수 있어, 특성 간 복잡한 비선형 관계가 있는 데이터에서 강점을 보인다. Gorishniy et al.의 벤치마크에서 FT-Transformer는 11개 데이터셋 중 6개에서 최고 성능을 보이며 평균 순위 1.8을 기록했다.

## 7.4 표현 학습: 오토인코더와 임베딩 추출

오토인코더(Autoencoder)는 입력을 저차원 잠재 공간으로 압축했다가 다시 복원하는 신경망이다. 이 과정에서 학습된 잠재 표현(latent representation)은 원본 데이터의 핵심 정보를 담고 있어 다양한 데이터 분석 태스크에 활용된다.

### 오토인코더의 구조와 원리

오토인코더는 인코더와 디코더로 구성된다. 인코더는 고차원 입력을 저차원 잠재 벡터로 압축하고, 디코더는 잠재 벡터에서 원본을 복원한다. 학습 목표는 재구성 오류(reconstruction error)를 최소화하는 것이다. 잠재 차원이 입력 차원보다 작으면, 오토인코더는 데이터의 핵심 구조만을 학습하게 된다. 이는 PCA의 비선형 확장으로 볼 수 있으며, 복잡한 데이터 구조를 포착하는 데 더 효과적이다. PCA가 선형 변환만 허용하는 반면, 오토인코더는 비선형 활성화 함수를 통해 더 복잡한 관계를 모델링할 수 있다.

[그림 7-11] 오토인코더 구조
![오토인코더 구조](../diagram/7-3-autoencoder-nn.png)

```python
class Autoencoder(nn.Module):
    def __init__(self, input_dim, latent_dim=8):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 32), nn.ReLU(),
            nn.Linear(32, 16), nn.ReLU(),
            nn.Linear(16, latent_dim)
        )
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 16), nn.ReLU(),
            nn.Linear(16, 32), nn.ReLU(),
            nn.Linear(32, input_dim)
        )

    def forward(self, x):
        z = self.encoder(x)
        return self.decoder(z)

    def get_latent(self, x):
        return self.encoder(x)
```

### 이상 탐지: 재구성 오류 활용

오토인코더 기반 이상 탐지는 정상 데이터만으로 학습하여 이상 데이터를 탐지하는 패턴이다. 정상 데이터의 분포를 학습한 오토인코더는 정상 데이터를 잘 복원하지만, 학습 시 보지 못한 이상 데이터는 제대로 복원하지 못해 재구성 오류가 크게 나타난다. 이 차이를 활용해 이상 여부를 판단한다.

[그림 7-12] 오토인코더 기반 이상 탐지 원리
```
[학습 단계 - 정상 데이터만]
정상 데이터 → 오토인코더 → 재구성
             (정상 패턴 학습)

[추론 단계]
정상 데이터 → 오토인코더 → 재구성 ──► 오류 작음 (정상 판정)
```

## 7.5 실무 파이프라인: PyTorch 기반 분석 흐름

PyTorch(PyTorch Documentation, 2024)는 동적 계산 그래프와 직관적인 Python API, 자동미분(autograd)을 제공하여 연구용 프로토타이핑과 실무 배포를 모두 지원하는 프레임워크이다. 앞서 다룬 MLP, TabNet, 오토인코더 등 모든 신경망 모델은 공통된 파이프라인 구조를 따른다. 이 절에서는 재현성 확보부터 학습 루프까지 실무 파이프라인을 정리한다.

모든 실험의 첫 단계는 재현성 확보다. 랜덤 시드를 고정하여 동일한 결과를 얻을 수 있도록 한다.

```python
import random
import numpy as np
import torch

SEED = 42
random.seed(SEED)
np.random.seed(SEED)
torch.manual_seed(SEED)
if torch.cuda.is_available():
    torch.cuda.manual_seed_all(SEED)
    torch.backends.cudnn.deterministic = True
    torch.backends.cudnn.benchmark = False
```

### 데이터 로딩: DataLoader 구성

PyTorch의 DataLoader는 배치 처리, 셔플링, 병렬 로딩을 자동화한다.

```python
from torch.utils.data import DataLoader, TensorDataset

# 텐서 변환
X_train_t = torch.FloatTensor(X_train)
y_train_t = torch.LongTensor(y_train)

# DataLoader 구성
train_loader = DataLoader(
    TensorDataset(X_train_t, y_train_t),
    batch_size=256,
    shuffle=True,
    num_workers=4,      # CPU 병렬 로딩
    pin_memory=True,    # CUDA 전송 최적화
    drop_last=True      # 마지막 불완전 배치 제외
)
```

`num_workers`는 CPU 병렬 로딩을, `pin_memory=True`는 GPU 전송 속도를 향상시킨다. `drop_last=True`는 마지막 배치가 배치 크기보다 작을 때 제외하여 배치 정규화 등에서 발생할 수 있는 문제를 방지한다. 배치 크기는 GPU 메모리가 허용하는 최대 크기에서 시작하여 조정한다.

### 과적합 방지: 검증 전략과 정규화

과적합을 방지하려면 적절한 검증 전략과 정규화 기법이 필요하다.

**검증 전략**: 데이터를 학습/검증/테스트로 분할하고, 검증 손실을 모니터링하여 과적합을 감지한다. 시계열 데이터에서는 시간 순서를 존중하는 분할을 사용해야 한다. 미래 정보가 학습에 포함되면 실제 예측 상황과 맞지 않기 때문이다.

<표 7-13> 검증 전략 비교

| 전략           | 방법                      | 적합한 상황     |
| -------------- | ------------------------- | --------------- |
| 홀드아웃       | 학습/검증/테스트 분할     | 대규모 데이터   |
| K-Fold CV      | K개 폴드로 교차 검증      | 중소규모 데이터 |
| 층화 K-Fold    | 클래스 비율 유지하며 분할 | 불균형 분류     |
| 시간 기반 분할 | 과거→미래 순서로 분할     | 시계열 데이터   |

**정규화 기법**:

L2 정규화(Weight Decay)는 가중치가 지나치게 커지는 것을 막는 기법이다. 손실 함수에 가중치 크기의 제곱합(λ||w||²)을 페널티로 추가하여, 모델이 소수의 특성에 과도하게 의존하지 않고 여러 특성을 골고루 활용하도록 유도한다. 마치 시험공부에서 한 과목만 파지 말고 전 과목을 고르게 공부하라는 것과 유사하다.

드롭아웃(Dropout)(Srivastava et al., 2014)은 학습 중 무작위로 일부 뉴런을 꺼버리는 기법이다. 예를 들어 드롭아웃 비율 0.3이면 매 학습 단계마다 30%의 뉴런이 무작위로 비활성화된다. 이렇게 하면 모델이 특정 뉴런에만 의존하지 않고, 여러 뉴런이 협력하여 예측하는 법을 배운다. 팀 프로젝트에서 특정 팀원이 빠져도 나머지가 해낼 수 있도록 역량을 분산시키는 것과 같다.

배치 정규화(Batch Normalization)는 각 층의 입력값을 평균 0, 분산 1로 정규화하여 학습을 안정화한다. 데이터 분포가 일정해지면 더 큰 학습률을 사용할 수 있어 학습 속도가 빨라진다. 다만 배치 크기가 너무 작으면(예: 8 이하) 평균과 분산 추정이 불안정해질 수 있다.

```python
import torch.nn as nn
import torch.optim as optim

# 모델 정의 (드롭아웃, 배치 정규화 포함)
model = nn.Sequential(
    nn.Linear(input_dim, 64),
    nn.BatchNorm1d(64),
    nn.ReLU(),
    nn.Dropout(0.3),
    nn.Linear(64, 32),
    nn.BatchNorm1d(32),
    nn.ReLU(),
    nn.Dropout(0.3),
    nn.Linear(32, num_classes)
)

# 옵티마이저 (Weight Decay 포함)
optimizer = optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-5)
scheduler = optim.lr_scheduler.ReduceLROnPlateau(optimizer, mode='min', patience=5)
```

### 조기 중단과 모델 저장

조기 중단(Early Stopping)은 검증 손실이 개선되지 않으면 학습을 중단하는 가장 간단하면서도 효과적인 정규화 기법이다.

```python
class EarlyStopping:
    def __init__(self, patience=10):
        self.patience = patience
        self.counter = 0
        self.best_loss = float('inf')
        self.best_state = None

    def update(self, val_loss, model):
        if val_loss < self.best_loss:
            self.best_loss = val_loss
            self.best_state = model.state_dict().copy()
            self.counter = 0
        else:
            self.counter += 1

    @property
    def should_stop(self):
        return self.counter >= self.patience
```

모델 체크포인트는 최고 성능 모델을 저장하여 나중에 복원할 수 있게 한다. state_dict를 사용하면 모델 구조와 분리하여 가중치만 저장/로드할 수 있다.

```python
# 최고 모델 저장
torch.save(model.state_dict(), 'best_model.pth')

# 모델 복원
model.load_state_dict(torch.load('best_model.pth'))
model.eval()  # 추론 모드로 전환
```

### 전체 학습 루프

앞서 설명한 요소들을 결합한 전체 학습 루프는 다음과 같다.

```python
early_stopping = EarlyStopping(patience=10)
scheduler = optim.lr_scheduler.ReduceLROnPlateau(optimizer, patience=5)

for epoch in range(max_epochs):
    # 학습
    model.train()
    for X_batch, y_batch in train_loader:
        X_batch, y_batch = X_batch.to(device), y_batch.to(device)
        optimizer.zero_grad()
        outputs = model(X_batch)
        loss = criterion(outputs, y_batch)
        loss.backward()
        optimizer.step()

    # 검증
    model.eval()
    with torch.no_grad():
        val_outputs = model(X_valid_t.to(device))
        val_loss = criterion(val_outputs, y_valid_t.to(device))

    # 학습률 스케줄링
    scheduler.step(val_loss)

    # 조기 중단 확인
    early_stopping.update(val_loss, model)
    if early_stopping.should_stop:
        print(f"Early stopping at epoch {epoch+1}")
        model.load_state_dict(early_stopping.best_state)
        break
```
