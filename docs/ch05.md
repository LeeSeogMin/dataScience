# 제5장: 앙상블 학습 - 정형 데이터 모델링과 해석

이 장에서는 배깅(bagging)과 부스팅(boosting)이 편향-분산 트레이드오프를 어떻게 바꾸는지, 그리고 정형 데이터(tabular data)에서 앙상블이 왜 강력한지 학습한다. 랜덤 포레스트가 부트스트랩과 특성 무작위 선택으로 트리 간 상관을 줄이는 원리를 이해하고, 이어서 XGBoost·LightGBM·CatBoost로 대표되는 그래디언트 부스팅 계열이 손실 최소화 관점에서 어떻게 구현되는지 정리한다. 또한 Optuna를 이용한 베이지안 최적화 기반 최적화를 실습으로 재현하고, TreeSHAP·PDP·ICE·Permutation Importance로 예측 근거를 해석하는 흐름까지 연결한다. 마지막으로 반사실적 설명(counterfactual explanations)과 LLM 임베딩 특성(텍스트)을 앙상블 입력으로 결합하는 방법을 통해, “모델을 더 잘 맞추는 것”과 “모델을 더 잘 설명하는 것”을 함께 다루는 실무 흐름을 완성한다.

## 5.1 앙상블 학습의 원리

앙상블 학습(Ensemble Learning)은 여러 개의 약한 학습기(weak learner)를 결합하여 단일 모델보다 강력한 예측 성능을 달성하는 방법론이다. 핵심 아이디어는 단순하다. 한 명의 전문가보다 여러 전문가의 의견을 종합하는 것이 더 정확한 판단을 내릴 수 있다는 "집단 지성(wisdom of the crowd)" 원리를 기계학습에 적용한 것이다.

### 왜 앙상블이 효과적인가?

단일 모델은 학습 데이터의 특정 패턴에 과적합되거나, 반대로 너무 단순하여 복잡한 패턴을 포착하지 못할 수 있다. 앙상블은 이러한 개별 모델의 약점을 상호 보완한다. 통계적 관점에서 앙상블의 효과를 이해할 수 있다. n개의 독립적인 모델이 있고, 각 모델의 오차가 평균 0, 분산 σ²를 따른다고 가정하자. 이 모델들의 예측을 평균하면 분산이 σ²/n으로 감소한다. 즉, 모델 수가 증가할수록 예측의 불확실성이 줄어든다.

### 배깅: 분산을 줄이는 전략

배깅(Bagging, Bootstrap Aggregating)은 Leo Breiman이 1996년에 제안한 방법으로(Breiman, 1996), 동일한 학습 알고리즘을 서로 다른 데이터 샘플에 적용하여 다양성을 확보한다. 핵심 메커니즘은 부트스트랩 샘플링이다. 원본 데이터에서 중복을 허용하여 무작위로 샘플을 추출하고, 각 부트스트랩 샘플로 독립적인 모델을 학습시킨다.
배깅은 원본 데이터 D로부터 중복을 허용하는 부트스트랩 샘플 Dᵦ를 여러 번 생성하고, 각 샘플마다 동일한 학습 알고리즘으로 개별 모델 hᵦ를 학습한 뒤, 예측 단계에서 분류는 다수결 투표로, 회귀는 평균으로 결합한다. 이 과정의 핵심은 개별 모델들이 서로 다른 학습 데이터를 보게 만들어 예측 오차의 분산을 낮추는 데 있다.
부트스트랩 샘플링에서 원본 데이터의 각 샘플이 특정 부트스트랩 샘플에 포함될 확률은 1 - (1 - 1/n)ⁿ이다. n이 충분히 크면 이 값은 약 1 - 1/e ≈ 0.632에 수렴한다. 이는 평균적으로 약 63.2%의 고유 샘플이 각 부트스트랩 샘플에 포함됨을 의미한다. 나머지 약 36.8%의 샘플은 해당 트리 학습에 사용되지 않았으므로 OOB(Out-of-Bag) 샘플로서 해당 트리의 성능 검증에 활용할 수 있다. 다만, 실제 비율은 샘플 크기와 무작위성에 따라 약간 변동할 수 있다. 배깅은 고분산(high variance) 모델, 즉 데이터 변화에 민감한 불안정한 모델에 효과적이다. 의사결정나무가 대표적인 예로, 단일 나무는 학습 데이터의 작은 변화에도 구조가 크게 달라질 수 있다. 배깅은 이러한 불안정성을 평균화하여 안정적인 예측을 제공한다.

### 부스팅: 편향을 줄이는 전략

부스팅(Boosting)은 약한 학습기를 순차적으로 학습시키되, 이전 모델이 잘못 예측한 샘플에 더 집중하는 방식이다. 배깅이 병렬적으로 독립 모델을 학습하는 것과 달리, 부스팅은 순차적으로 모델을 개선해 나간다. 부스팅의 핵심 아이디어는 이전 모델의 약점을 보완하는 것이다. 구체적으로, 첫 번째 모델이 오분류한 샘플에 더 높은 가중치를 부여하여 두 번째 모델이 해당 샘플에 더 집중하도록 한다. 이 과정을 반복하면 개별 모델은 약하지만 이들을 결합한 최종 모델은 강력한 예측 능력을 갖게 된다. 마치 여러 번의 시험을 거쳐 틀린 문제를 집중 복습하는 것처럼, 부스팅은 모델이 어려워하는 샘플들을 점진적으로 학습한다.
대표적인 부스팅 알고리즘인 AdaBoost(Adaptive Boosting)(Freund & Schapire, 1997)는 처음에 모든 샘플에 동일한 가중치 wᵢ = 1/n을 부여하고 약한 학습기를 학습한 뒤, 오분류된 샘플의 가중치를 높여 다음 학습기가 어려운 샘플에 더 집중하도록 만든다. 이를 T번 반복하면 각 단계의 약한 학습기는 이전 단계의 실수를 보완하게 되고, 최종 예측은 각 학습기의 성능에 비례하는 가중 투표로 결합된다. 부스팅은 고편향(high bias) 모델, 즉 너무 단순하여 데이터 패턴을 충분히 학습하지 못하는 모델에 효과적이다. 약한 학습기를 순차적으로 결합하여 복잡한 패턴도 학습할 수 있게 된다.

### 편향-분산 트레이드오프 관점

모델의 예측 오차는 편향(bias), 분산(variance), 그리고 줄일 수 없는 노이즈로 분해된다. 요약하면, 배깅은 "불안정한 모델을 안정화"하고, 부스팅은 "단순한 모델을 강화"한다.

<표 5-1> 배깅과 부스팅의 편향-분산 특성

| 특성          | 배깅                    | 부스팅                         |
| ------------- | ----------------------- | ------------------------------ |
| 주요 효과     | 분산 감소               | 편향 감소                      |
| 학습 방식     | 병렬 (독립적)           | 순차 (의존적)                  |
| 기본 모델     | 고분산 모델 (깊은 트리) | 고편향 모델 (얕은 트리)        |
| 과적합 위험   | 낮음                    | 높음 (반복 횟수 증가 시)       |
| 노이즈 민감도 | 낮음                    | 높음 (노이즈를 학습할 수 있음) |

### 앙상블의 다양성 확보

앙상블이 효과적이려면 개별 모델들이 서로 다른 예측을 해야 한다. 모든 모델이 동일한 예측을 한다면 앙상블의 이점이 없다. 실무에서는 (1) 배깅처럼 학습 데이터를 다르게 샘플링하거나, 부스팅처럼 샘플 가중치를 조정해 각 모델이 서로 다른 오류 패턴에 집중하게 만들고, (2) 랜덤 포레스트처럼 각 분할에서 특성의 부분집합만 사용해 특정 강한 특성에 트리가 과도하게 의존하는 것을 줄이며, (3) 서로 다른 알고리즘의 예측을 평균하거나 스태킹해 서로 다른 귀납 편향을 결합하는 방식으로 다양성을 확보한다.
앙상블에서 다양성과 개별 정확도는 트레이드오프 관계에 있다. 너무 다양한 모델을 사용하면 개별 정확도가 낮아질 수 있고, 정확도만 추구하면 모델들이 유사해진다. 실무에서는 적절한 균형점을 찾는 것이 중요하다.

## 5.2 랜덤 포레스트

랜덤 포레스트(Random Forest)는 Leo Breiman이 2001년에 제안한 알고리즘으로(Breiman, 2001), 배깅에 "특성 무작위 선택"을 추가하여 트리 간 상관관계를 줄인 개선된 앙상블 방법이다. 정형 데이터 분석에서 가장 널리 사용되는 알고리즘 중 하나로, 높은 정확도와 해석 가능성을 동시에 제공한다.

### 랜덤 포레스트의 핵심 메커니즘

일반 배깅에서는 각 트리가 전체 특성을 사용하여 분할점을 찾는다. 그러나 만약 특정 특성이 매우 강력한 예측력을 가진다면, 모든 트리가 해당 특성을 루트 노드로 선택하게 되어 트리 간 유사성이 높아진다. 이는 앙상블의 다양성을 해치고 분산 감소 효과를 약화시킨다.
랜덤 포레스트는 이 문제를 해결하기 위해 각 분할 시점에서 전체 특성 중 일부만 무작위로 선택하여 최적 분할을 찾는다. 전체 특성 수가 p개일 때, 일반적으로 분류에서는 √p개, 회귀에서는 p/3개의 특성만 고려한다.
랜덤 포레스트는 원본 데이터에서 부트스트랩 샘플을 생성해 트리를 학습하되, 각 노드 분할 시점에 √p(분류) 또는 p/3(회귀)개의 특성만 무작위로 후보로 두고 그중 최적 분할점을 선택한다. 이렇게 학습된 트리를 B개까지 반복 생성한 뒤, 예측 단계에서 분류는 다수결, 회귀는 평균으로 결합하여 최종 예측을 만든다.

### OOB(Out-of-Bag) 평가

랜덤 포레스트의 장점 중 하나는 별도의 검증 세트 없이 모델 성능을 추정할 수 있다는 것이다. 각 부트스트랩 샘플에 포함되지 않은 약 36.8%의 OOB 샘플은 해당 트리에 대해 "보지 못한 데이터"이다. OOB 평가는 각 샘플 xᵢ에 대해, xᵢ를 학습에 포함하지 않은 트리들의 예측만 집계해 OOB 예측을 만들고, 이를 실제 값과 비교하여 OOB 오차를 계산하는 방식이다. 즉, 학습 과정에서 자연스럽게 생기는 “검증용 샘플”을 이용해 추가 데이터 분할 없이 일반화 성능을 추정한다. OOB 오차는 교차 검증과 유사한 신뢰도를 가지면서도 추가 계산 비용이 거의 없다는 장점이 있다. 이는 하이퍼파라미터 튜닝 시 검증 전략으로 활용할 수 있다.

### 핵심 하이퍼파라미터

랜덤 포레스트의 성능을 좌우하는 주요 하이퍼파라미터를 살펴본다.

<표 5-2> 랜덤 포레스트 핵심 하이퍼파라미터

| 파라미터          | 설명                       | 기본값  | 튜닝 가이드                             |
| ----------------- | -------------------------- | ------- | --------------------------------------- |
| n_estimators      | 트리 개수                  | 100     | 많을수록 좋으나 수익 체감, 100-500 권장 |
| max_depth         | 트리 최대 깊이             | None    | None(무제한) 또는 10-30                 |
| min_samples_split | 분할에 필요한 최소 샘플 수 | 2       | 과적합 시 증가 (5-20)                   |
| min_samples_leaf  | 리프 노드 최소 샘플 수     | 1       | 과적합 시 증가 (1-10)                   |
| max_features      | 분할 시 고려할 특성 수     | sqrt(p) | 'sqrt', 'log2', 또는 0.3-0.7            |
| bootstrap         | 부트스트랩 샘플링 사용     | True    | True 권장                               |
| oob_score         | OOB 점수 계산              | False   | True로 설정하여 모니터링                |

n_estimators는 트리 개수로, 일반적으로 많을수록 성능이 향상되지만 일정 수준 이상에서는 개선 폭이 작아진다. 계산 시간과 성능의 균형을 고려하여 100-500 사이로 설정하는 것이 일반적이다. OOB 오차를 모니터링하면서 오차가 수렴하는 지점을 찾는 것이 좋다. max_features는 다양성과 개별 트리 성능 간의 트레이드오프를 결정한다. 값이 작을수록 트리 간 상관관계가 낮아져 다양성이 증가하지만, 개별 트리의 예측력은 약해질 수 있다. 분류에서는 sqrt(p), 회귀에서는 p/3이 경험적으로 좋은 기본값이다.

### 랜덤 포레스트의 장단점

랜덤 포레스트는 하이퍼파라미터에 덜 민감해 기본 설정에서도 준수한 성능을 내는 경우가 많고, 배깅 기반이라 과적합에도 비교적 강건하다. 또한 학습 과정에서 얻는 특성 중요도를 활용하면 모델을 완전히 투명하게 만들 수는 없더라도, 어떤 입력이 예측에 기여했는지 빠르게 점검할 수 있다. 반면 트리를 많이 저장하기 때문에 메모리 사용량이 커질 수 있고, 추론 시 다수의 트리를 순회해야 하므로 지연 시간이 민감한 환경에서는 부담이 될 수 있다. 아울러 트리 기반 모델 일반의 특성으로 학습 범위를 벗어난 외삽(extrapolation)이 약하며, 희소 고차원 데이터에서는 경우에 따라 그래디언트 부스팅 계열이 더 유리할 수 있다.

### 랜덤 포레스트 실습: OOB 평가와 특성 중요도

아래 코드는 California Housing 데이터에서 랜덤 포레스트를 학습하고, n_estimators 변화에 따른 OOB 오차 수렴과 특성 중요도를 확인한다.

```python
from sklearn.ensemble import RandomForestRegressor

model = RandomForestRegressor(n_estimators=300, oob_score=True, random_state=42, n_jobs=-1)
model.fit(X_train, y_train)
print(f"OOB Score: {model.oob_score_:.4f}")
```

## 5.3 그래디언트 부스팅

그래디언트 부스팅(Gradient Boosting)은 부스팅 원리에 경사 하강법 관점을 결합한 알고리즘으로, 정형 데이터에서 강력한 베이스라인으로 널리 활용된다. XGBoost, LightGBM, CatBoost는 이 원리를 효율적으로 구현한 대표 라이브러리이며, 실무와 경진대회 환경 모두에서 자주 선택된다.

### 그래디언트 부스팅의 원리

![경사하강법](../diagram/5-1-gradient.png)

**경사하강법 or 그래디언트(Gradient Descent)**

경사하강법은 손실 함수의 값을 최소화하는 파라미터(가중치, 편향)를 찾기 위한 최적화 알고리즘이다. 손실 함수(MSE 등)의 접선 기울기, 즉 그래디언트(gradient)를 계산해, 파라미터를 기울기가 감소하는 방향으로 반복적으로 업데이트한다. 이 과정을 통해 오차가 최소가 되는 지점에 수렴한다.

**그래디언트 부스팅**
그래디언트 부스팅(Friedman, 2001)은 여러 약한 학습기(주로 의사결정나무)를 순차적으로 결합해 최종 모델 𝐹(𝑥)를 만든다. 각 단계에서 이전 모델의 예측 오차(잔차)를 새로운 모델이 학습하며, 𝐹ₘ(𝑥) = 𝐹ₘ₋₁(𝑥) + γₘ ℎₘ(𝑥) 형태로 점진적으로 성능을 개선한다. 즉, 첫 번째 모델이 놓친 부분을 두 번째 모델이 보완하고, 이후 단계에서도 반복적으로 실수를 수정해 나간다. 이 방식은 앙상블의 효과와 경사하강법의 최적화 원리를 결합한다.

### XGBoost: 극한의 최적화

XGBoost(eXtreme Gradient Boosting)는 2016년 Tianqi Chen이 발표한 그래디언트 부스팅 라이브러리로, 빠른 속도와 높은 성능 때문에 실무와 대회에서 널리 사용된다. XGBoost의 핵심은 "예측 오차(손실)"와 "모델 복잡도(정규화)"를 동시에 고려해, 과적합을 억제하면서도 높은 예측력을 달성하는 데 있다.

목적함수는 다음과 같이 두 부분으로 구성된다.

L(φ) = Σᵢ l(yᵢ, ŷᵢ) + Σₖ Ω(fₖ)

여기서  
첫 번째 항(Σᵢ l(yᵢ, ŷᵢ))은 실제값과 예측값의 차이, 즉 **예측 오차(손실)**를 의미한다.  
두 번째 항(Σₖ Ω(fₖ))은 트리의 복잡도를 패널티로 주는 **정규화 항**이다.

정규화 항은 다음과 같이 정의된다.

Ω(f) = γT + ½λ||w||²

여기서  
γT는 트리의 리프 노드 개수(T)가 많아질수록 패널티가 커지게 하여, 지나치게 복잡한 트리 생성을 억제한다.  
½λ||w||²는 각 리프 노드의 예측값(w)이 너무 크거나 극단적으로 치우치지 않도록 조절한다(L2 정규화).

즉, XGBoost는 "예측을 잘하면서도, 불필요하게 복잡하지 않은 트리"를 만들도록 설계되어 있다.  
이로써 과적합 위험을 줄이면서도 높은 성능을 얻을 수 있다.

또한 XGBoost는 계산 효율을 위해 "2차 테일러 전개"라는 수학적 근사법을 사용한다.  
복잡한 손실 함수를 직선(1차)이나 포물선(2차)로 단순화하여, 분할점 탐색을 더 빠르고 정확하게 수행한다.

추가적으로, XGBoost는

- 데이터를 효율적으로 저장하고 읽는 구조(히스토그램, 스케치)
- 여러 CPU 코어를 동시에 활용하는 병렬 처리
- 대용량 데이터도 처리할 수 있는 시스템 최적화  
  등 다양한 기술을 적용해, 실제 데이터 환경에서도 빠르고 안정적으로 동작한다.

```text
L1 정규화는 일부 리프 노드의 가중치를 0으로 만들어 불필요한 특성을 자동으로 제거하는 효과가 있다. 예를 들어, 불필요한 특성이 많을 때 L1 정규화를 적용하면 모델이 실제로 중요한 특성만 남기고 나머지는 무시하게 된다. 반면 L2 정규화는 모든 리프 노드의 가중치를 작게 유지해, 과적합을 방지하고 모델의 일반화 성능을 높인다. 실제 실습에서는 reg_alpha(L1)와 reg_lambda(L2) 파라미터를 조정해, 복잡한 모델이 데이터에 과도하게 적합되는 것을 막을 수 있다.
```

### LightGBM: 대규모 데이터를 위한 경량화

LightGBM(Light Gradient Boosting Machine)은 Microsoft에서 2017년에 발표한 라이브러리로(Ke et al., 2017), 대규모 데이터와 고차원 특성에 최적화되어 있다.

LightGBM의 핵심 혁신은 두 가지이다.

**GOSS(Gradient-based One-Side Sampling)**: 모델이 잘못 예측한 샘플(오차가 큰 샘플)은 모두 남기고, 잘 맞춘 샘플(오차가 작은 샘플)은 일부만 골라서 학습에 사용한다. 즉, 중요한 샘플은 최대한 반영하고, 덜 중요한 샘플은 줄여서 계산을 빠르게 한다.
**EFB(Exclusive Feature Bundling)**: 동시에 값이 0이 아닌 특성(컬럼)이 거의 겹치지 않는 경우, 여러 특성을 하나로 묶어서 처리한다. 예를 들어, 대부분의 값이 0인 특성들이 서로 겹치지 않으면 한 컬럼에 합칠 수 있다. 이렇게 하면 특성(컬럼) 수가 줄어들어 메모리와 계산 속도가 크게 개선된다.
LightGBM은 트리를 성장시킬 때 "리프 중심 성장(leaf-wise growth)" 방식을 사용한다. 기존의 "레벨 중심 성장(level-wise)"은 트리의 모든 노드를 같은 깊이로 한 번에 분할하지만, 리프 중심 성장은 현재 리프(끝 노드) 중에서 손실감소가 가장 큰 노드를 먼저 분할한다. 이렇게 하면 같은 리프 개수로도 더 낮은 손실을 얻을 수 있지만, 트리가 한쪽으로 깊게 자라 과적합이 발생할 수 있으므로 max_depth(최대 깊이) 제한을 반드시 두는 것이 중요하다.

### CatBoost: 범주형 특성의 전문가

CatBoost(Prokhorenkova et al., 2018)는 범주형 특성 처리에 특화된 그래디언트 부스팅 라이브러리다. **Ordered Target Encoding**: 범주형 특성(예: "도시"가 "서울")의 타깃 인코딩에서, 일반 방식은 "서울"에 속한 모든 샘플의 타깃 평균을 그대로 사용한다. 하지만 이때 해당 샘플의 실제 타깃 값이 평균 계산에 포함되어 데이터 누수(leakage)가 발생할 수 있다. CatBoost의 Ordered Target Encoding은 이 문제를 방지하기 위해 데이터를 무작위로 섞은 뒤, 각 샘플의 인코딩 값을 "자신보다 앞선 동일 범주 샘플들의 타깃 평균"으로 계산한다. 즉, "도시=서울"인 샘플의 인코딩은 앞선 "서울" 샘플들의 타깃 평균(여기에 전체 타깃 평균(prior)과 스무딩 파라미터(α)를 더함)으로 정해진다. 이 방식은 각 샘플이 자신의 타깃 값을 인코딩에 포함하지 않으므로 데이터 누수가 없다. 스무딩은 데이터가 적은 범주에서 인코딩 값이 극단적으로 치우치는 것을 방지한다. 인코딩 공식은 다음과 같다.

```
인코딩(xᵢ) = (∑_{j < i} [categoryⱼ = categoryᵢ] × yⱼ + prior × α) ÷ (count_{j < i} + α)
```

여기서 prior는 전체 타깃 평균, α는 스무딩 강도이다. 예를 들어 prior=0.5, α=10, 앞선 "서울" 샘플 2개의 타깃 합이 1.2라면 인코딩 값은 (1.2 + 0.5×10) ÷ (2+10) = 0.6이 된다. 데이터가 적을수록 prior(전체 평균)에 더 가까워져 과적합을 막는다.

**Ordered Boosting**: 전통적인 그래디언트 부스팅에서도 유사한 누수 문제가 있다. 잔차를 계산할 때 전체 데이터로 학습된 모델을 사용하면, 해당 샘플이 이미 모델 학습에 사용되었기 때문이다. CatBoost는 각 샘플의 잔차를 해당 샘플이 포함되지 않은 모델로 계산하여 이 문제를 해결한다.
**Symmetric Trees**: CatBoost는 대칭 의사결정나무(Symmetric Decision Tree)를 사용한다. 같은 깊이의 모든 노드가 동일한 분할 조건을 사용하므로 추론 속도가 매우 빠르다. 대칭 의사결정나무는 트리의 같은 깊이에 있는 모든 노드가 동일한 특성과 분할 조건을 사용하여 데이터를 분할하는 구조이다.

### 그래디언트 부스팅의 과적합 방지

그래디언트 부스팅은 높은 예측력을 갖지만 과적합에 취약할 수 있으므로, 실무에서는 여러 방어 기법을 조합해 사용한다. 첫째, 학습률(learning_rate)을 0.01~0.1 수준으로 낮추면 각 트리의 영향력이 줄어들어 모델이 점진적으로 학습하게 되며, 대신 더 많은 트리가 필요하다. 둘째, 조기 종료(early stopping)를 적용하면 검증 세트의 성능이 일정 횟수 이상 개선되지 않을 때 학습을 자동으로 중단해 불필요한 반복을 방지할 수 있다. 셋째, 서브샘플링(subsample, colsample_bytree 등)으로 각 트리마다 일부 데이터나 특성만 사용하면 모델의 다양성이 증가하고 과적합 위험이 줄어든다. 넷째, 트리의 복잡도를 제한하는 max_depth, min_child_weight, min_samples_leaf 같은 파라미터와, XGBoost의 γ(분할 최소 이득), λ·α(L1/L2 정규화) 등 정규화 항을 함께 조정하면 트리가 지나치게 복잡해지는 것을 막을 수 있다. 이러한 기법들은 단독으로도 효과가 있지만, 낮은 학습률과 조기 종료, 서브샘플링, 트리 제약을 함께 적용하는 것이 실무에서 가장 널리 쓰이는 과적합 방지 전략이다.

## 5.4 하이퍼파라미터 최적화 전략

앙상블 모델의 성능은 하이퍼파라미터 설정에 크게 좌우된다. 이 절에서는 그리드 서치의 한계를 이해하고, 베이지안 최적화 기반의 효율적인 튜닝 전략을 학습한다.

### 하이퍼파라미터 탐색 방법 비교

**그리드 서치(Grid Search)**: 가능한 모든 하이퍼파라미터 조합을 체계적으로 탐색한다. 장점은 구현이 단순하고 최적점을 놓치지 않는다는 것이다. 단점은 차원의 저주로 인해 파라미터 수가 증가하면 탐색 시간이 기하급수적으로 증가한다는 점이다. 3개 파라미터에 각각 10개 후보가 있으면 10³ = 1,000번의 학습이 필요하다.
**랜덤 서치(Random Search)**: 하이퍼파라미터 공간에서 무작위로 조합을 샘플링한다. Bergstra & Bengio(2012)의 연구에 따르면, 대부분의 경우 그리드 서치보다 효율적이다. 그 이유는 실제로 성능에 영향을 미치는 파라미터는 일부에 불과하기 때문이다. 랜덤 서치는 중요한 파라미터의 더 넓은 범위를 탐색할 수 있다.
**베이지안 최적화(Bayesian Optimization)**: 이전 탐색 결과를 바탕으로 다음에 탐색할 지점을 "지능적으로" 선택한다. 목적 함수(모델 성능)를 확률 모델(주로 가우시안 프로세스)로 근사하고, 획득 함수(acquisition function)를 최대화하는 지점을 선택한다.

![하이퍼 파라미터 튜닝 비교](../diagram/5-tuning.png)

<표 5-5> 하이퍼파라미터 탐색 방법 비교

| 방법            | 탐색 효율 | 병렬화 | 구현 복잡도 | 적합한 상황               |
| --------------- | --------- | ------ | ----------- | ------------------------- |
| 그리드 서치     | 낮음      | 용이   | 매우 쉬움   | 파라미터 2-3개, 후보 적음 |
| 랜덤 서치       | 중간      | 용이   | 쉬움        | 빠른 탐색, 초기 범위 설정 |
| 베이지안 최적화 | 높음      | 가능   | 중간        | 학습 비용이 큰 모델       |

### Optuna를 활용한 베이지안 최적화

Optuna(Akiba et al., 2019)는 베이지안 최적화 기반의 하이퍼파라미터 튜닝 프레임워크다. Optuna의 베이지안 최적화는 "이전 탐색 결과를 바탕으로 다음 시도할 파라미터를 똑똑하게 고르는 방법"이다. 그리드 서치처럼 모든 조합을 무작정 시도하는 대신, 좋은 결과가 나왔던 영역을 더 집중적으로 탐색한다. 예를 들어, 50번의 시도 중에서 RMSE가 낮게 나온 파라미터 근처를 더 자주 시도해 빠르게 최적값에 접근한다. Optuna는 파라미터 후보를 직접 코드에서 지정할 수 있고, 성능이 낮은 시도는 중간에 자동으로 중단(Pruning)해 시간도 절약된다. 실무에서는 "최적화 과정을 자동으로 기록하고, 결과를 그래프로 확인"할 수 있어, 어떤 파라미터가 중요한지 한눈에 파악할 수 있다.

Optuna의 기본 사용 패턴은 다음과 같다.

```python
import optuna
study = optuna.create_study(direction="maximize")
study.optimize(objective, n_trials=50)
best_params = study.best_params
```

### XGBoost 핵심 하이퍼파라미터 가이드

XGBoost 튜닝에서 가장 중요한 파라미터들과 권장 탐색 범위를 정리한다.

<표 5-7> XGBoost 핵심 하이퍼파라미터 튜닝 가이드

| 파라미터         | 설명                | 탐색 범위      | 우선순위 |
| ---------------- | ------------------- | -------------- | -------- |
| n_estimators     | 트리 개수           | 100-1000       | 높음     |
| learning_rate    | 학습률              | 0.01-0.3 (log) | 높음     |
| max_depth        | 트리 깊이           | 3-10           | 높음     |
| min_child_weight | 리프 최소 가중치 합 | 1-10           | 중간     |
| subsample        | 샘플 샘플링 비율    | 0.6-1.0        | 중간     |
| colsample_bytree | 특성 샘플링 비율    | 0.6-1.0        | 중간     |
| reg_alpha (L1)   | L1 정규화           | 1e-8-10 (log)  | 낮음     |
| reg_lambda (L2)  | L2 정규화           | 1e-8-10 (log)  | 낮음     |
| gamma            | 분할 최소 이득      | 0-5            | 낮음     |

실무에서는 먼저 학습률과 트리 개수를 고정한 상태에서 트리 복잡도를 좌우하는 max_depth, min_child_weight 같은 파라미터를 빠르게 탐색하고, 과적합 조짐이 보이면 subsample, colsample_bytree를 낮추거나 reg_alpha, reg_lambda를 높여 정규화를 강화하는 방식이 흔하다. 이후에는 성능이 좋았던 구간 주변으로 탐색 범위를 좁혀 정밀 튜닝을 수행하고, 마지막 단계에서 learning_rate를 더 낮추는 대신 n_estimators를 늘려 일반화 성능을 끌어올리는 전략을 고려한다.

### 조기 종료와 콜백 활용

실무에서는 n_estimators를 크게 설정하고 조기 종료(early stopping)를 활용하는 것이 효율적이다.

```python
model = xgb.XGBClassifier(n_estimators=1000, learning_rate=0.05, early_stopping_rounds=50, eval_metric="auc")
model.fit(X_train, y_train, eval_set=[(X_valid, y_valid)], verbose=False)
best_round = model.best_iteration
```

조기 종료를 사용하면 n_estimators를 튜닝할 필요가 없어지며, 검증 성능이 더 이상 개선되지 않는 시점에 자동으로 학습이 중단된다.

### LightGBM 핵심 하이퍼파라미터

LightGBM은 XGBoost와 유사하지만 일부 파라미터 이름과 기본값이 다르다. LightGBM 특유의 주요 파라미터를 정리한다.

<표 5-8> LightGBM 핵심 하이퍼파라미터

| 파라미터         | 설명               | 탐색 범위            | XGBoost 대응            |
| ---------------- | ------------------ | -------------------- | ----------------------- |
| num_leaves       | 리프 노드 최대 수  | 20-100               | 2^max_depth             |
| max_depth        | 트리 최대 깊이     | -1(무제한) 또는 3-12 | max_depth               |
| min_data_in_leaf | 리프 최소 샘플 수  | 10-100               | min_child_weight (유사) |
| feature_fraction | 특성 샘플링 비율   | 0.6-1.0              | colsample_bytree        |
| bagging_fraction | 데이터 샘플링 비율 | 0.6-1.0              | subsample               |
| lambda_l1        | L1 정규화          | 0-10                 | reg_alpha               |
| lambda_l2        | L2 정규화          | 0-10                 | reg_lambda              |

LightGBM 튜닝에서 가장 중요한 파라미터는 num_leaves이다. 리프 중심 성장 방식에서 트리 복잡도를 직접 제어하는 파라미터이기 때문이다. 일반적으로 num_leaves < 2^max_depth로 설정하여 과적합을 방지한다. 예를 들어 max_depth=7이면 num_leaves는 70-100 정도가 적절하다.

### 교차 검증 전략

앙상블 모델의 하이퍼파라미터 튜닝에서는 적절한 검증 전략 선택이 매우 중요하다. 가장 널리 사용되는 방법은 K-Fold 교차 검증으로, 데이터가 충분할 때는 5-Fold, 데이터가 부족할 때는 10-Fold를 적용하는 것이 일반적이다. 분류 문제에서 클래스 불균형이 존재한다면 Stratified K-Fold를 활용해 각 폴드의 클래스 비율을 전체와 일치시키는 것이 바람직하다. 시계열 데이터의 경우에는 Time Series Split을 사용해 미래 데이터가 학습에 포함되는 데이터 누수를 방지해야 한다. 또한 동일 그룹(예: 같은 사용자)의 데이터가 학습과 검증에 동시에 포함되면 안 되는 상황에서는 Group K-Fold를 적용해 그룹 단위로 데이터를 분리하는 것이 권장된다. 실무에서는 데이터 특성과 목적에 따라 이러한 검증 전략을 적절히 선택해 모델의 일반화 성능을 신뢰성 있게 평가하는 것이 중요하다.

## 5.5 트리 기반 모델의 심화 해석

앙상블 모델은 "블랙박스"라는 인식이 있지만, 트리 기반 모델은 구조적 특성을 이용해 비교적 효율적인 해석 기법을 적용할 수 있다. 이 절에서는 불순도 기반 중요도(MDI), Permutation Importance, SHAP(특히 트리 모델에 최적화된 TreeSHAP), 그리고 PDP·ICE를 이용해 "전역적으로 무엇이 중요했는가"와 "개별 샘플에서 어떤 요인이 예측을 바꿨는가"를 함께 해석한다. 해석 가능성에 대한 포괄적인 논의는 Molnar(2022)를 참고할 수 있다.

### 트리 기반 변수 중요도

**불순도 감소 기반 중요도(MDI, Mean Decrease in Impurity)**: 각 특성이 트리 분할에서 불순도(지니 또는 엔트로피)를 얼마나 감소시켰는지를 측정한다. 계산이 빠르고 학습 과정에서 자연스럽게 얻어지는 장점이 있다. 그러나 MDI는 몇 가지 한계가 있다. 첫째, 카디널리티가(cardinality) 높은 특성(값이 많은 범주형 특성)에 편향된다. 분할 기회가 많을수록 중요도가 높게 측정되기 때문이다. 예를 들어, 1000개의 고객 ID가 있다면, 트리는 각 ID별로 분할을 시도할 수 있으므로 중요도가 실제보다 높게 측정될 수 있다. 둘째, 특성 간 상관관계가 있으면 중요도가 분산되어 개별 중요도가 과소평가될 수 있다.
**순열 중요도(Permutation Importance)**: 특성 값을 무작위로 섞었을 때 모델 성능이 얼마나 하락하는지를 측정한다. 특성과 타깃 간의 관계를 끊었을 때 성능 저하가 크면 해당 특성이 중요하다고 해석한다. 순열 중요도의 장점은 모델에 무관하게(model-agnostic) 적용 가능하고, 실제 예측 성능에 기반한다는 점이다. 단점은 상관된 특성들 간에 중요도가 분산될 수 있고, 계산 비용이 상대적으로 높다는 점이다.
**SHAP(SHapley Additive exPlanations)**: SHAP(Lundberg & Lee, 2017)는 게임 이론의 Shapley 값을 기계학습 해석에 적용한 방법이다. 각 특성의 기여도를 공정하게 배분하는 수학적 기반이 있어 해석의 일관성과 신뢰성이 높다. Shapley 값의 핵심 아이디어는 "협력 게임에서 각 플레이어의 기여도를 공정하게 분배하는 방법"이다. 기계학습에서는 각 특성이 플레이어이고, 예측값이 게임의 결과이다.
집값 예측 모델에서 Latitude(위도) 특성이 SHAP 값 0.45로 가장 높게 나왔다면, 이는 위도가 높을수록(북쪽일수록) 예측 집값이 평균적으로 크게 변동한다는 의미다. 즉, Latitude가 모델의 예측에 가장 큰 영향을 준다는 해석이 가능하다. 반면 MedInc(중위 소득) 특성의 SHAP 값이 0.41이라면, 소득이 높을수록 집값 예측이 크게 증가하는 경향을 보인다는 뜻이다. SHAP은 이런 식으로 각 특성이 예측값에 얼마나, 어떤 방향으로 기여했는지 수치로 보여주기 때문에, "왜 이런 결과가 나왔는가?"를 데이터 기반으로 쉽게 설명할 수 있다.

```text
SHAP은 모델 종류에 따라 계산 방식이 달라진다. 트리 기반 앙상블에서는 TreeSHAP이 트리 구조를 이용해 SHAP 값을 효율적으로 계산하며, 반대로 KernelSHAP은 모델을 “질의 가능한 블랙박스”로 보고 근사적으로 Shapley 값을 추정한다. 실무에서는 트리 모델을 해석할 때 TreeSHAP을 우선 사용하고, 트리가 아닌 모델(선형회귀, 로지스틱 회귀, SVM, 신경망 등)이나 통일된 기준이 필요한 경우에 KernelSHAP을 사용하는 것이 일반적이다.
```

```python
import numpy as np
import shap
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)
global_importance = np.abs(shap_values).mean(axis=0)
```

SHAP의 주요 시각화는 서로 보완적으로 사용된다. Summary Plot(Bar)은 각 특성의 평균 |SHAP|을 막대 길이로 표시해 전역 중요도를 직관적으로 보여주고, Summary Plot(Dot)은 샘플별 SHAP 분포와 특성값을 색상으로 함께 나타내어 특성값이 클 때 예측을 올리는지 내리는지의 방향성과 분포를 확인하게 해준다. Dependence Plot은 특정 특성값과 그 특성의 SHAP 값을 산점도로 그려 비선형성 및 상호작용 징후를 드러내며 색상으로 다른 상호작용 특성의 효과를 표현한다. Waterfall Plot은 개별 예측을 베이스값에서 시작해 각 특성 기여도를 순차적으로 누적해 보여주므로 예측이 단계별로 어떻게 형성되었는지 설명한다. Force Plot은 양(빨강)과 음(파랑) 기여를 일렬로 배치해 로컬 예측의 핵심 기여 요소를 한눈에 파악할 수 있게 한다.


### PDP와 ICE: 전역 효과와 개별 효과

부분 의존도 플롯(Partial Dependence Plot, PDP)은 "특정 특성을 변화시킬 때 예측의 평균적 변화가 어떻게 되는가"를 보여주는 전역적(평균적) 해석 도구이고, 개별 조건 기대(Individual Conditional Expectation, ICE)(Goldstein et al., 2015)는 각 샘플별 곡선을 함께 그려 "어떤 집단에서는 효과가 다르게 나타나는가"를 확인하는 도구이다. 따라서 상호작용이나 이질성이 의심될 때는 PDP만 보지 말고 ICE까지 함께 확인하는 것이 안전하다. SHAP도 개별 예측의 근거까지 포함한 상세 해석을 제공하지만 PDP, ICE가 상호작용 등 보다 자세한 정보를 제공한다.

[그림 5-7] PDP와 ICE 비교 (상위 특성: Latitude)
![PDP/ICE](../diagram/pdp_ice_top_feature.png)

그림 5-7에서 파란색 점선은 PDP(전역 평균 효과), 연한 파란색 실선들은 ICE(개별 샘플 효과)를 나타낸다. PDP 곡선을 보면 Latitude가 낮은 남부에서 예측값이 높고, 북쪽으로 갈수록 낮아지는 패턴이 뚜렷하다. ICE 곡선들은 대체로 PDP와 같은 방향을 따르지만, 남부 구간(32-35°)에서 개별 선들의 분산이 넓게 퍼져 있어 샘플 간 이질성이 크다는 것을 알 수 있다. 이는 같은 위도라도 소득, 주거밀도 등 다른 특성에 따라 집값 효과가 달라지는 상호작용이 존재함을 시사한다.

## 5.6 Counterfactual Explanations

SHAP이 “왜 이런 예측이 나왔는가”를 기여도 관점에서 설명한다면, 반사실적 설명(counterfactual explanations)은 “무엇을 바꾸면 예측이 달라지는가”를 변화량 관점에서 설명한다. 두 관점은 경쟁 관계가 아니라 보완 관계다. 실무에서는 (1) SHAP으로 중요한 변수를 좁히고, (2) 그 변수들을 실제로 어느 방향·얼마나 바꿔야 목표 결과가 가능한지(counterfactual)를 확인하는 흐름이 의사결정에 더 직접적이다.

### DiCE: Diverse Counterfactual Explanations

DiCE(Mothilal et al., 2020)는 한 개의 "정답 같은 반사실"이 아니라, 여러 개의 다양한(different) 반사실 후보를 생성하는 접근을 제공한다. 같은 목표를 달성하더라도 바꿀 수 있는 변수가 여러 개일 수 있고, 현업에서는 "조정 가능 변수(actionable)"가 무엇인지가 상황마다 다르기 때문이다. 예를 들어 대출 심사라면 “소득”은 조정 가능하지만 “나이”는 조정 불가능하므로, 반사실 생성 시 제약을 반영하는 것이 중요하다.

```python
import dice_ml
dice = dice_ml.Dice(data, model, method="random")
exp = dice.generate_counterfactuals(query_x, total_CFs=3, desired_class="opposite")
cf_df = exp.cf_examples_list[0].final_cfs_df
```

## 5.7 정형 데이터 모델링: XGBoost와 딥러닝 비교

정형 데이터에서 트리 기반 부스팅이 강력하다는 주장은 특정 경진대회 결과에 기대기보다, 공개된 벤치마크 연구를 통해 확인하는 편이 안전하다. Grinsztajn et al.(2022)은 중·소규모 정형 데이터셋 다수에서 트리 기반 모델이 다양한 딥러닝 아키텍처보다 일관되게 우수하거나 경쟁력 있음을 보고했고, Shwartz-Ziv & Armon(2022)도 유사한 결론을 제시했다. 정형 데이터에 대한 딥러닝 접근법에 대한 포괄적 리뷰는 Borisov et al.(2022)을 참고할 수 있다. 다만 이러한 결론은 데이터 크기, 특성 구조, 목적 함수, 전처리와 검증 설계에 의해 달라질 수 있으므로, 본 절의 비교는 “항상 우열이 고정된다”기보다 “어떤 조건에서 무엇이 유리해지는가”를 정리하는 데 목적이 있다.

### XGBoost가 강한 이유

**트리 구조의 특성 공간 분할**: 의사결정나무는 특성 공간을 축에 평행한 직사각형 영역으로 분할한다. 이러한 분할 방식은 정형 데이터의 일반적인 패턴(특정 임계값 기준 분류)과 잘 맞는다. 예를 들어 "연소득 5천만 원 이상이고 신용점수 700점 이상이면 대출 승인"과 같은 비즈니스 규칙은 트리로 자연스럽게 표현된다. 반면 신경망은 부드러운 결정 경계를 학습하는데, 이는 이미지나 텍스트처럼 연속적인 패턴에는 적합하지만, 불연속적인 임계값 기반 패턴에는 비효율적일 수 있다.
**특성 간 상호작용 포착**: 그래디언트 부스팅은 트리 분할을 통해 자연스럽게 특성 간 상호작용을 학습한다. 루트 노드에서 특성 A로 분할하고, 자식 노드에서 특성 B로 분할하면 A와 B의 상호작용이 포착된다. 신경망도 상호작용을 학습할 수 있지만, 더 많은 데이터와 적절한 아키텍처가 필요하다.
**결측값과 이상치 처리**: XGBoost는 결측값을 처리하는 기본 메커니즘이 내장되어 있다. 학습 시 결측값이 있는 샘플을 왼쪽 또는 오른쪽 자식 노드로 보내는 최적 방향을 학습한다. 이상치 역시 트리 분할에 의해 별도의 영역으로 분리되어 다른 샘플에 영향을 주지 않는다. 신경망은 결측값 처리를 위한 별도의 전처리가 필요하고, 이상치에 민감할 수 있다.
**적은 데이터에서의 효율성**: 그래디언트 부스팅은 수천~수만 개의 샘플로도 좋은 성능을 보인다. 반면 딥러닝은 일반적으로 더 많은 데이터가 필요하며, 적은 데이터에서는 과적합 위험이 높다.

### 딥러닝이 유리한 상황

그럼에도 정형 데이터에서 딥러닝이 우위를 보이는 상황이 있다. 샘플 수가 매우 크고(예: 수백만 이상) 특성이 많아 표현력과 파라미터 효율이 중요해질 때는 딥러닝이 경쟁력을 갖기 쉽다. 또한 텍스트·이미지 같은 비정형 특성이 정형 특성과 함께 존재하는 경우에는, 멀티모달 학습이나 표현 학습을 통해 엔드-투-엔드로 이점을 얻을 수 있다.

운영 관점에서는 실시간으로 모델을 자주 업데이트해야 하는 온라인 학습 상황에서 확률적 경사 하강법 기반 모델이 적합할 수 있고, TabNet(Arik & Pfister, 2021), FT-Transformer(Gorishniy et al., 2021)처럼 정형 데이터에 특화된 아키텍처가 특정 데이터셋에서 강점을 보이는 사례도 보고되고 있다. 따라서 최종 선택은 데이터 규모와 구조, 추론 지연 제약, 해석 요구사항을 함께 고려해 실험으로 확인하는 것이 바람직하다.

<표 5-13> XGBoost vs 딥러닝 선택 가이드

| 상황                   | 권장 모델                           |
| ---------------------- | ----------------------------------- |
| 샘플 수 < 10,000       | XGBoost/LightGBM                    |
| 샘플 수 10,000-100,000 | XGBoost/LightGBM (딥러닝 실험 가능) |
| 샘플 수 > 100,000      | 둘 다 실험 후 결정                  |
| 범주형 특성 다수       | CatBoost > XGBoost > 딥러닝         |
| 텍스트/이미지 포함     | 딥러닝 (멀티모달)                   |
| 해석 가능성 필요       | XGBoost + SHAP                      |
| 빠른 프로토타이핑      | XGBoost (튜닝 적음)                 |

## 5.8 앙상블과 LLM의 결합 전략

LLM(Large Language Model)의 등장으로 텍스트 데이터에서 풍부한 특성을 추출할 수 있게 되었다. 이 절에서는 LLM 임베딩을 정형 데이터 모델과 결합하는 하이브리드 전략을 다룬다. 많은 실무 데이터셋에는 정형 특성과 텍스트 특성이 함께 존재한다. 예를 들어 이커머스 고객 데이터에는 구매 금액, 방문 횟수 같은 수치 특성과 함께 상품 리뷰, 고객 문의 텍스트가 있다. 전통적으로 텍스트는 TF-IDF나 Bag-of-Words로 벡터화하여 정형 모델에 입력했다. 그러나 이 방식은 단어의 의미나 문맥을 포착하지 못하는 한계가 있다. LLM 임베딩은 텍스트의 의미적 표현을 고차원 벡터로 추출한다. 이 임베딩을 기존 정형 특성과 결합하면 텍스트 정보를 효과적으로 활용할 수 있다. 텍스트 임베딩에는 Sentence-BERT(Reimers & Gurevych, 2019) 등이 널리 사용된다.

### 파이프라인 설계와 차원 축소

LLM + XGBoost 하이브리드 파이프라인은 텍스트를 임베딩으로 변환해 수치 특성과 결합한 뒤, 그 결합 특성을 입력으로 트리 기반 모델을 학습하는 흐름으로 이해할 수 있다. 실무에서는 먼저 로컬 임베딩(Sentence-BERT 계열)으로 빠르게 가능성을 확인하고, 필요할 때에만 상용 임베딩 API를 검토하는 식으로 비용과 운영 복잡도를 관리하는 편이 일반적이다. LLM 임베딩은 보통 384-1536차원으로 특성 수가 매우 많아지므로, PCA 같은 차원 축소 기법을 적용해 학습 시간 증가와 과적합 위험을 완화한다. 아래는 텍스트 임베딩 추출과 차원 축소, 정형 특성 결합의 핵심 코드이다.

```python
from sentence_transformers import SentenceTransformer
from sklearn.decomposition import PCA
import xgboost as xgb

emb = SentenceTransformer("all-MiniLM-L6-v2").encode(df["review_text"].tolist())
pca = PCA(n_components=50)
emb_reduced = pca.fit_transform(emb)
X = np.hstack([df[numeric_features].to_numpy(), emb_reduced])
model = xgb.XGBClassifier(n_estimators=300, learning_rate=0.05, max_depth=4)
model.fit(X_train, y_train)
```

_전체 코드는 practice/chapter5/code/5-8-llm-xgboost.py 참고_

본 실습에서는 이커머스 구매 예측 데이터를 생성하여 실험했다. 정형 특성으로는 나이, 누적 구매금액, 방문횟수, 장바구니 상품수, 최근 방문 경과일, 평균 세션 시간, 회원등급, 접속기기를 사용했고, 텍스트 특성으로는 상품 리뷰, 고객 문의, 검색 키워드를 결합했다. Sentence-BERT 임베딩 384차원을 PCA로 50차원으로 축소했을 때 분산 설명 비율 합이 0.9782였으며, 임베딩 추출에는 약 14초가 소요되었다.

<표 5-14> LLM 임베딩 + XGBoost 하이브리드 결과 (이커머스 구매 예측)

| -------------------- | -------- | ------- | -------- |

이번 실험에서는 accuracy는 소폭 하락했지만 roc_auc와 f1_score는 의미 있게 개선되었다. 리뷰 텍스트의 긍정·부정 감정, 검색 키워드의 구매 의도 신호, 고객 문의 내용이 임베딩을 통해 효과적으로 반영된 결과다. 특히 불균형 데이터(구매율 약 30%)에서 양성 클래스 탐지력을 나타내는 f1_score가 56.7% 향상된 점이 주목할 만하다. 이는 텍스트에 담긴 감정·의도 신호가 구매 예측에 실질적인 정보를 제공했음을 시사한다.

### 하이브리드 전략의 장단점

하이브리드 접근법의 장점은 텍스트의 의미 정보를 임베딩으로 요약해 정형 모델이 활용할 수 있게 만들면서도, 트리 기반 모델이 가진 정형 데이터 처리 강점과 운영 편의성을 상당 부분 유지할 수 있다는 점이다. 특히 엔드-투-엔드 딥러닝을 처음부터 설계하는 것보다 학습과 실험이 단순해지는 경우가 많고, 필요하면 SHAP 등으로 임베딩 차원까지 포함한 기여도를 추적하는 것도 가능하다. 반면 임베딩 추출 자체가 추가 비용과 계산 지연을 만들 수 있고, 임베딩 품질이 다운스트림(임베딩 이후의 분류·회귀 등 후속 작업) 성능을 좌우할 수 있으며, 고차원 임베딩이 결합되면서 특성 수가 급격히 증가해 과적합과 운영 복잡도가 커질 수 있다. 따라서 차원 축소나 임베딩 캐싱(동일 텍스트의 임베딩을 저장해 재사용), 폴백 모델(임베딩 서비스 장애 시 정형 특성만으로 동작하는 대체 모델) 준비 같은 운영 설계를 함께 고려해야 한다.

<표 5-15> 임베딩 옵션 비교

| 옵션                   | 비용                      | 품질 | 속도 | 적합한 상황                        |

상용 API는 가격과 약관이 수시로 변경될 수 있으므로, 실제 도입 시점에는 반드시 벤더의 공식 가격·정책 문서를 확인해야 한다.

### 실시간 서빙 고려사항

프로덕션 환경에서 하이브리드 모델을 서빙할 때는 임베딩 계산이 병목이 되지 않도록 동일 텍스트에 대한 임베딩 캐싱을 우선 설계하고, 실시간이 아닌 배치 예측이라면 임베딩을 사전에 계산해 저장하는 전략을 함께 검토한다. 또한 운영 환경의 지연 시간 제약을 맞추기 위해 트리 수를 줄이거나 모델을 경량화하는 선택이 필요할 수 있으며, 상용 임베딩 API를 쓴다면 장애 시 텍스트 특성 없이도 동작하는 폴백 모델을 준비해 가용성을 확보하는 편이 안전하다.
