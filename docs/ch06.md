# 제6장: 토픽 모델링 - 대량 문서에서 주제 자동 발견

이 장에서는 토픽 모델링이 실제 비즈니스 문제에서 어떤 역할을 하고, 각 알고리즘이 "주제"를 어떤 방식으로 발견하는지 학습한다. 먼저 전통적인 LDA(Latent Dirichlet Allocation)가 단어 빈도를 기반으로 잠재 토픽을 추론하는 원리와 한계를 이해하고, 이어서 BERTopic이 임베딩 기반 의미적 유사성으로 어떻게 더 정확한 토픽을 발견하는지 살펴본다. 또한 LLM을 활용한 토픽 자동 레이블링과 시간에 따른 토픽 변화를 추적하는 동적 토픽 모델링을 다룬다. 

## 6.1 토픽 모델링이 해결하는 문제

### 6.1.1 토픽 모델링이란?

토픽 모델링(Topic Modeling)은 비지도학습의 대표적인 텍스트 분석 기법으로, 레이블 없이 대량의 문서에서 잠재된 주제(토픽)를 자동으로 발견하는 방법론이다. 핵심 질문은 **"수천 개의 문서에서 사람들이 주로 이야기하는 주제는 무엇인가?"**이다.

토픽 모델링의 산출물은 두 가지다. 첫째는 **토픽별 키워드 분포**로, 각 토픽이 어떤 단어들로 구성되는지를 보여준다. 예를 들어 "배송, 택배, 도착, 늦음"이라는 키워드 집합은 "배송 관련" 토픽을 나타낸다. 둘째는 **문서별 토픽 분포**로, 각 문서가 어떤 토픽들의 조합으로 구성되는지를 보여준다. "배송이 늦었지만 상품 품질은 좋았어요"라는 리뷰는 "배송" 토픽 60%와 "품질" 토픽 40%로 구성될 수 있다.

### 6.1.2 군집화와 토픽 모델링의 차이

4장에서 다룬 군집화(Clustering)와 토픽 모델링은 모두 비지도학습이지만, 중요한 차이가 있다.

<표 6-1> 군집화 vs 토픽 모델링 비교

| 구분          | 군집화                         | 토픽 모델링                           |
| ------------- | ------------------------------ | ------------------------------------- |
| 할당 방식     | 하드 할당 (문서 → 하나의 군집) | 소프트 할당 (문서 → 여러 토픽의 혼합) |
| 핵심 산출물   | 군집 레이블                    | 토픽-단어 분포, 문서-토픽 분포        |
| 해석 방식     | 군집 내 문서의 공통점 파악     | 토픽별 키워드로 주제 해석             |
| 대표 알고리즘 | K-Means, HDBSCAN               | LDA, BERTopic                         |

군집화는 "이 문서는 어느 그룹에 속하는가?"라는 질문에 답한다. 반면 토픽 모델링은 "이 문서는 어떤 주제들의 조합인가?"라는 질문에 답한다. 실무에서는 두 접근을 상호 보완적으로 사용할 수 있다. BERTopic은 실제로 내부적으로 HDBSCAN 군집화를 사용한 후 각 군집을 토픽으로 해석한다.

### 6.1.3 비즈니스 응용 사례

토픽 모델링은 다양한 비즈니스 영역에서 활용된다. 고객 VOC(Voice of Customer) 분석은 토픽 모델링의 가장 대표적인 활용 사례로, 수천 건의 고객 리뷰나 문의를 수동으로 분류하는 것은 비현실적이지만 토픽 모델링을 적용하면 "배송 불만", "품질 문제", "가격 만족", "고객 서비스 불만" 등의 주요 이슈를 자동으로 발견할 수 있다. 이를 통해 제품팀은 품질 개선 우선순위를 정하고, 물류팀은 배송 프로세스를 점검하며, CS팀은 주요 불만 유형에 대한 응대 스크립트를 준비할 수 있다.

뉴스 트렌드 분석에서는 매일 쏟아지는 수천 건의 뉴스 기사에서 주요 이슈를 자동으로 파악하는데, 언론사는 독자 관심사를 파악하고 기업은 자사 관련 여론을 모니터링하며 정책 연구기관은 사회적 아젠다 변화를 추적할 수 있다. 특히 동적 토픽 모델링을 적용하면 시간에 따른 토픽 변화를 추적하여 이슈의 부상과 쇠퇴를 파악할 수 있다. 연구 동향 파악에서는 학술 논문 초록에 토픽 모델링을 적용하여 연구 분야의 흐름을 분석하고, 어떤 주제가 최근 급부상하고 있는지, 어떤 주제가 쇠퇴하고 있는지를 파악하여 연구 방향 설정에 활용한다.

### 6.1.4 토픽 모델링의 두 패러다임

토픽 모델링은 크게 두 가지 패러다임으로 발전해왔다. 전통적 접근법인 LDA(Latent Dirichlet Allocation)는 단어 빈도(Bag-of-Words)를 기반으로 토픽을 추론하는 방식으로, Blei et al. (2003)이 제안한 이후 오랫동안 토픽 모델링의 표준으로 사용되어왔다. LDA는 확률적 생성 모델로서 문서가 어떻게 생성되었는지를 역추론하여 잠재 토픽을 발견하며, 각 문서를 토픽들의 확률적 혼합으로 표현한다.

한편 현대적 접근법인 BERTopic은 임베딩(Embedding) 기반으로 의미적 유사성에 따라 토픽을 발견한다. 2장에서 간략히 설명한 Sentence-BERT 임베딩 (Reimers & Gurevych, 2019)을 활용하여 문맥을 반영한 토픽 분석이 가능하며, Grootendorst (2022)가 제안한 이 방법론은 LDA의 여러 한계를 극복한다. 물론 BERTopic 이외에도 다양한 임베딩 기반의 토픽 모델링 기법이 존재한다. 문서 임베딩을 직접 클러스터링해 토픽을 찾는 Top2Vec, 단어 임베딩과 토픽 분포를 함께 학습하는 LDA2Vec, 사전학습 언어모델의 문맥 임베딩을 변분 토픽모형에 결합한 CTM(Contextualized Topic Models)과 ETM(Embedded Topic Model) 같은 신경망 기반 토픽모델이 대표적인 예이다. 이 책에서는 이 중 대표적인 의미 기반 토픽 모델링으로 BERTopic을 다루기로 한다. 

<표 6-2> LDA vs BERTopic 개요

| 구분         | LDA                      | BERTopic           |
| ------------ | ------------------------ | ------------------ |
| 입력 표현    | Bag-of-Words (단어 빈도) | 임베딩 (의미 벡터) |
| 문맥 이해    | 불가                     | 가능               |
| 토픽 수 결정 | 사전 지정 필수           | 자동 결정          |
| 짧은 텍스트  | 취약                     | 강건               |

### 6.1.5 AI 융합 포인트

토픽 모델링 분야에서도 AI 기술의 발전에 따라 방법론이 진화하고 있다.

<표 6-3> 토픽 모델링의 AI 시대 진화

| 전통 방식            | AI 시대 진화           | 개선 효과              |
| -------------------- | ---------------------- | ---------------------- |
| LDA (단어 빈도 기반) | BERTopic (임베딩 기반) | 문맥 이해, 동의어 처리 |
| 수동 토픽 명명       | LLM 자동 레이블링      | 일관성, 확장성         |
| 정적 토픽 분석       | 동적 토픽 모델링       | 시간별 트렌드 추적     |
| TF-IDF 키워드 추출   | c-TF-IDF + 의미 기반   | 토픽 대표성 향상       |
| 사전 K 지정          | 자동 토픽 수 결정      | 분석 효율성            |

텍스트 임베딩과 우리가 잘 알고 있는 프롬프트 엔지니어링 기법이 토픽 모델링에 직접 적용된다. 특히 Sentence-BERT 임베딩은 BERTopic의 핵심 구성 요소이며, Zero-shot/Few-shot 기법은 LLM 토픽 레이블링에 활용된다.

## 6.2 LDA: 단어 빈도로 주제 추론하는 원리

### 6.2.1 LDA의 핵심 아이디어

LDA(Latent Dirichlet Allocation)는 생성 모델(Generative Model) 관점에서 토픽을 추론한다. 핵심 아이디어는 "문서가 어떻게 만들어졌는가?"를 역추론하는 것이다. LDA는 두 가지 핵심 가정에 기반한다. 첫째, 각 문서는 여러 토픽의 혼합으로 구성된다. 예를 들어 하나의 리뷰가 "배송 60% + 품질 40%"처럼 여러 토픽의 조합으로 표현될 수 있다. 둘째, 각 토픽은 단어들의 확률 분포로 정의된다. "배송" 토픽에서는 "택배", "도착", "늦음" 등의 단어가 높은 확률로 등장하고, "품질" 토픽에서는 "불량", "만족", "재구매" 등의 단어가 높은 확률로 등장한다.

이러한 가정에 따라 LDA는 문서 생성 과정을 다음과 같이 시뮬레이션한다.

```
1. 문서 d의 토픽 분포 θ_d를 디리클레 분포에서 샘플링
2. 각 단어 위치 n에 대해:
   a. 토픽 z_n을 θ_d에서 샘플링 (이 단어가 어느 토픽에 속하는지 결정)
   b. 선택된 토픽 z_n의 단어 분포 φ_z에서 단어 w_n을 샘플링
```

실제 학습에서는 이 과정을 역으로 수행한다. 관측된 단어들로부터 잠재 변수(토픽 할당 z, 토픽 분포 θ, 단어 분포 φ)를 추론한다.

디리클레 분포는 각 문서가 어떤 토픽들을 얼마나 섞어 갖게 될지 정해주는 '레시피의 가이드'로 생각하면 직관적이다. 예를 들어 여러 색 페인트(토픽)를 섞어 한 폭의 그림(문서)을 그린다고 하면, 파라미터 α는 팔레트에 미리 담아 놓은 각 색의 양, 즉 '유사 카운트'이다. α가 작으면 한두 색이 압도적으로 많은 선명한 그림이 나와 문서가 소수 토픽에 집중되고(희소), α=1이면 색들이 무작위로 고르게 섞이며(균등), α가 크면 여러 색이 골고루 어우러져 복합적인 색조의 문서가 생성된다. 실무에서는 문서가 소수 토픽에 집중되도록 하려면 작은 α를, 여러 토픽이 혼합된 문서를 원하면 큰 α를 선택한다.

그러나 LDA에는 몇 가지 유의한 한계가 있다. 우선 Bag‑of‑Words 가정으로 문맥을 고려하지 못해 동음이의어와 같은 의미 차이를 구분하지 못하며, 이는 문맥 의존 표현이 많은 언어에서 특히 문제를 일으킨다. 또한 통계적 추론에 의존하기 때문에 트윗이나 짧은 리뷰처럼 단어 수가 적은 텍스트에서는 동시 출현 정보가 부족하여 토픽 추론이 불안정해진다. 실무에서는 적절한 토픽 수 K를 사전에 알기 어려워 여러 K에 대해 모델을 학습하고 일관성 점수를 비교해야 하며, 이 과정은 계산 비용과 시간을 증가시킨다. 마지막으로 “배송”, “택배”, “배달”처럼 의미상 유사한 단어들을 별개로 처리해 토픽의 대표성이 분산되는 문제가 있어, 이러한 한계를 보완하기 위해 의미 기반 임베딩을 활용하는 BERTopic과 같은 현대적 기법이 도입되었다.

### 6.2.2 최적 토픽 수(K) 선택

토픽 수 K를 정할 때는 하나의 지표만 의존하지 않고 여러 방법을 병행하는 것이 권장된다. 일반적으로 사용되는 방법으로는 엘보우(Elbow) 검정, 실루엣(Silhouette) 점수, 확률모형의 퍼플렉시티(Perplexity) 탐색, 그리고 토픽 일관성(Coherence) 최적화가 있다. 엘보우 방식은 K에 따른 평가 지표(일관성·퍼플렉시티 등)를 그려서 증가율이 급격히 둔화되는 지점을 선택하는 방법이고, 실루엣 점수는 임베딩/군집 기반(예: BERTopic의 차원 축소+클러스터링)에서 각 문서의 군집 응집도와 분리도를 수치로 평가하여 높은 값을 갖는 K를 선호한다. 퍼플렉시티는 LDA 같은 생성모형에서 낮을수록 좋지만, 사람의 해석성과 항상 일치하지는 않으므로 단독 지표로 쓰기에는 한계가 있다.

### 6.2.3 토픽 평가지표 
토픽 평가는 일관성 지표와 다양성 지표를 함께 사용하여 판단하는 것이 바람직하다. 
일관성 지표는 C_v·UMass·NPMI 등으로, 토픽의 상위 단어들 간 공동 발생 및 의미적 유사성을 수치화하여 "토픽이 해석 가능하게 응집되어 있는가"를 평가한다. 예를 들어 C_v는 상위 K개 단어 쌍의 유사도 평균으로 정의할 수 있으며, 수식으로는 C_v = (2 / (K(K-1))) · Σ_{i<j} sim(w_i, w_j)로 표현된다(여기서 w_i, w_j는 상위 단어, sim은 유사도 함수). 경험적으로 C_v가 약 0.4 이상이면 수용 가능, 0.5 이상이면 양호한 수준으로 간주하는 경우가 많다. 
다양성 지표는 토픽 간 중복을 줄였는지 보는 척도로, 대표적으로 각 토픽의 상위 N개 단어들에서 고유 단어 비율(Topic Diversity)이나 토픽 간 Jaccard/중복률을 사용한다. 실무에서는 여러 지표(C_v·NPMI·UMass·토픽 다양성·실루엣 등)를 K별로 비교하여 엘보우나 최고점 근처를 후보로 선정하고, 최종적으로 대표 문서와 키워드를 사람이 검토해 해석 가능성을 확인하는 절차를 권장한다.

| NPMI (일관성)   | 0.0273  | -1~1 범위, 높을수록 좋음       | 낮음        |
| 토픽 다양성     | 0.7250  | 1에 가까울수록 좋음 (중복 적음)| 양호        |

C_v 일관성 점수(0.55)는 양호한 수준으로, 각 토픽의 상위 키워드들이 의미적으로 응집되어 있음을 보여준다. 토픽 다양성(0.73)도 양호하여 토픽 간 키워드 중복이 적다. 반면 NPMI(0.03)는 상대적으로 낮은데, 이는 일부 토픽에서 일반적인 단어(all, one 등)가 포함되어 발생한 것으로 보인다. UMass(-1.42)는 음수값이 일반적이며, 절대값이 작을수록 좋다. 각 토픽이 실제 카테고리와 대체로 잘 대응되지만, 일부 문서는 여러 토픽에 걸쳐 분포한다. 이는 LDA의 소프트 할당 특성을 보여준다.

## 6.3 BERTopic: 의미적 유사성으로 주제 발견

[그림 6-1] BERTopic 5단계 파이프라인 (임베딩 → UMAP → HDBSCAN → 토큰화 → c-TF-IDF)
![](../diagram/6-process.svg)

### 6.3.1 BERTopic의 핵심 아이디어

BERTopic은 Maarten Grootendorst가 2022년에 제안한 임베딩 기반 토픽 모델링 기법이다 (Grootendorst, 2022). LDA가 단어 빈도에 의존하는 것과 달리, BERTopic은 **의미적 유사성**을 기반으로 토픽을 발견한다. 핵심 통찰은 "의미적으로 유사한 문서를 먼저 군집화하고, 그 후에 각 군집을 대표하는 키워드를 추출한다"는 것이다. 4장에서 다룬 임베딩 군집화(BERT 임베딩 + HDBSCAN)를 토픽 모델링에 적용한 것으로 볼 수 있다. 다만 BERTopic은 군집화 후 c-TF-IDF를 통해 토픽별 대표 키워드를 추출하는 단계가 추가된다.

### 6.3.2 BERTopic 5단계 파이프라인

BERTopic은 5개의 모듈화된 단계로 구성되며, 각 단계는 독립적으로 교체할 수 있어 유연성이 높다. 

1. 1단계인 임베딩(Embedding)에서는 문서를 밀집 벡터(dense vector)로 변환한다. 기본적으로 Sentence-BERT(all-MiniLM-L6-v2 또는 한국어 임베딩 모델)를 사용하며, 각 문서는 384차원의 벡터로 변환되어 의미적 유사성을 수치로 표현할 수 있게 된다.

2. 2단계인 차원 축소(UMAP)에서는 고차원 임베딩을 저차원으로 투영한다. UMAP(Uniform Manifold Approximation and Projection)은 2장과 4장에서 다룬 차원 축소 기법으로, 국소 구조와 전역 구조를 모두 보존하면서 기본 설정상 5차원으로 축소한다. 

**UMAP 파라미터:**

| 파라미터     | 의미                | 권장값                 |
| ------------ | ------------------- | ---------------------- |
| n_neighbors  | 지역 구조 보존 정도 | 15 (기본값)            |
| n_components | 출력 차원           | 5 (BERTopic 기본값)    |
| min_dist     | 점들 간 최소 거리   | 0.0 (밀집 군집 허용)   |
| metric       | 거리 측정 방식      | cosine (임베딩에 적합) |

n_neighbors가 작으면 지역 구조에 집중하고, 크면 전역 구조를 더 반영한다. 토픽 모델링에서는 의미적으로 유사한 문서들의 지역 군집을 발견하는 것이 목표이므로 기본값 15가 적합하다.

3. 3단계인 군집화(HDBSCAN)에서는 저차원 공간에서 문서를 군집화한다. 4장에서 다룬 HDBSCAN은 밀도 기반 군집화로, 비구형 군집을 탐지하고 노이즈를 자동으로 분리하며, 이 단계에서 각 군집이 하나의 토픽 후보가 된다.

**HDBSCAN 파라미터:**

| 파라미터                 | 의미             | 권장값                       |
| ------------------------ | ---------------- | ---------------------------- |
| min_cluster_size         | 최소 군집 크기   | 15 (데이터 크기에 따라 조정) |
| min_samples              | 코어 포인트 밀도 | 10                           |
| cluster_selection_method | 군집 선택 방식   | 'eom' (초과 질량)            |

min_cluster_size가 크면 적은 수의 큰 토픽이, 작으면 많은 수의 작은 토픽이 생성된다. 노이즈로 분류되는 문서(-1 토픽)가 너무 많으면 min_cluster_size를 줄이고, 토픽이 너무 세분화되면 늘린다.

**Optuna를 활용한 파라미터 자동 튜닝**

UMAP과 HDBSCAN의 파라미터는 데이터 특성에 따라 최적값이 달라지므로, 베이지안 최적화 라이브러리인 Optuna를 활용하면 체계적으로 최적 파라미터를 탐색할 수 있다. 핵심 아이디어는 토픽 일관성(Coherence)은 최대화하고 노이즈(아웃라이어) 비율은 최소화하는 다목적 최적화(Multi-Objective Optimization)를 수행하는 것이다.

```python
import optuna
from bertopic import BERTopic

def objective(trial):
    # 튜닝 대상 파라미터 범위 지정
    min_cluster_size = trial.suggest_int("min_cluster_size", 10, 30)
    min_samples = trial.suggest_int("min_samples", 5, 20)
    n_neighbors = trial.suggest_int("n_neighbors", 5, 25)
    n_components = trial.suggest_int("n_components", 5, 15)

    model = BERTopic(
        min_topic_size=min_cluster_size,
        umap_model=UMAP(n_neighbors=n_neighbors, n_components=n_components),
        hdbscan_model=HDBSCAN(min_cluster_size=min_cluster_size, min_samples=min_samples)
    )
    topics, _ = model.fit_transform(docs)

    # 평가 지표: 일관성 점수와 아웃라이어 비율
    coherence = calculate_coherence(model, docs)
    outlier_ratio = topics.count(-1) / len(topics)
    return coherence, outlier_ratio

# 다목적 최적화 실행
study = optuna.create_study(directions=["maximize", "minimize"])
study.optimize(objective, n_trials=50)
```

Optuna의 베이지안 최적화는 무작위 탐색보다 효율적으로 파라미터 공간을 탐색하며, 파레토 최적해(Pareto Front)를 제공하여 일관성과 노이즈 비율 간의 트레이드오프를 분석할 수 있다. 실무에서는 50~100회 정도의 트라이얼을 수행하면 충분히 좋은 파라미터를 찾을 수 있으며, 전체 구현 코드는 practice/chapter6/code/ 디렉토리에서 확인할 수 있다.

4. 4단계인 토큰화(Tokenization)에서는 각 군집 내 문서들의 단어를 추출하고 CountVectorizer를 사용하여 단어 빈도를 계산한다. 마지막 5단계인 토픽 표현(c-TF-IDF)에서는 각 토픽을 대표하는 키워드를 추출한다. c-TF-IDF(class-based TF-IDF)는 토픽별로 특징적인 단어를 찾는 기법이다.

c-TF-IDF_i,c = tf_i,c · log(1 + A / tf_i)

여기서 tf_i,c는 토픽 c에서 단어 i의 빈도, A는 전체 문서의 평균 단어 수, tf_i는 전체에서 단어 i의 빈도다. 이 수식은 "해당 토픽에서 특별히 자주 등장하는 단어"에 높은 점수를 부여한다.

5. 5단계(c-TF‑IDF — 토픽 표현): 군집별 문서들을 합쳐 하나의 클래스(토픽) 문서로 보고, 각 토픽에 대해 클래스 기반 TF‑IDF를 계산하여 해당 토픽을 대표하는 키워드를 추출한다. 이 과정에서는 토픽 내에서 빈번하게 등장하면서 전체 코퍼스에서는 상대적으로 적게 등장하는 단어에 높은 가중치를 부여하므로, 각 군집을 잘 설명하는 특징어를 얻을 수 있다. 실무에서는 토픽별 상위 N개 키워드와 대표 문서들을 함께 검토하여 해석 가능성을 확보한다.

6. 6단계(토픽 표현 보정 — 선택적): 추출된 키워드 기반 표현을 KeyBERT 스타일의 방법이나 LLM(예: GPT/Claude) 기반의 representation_model로 후처리하여 토픽 이름을 자동 생성하거나 표현을 정제한다. 이 단계는 레이블의 일관성·가독성·도메인 적합성을 높이기 위한 선택적 절차로, nr_docs·prompt·few‑shot 예시 등을 조정하여 출력 스타일을 제어할 수 있다.

### 6.3.4 선택 가이드

LDA와 BERTopic은 각각의 장점과 적합한 상황이 다르므로, 분석 목적과 데이터 특성에 따라 적절한 방법론을 선택해야 한다. LDA는 문서가 충분히 길고 수백 단어 이상일 때, 전통적인 확률적 해석이 필요할 때, 계산 자원이 제한적이고 GPU가 없을 때, 그리고 문서-토픽 확률 분포가 명시적으로 필요할 때 적합하다. 특히 LDA는 각 문서가 토픽들의 확률적 혼합으로 표현되므로, "이 문서는 A 토픽 60%, B 토픽 30%, C 토픽 10%로 구성된다"와 같은 해석이 가능하다는 장점이 있다.

반면 BERTopic은 짧은 텍스트(트윗, 리뷰, 댓글)를 분석할 때, 문맥에 따라 의미가 달라지는 단어가 많을 때, 토픽 수를 사전에 알 수 없을 때, 그리고 동적 토픽 분석이나 LLM 레이블링이 필요할 때 더 적합하다. BERTopic은 임베딩 기반이므로 "은행"이라는 단어가 금융 맥락인지 강가 맥락인지를 문맥에 따라 구분할 수 있으며, HDBSCAN의 특성상 토픽 수를 자동으로 결정하여 분석자의 주관적 판단을 최소화한다.

이 중 `visualize_topics()`는 2D 공간에서 토픽의 크기(원의 크기)와 토픽 간 거리를 한눈에 보여주어 전체적인 토픽 구조를 파악하는 데 유용하다. `visualize_topics_per_class()`는 실제 레이블이 있는 데이터에서 토픽이 각 클래스에 어떻게 분포하는지 확인할 때 활용한다. 모든 시각화는 HTML 파일로 저장되어 브라우저에서 인터랙티브하게 탐색할 수 있다. 독자는 실습 코드를 실행하여 각 시각화의 결과를 직접 확인해보기 바란다.

**노이즈 문서 처리 전략**

HDBSCAN의 노이즈 문서(-1 토픽)를 처리하는 방법은 여러 가지가 있으며, 분석 목적에 따라 적절한 전략을 선택해야 한다. 가장 단순한 방법은 노이즈 문서를 분석에서 제외하는 것인데, 이는 구현이 간단하지만 잠재적으로 중요한 정보를 잃을 수 있다는 단점이 있다. 두 번째 방법은 `reduce_outliers()` 메서드를 사용하여 노이즈 문서를 가장 유사한 토픽에 재할당하는 것이다. 이 방법은 확률 분포 기반으로 각 문서와 가장 유사한 토픽을 찾아 할당하므로, 노이즈로 분류되었지만 실제로는 특정 토픽과 관련이 있는 문서들을 복구할 수 있다. 세 번째 방법은 노이즈 문서를 별도로 검토하여 새로운 패턴을 발견하는 것으로, 때로는 노이즈로 분류된 문서들이 기존 토픽에 속하지 않는 새로운 주제를 다루고 있을 수 있기 때문이다.

```python
# 노이즈 문서를 가장 유사한 토픽에 재할당
new_topics = topic_model.reduce_outliers(
    documents,
    topics,
    strategy="distributions"  # 확률 분포 기반 재할당
)
```

`reduce_outliers`를 실행하면 노이즈로 분류되었던 문서 대부분이 기존 토픽에 재할당된다. 그 결과, 각 토픽의 문서 수가 증가하고 노이즈(-1 토픽)는 거의 0에 가깝게 줄어든다.

<표 6-8> reduce_outliers 적용 전후 토픽별 문서 수 변화

| 토픽 | 자동 레이블 | 문서 수 (적용 전) | 문서 수 (적용 후) | 변화량 |
| --- | --- | --- | --- | --- |
| -1 | (노이즈) | 218 | 2 | -216 |
| 0 | Computer Graphics | 528 | 544 | +16 |
| 1 | Sports/Baseball | 462 | 468 | +6 |
| 2 | Medical/Health | 411 | 511 | +100 |
| 3 | Politics | 275 | 369 | +94 |
| **합계** | | **1894** | **1894** | |

이 과정을 통해 모든 문서를 특정 토픽에 할당하여 LDA와 유사하게 전체 데이터셋을 활용하는 분석이 가능해집니다. 다만, 노이즈를 기존 토픽에 배분하는 LDA와 달리 BERTopic은 새로운 토픽을 생성하기 때문에 보다 정확한 토픽분류가 가능하다.

**토픽 축소 및 병합**

HDBSCAN은 데이터의 밀도 구조에 따라 토픽 수를 자동으로 결정하므로, 때로는 수십 개 이상의 세분화된 토픽이 생성될 수 있다. 이 경우 유사한 토픽을 병합하여 해석 가능성을 높이는 것이 실무에서 유용하다. BERTopic은 `reduce_topics()` 메서드를 통해 토픽 임베딩 간 유사성을 기반으로 유사 토픽을 자동으로 병합하는 기능을 제공한다.

```python
# 원래 토픽 수 확인
original_topics = topic_model.get_topic_info()
print(f"원래 토픽 수: {len(original_topics) - 1}")  # -1은 노이즈 토픽 제외

# 토픽 수를 10개로 축소
topic_model.reduce_topics(documents, nr_topics=10)

# 축소 후 토픽 수 확인
reduced_topics = topic_model.get_topic_info()
print(f"축소된 토픽 수: {len(reduced_topics) - 1}")
```

토픽 축소 시 BERTopic은 c-TF-IDF 벡터 간 코사인 유사도가 높은 토픽들을 우선적으로 병합한다. 예를 들어 "야구"와 "축구" 토픽이 별도로 존재할 때 토픽 수를 줄이면 "스포츠"라는 상위 토픽으로 병합될 수 있다. 병합 과정에서 각 토픽의 키워드는 재계산되어 병합된 토픽을 대표하는 새로운 키워드 집합이 생성된다.

토픽 축소는 다음과 같은 상황에서 유용하다. 첫째, 생성된 토픽이 너무 세분화되어 전체적인 주제 구조를 파악하기 어려울 때 사용한다. 둘째, 비즈니스 보고서나 대시보드에서 관리 가능한 수준의 토픽 수가 필요할 때 적용한다. 셋째, 계층적 분석을 위해 상위 수준의 토픽 구조를 먼저 파악하고자 할 때 활용한다. 다만 과도한 축소는 의미 있는 토픽 구분을 잃게 할 수 있으므로, `visualize_hierarchy()` 메서드로 토픽 간 계층 구조를 먼저 확인한 후 적절한 축소 수준을 결정하는 것이 권장된다.

## 6.4 LLM 기반 토픽 라벨링

### 6.4.1 토픽 레이블링의 도전

토픽 모델링의 산출물은 토픽별 키워드 리스트다. 예를 들어 ["배송", "택배", "도착", "늦음", "빠름"]이라는 키워드 집합을 얻었다면, 이를 어떻게 해석해야 할까? "배송 관련"이라고 명명할 수도 있고, "배송 만족도"라고 명명할 수도 있다.

전통적으로 토픽 레이블링은 도메인 전문가가 키워드를 보고 수동으로 수행했다. 그러나 이 방식에는 여러 한계가 있다. 먼저 주관성 문제가 있는데, 동일한 키워드 집합을 보더라도 분석자에 따라 다른 이름을 붙일 수 있어 결과의 재현성이 떨어진다. 또한 확장성 문제도 심각한데, 토픽 수가 수십 개 이상으로 늘어나면 수동 레이블링은 시간과 비용 면에서 비현실적이 된다. 마지막으로 일관성 문제가 있는데, 동일한 분석자라도 시간이 지남에 따라 레이블링 기준이 미묘하게 변할 수 있어 종단적 분석의 신뢰성이 저하될 수 있다.

### 6.4.2 BERTopic의 LLM 통합

BERTopic 0.15 버전부터 LLM을 활용한 토픽 레이블링을 지원한다. `representation_model` 파라미터를 통해 OpenAI GPT, Claude 등 다양한 LLM을 연동할 수 있다.

```python
from bertopic.representation import OpenAI

# LLM 표현 모델 생성
representation_model = OpenAI(
    model="gpt-3.5-turbo",
    chat=True,
    prompt="다음 키워드들을 바탕으로 토픽 이름을 5단어 이내로 지어주세요: [KEYWORDS]"
)

# BERTopic에 LLM 연동
topic_model = BERTopic(representation_model=representation_model)
```

LLM은 단순히 키워드를 나열하는 것을 넘어, 키워드 간의 관계를 파악하고 적절한 추상화 수준의 이름을 생성한다. 예를 들어 ["신용카드", "할부", "결제", "승인"]이라는 키워드에 대해 "결제 관련 문의"라는 레이블을 생성할 수 있다.

### 6.4.3 프롬프트 엔지니어링

LLM 레이블링의 품질은 프롬프트에 크게 의존한다. 효과적인 프롬프트를 설계하기 위한 기법들을 살펴보자. 

먼저 도메인 컨텍스트를 제공하면 LLM이 더 적절한 용어를 사용하고, 해당 분야에서 의미 있는 분류를 수행한다.

```
당신은 이커머스 고객 리뷰 분석 전문가입니다.
다음 키워드들로 표현되는 고객 불만 유형을 분류하세요:
키워드: [배송, 택배, 늦음, 지연, 배달]
```

구조화된 출력을 요청하면 후처리가 용이하고, 일관된 형식의 레이블을 얻을 수 있다.

**출력 형식 지정**

```
다음 형식으로 응답하세요:
{
  "label": "토픽 이름 (5단어 이내)",
  "confidence": 0.0-1.0,
  "reasoning": "레이블 선택 이유"
}
```

Few-shot 예시를 제공하면 원하는 레이블 스타일과 추상화 수준을 LLM에게 전달할 수 있다.

```
예시:
- ["배송", "택배", "도착"] → "배송 서비스"
- ["품질", "불량", "교환"] → "제품 품질 문제"
- ["환불", "취소", "반품"] → "주문 취소/환불"

```

### 6.4.4 레이블링 품질 평가

LLM 레이블링의 품질을 평가하기 위해서는 여러 측면에서 검토가 필요하다. 먼저 일관성 검토를 통해 유사한 키워드 집합에 대해 일관된 레이블이 생성되는지 확인해야 한다. 예를 들어 ["배송", "택배", "도착"]과 ["배달", "택배", "수령"]이라는 두 키워드 집합에 대해 유사한 레이블이 부여되는지 살펴본다. 또한 도메인 적합성을 검토하여 생성된 레이블이 해당 비즈니스 도메인에서 의미 있고 실무자가 이해할 수 있는 용어인지 확인해야 한다. 마지막으로 샘플 검증을 통해 각 토픽의 대표 문서를 실제로 읽어보고 레이블이 문서 내용을 적절하게 대표하는지 수동으로 확인하는 과정이 필요하다. 레이블링 결과가 만족스럽지 않으면 프롬프트를 조정하거나 Few-shot 예시를 추가하여 품질을 개선할 수 있다. API 비용이 부담되거나 오프라인 환경에서는 규칙 기반 레이블링을 사용할 수 있다. 이 방식은 도메인 지식을 직접 인코딩해야 하지만, API 비용 없이 빠르게 레이블링할 수 있다.

## 6.5 동적 토픽: 토픽 생명주기 분석

### 6.5.1 동적 토픽 모델링이란?

동적 토픽 모델링 (Blei & Lafferty, 2006)은 전체 코퍼스를 단일 시점으로 보는 정적 분석과 달리, 문서에 타임스탬프를 부여하여 시간에 따른 토픽의 등장·증가·감소를 추적하는 방법론이다. 실무적 응용으로는 소셜 미디어의 신흥 이슈 조기 탐지, 브랜드·여론 변화 모니터링, 정책 시행 효과 분석, 학술 연구 주제의 진화 관찰 등이 있으며, 시계열 빈도(예: 월별·분기별)로 토픽 빈도를 집계해 추세 패턴(급격 상승/완만 상승/하락/주기성/안정)을 식별한다. 
구현 측면에서는 문서→임베딩→토픽 할당 후 timestamps를 기준으로 binning(nr_bins 등)을 수행하고 BERTopic의 topics_over_time 같은 API로 시계열을 생성·시각화한다. 해석 시에는 자동 지표(빈도·상관·변화율)와 대표 문서·키워드의 사람 검증을 결합해 급등 원인·주제 전환·노이즈 여부를 판단하고, 필요시 토픽 재명명·세분화·병합 또는 노이즈 재할당(reduce_outliers 등)을 적용한다. 

### 6.5.2 BERTopic의 동적 토픽 분석

BERTopic은 `topics_over_time()` 메서드를 통해 동적 토픽 분석을 지원한다. 이 기능은 각 문서에 타임스탬프를 제공하면 시간에 따른 토픽 빈도 변화를 계산한다.

**주요 파라미터:**

`nr_bins` 파라미터는 시간을 몇 개의 구간으로 나눌지를 결정한다. 3년치 데이터를 분석할 때 `nr_bins=36`으로 설정하면 월별 분석이, `nr_bins=12`로 설정하면 분기별 분석이 된다.

_구체적인 코드와 실행 결과는 6.5.4절의 통합 실습에서 확인할 수 있다._

### 6.5.3 토픽 변화 시각화 및 해석
동적 토픽 분석 결과를 해석할 때는 체계적인 프레임워크를 활용하는 것이 효과적이다. 
첫 번째 단계는 추세 유형을 식별하는 것이다. 급격한 상승은 짧은 기간에 빈도가 급증하는 패턴으로, 새로운 이슈 발생이나 위기 상황을 나타낸다. 점진적 상승은 완만하게 증가하는 패턴으로, 장기적으로 관심이 증가하는 주제를 의미한다. 급격한 하락은 짧은 기간에 빈도가 급감하는 패턴으로, 이슈가 해결되었거나 관심이 다른 곳으로 이동했음을 나타낸다. 주기적 변동은 규칙적으로 등락하는 패턴으로, 계절성이나 정기 이벤트와 관련된 주제에서 나타난다. 안정적 패턴은 일정한 빈도를 유지하는 것으로, 지속적으로 관심을 받는 주제를 의미한다.


BERTopic은 `visualize_topics_over_time()` 메서드로 시간에 따른 토픽 변화를 시각화할 수 있다.

```python
# 상위 5개 토픽의 시간별 변화 시각화
fig = topic_model.visualize_topics_over_time(
    topics_over_time,
    top_n_topics=5
)
fig.show()
```

두 번째 단계는 아래와 같이 토픽 간 관계를 분석하는 것이다. 상관관계가 높은 토픽 쌍은 함께 언급되는 경향이 있다. 예를 들어 "코로나"와 "재택근무"의 높은 양의 상관관계는 팬데믹이 업무 환경 변화를 촉발했음을 나타낸다.

```python
# 토픽 간 시계열 상관관계 계산
import pandas as pd

# topics_over_time을 피벗하여 토픽별 시계열 생성
pivot_df = topics_over_time.pivot(
    index='Timestamp',
    columns='Topic',
    values='Frequency'
).fillna(0)

# 상관관계 행렬
correlation_matrix = pivot_df.corr()
```

**토픽 생명 주기 분석**

동적 토픽 분석에서 한 단계 더 나아가, 각 토픽의 **출현(emergence), 최고점(peak), 소멸(decline)** 시점을 자동으로 식별하는 생명 주기 분석을 수행할 수 있다. 이는 특정 이슈가 언제 처음 등장했는지, 언제 가장 활발히 논의되었는지, 언제 관심이 사라졌는지를 정량적으로 파악하는 데 유용하다.

```python
def analyze_topic_lifecycle(topics_over_time, topic_model, threshold=0.01):
    """토픽의 출현, 최고점, 소멸 시점을 분석"""
    topic_lifecycle = {}

    unique_topics = [t for t in topics_over_time["Topic"].unique() if t != -1]

    for topic in unique_topics:
        topic_data = topics_over_time[topics_over_time["Topic"] == topic]

        # 출현: 빈도가 임계값을 초과하는 첫 시점
        above_threshold = topic_data[topic_data["Frequency"] > threshold]
        emergence = above_threshold.iloc[0]["Timestamp"] if len(above_threshold) > 0 else None

        # 최고점: 빈도가 최대인 시점
        peak_idx = topic_data["Frequency"].idxmax()
        peak = topic_data.loc[peak_idx, "Timestamp"]
        peak_freq = topic_data.loc[peak_idx, "Frequency"]

        # 소멸: 최고점 이후 빈도가 임계값 아래로 떨어지는 첫 시점
        after_peak = topic_data.loc[peak_idx:]
        below_threshold = after_peak[after_peak["Frequency"] < threshold]
        decline = below_threshold.iloc[0]["Timestamp"] if len(below_threshold) > 0 else None

        topic_lifecycle[topic] = {
            "emergence": emergence,
            "peak": peak,
            "peak_frequency": peak_freq,
            "decline": decline,
            "keywords": [w for w, _ in topic_model.get_topic(topic)[:5]]
        }

    return topic_lifecycle
```

## 6.6 Multimodal Topic Modeling: 텍스트와 이미지 통합 분석

### 6.6.1 멀티모달 토픽 모델링이란?

전통적인 토픽 모델링은 텍스트만을 대상으로 주제를 발견한다. 그러나 현실 세계의 데이터는 단일 모달리티로 존재하지 않는다. 소셜 미디어 게시물은 이미지와 캡션이 함께 있고, 전자상거래 상품은 제품 사진과 설명이 결합되어 있으며, 뉴스 기사는 본문과 관련 이미지가 동반된다. 멀티모달 토픽 모델링(Multimodal Topic Modeling)은 텍스트와 이미지를 동시에 분석하여 더 풍부하고 정확한 토픽을 발견하는 기법이다.

멀티모달 토픽 모델링의 핵심 아이디어는 **텍스트와 이미지를 동일한 벡터 공간에 임베딩**하는 것이다. CLIP(Contrastive Language-Image Pre-training) (Radford et al., 2021)과 같은 멀티모달 모델은 텍스트와 이미지를 512차원의 동일한 공간에 투영하여, 의미적으로 유사한 텍스트와 이미지가 가까운 위치에 배치되도록 학습되었다. 예를 들어 "해변에서 서핑하는 사람" 텍스트와 실제 서핑 이미지는 벡터 공간에서 가까운 거리에 위치한다.

### 6.6.2 비즈니스 활용 사례

멀티모달 토픽 모델링은 다양한 실무 상황에서 활용된다. 첫째, 소셜 미디어 분석에서는 인스타그램, 트위터 같은 플랫폼에서 이미지와 캡션을 함께 분석하여 트렌드를 파악하고, 텍스트만으로는 포착하기 어려운 시각적 트렌드(예: 특정 색상, 스타일)를 발견할 수 있다. 둘째, 전자상거래 상품 분류에서는 상품 이미지와 설명을 결합하여 더 정확한 카테고리 분류와 유사 상품 추천이 가능하며, "빈티지 스타일 원목 테이블"이라는 텍스트와 실제 제품 사진을 함께 분석하면 단일 모달리티보다 정확한 분류가 가능하다. 셋째, 뉴스 및 미디어 모니터링에서는 기사 본문과 이미지를 함께 분석하여 텍스트-이미지 불일치 패턴을 감지하거나 시각적 내러티브를 비교함으로써 여론 흐름을 더 풍부하게 이해할 수 있으며, 이렇게 얻은 인사이트를 가짜 뉴스 판별 모델의 입력으로 확장하는 식으로 종합적인 분석이 가능하다.

### 6.6.3 BERTopic의 Multimodal 지원

BERTopic 0.15 버전부터 멀티모달 토픽 모델링을 공식 지원한다. `bertopic[vision]` 패키지를 설치하면 이미지 처리 기능을 사용할 수 있다. BERTopic의 멀티모달 분석은 두 가지 시나리오를 지원한다.

<표 6-13> BERTopic Multimodal 분석 시나리오

| 시나리오 | 입력 데이터 | 임베딩 방식 | 활용 사례 |
| --- | --- | --- | --- |
| 텍스트 + 이미지 | 문서-이미지 쌍 | CLIP 결합 임베딩 | 소셜 미디어 분석 |
| 이미지 단독 | 이미지만 | CLIP 이미지 임베딩 + 캡션 생성 | 상품 이미지 분류 |

**텍스트와 이미지 결합 분석**

각 문서에 대응하는 이미지가 있을 때 사용하는 방식으로, 인스타그램 게시물이나 상품 설명-사진 쌍 등이 대표적 사례다. 이 방식의 핵심 원리는 CLIP 모델을 활용하여 텍스트와 이미지 각각의 임베딩을 생성한 후 평균화하여 결합 임베딩을 만드는 것이다. 텍스트 임베딩과 이미지 임베딩이 동일한 벡터 공간에서 정의되기 때문에 이러한 평균화가 가능하며, 생성된 결합 임베딩을 기반으로 UMAP 차원 축소와 HDBSCAN 군집화를 수행하여 최종 토픽을 도출한다.

```python
from bertopic import BERTopic
from bertopic.backend import MultiModalBackend
from bertopic.representation import VisualRepresentation

# 1. 멀티모달 임베딩 모델 (CLIP 기반)
embedding_model = MultiModalBackend('clip-ViT-B-32', batch_size=32)

# 2. 시각적 토픽 표현 모델
visual_model = VisualRepresentation()
representation_model = {
    "Visual_Aspect": visual_model
}

# 3. BERTopic 생성 및 학습
topic_model = BERTopic(
    embedding_model=embedding_model,
    representation_model=representation_model,
    verbose=True
)

# 4. 문서와 이미지를 함께 학습
topics, probs = topic_model.fit_transform(
    documents=captions,  # 텍스트 캡션
    images=image_paths   # 이미지 경로 리스트
)
```

결합 임베딩을 통해 "강아지와 산책"이라는 캡션과 실제 산책 이미지가 함께 군집화되어, 텍스트만으로는 구분하기 어려운 "반려동물 일상" 토픽을 더 정확하게 발견할 수 있다.

**이미지 단독 토픽 모델링**

텍스트 데이터 없이 이미지만으로 토픽을 발견하는 방식으로, 대량의 제품 사진, 의료 영상, 위성 이미지 등 텍스트 레이블이 없는 이미지 컬렉션 분석에 유용하다. 이 방식의 핵심 원리는 CLIP으로 이미지 임베딩을 생성하고 군집화한 후, 각 토픽의 대표 이미지에 대해 Image-to-Text 모델(예: ViT-GPT2)로 자동 캡션을 생성하여 토픽을 설명하는 것이다. 이를 통해 사람이 직접 레이블을 부여하지 않아도 이미지 군집의 특성을 자연어로 표현할 수 있다.

```python
from bertopic.representation import VisualRepresentation

# 이미지-텍스트 변환을 포함한 시각 표현 모델
representation_model = {
    "Visual_Aspect": VisualRepresentation(
        image_to_text_model="nlpconnect/vit-gpt2-image-captioning"
    )
}

# BERTopic 학습 (documents=None으로 이미지만 사용)
topic_model = BERTopic(
    embedding_model=embedding_model,
    representation_model=representation_model
)
topics, probs = topic_model.fit_transform(
    documents=None,  # 텍스트 없음
    images=images    # 이미지 데이터
)
```

| 1 | footwear, leather, boots, flat | 신발 형태, 가죽/캔버스 질감 | 신발류 | 4 |
| 2 | strap, gold, chain, bag, canvas | 가방 형태, 금속 스트랩 | 가방/액세서리 | 4 |

흥미로운 결과로, 드레스와 재킷이 동일한 Topic 0으로 군집화되었다. 이는 CLIP 모델이 "착용하는 의류"라는 시각적 공통점을 포착했기 때문이다. 반면 신발(Topic 1)과 가방(Topic 2)은 각각 독립적인 토픽으로 분리되어, 형태적 차이가 명확한 카테고리는 별도 토픽으로 구분됨을 확인할 수 있다. 단일 모달리티(텍스트만) 분석 시에는 "leather" 키워드가 재킷, 신발, 가방에 공통으로 나타나 토픽이 혼동되는 경향이 있으나, 이미지 데이터를 결합한 멀티모달 분석에서는 시각적 형태의 차이가 벡터 공간에 반영되어 기능별 카테고리를 효과적으로 분리해냈다.

**Multimodal vs Unimodal 토픽 품질 비교**

동일한 20개 샘플 데이터에 대해 분석 방식별 품질을 비교한 결과다.

<표 6-15> Unimodal vs Multimodal 토픽 품질 비교

| 평가 지표 | 텍스트 단독 | 이미지 단독 | 멀티모달 (결합) |
| --- | --- | --- | --- |
| 토픽 분별력 | 보통 (동의어 혼동) | 보통 (질감 유사성) | **우수 (의미+시각 결합)** |
| 키워드 대표성 | 상 (문구 중심) | 하 (자동 캡션 의존) | **최상 (정교한 키워드)** |
| 노이즈 발생률 | 10% (의미 모호) | 15% (시각적 노이즈) | **0% (패턴 명확)** |

멀티모달 접근은 텍스트의 구체성과 이미지의 시각적 맥락을 동시에 활용하므로, 데이터 양이 적은 상황에서도 단일 모달리티보다 훨씬 응집력 있는 토픽을 도출함을 확인할 수 있다.

### 6.6.5 실무 적용 가이드

멀티모달 토픽 모델링을 실무에 적용할 때는 데이터 준비, 모델 선택, 성능 최적화 세 가지 측면을 고려해야 한다. 데이터 준비 단계에서는 텍스트와 이미지가 1:1로 대응되어야 하며, 이미지 경로를 리스트로 전달하면 전체 이미지를 메모리에 로드하는 것보다 효율적이다. 또한 이미지 크기를 사전에 224x224 등으로 정규화하면 처리 속도가 크게 향상되고 모델 입력 형식과 일치시킬 수 있다.

모델 선택에서는 분석 목적과 가용 리소스를 고려해야 한다. CLIP ViT-B/32는 일반적인 용도에 적합하고 처리 속도가 빠르며, CLIP ViT-L/14는 더 높은 정확도를 제공하지만 더 많은 계산 리소스를 필요로 한다. 의료, 패션 등 특수한 도메인에서는 해당 영역에 특화된 CLIP 모델을 사용하면 성능이 향상될 수 있다.

성능 최적화 측면에서는 배치 크기를 조정하여 GPU 메모리 활용을 최적화하는 것이 중요하다. 대규모 데이터셋을 다룰 때는 임베딩을 미리 계산하여 저장해두면 반복 분석 시 시간을 절약할 수 있다. 또한 이미지 전처리 파이프라인을 병렬화하면 I/O 병목을 줄이고 전체 처리 속도를 향상시킬 수 있다.
