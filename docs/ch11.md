# 제11장: 그래프 분석 - 네트워크에서 패턴 발견

관계 데이터는 전통적인 테이블 형식으로는 표현하기 어려운 복잡한 구조적 정보를 담고 있으므로, 고객 간 상호작용이나 조직 내 협업 관계, 그리고 금융 거래 네트워크 등에서 숨겨진 패턴을 발견하기 위해서는 그래프 분석이 필수적이다. 본 장에서는 네트워크에서 영향력 있는 노드를 식별하고 숨겨진 그룹을 탐지하며, 나아가 미래의 관계를 예측하는 핵심 기법들을 학습한다. 더불어 노드를 벡터로 표현하는 그래프 임베딩 기법과 이를 활용한 사용자-아이템 관계 분석 기반의 추천 시스템까지 포괄적으로 다룬다. 

각주: 11장과 12장은 그래프 데이터과학 분야의 일부를 압축적으로 소개하고 있다. 최근의 그래프 데이터과학은 책 한권의 분량으로도 다 소개가 어려운 새로운 분야이다. 미국의 팔란티어 기업에 의해 유명해진 온톨로지와도 연관되어 있고 최근의 몇몇 국내 IT기업이 진출하려는 분야이기도 하다. 철학적인 베이스가 탄탄한 매우 타당하고 의미있는 방법론이지만 최근의 LLM이 추론분야에서 급속히 발전하면서 다소 논란이 되고 있는 분야이기도 하다. 그럼에도 불구하고 그래프과학 적용이 보다 타당한 부분이 있기 때문에 두 장에 걸쳐 간략히 소개하기로 한다. 그래프 과학은 추후 집필 예정이다. 

## 11.1 그래프 분석이 해결하는 문제

### 11.1.1 왜 관계를 분석하는가

데이터 사이언스 영역에서의 분석은 대부분 개별 관측치의 속성에 집중하는 경향이 있어, 고객의 나이나 구매 금액, 방문 빈도 같은 특성을 분석하여 행동을 예측하곤 한다. 그러나 이러한 접근법은 관측치 간의 관계에서 발생하는 중요한 정보를 간과할 위험이 있다. 반면 그래프 분석은 "누가 누구와 연결되어 있는가"라는 관계 정보를 활용함으로써, 개별 속성만으로는 포착하기 어려운 복잡한 패턴을 규명해낸다. 여기서 그래프(Graph)란 노드(Node)와 엣지(Edge)로 구성된 구조를 의미하는데, 노드는 사용자나 제품, 문서와 같은 분석 대상 개체를 나타내며, 엣지는 친구 관계나 구매, 인용 등 개체 간의 관계를 표현한다. 또한 네트워크(Network)는 이러한 그래프의 실세계 응용을 지칭하는 용어로, 소셜 네트워크나 인용 네트워크, 생물학적 네트워크 등이 그 대표적인 예시다.

### 11.1.2 그래프의 기본 구성 요소

그래프는 노드와 엣지로 시작된다. 노드는 분석 대상(사람, 도시, 웹페이지 등), 엣지는 이들 사이의 관계(친구, 도로, 통화 등)를 나타낸다. 노드는 정보를 담고, 엣지는 방향성·강도 등 다양한 정보를 가질 수 있다.

### 11.1.4 비즈니스 응용 사례

**소셜 네트워크 분석**: 마케팅 캠페인 수행 시 영향력 있는 사용자(인플루언서)를 식별하여 바이럴 마케팅의 시발점으로 활용할 수 있다. 연결 중심성이 높은 사용자는 더 많은 사람에게 도달할 잠재력을 지니며, 매개 중심성이 높은 사용자는 서로 다른 커뮤니티를 연결하는 브리지 역할을 수행한다.

**조직 네트워크 분석**: 이메일 교환이나 협업 패턴을 분석함으로써 정보 흐름의 병목점을 파악할 수 있다. 만약 특정 직원이 과도하게 많은 정보 흐름을 중개하고 있다면 이는 조직의 잠재적 취약점이 될 수 있는데, 해당 직원의 이직이 정보 단절을 유발할 수 있기 때문이다.

**금융 사기 탐지**: 거래 네트워크상에서 비정상적인 패턴을 탐지하는 데에도 활용된다. 정상적인 거래가 특정 구조적 특성(예: 별형 패턴)을 보이는 것과 대조적으로, 자금 세탁 네트워크는 순환 구조나 빠른 다단계 이동과 같은 특이한 양상을 띠는 경향이 있다.

**추천 시스템**: 사용자-아이템 이분 그래프(Bipartite Graph)에서 유사한 사용자나 아이템을 찾아내어 개인화된 추천을 생성한다. Netflix나 Amazon과 같은 주요 플랫폼의 추천 엔진은 본질적으로 이러한 그래프 분석에 기반을 두고 있다.

### 11.1.5 그래프 분석의 핵심 도구와 예제 네트워크

이 장에서는 그래프 분석의 대표적인 벤치마크 데이터셋인 Zachary's Karate Club을 활용한다. 이 데이터셋은 1977년 Wayne Zachary가 미국 대학 가라테 클럽의 사회적 관계를 2년간 관찰하여 수집한 것으로, 34명의 멤버(노드)와 78개의 친분 관계(엣지)로 구성된다. 클럽은 강사 Mr. Hi(노드 0)와 회장 Officer(노드 33) 간의 갈등으로 인해 결국 두 그룹으로 분열되었으며, 이 역사적 사건은 네트워크 구조가 집단의 분열을 어떻게 예측할 수 있는지를 보여주는 고전적 사례가 되었다. ## 11.2 중심성 분석: 영향력 있는 노드 찾기

### 11.2.1 핵심 질문: 네트워크에서 가장 중요한 노드는 무엇인가?

중심성(Centrality)은 네트워크 내에서 노드의 중요도를 정량화하는 척도이지만, '중요하다'는 개념은 분석 맥락에 따라 다양하게 정의될 수 있다. 예컨대 많은 친구를 가진 사람이 중요한지, 혹은 서로 다른 그룹을 연결하는 사람이 중요한지는 관점에 따라 다르다. 따라서 중심성 지표는 이러한 다각적인 관점에서 노드의 중요도를 측정하도록 설계되었다.

### 11.2.2 4가지 중심성 지표

**연결 중심성(Degree Centrality)**: "이 노드는 몇 명과 직접 연결되어 있는가?"

가장 직관적인 중심성 지표로, 연결이 많을수록 중요하다고 본다. 수식은 다음과 같다.

Cᴰ(v) = deg(v) / (n - 1)

deg(v)는 노드 v의 연결 수, n은 전체 노드 수다. 분모 (n-1)은 가능한 최대 연결 수로, 이를 나눔으로써 0과 1 사이의 값으로 정규화한다. 예를 들어 10명으로 구성된 네트워크에서 어떤 사람이 6명과 연결되어 있다면, 연결 중심성은 6/9 = 0.67이다. 소셜 네트워크에서 팔로워가 많은 인플루언서나 조직에서 여러 부서와 협업하는 직원이 높은 연결 중심성을 갖는다.

**매개 중심성(Betweenness Centrality)**: "다른 사람들이 서로 연락하려면 이 노드를 얼마나 거쳐야 하는가?"

네트워크에서 정보가 흐를 때 "중개자" 역할을 하는 노드를 찾는다. 수식은 다음과 같다.

Cᴮ(v) = Σₛ≠ᵥ≠ₜ (σₛₜ(v) / σₛₜ)

σₛₜ는 노드 s에서 t까지의 최단 경로 개수이고, σₛₜ(v)는 그 중 노드 v를 통과하는 경로의 개수다. 즉, "s에서 t로 가는 최단 경로 중 몇 퍼센트가 v를 지나는가"를 모든 노드 쌍에 대해 합산한 값이다. 예를 들어 A-B-C가 일렬로 연결된 네트워크에서 B의 매개 중심성은 최대가 된다. A와 C가 소통하려면 반드시 B를 거쳐야 하기 때문이다. 매개 중심성이 높은 노드가 이탈하면 네트워크가 단절되거나 소통 경로가 크게 늘어날 수 있어, 조직에서 핵심 인력을 파악하는 데 유용하다.

**근접 중심성(Closeness Centrality)**: "이 노드에서 다른 모든 노드까지 평균적으로 얼마나 빨리 도달할 수 있는가?"

네트워크의 "지리적 중심"에 위치한 노드를 찾는다. 수식은 다음과 같다.

Cᶜ(v) = (n - 1) / Σᵤ d(v, u)

d(v, u)는 노드 v에서 u까지의 최단 거리(거쳐야 하는 엣지 수)다. 분모는 v에서 다른 모든 노드까지 거리의 합이므로, 이 값이 작을수록 근접 중심성이 높아진다. 예를 들어 별 모양 네트워크에서 중앙 노드는 다른 모든 노드와 거리가 1이므로 근접 중심성이 최대다. 반면 가장자리 노드는 중앙을 거쳐야 다른 노드에 도달할 수 있어 근접 중심성이 낮다. 정보를 빠르게 전파해야 할 때 어디서 시작할지 결정하는 데 활용된다.

**PageRank**: "이 노드는 중요한 노드들로부터 얼마나 많은 연결을 받고 있는가?"

Google의 웹페이지 순위 알고리즘에서 유래한 지표로, 단순한 연결 수가 아니라 "누구로부터 연결받았는가"를 고려한다. 수식은 다음과 같다.

PR(v) = (1 - d)/n + d · Σᵤ∈In(v) PR(u) / |Out(u)|

d는 감쇠 계수(보통 0.85), In(v)는 v로 들어오는 링크를 가진 노드들, Out(u)는 u에서 나가는 링크 수다. 핵심 아이디어는 "중요한 페이지가 링크한 페이지도 중요하다"는 것이다. 예를 들어 무명 블로그 100개에서 링크를 받는 것보다 네이버 메인에서 링크 1개를 받는 것이 PageRank를 더 크게 높인다. 이 재귀적 특성 덕분에 스팸 링크에 강건하며, 학술 인용 네트워크에서 논문의 실질적 영향력을 평가하는 데도 효과적이다.

<표 11-2> 중심성 지표 비교

| 지표        | 측정 관점          | 계산 복잡도      | 적합한 시나리오          |
| ----------- | ------------------ | ---------------- | ------------------------ |
| 연결 중심성 | 직접 연결 수       | O(n)             | 인기도, 활동성 측정      |
| 매개 중심성 | 정보 흐름 중개     | O(n²) 또는 O(nm) | 병목점 식별, 브리지 역할 |
| 근접 중심성 | 네트워크 중심 위치 | O(n²)            | 정보 전파 속도           |
| PageRank    | 재귀적 중요도      | O(k·m), k=반복   | 인용 영향력, 웹 순위     |

### 11.2.3 실습: Karate Club 네트워크 중심성 분석

Zachary의 Karate Club은 34명의 멤버로 구성된 실제 소셜 네트워크로, 두 리더(Mr. Hi와 Officer) 간의 갈등으로 인해 클럽이 분열된 역사적 사례를 담고 있다. 이 데이터셋에서 4가지 중심성을 계산함으로써 두 리더의 구조적 위치를 심층적으로 분석해 본다.

```python
# Karate Club 중심성 분석 (핵심 코드)
import networkx as nx

G = nx.karate_club_graph()

# 4가지 중심성 계산
centrality = {
    "degree": nx.degree_centrality(G),
    "betweenness": nx.betweenness_centrality(G),
    "closeness": nx.closeness_centrality(G),
    "pagerank": nx.pagerank(G, alpha=0.85)
}
```

## 11.3 커뮤니티 탐지: 숨겨진 그룹 발견

### 11.3.1 핵심 질문: 네트워크에서 촘촘하게 연결된 그룹은 어떤 것인가?

커뮤니티(Community)는 내부적으로는 밀접하게 연결되어 있으나 외부와는 드물게 연결된 노드 집합을 의미하며, 소셜 네트워크의 친구 그룹이나 학술 네트워크의 연구 분야, 조직 네트워크의 부서 등이 그 대표적인 예이다. 커뮤니티 탐지는 네트워크의 메조(meso) 수준 구조를 발견하여 그룹 간 상호작용 패턴을 이해하는 데 있어 핵심적인 역할을 수행한다.

### 11.3.2 모듈성(Modularity): 커뮤니티 품질 척도

모듈성(Modularity)은 커뮤니티 분할의 품질을 정량화하는 지표로, 동일 커뮤니티 내 실제 엣지 수가 무작위 그래프에서 기대되는 엣지 수보다 얼마나 많은지를 측정한다.

Q = (1/2m) · Σᵢⱼ (Aᵢⱼ - kᵢkⱼ/2m) · δ(cᵢ, cⱼ)

여기서 Aᵢⱼ는 인접 행렬, kᵢ는 노드 i의 연결 수, m은 총 엣지 수, 그리고 δ(cᵢ, cⱼ)는 i와 j가 같은 커뮤니티일 때 1의 값을 갖는다. 모듈성 Q는 -0.5에서 1 사이의 값을 가지며, 통상적으로 0.3 이상이면 유의미한 커뮤니티 구조가 존재한다고 해석한다.

직관적으로 해석하면, 모듈성은 "커뮤니티 내부 연결 밀도가 우연히 기대되는 수준보다 얼마나 높은가"를 측정하는 것으로, 노드들을 무작위로 그룹화하면 Q ≈ 0이 되는 반면 실제 그룹 구조가 존재하면 Q > 0이 된다.

### 11.3.3 Louvain 알고리즘

Louvain 알고리즘(Blondel et al., 2008)은 대규모 네트워크에서 효율적으로 커뮤니티를 탐지하기 위한 탐욕적(greedy) 최적화 방법으로, 모듈성을 최대화하기 위해 두 단계의 과정을 반복적으로 수행한다. 이 과정은 더 이상 모듈성이 증가하지 않을 때까지 지속되며, 이를 통해 계층적 커뮤니티 구조를 발견할 수 있다. 시간 복잡도는 O(n log n) 수준으로, 수백만 노드 규모의 네트워크에도 적용 가능하다는 장점이 있다.

**1단계 (지역 최적화)**: 각 노드를 이웃 커뮤니티로 이동시켰을 때 모듈성 증가가 가장 큰 커뮤니티로 할당하며, 더 이상 개선이 없을 때까지 이 과정을 반복한다.

**2단계 (네트워크 압축)**: 각 커뮤니티를 하나의 "슈퍼 노드"로 축소하여 새로운 그래프를 구성한다. 이때 커뮤니티 내부 엣지는 슈퍼 노드의 자기 루프가 되고, 커뮤니티 간 엣지는 슈퍼 노드 간의 엣지로 변환된다.

| 특성        | Louvain                  | 스펙트럴 군집화           |
| ----------- | ------------------------ | ------------------------- |
| 접근법      | 탐욕적 최적화            | 그래프 Laplacian 고유벡터 |

| 결과 안정성 | 시드에 따라 변동 가능    | 안정적                    |
| 계층 구조   | 지원 (해상도 조절)       | 미지원                    |

## 11.4 링크 예측: 미래 관계 예측

### 11.4.1 핵심 질문: 앞으로 어떤 두 노드가 연결될 가능성이 높은가?

링크 예측(Link Prediction)은 현재 존재하지 않는 엣지가 미래에 형성될 가능성을 추정하는 기법이다. 이는 소셜 네트워크의 "알 수도 있는 사람" 추천이나 학술 네트워크의 잠재적 협업자 식별, 생물학 네트워크의 단백질 상호작용 예측 등 다양한 분야에 활용된다. 핵심적인 아이디어는 네트워크 구조상에서 노드 쌍의 유사성을 측정하여 연결 가능성을 점수화하는 데 있다.

### 11.4.2 구조적 유사도 지표

**공통 이웃(Common Neighbors)**: 두 노드가 공유하는 이웃의 수가 많을수록 연결 가능성이 높다는 원리다. N(u)를 노드 u의 이웃 집합이라 할 때, 직관적으로 공통 친구가 많은 두 사람은 서로 알게 될 가능성이 높다고 볼 수 있다.

CN(u, v) = |N(u) ∩ N(v)|

**Jaccard 유사도**: 공통 이웃의 수를 전체 이웃의 합집합 크기로 정규화하여 노드 연결 수의 영향을 보정한다. 연결 수가 많은 허브 노드는 필연적으로 공통 이웃이 많을 수밖에 없으므로, Jaccard 유사도는 이러한 편향을 완화하는 역할을 한다.

Jaccard(u, v) = |N(u) ∩ N(v)| / |N(u) ∪ N(v)|

**Adamic-Adar (AA) 지수**: 공통 이웃의 희귀성에 가중치를 부여하는 방식이다. 즉, 연결이 적은(희귀한) 공통 이웃에 더 높은 가중치를 준다. 친구가 많은 사람보다는 친구가 적은 사람을 통해 연결되는 것이 더 강력한 신호라는 가정에 기반하며, 예를 들어 유명인과의 공통 친구보다 일반인 친구를 공유하는 것이 관계 예측에 더 유의미하다는 것이다.

AA(u, v) = Σₖ∈N(u)∩N(v) 1 / log(|N(k)|)

**Preferential Attachment**: 연결 수가 많은 노드끼리 더 연결되기 쉽다는 "부익부" 가설에 기반한다. 이는 웹 링크나 인용 네트워크에서 흔히 관찰되는 "부익부(rich-get-richer)" 현상을 반영한 지표다.

PA(u, v) = |N(u)| · |N(v)|

<표 11-11> 링크 예측 지표 비교

| 지표                    | 핵심 아이디어       | 장점                  | 단점                     |
| ----------------------- | ------------------- | --------------------- | ------------------------ |
| 공통 이웃               | 공유 친구 수        | 단순, 직관적          | 허브 노드에 편향         |
| Jaccard                 | 정규화된 공통 이웃  | 허브 편향 보정        | 희소 네트워크에서 0 다수 |
| Adamic-Adar             | 희귀 공통 이웃 강조 | 정보성 있는 연결 포착 | 계산 비용 약간 높음      |
| Preferential Attachment | 연결 수 곱          | 성장 네트워크에 적합  | 구조 정보 미활용         |

**소셜 네트워크 친구 추천**: LinkedIn의 "알 수도 있는 사람"이나 Facebook의 "친구 추천" 기능은 링크 예측의 대표적 응용 사례다. 이는 공통 친구가 많고 유사한 프로필을 가진 사용자를 우선적으로 추천하는 알고리즘에 기반한다.

## 11.5 그래프 임베딩: 노드를 벡터로 표현

### 11.5.1 핵심 질문: 노드를 저차원 벡터로 어떻게 표현할 수 있는가?

그래프 임베딩(Graph Embedding)은 노드를 저차원 벡터 공간에 매핑함으로써 그래프 구조를 벡터 연산으로 다룰 수 있게 하는 기법이다. 임베딩 공간에서 가까운 노드는 원본 그래프에서도 유사한 관계(예: 연결되어 있거나 이웃이 비슷한 경우)를 갖게 되며, 이를 통해 노드 분류나 클러스터링, 시각화, 링크 예측 등 다양한 다운스트림 작업에 효과적으로 활용할 수 있다.

### 11.5.2 DeepWalk: 랜덤 워크 + Skip-gram

DeepWalk(Perozzi et al., 2014)는 그래프에서 랜덤 워크(Random Walk)를 수행하여 노드 시퀀스를 생성하고, 이를 Word2Vec의 Skip-gram 모델로 학습하는 방식이다. 여기서 랜덤 워크란 현재 노드에서 연결된 이웃 중 하나를 무작위로 선택하여 이동하는 과정을 반복하는 것으로, 시계열 분석에서 다음 값이 현재 값에 무작위 변동을 더한 형태로 결정되는 랜덤 워크 모형과 동일한 원리다. Skip-gram은 중심 노드가 주어졌을 때 주변 노드를 예측하도록 학습되므로, 결과적으로 자주 함께 등장하는 노드들은 유사한 임베딩 벡터를 갖게 된다.

**알고리즘 절차**:
1. 각 노드에서 시작하여 k번의 랜덤 워크를 수행한다.
2. 워크 길이 L만큼 이웃 노드로 무작위 이동한다.
3. 이렇게 생성된 노드 시퀀스를 마치 "문장"처럼 취급한다.
4. Skip-gram 모델을 사용하여 각 노드의 임베딩을 학습한다.

### 11.5.3 Node2Vec: BFS/DFS 균형 조정

Node2Vec(Grover & Leskovec, 2016)은 DeepWalk를 확장하여 탐색 전략을 정교하게 제어하기 위해 두 개의 하이퍼파라미터 p와 q를 도입하였다. 여기서 p(Return Parameter)는 이전 노드로 돌아갈 확률을 조절하며, q(In-out Parameter)는 외부로 나갈 확률을 결정한다. p가 작으면 BFS처럼 지역 구조를 탐색하는 경향이 강해지고, 반면 p가 크면 멀리 탐색하는 성향을 보인다. 마찬가지로 q가 작으면 DFS처럼 멀리 탐색하고, q가 크면 지역에 머무르는 특성을 갖는다.

<표 11-14> Node2Vec 파라미터 조합

| 설정         | 탐색 특성       | 포착하는 유사성    |
| ------------ | --------------- | ------------------ |
| p=1, q=1     | 균형 (DeepWalk) | 구조적 + 커뮤니티  |
| p < 1, q > 1 | BFS-like        | 지역 이웃 구조     |
| p > 1, q < 1 | DFS-like        | 전역 커뮤니티 소속 |

| p=0.5, q=2 (BFS-like) | 0.600                     |
| p=2, q=0.5 (DFS-like) | 0.600                     |

Karate Club과 같이 작고 밀집된 네트워크에서는 파라미터 설정에 따른 차이가 크지 않을 수 있다. 그러나 대규모 희소 네트워크에서는 BFS-like 설정이 지역 구조 포착에 유리한 반면, DFS-like 설정은 전역 커뮤니티 포착에 더 효과적일 수 있다.

### 11.5.6 그래프 임베딩의 활용

**유사 노드 검색**: 임베딩 공간에서 코사인 유사도나 유클리드 거리를 이용하여 가장 가까운 노드를 찾을 수 있다. 이는 유사 사용자 추천이나 관련 제품 검색 등에 널리 활용된다.

**노드 분류**: 임베딩 벡터를 특성(feature)으로 사용하여 노드의 레이블을 예측하는 작업이다. 예를 들어 소셜 네트워크에서 사용자의 관심사나 악성 계정 여부를 분류하는 데 사용될 수 있다.

**그래프 시각화**: 64차원과 같은 고차원 임베딩을 t-SNE나 UMAP을 통해 2차원으로 축소하면, 커뮤니티 구조를 직관적으로 시각화할 수 있다. 이때 동일 커뮤니티에 속한 노드들은 2차원 공간에서도 서로 가깝게 위치하게 된다.

<표 11-17> 그래프 임베딩 방법론 비교

| 방법     | 핵심 아이디어         | 장점                   | 단점                     |
| -------- | --------------------- | ---------------------- | ------------------------ |
| DeepWalk | 랜덤 워크 + Skip-gram | 구현 간단, 확장성      | BFS/DFS 균형 조절 불가   |
| Node2Vec | p, q로 탐색 전략 조절 | 유연한 유사성 정의     | 하이퍼파라미터 튜닝 필요 |
| LINE     | 1차/2차 근접성 보존   | 대규모 네트워크 효율적 | 고차 근접성 미반영       |
| SDNE     | 오토인코더 기반       | 비선형 구조 포착       | 계산 비용 높음           |

Node2Vec은 소규모에서 중규모 네트워크에 이르기까지 가장 널리 사용되는 방법으로, 파라미터 조정을 통해 다양한 유사성 개념을 포착할 수 있다는 장점이 있다. 반면 대규모 네트워크의 경우 LINE이나 분산 학습이 가능한 변형 모델을 고려하는 것이 좋다.

## 11.6 추천 시스템: 사용자-아이템 관계 분석

### 11.6.1 핵심 질문: 사용자가 어떤 아이템을 선호할까?

추천 시스템(Recommendation System)은 사용자의 과거 행동 데이터를 분석하여 선호할 만한 아이템을 예측하는 기술이다. 사용자-아이템 상호작용은 이분 그래프(Bipartite Graph)로 모델링할 수 있는데, 이러한 그래프 분석 관점에서 볼 때 추천은 링크 예측 문제의 특수한 형태로 해석될 수 있다.

### 11.6.2 협업 필터링: 유사한 사용자/아이템 활용

협업 필터링(Collaborative Filtering)은 사용자 간 또는 아이템 간의 유사성을 기반으로 추천을 수행한다. 가장 널리 사용되는 구현 방식은 K-최근접 이웃(K-Nearest Neighbors, KNN) 알고리즘으로, 유사도가 가장 높은 K명의 이웃을 찾아 그들의 평점을 가중 평균하여 예측한다. 사용자 간 유사도를 계산하면 User-KNN, 아이템 간 유사도를 계산하면 Item-KNN이 된다.

**사용자 기반(User-KNN)**: "나와 비슷한 사용자가 좋아한 아이템을 추천한다"는 원리다. 사용자 u의 아이템 i에 대한 예측 평점은 아래 식과 같이 계산된다. 여기서 μᵤ는 사용자 u의 평균 평점, sim(u, v)는 사용자 간 유사도(코사인, 피어슨 등), N(u)는 u와 유사한 이웃 집합을 나타낸다.

r̂ᵤᵢ = μᵤ + Σᵥ∈N(u) sim(u, v) · (rᵥᵢ - μᵥ) / Σᵥ∈N(u) |sim(u, v)|

**아이템 기반(Item-KNN)**: "내가 좋아한 아이템과 비슷한 아이템을 추천한다"는 접근법이다. Amazon의 "이 상품을 본 고객이 함께 본 상품" 기능이 대표적인 예시이며, 사용자의 이력 변화에 덜 민감하기 때문에 대규모 시스템에서 선호되는 경향이 있다.

<표 11-18> 사용자 기반 vs 아이템 기반 협업 필터링

| 특성          | 사용자 기반                 | 아이템 기반                  |
| ------------- | --------------------------- | ---------------------------- |
| 유사도 계산   | 사용자 간                   | 아이템 간                    |
| 확장성        | 사용자 증가 시 재계산       | 아이템 유사도 사전 계산 가능 |
| 설명 가능성   | "비슷한 취향의 사용자가..." | "이 상품과 비슷한 상품이..." |
| 적합 시나리오 | 아이템 > 사용자             | 사용자 > 아이템              |

### 11.6.3 행렬 분해(Matrix Factorization): 잠재 요인 모델

행렬 분해의 핵심 아이디어는 사용자와 아이템을 동일한 "취향 공간"에 배치하는 것이다. 예를 들어 영화 추천에서 k=3차원을 사용한다면, 각 차원이 "액션 선호도", "로맨스 선호도", "예술영화 선호도"를 나타낸다고 상상할 수 있다. 사용자 A가 [0.9, 0.2, 0.1]이고 영화 X가 [0.8, 0.3, 0.2]라면, 두 벡터가 가까우므로 A는 X를 좋아할 가능성이 높다.

수식으로 표현하면, 사용자-아이템 평점 행렬 R을 두 개의 저차원 행렬로 분해한다.

R ≈ P · Qᵀ

여기서 P는 사용자 행렬(각 행이 사용자의 잠재 벡터), Q는 아이템 행렬(각 행이 아이템의 잠재 벡터)이다. 사용자 u가 아이템 i에 줄 예측 평점은 두 벡터의 내적으로 계산된다.

r̂ᵤᵢ = pᵤ · qᵢᵀ = Σₖ pᵤₖ · qᵢₖ

실제로 학습된 잠재 요인이 항상 "액션", "로맨스"처럼 명확하게 해석되지는 않는다. 데이터에서 자동으로 학습된 추상적 패턴일 수 있으며, 이러한 특성은 모델이 사람이 인지하지 못한 미묘한 선호 패턴까지 포착할 수 있게 해준다.

**SVD (Singular Value Decomposition)**: "특이값 분해"라는 이름의 선형대수 기법에서 유래했으나, 추천 시스템에서 사용하는 SVD는 원래의 수학적 SVD와는 다르다. 추천 시스템의 SVD는 결측치(사용자가 보지 않은 영화)가 대부분인 희소 행렬을 다루기 위해 변형된 방법으로, "알려진 평점만으로 P, Q를 학습"하는 것이 핵심이다.

학습 과정을 직관적으로 설명하면 다음과 같다. 사용자 1이 영화 A에 4점을 줬다면, p₁ · qₐᵀ가 4에 가까워지도록 P와 Q를 조금씩 조정한다. 이 과정을 모든 알려진 평점에 대해 반복하면, 결국 "아직 보지 않은 영화"에 대해서도 예측이 가능해진다. Netflix Prize(2009) 우승 솔루션의 핵심 기법이기도 하다.

min Σ(u,i)∈Known (rᵤᵢ - pᵤ · qᵢᵀ)² + λ(||pᵤ||² + ||qᵢ||²)

위 수식에서 첫 번째 항은 "예측 오차를 최소화"하고, 두 번째 항(정규화)은 "벡터 값이 지나치게 커지는 것을 방지"하여 과적합을 억제한다.

<표 11-19> 협업 필터링 vs 행렬 분해 비교

| 특성        | 협업필터링        | 행렬 분해    |
| ----------- | ------------------- | ------------------ |
| 학습 방식   | 유사도 직접 계산    | 잠재 요인 학습     |
| 메모리      | 전체 평점 행렬 저장 | 압축된 요인 저장   |
| 확장성      | 대규모 어려움       | 대규모 가능        |
| 희소성 처리 | 공통 아이템 필요    | 잠재 공간에서 보간 |
| Cold-start  | 취약                | 취약 (동일)        |

### 11.6.4 Cold-start 문제

Cold-start 문제는 신규 사용자나 아이템의 경우 과거 상호작용 데이터가 부재하여 협업 필터링이나 행렬 분해만으로는 적절한 추천을 제공하기 어려운 현상을 말한다. 이를 해결하기 위해 아이템의 장르·태그·설명과 사용자 인구통계·프로필 같은 메타데이터를 활용한 콘텐츠 기반 추천을 적용하거나, 협업 필터링과 결합한 하이브리드 모델을 도입하여 상호보완하는 전략이 효과적이다. 초기에는 전체 인기 아이템을 기본 추천으로 제공하고 온보딩 설문이나 능동적 피드백 수집을 통해 빠르게 선호 신호를 확보해야 하며, 텍스트·카테고리·이미지 등 사이드 정보를 임베딩에 포함하거나 탐험(explore) 전략을 병행하면 cold-start 상황에서도 개인화 품질을 개선할 수 있다.

### 11.6.5 추천 시스템의 평가 지표

추천 시스템의 성능을 평가하기 위해 다양한 지표가 사용된다.

**정확도 지표**:
- **RMSE (Root Mean Square Error)**: 예측 평점과 실제 평점 간의 차이를 나타내며, 값이 작을수록 예측이 정확함을 의미한다.
- **MAE (Mean Absolute Error)**: 평균 절대 오차로, RMSE에 비해 이상치(outlier)에 덜 민감하다는 특징이 있다.

**랭킹 지표**:
- **Precision@K**: 상위 K개 추천 목록 중 실제로 사용자가 선호한 아이템의 비율이다.
- **Recall@K**: 사용자가 선호하는 전체 아이템 중에서 상위 K개 추천에 포함된 아이템의 비율을 의미한다.
- **NDCG (Normalized Discounted Cumulative Gain)**: 추천된 아이템의 순위까지 고려하여 추천 품질을 평가하는 지표다.

**다양성 지표**:
- **Coverage**: 전체 아이템 중 추천 가능한 아이템의 비율을 나타내며, 인기 아이템에만 추천이 편중될 경우 낮아진다.
- **Diversity**: 추천 목록 내 아이템 간의 다양성을 측정한다.

실무에서는 오프라인 평가(RMSE, NDCG)와 온라인 A/B 테스트(클릭률, 구매 전환율)를 병행하는 것이 일반적이다. 이는 오프라인 지표가 우수하더라도 실제 비즈니스 성과와 반드시 일치하지는 않을 수 있기 때문이다.

<표 11-20> 추천 시스템 평가 지표 요약

| 지표 유형 | 지표                | 측정 내용          |
| --------- | ------------------- | ------------------ |
| 정확도    | RMSE, MAE           | 평점 예측 정확도   |
| 랭킹      | Precision@K, NDCG   | 추천 순위 품질     |
| 다양성    | Coverage, Diversity | 추천 범위와 다양성 |
| 비즈니스  | CTR, 전환율         | 실제 사용자 행동   |
