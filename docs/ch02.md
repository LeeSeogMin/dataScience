# 제2장: 데이터 전처리와 특성 공학 - 모델 성능을 좌우하는 원리

본 장에서는 데이터 전처리와 특성 공학(Feature Engineering)이 모델 성능에 어떤 경로로 영향을 미치는지 학습한다. 결측치 처리와 스케일링을 중심으로, 알고리즘이 입력을 어떻게 해석하는지에 따라 전처리 선택이 달라지는 원리를 이해하도록 하겠다. 또한 PCA, t-SNE, UMAP을 비교하여 차원 축소의 목적과 방법을 구분하고, 분석 목적에 맞는 선택 기준을 익힌다. 마지막으로 텍스트와 범주형 변수를 임베딩으로 수치화하는 관점을 정리하여, AI 시대의 특성 표현 방식이 기존 전처리와 어떻게 연결되는지 살펴본다.

## 2.1 결측치 처리: 모델이 결측을 싫어하는 이유

데이터 분석의 첫 단계는 데이터 품질을 확인하는 것이다. 현실 세계의 데이터에서 결측치(missing value)는 피할 수 없다. 설문 응답 거부, 센서 오류, 시스템 장애, 데이터 입력 누락 등 다양한 이유로 결측이 발생한다. 그런데 결측치를 어떻게 처리하느냐에 따라 분석 결과가 크게 달라질 수 있으며, 잘못된 처리는 편향된 결론을 초래한다.

### 2.1.1 결측치가 모델 학습을 방해하는 이유

대부분의 머신러닝 알고리즘은 결측치를 직접 처리할 수 없다. 선형 회귀에서 Xβ를 계산하려면 모든 X 값이 존재해야 한다. K-Means에서 중심점까지 거리를 계산하려면 모든 차원의 좌표가 필요하다. 신경망에서 가중합을 계산하려면 모든 입력이 정의되어야 한다.

결측치가 있으면 해당 행을 계산에서 제외하거나, 어떤 값으로 대체해야 한다. 제외하면 정보 손실이 발생하고, 대체하면 인위적인 패턴이 도입될 수 있다. 어느 쪽이든 트레이드오프가 존재한다.

일부 알고리즘은 결측치를 내재적으로 처리한다. XGBoost, LightGBM 등 그래디언트 부스팅 알고리즘은 분할 시 결측치를 별도 경로로 처리한다. 결측 여부 자체가 정보가 되어 분할 기준으로 활용될 수 있는 것이다. 그러나 이 경우에도 결측 패턴이 학습/테스트 데이터에서 일관적이어야 한다.

### 2.1.2 결측의 세 가지 유형

결측치를 이해하려면 결측이 발생하는 메커니즘을 파악해야 한다. 통계학에서는 결측을 세 가지 유형으로 분류한다.

**완전 무작위 결측(MCAR, Missing Completely At Random)**은 결측 발생이 어떤 변수와도 관련이 없는 경우다. 설문지가 우편 배송 중 분실되어 일부 응답이 누락된 경우가 예시다. MCAR에서는 결측이 있는 데이터와 없는 데이터가 통계적으로 동일한 모집단을 대표한다. 이 경우 단순 삭제(listwise deletion)를 해도 편향이 발생하지 않지만, 표본 크기 감소로 효율성이 떨어진다.

**무작위 결측(MAR, Missing At Random)**은 결측 발생이 관측된 다른 변수에 의존하는 경우다. 고소득자가 소득 정보 공개를 꺼려 결측이 발생하되, 소득과 상관있는 직업 정보는 관측되는 경우다. MAR에서는 관측된 변수를 조건으로 하면 결측이 무작위가 된다. 다중 대체법(multiple imputation)이 효과적이다(van Buuren, 2018).

**비무작위 결측(MNAR, Missing Not At Random)**은 결측 발생이 결측된 값 자체에 의존하는 경우다. 우울증 환자가 증상이 심할수록 설문 응답을 포기하는 경우가 예시다. MNAR은 처리가 가장 어려우며, 결측 메커니즘을 명시적으로 모형화해야 한다(Little & Rubin, 2019).

실무에서는 결측 유형을 정확히 판단하기 어려운 경우가 많다. 그러나 최소한 결측 패턴을 탐색하고, 결측이 특정 그룹에 집중되어 있는지 확인해야 한다.

### 2.1.3 결측치 처리 전략

결측치 처리 전략은 크게 삭제, 단순 대체, 모델 기반 대체, 결측 지시 변수의 네 가지로 구분된다.

삭제(Deletion)는 가장 단순한 방법으로, 결측이 있는 행을 제거하는 것이다. 결측 비율이 낮고(5% 이하) MCAR이 가정될 때 적합하다. 그러나 결측 비율이 높거나 여러 변수에 결측이 분산되어 있으면 대부분의 데이터가 삭제될 수 있다. 한편 변수 삭제는 특정 변수의 결측 비율이 매우 높을 때 고려하는데, 결측이 70% 이상인 변수는 정보량이 제한적이므로 제외하는 것이 나을 수 있다. 다만 결측 자체가 의미 있는 정보일 수 있으므로 도메인 지식에 기반한 판단이 필요하다.

단순 대체(Simple Imputation)는 결측치를 특정 값으로 대체하는 방법이다. 수치형 변수는 평균, 중앙값, 최빈값으로, 범주형 변수는 최빈값이나 별도 범주('Unknown')로 대체한다. 평균 대체는 변수의 평균을 유지하지만 분산을 과소추정하고 다른 변수와의 상관관계를 약화시킨다. 결측 비율이 높으면 분포가 왜곡된다. 이에 비해 중앙값 대체는 이상치에 강건하여 왜도가 큰 분포에서 평균보다 적합하다. 소득이나 가격 등 오른쪽으로 치우친 분포에서 선호된다. 그룹별 대체는 관련 변수의 그룹 내에서 통계량을 계산하는 방식으로, 성별로 구분하여 남성 평균과 여성 평균으로 각각 대체하는 것이 예시다. 이 방법은 MAR 상황에서 단순 대체보다 편향을 줄인다.

모델 기반 대체(Model-based Imputation)는 결측치를 예측 문제로 접근하여 다른 변수를 사용해 결측값을 예측한다. K-최근접이웃(KNN) 대체는 결측이 없는 변수들을 기준으로 유사한 K개 샘플을 찾아 그들의 값으로 대체하는데, 변수 간 관계를 활용하므로 단순 대체보다 정교하다. 반복 대체(Iterative Imputation, MICE)는 각 변수의 결측을 다른 변수들의 함수로 모형화하고 이를 반복적으로 업데이트한다. 이 방법은 변수 간 상관관계를 보존하며 불확실성을 반영할 수 있다.

결측 지시 변수(Missing Indicator)는 결측 여부를 나타내는 이진 변수를 추가로 생성하는 방법이다. 원래 변수는 평균 등으로 대체하되 결측이었는지 여부를 별도 변수로 기록한다. 결측 자체가 예측에 유용한 정보일 때 효과적이다. 예를 들어 신용평가에서 특정 정보의 미제공이 위험 신호일 수 있다.

## 2.2 스케일링: 거리 기반 알고리즘이 필요로 하는 것

변수들의 측정 단위와 범위가 다르면 일부 알고리즘의 성능이 저하된다. 소득(수백만 원)과 나이(수십 년)를 함께 사용할 때, 소득의 절대적 차이가 나이보다 훨씬 크므로 거리 계산에서 소득이 지배적이 된다. 스케일링(Scaling)은 이러한 문제를 해결하기 위해 변수들을 비슷한 범위로 변환한다.

### 2.2.1 스케일링이 필수인 알고리즘

**거리 기반 알고리즘**은 샘플 간 거리를 계산한다. K-최근접이웃(KNN), K-Means 군집화, SVM, DBSCAN 등이 여기에 해당한다. 유클리드 거리 √(Σ(xᵢ - yᵢ)²)에서 범위가 큰 변수가 거리를 지배한다.

예를 들어 집값 예측에서 방 개수(0-10)와 면적(0-10,000)을 사용한다고 하자. 두 집의 방 개수가 1 차이 나고 면적이 100 차이 난다면, 유클리드 거리는 √(1² + 100²) ≈ 100이 되어 방 개수의 영향은 거의 없다. 스케일링 없이는 KNN이 거의 면적만으로 이웃을 결정하게 된다.

**기울기 기반 최적화로 학습하는 모델(신경망 포함)**은 입력 변수들의 크기가 제각각이면 학습이 느려지거나 불안정해질 수 있다. 큰 값의 변수가 업데이트를 좌우해 최솟값까지 가는 길이 지그재그가 되기 때문이다. 스케일링으로 변수 크기를 맞추면 더 곧은 경로로 내려가며 수렴이 빨라진다. 특히 신경망에서 시그모이드나 tanh 같은 활성화 함수를 쓸 때는, 입력이 너무 크거나 작으면 출력이 끝값에 붙어 “거의 안 변하는 구간”이 생길 수 있으므로 입력 스케일을 맞추는 것이 도움이 된다.

### 2.2.2 스케일링이 불필요한 알고리즘

트리 기반 알고리즘은 변수를 분할점(threshold) 기준으로 이진 분할한다. 분할점은 변수 값의 순서에만 의존하므로 스케일이 바뀌어도 분할 결과가 동일하다. 의사결정나무, 랜덤 포레스트, XGBoost, LightGBM 등은 스케일링 없이 사용해도 성능이 동일하다. 이와 유사하게 나이브 베이즈도 각 변수의 조건부 확률을 독립적으로 추정하므로 스케일의 영향을 받지 않는다.

### 2.2.3 스케일링 방법

**표준화(Standardization, Z-score normalization)**는 평균을 0, 표준편차를 1로 변환한다:

z = (x - μ) / σ

표준화된 값은 원래 값이 평균에서 몇 표준편차 떨어져 있는지를 나타낸다. 대부분의 값이 -3에서 +3 사이에 분포하지만, 이상치가 있으면 범위가 넓어질 수 있다. 정규분포를 가정하는 알고리즘에 적합하며, 이상치에 민감하지만 분포 형태를 보존한다.

**Min-Max 정규화**는 값을 [0, 1] 또는 [-1, 1] 범위로 변환한다:

x_scaled = (x - x_min) / (x_max - x_min)

모든 값이 지정된 범위 내에 있으므로 신경망의 입력으로 적합하다. 그러나 이상치가 있으면 대부분의 값이 좁은 범위에 압축되는 문제가 있다. 예를 들어 대부분 0-100 범위인데 하나의 이상치가 10,000이면, 정상 값들은 0-0.01 범위에 몰리게 된다.

**Robust 스케일링**은 중앙값과 사분위범위(IQR)를 사용한다:

x_scaled = (x - median) / IQR

중앙값과 IQR은 이상치에 강건하므로, 이상치가 있는 데이터에 적합하다. 분포의 중심과 산포를 이상치에 영향받지 않고 조정한다.

**MaxAbs 스케일링**은 절대값의 최대로 나눈다:

x_scaled = x / |x_max|

희소 행렬(sparse matrix)에서 0 값을 보존하므로 텍스트 데이터의 TF-IDF 행렬에 유용하다. 0이 의미 있는 데이터에서 사용한다.

### 2.2.4 스케일링 방법 선택 가이드

<표 2-2> 스케일링 방법별 특징 비교

| 방법             | 범위              | 이상치 영향 | 적합한 상황                       |
| ---------------- | ----------------- | ----------- | --------------------------------- |
| 표준화 (Z-score) | 제한 없음         | 민감        | 정규분포 가정, SVM, 로지스틱 회귀 |
| Min-Max          | [0,1] 또는 [-1,1] | 매우 민감   | 신경망, 이상치 없을 때            |
| Robust           | 제한 없음         | 강건        | 이상치 있을 때                    |
| MaxAbs           | [-1,1]            | 민감        | 희소 행렬, TF-IDF                 |

### 2.2.5 주의사항: 데이터 누수 방지

스케일링의 파라미터(평균, 표준편차, 최소값, 최대값 등)는 **반드시 훈련 데이터에서만 계산**해야 한다. 테스트 데이터의 통계량을 사용하면 미래 정보가 누수(data leakage)된다.

```python
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)
```

교차검증에서도 각 폴드의 훈련 세트에서 스케일링 파라미터를 계산해야 한다. scikit-learn의 Pipeline을 사용하면 이 과정이 자동으로 처리된다.

## 2.3 차원 축소: 고차원의 저주를 해결하는 방법

고차원 데이터는 여러 문제를 야기한다. 변수가 많으면 모델 복잡도가 증가하고, 과적합 위험이 높아지며, 계산 비용이 커진다. 더 근본적으로, 고차원 공간에서는 직관이 작동하지 않는 "차원의 저주(curse of dimensionality)" 현상이 발생한다.
각주: 과적합은 모델이 학습 데이터의 패턴뿐 아니라 우연한 잡음까지 외워서, 학습 성능은 좋지만 새 데이터에서는 성능이 떨어지는 현상이다. 쉽게 말해 “문제의 원리를 배운 게 아니라 기출문제 답을 외운 상태”에 가깝다.

### 2.3.1 차원의 저주

고차원 공간에서 데이터 포인트들은 점점 멀어진다. n차원 단위 초입방체에서 무작위로 두 점을 뽑을 때, 기대 거리는 √(n/6)에 비례하여 증가한다. 차원이 높아질수록 모든 점이 서로 비슷하게 멀어지므로, "가까운 이웃"의 개념이 무의미해진다.

또한 고차원에서는 데이터가 공간의 구석(corner)에 집중된다. 2차원에서 단위 원은 단위 정사각형 면적의 π/4 ≈ 78.5%를 차지한다. 그러나 10차원에서 단위 초구는 단위 초입방체 부피의 약 0.25%만 차지한다. 대부분의 데이터가 "가장자리"에 있으므로, 중심 근처에서 정의된 거리 개념이 유용하지 않다.

이러한 현상 때문에 고차원에서는 더 많은 샘플이 필요하다. 차원이 d인 공간을 k개 구간으로 나누면 k^d개 셀이 생긴다. 10개 변수를 각 10개 구간으로 나누면 100억 개 셀이 된다. 각 셀에 의미 있는 샘플 수를 확보하려면 천문학적 데이터가 필요하다.

### 2.3.2 차원 축소의 목적

차원 축소는 고차원 데이터를 저차원으로 변환하면서 중요한 정보를 보존한다. 두 가지 주요 목적이 있다.

**시각화**: 인간은 3차원까지만 직관적으로 이해할 수 있다. 100차원 데이터의 구조를 파악하려면 2-3차원으로 투영해야 한다. 군집 구조, 이상치, 클래스 간 분리 등을 시각적으로 확인할 수 있다.

**전처리**: 후속 모델의 입력 차원을 줄여 계산 효율성을 높이고, 노이즈를 제거하며, 과적합을 방지한다. 특히 샘플 수보다 변수 수가 많은 경우(p > n) 차원 축소가 필수적이다.

### 2.3.3 PCA: 분산을 최대화하는 선형 변환

**주성분 분석(Principal Component Analysis, PCA)**은 가장 널리 사용되는 선형 차원 축소 방법이다. PCA는 데이터의 분산을 최대한 보존하는 방향으로 새로운 좌표축(주성분)을 찾는다.

첫 번째 주성분(PC1)은 데이터의 분산이 최대인 방향이다. 두 번째 주성분(PC2)은 PC1과 직교하면서 남은 분산이 최대인 방향이다. 이 과정을 반복하여 원래 변수 수만큼 주성분을 얻을 수 있으며, 처음 몇 개의 주성분이 전체 분산의 대부분을 설명한다. 수학적으로 PCA는 공분산 행렬의 고유값 분해(eigendecomposition)다. 공분산 행렬 Σ의 고유벡터가 주성분 방향이고, 고유값이 해당 주성분이 설명하는 분산의 크기다. PCA는 한마디로 “데이터 구름을 가장 잘 펼쳐 보이는 방향으로 좌표축을 새로 돌려 잡는 방법”이다. PC1은 데이터 점들을 그 축에 투영했을 때 퍼짐(정보)이 가장 크게 남는 방향, 즉 한 줄로 요약해도 가장 많은 차이를 담는 방향이다. PC2는 PC1과 90도로 겹치지 않게 세우되, PC1로 설명하고 남은 퍼짐을 가장 많이 담는 두 번째 방향이고, 이런 식으로 서로 직각인 새 축들을 순서대로 찾는다. 그래서 앞의 몇 개 축만 써도 데이터의 “대부분의 변화”가 남고, 나머지는 상대적으로 작은 흔들림(세부/잡음)에 불과하다. 수학적으로 공분산 행렬을 고유값 분해한다는 말은, “함께 변하는 정도(공분산)를 가장 깔끔하게 분리해 주는 축(고유벡터)을 찾고, 그 축 방향으로 얼마나 퍼져 있는지(고유값=분산 크기)를 읽는다”는 뜻이다.

각주: 여기서 **직교(직각)**는 “두 방향이 90도로 꺾여 있어서 서로 겹치지 않는다”는 의미이다. 종이에 그린 **가로축(x)과 세로축(y)**처럼, 가로로만 움직이면 세로 방향 변화는 없고(서로 영향을 안 주고), 세로로만 움직이면 가로 방향 변화는 없는 것과 같은 이치이다. PCA에서 “PC2는 PC1과 직교”라는 말은, PC1이 이미 담아낸 변화와 겹치는(중복되는) 방향은  피하고, 그와 완전히 다른 방향에서 남은 퍼짐을 가장 많이 담겠다는 의미이다.

PCA의 주요 장점은 계산 효율성과 결과의 일관성이다. 시간 복잡도가 O(min(p³, n³))로 대규모 데이터에서도 실용적이며, 동일한 데이터에 대해 항상 같은 결과를 반환하므로 재현성이 보장된다. 또한 주성분들이 서로 직교하기 때문에 다중공선성 문제가 자동으로 해결되고, 각 주성분이 설명하는 분산 비율을 통해 적절한 차원 수를 선택할 수 있다. 반면 PCA에는 몇 가지 한계가 존재한다. 선형 변환이므로 비선형 관계를 포착하지 못하며, 분산이 크다고 해서 반드시 예측에 중요한 것은 아니다. PCA는 레이블 정보를 사용하지 않는 비지도 학습이기 때문이다. 또한 주성분은 원래 변수들의 선형 결합이므로 개별 주성분의 의미를 해석하기 어렵다는 단점도 있다.

### 2.3.4 t-SNE: 지역 구조를 보존하는 비선형 변환

**t-SNE(t-distributed Stochastic Neighbor Embedding)**(van der Maaten & Hinton, 2008)는 고차원 데이터를 2-3차원으로 시각화할 때, "가까운 이웃 관계"가 최대한 유지되도록 점들을 배치하는 비선형 차원 축소 방법이다. t-SNE는 거리 자체를 그대로 옮기기보다, 각 점의 주변 이웃이 누구인지에 집중하기 위해 고차원에서의 유사도를 확률로 바꾼다. 구체적으로 점 i가 점 j를 이웃으로 볼 확률 p_{j|i}를 가우시안(정규) 분포 형태로 정의한다:

p_{j|i} ∝ exp(-||x_i - x_j||² / (2σ_i²))

그다음 저차원 좌표 y에서도 비슷한 확률 q_{ij}를 정의하되, 여기서는 Student의 t-분포(꼬리가 두꺼운 분포)를 사용한다:

q_{ij} ∝ (1 + ||y_i - y_j||²)⁻¹

그리고 고차원의 확률 분포 P={p_{ij}}와 저차원의 확률 분포 Q={q_{ij}}가 비슷해지도록 KL-발산(Kullback-Leibler divergence) D_KL(P||Q)를 최소화하면서 y를 최적화한다. 이 과정은 고차원에서 매우 가까운 이웃(큰 p_{ij})이 저차원에서도 가깝게 유지되도록 강하게 유도한다. t-분포를 쓰는 이유는 "밀림 현상(crowding problem)"을 완화하기 위해서다. 고차원에서의 중간 거리들이 2차원으로 압축되면 서로 비슷한 거리로 뭉치기 쉬운데, t-분포는 멀리 떨어진 점에도 상대적으로 더 큰 확률을 부여하므로 군집 사이를 벌릴 여유를 제공해 구조가 덜 뭉개지게 한다.
t-SNE를 사용할 때는 몇 가지 주의점이 있다. 먼저 결과가 확률적이어서 초기화에 따라 매번 다른 결과를 얻을 수 있다. 또한 t-SNE는 전역 구조보다 지역 구조 보존에 최적화되어 있어서 군집 간 상대적 거리나 위치는 신뢰하기 어렵다. perplexity 파라미터가 결과에 큰 영향을 미치므로 여러 값(보통 5-50)을 시도해봐야 한다. 한편 이미 학습된 투영에 새로운 데이터를 추가하기 어려우므로 표본 외(out-of-sample) 확장이 제한적이다. 마지막으로 축 간 거리나 군집 크기는 임의적이므로 절대적 해석에 주의해야 한다.

각주: KL-발산(Kullback-Leibler divergence)은 두 확률 분포가 얼마나 다른지를 측정하는 비대칭적 거리 함수다. D_KL(P||Q) = Σ p_i log(p_i/q_i)로 정의되며, P와 Q가 동일하면 0이고, 차이가 클수록 값이 커진다. t-SNE는 고차원과 저차원의 이웃 확률 분포가 일치하도록 이 값을 최소화한다. 한편 perplexity는 각 점이 고려하는 유효 이웃 개수를 조절하는 하이퍼파라미터로, 값이 작으면 매우 가까운 이웃만, 크면 더 넓은 범위의 이웃을 고려한다. 일반적으로 데이터 크기의 제곱근 정도를 시작점으로 삼아 5-50 범위에서 탐색한다.

### 2.3.5 UMAP: t-SNE의 대안

**UMAP(Uniform Manifold Approximation and Projection)**(McInnes et al., 2018)은 t-SNE와 유사한 목적을 가지지만 더 빠르고 전역 구조를 더 잘 보존한다. 이 방법은 위상수학(topology)의 개념을 차용하여 데이터의 구조를 표현한다. UMAP의 핵심 아이디어는 데이터가 고차원 공간에 임베딩된 저차원 다양체(manifold) 위에 균일하게 분포한다는 가정이다. 이 다양체의 구조를 포착하기 위해 UMAP은 두 단계로 작동한다. 먼저 고차원에서 각 점의 k-최근접이웃을 찾아 가중 그래프를 구성한다. 이때 각 점은 자신만의 거리 척도를 가지는데, 이웃이 적어도 하나는 "가깝게" 보이도록 거리를 조정한다. 이렇게 구성된 지역 연결성(local connectivity)을 퍼지 집합(fuzzy simplicial set)으로 통합하면 전체 데이터의 위상 구조가 표현된다.
두 번째 단계에서는 저차원 공간에서도 유사한 그래프를 구성하고, 고차원 그래프와의 교차 엔트로피(cross-entropy)를 최소화하도록 저차원 좌표를 최적화한다. t-SNE가 KL-발산을 사용하는 것과 달리, UMAP의 교차 엔트로피는 "연결되어야 할 점이 떨어져 있는 경우"와 "연결되지 않아야 할 점이 붙어 있는 경우" 모두에 패널티를 부여한다. 이 대칭적 특성 덕분에 전역 구조가 더 잘 보존된다.
UMAP의 주요 하이퍼파라미터는 n_neighbors와 min_dist다. n_neighbors는 지역 이웃의 개수로, t-SNE의 perplexity와 유사한 역할을 한다. 값이 작으면 매우 지역적인 구조에 집중하고, 크면 더 넓은 맥락을 고려한다. 보통 5-50 범위에서 선택하며, 15가 기본값으로 자주 사용된다. min_dist는 저차원에서 점들이 얼마나 가깝게 뭉칠 수 있는지를 제어한다. 0에 가까우면 군집이 빽빽하게 모이고, 크면 점들이 더 퍼진다. 시각화 목적에서는 0.1 정도가 적당하다. UMAP이 t-SNE보다 빠른 이유는 최적화 방식의 차이에 있다. t-SNE는 모든 점 쌍의 상호작용을 계산해야 하므로 O(n²) 복잡도를 가지는 반면, UMAP은 근사 최근접이웃 탐색과 확률적 경사 하강법을 사용하여 O(n) 수준으로 확장된다. 100만 개 이상의 데이터에서도 합리적인 시간 내에 결과를 얻을 수 있다.
실무에서 UMAP의 또 다른 장점은 표본 외(out-of-sample) 확장이 용이하다는 점이다. 학습된 변환을 새로운 데이터에 적용할 수 있어, 스트리밍 데이터나 대규모 배치 처리에 적합하다. 다만 UMAP도 확률적 알고리즘이므로 random_state를 고정하지 않으면 실행마다 결과가 달라질 수 있으며, 하이퍼파라미터에 따라 시각화 결과가 크게 달라지므로 여러 설정을 탐색해봐야 한다.

### 2.3.6 차원 축소 방법 선택 가이드

차원 축소 방법 선택은 목적에 따라 달라진다. 시각화가 목적인 경우, 군집 구조를 탐색하려면 UMAP이나 t-SNE가 적합하다. 데이터의 연속적인 변화(예: 시간에 따른 변화)를 확인하려면 PCA의 처음 2-3개 주성분을 사용하는 것이 좋다. 대규모 데이터셋에서는 UMAP이 t-SNE보다 훨씬 빠르므로 선호된다. 전처리가 목적인 경우에는 대부분 PCA가 적합하다. 선형 관계를 보존하면서 차원을 줄일 수 있고, 새로운 데이터를 쉽게 투영할 수 있으며, 다중공선성을 제거하는 효과도 있기 때문이다. 후속 모델의 입력으로 사용할 때도 해석 가능성과 out-of-sample 확장 용이성 면에서 PCA가 유리하다.

<표 2-3> 차원 축소 방법 선택 플로우

```
시작
 │
 ├─ 목적이 시각화인가?
 │   ├─ 예 → 지역 구조 중요?
 │   │       ├─ 예 → UMAP 또는 t-SNE
 │   │       └─ 아니오 → PCA
 │   │
 │   └─ 아니오 → 후속 모델 입력용
 │               ├─ 선형 관계 충분 → PCA
 │               └─ 비선형 필요 → UMAP
```

### 2.3.7 실습: 차원 축소 비교

실습에서는 손글씨 숫자 데이터셋에 PCA, t-SNE, UMAP을 적용해 2차원 임베딩을 만들고, 처리 시간과 임베딩 품질(예: 실루엣 계수)을 동일한 기준으로 비교한다. 수치 결과는 데이터 크기, 하이퍼파라미터, 실행 환경에 따라 달라지므로, 본문에는 특정 값을 고정해 제시하지 않고 스크립트 실행으로 확인한다.

## 2.4 특성 공학: 모델이 학습하기 쉬운 표현 만들기

모델 성능은 데이터 표현(representation)에 크게 의존한다. 같은 정보라도 어떻게 표현하느냐에 따라 모델이 패턴을 학습하기 쉬울 수도, 어려울 수도 있다. 특성 공학(Feature Engineering)은 원시 데이터를 모델이 효과적으로 학습할 수 있는 형태로 변환하는 과정이다.

### 2.4.1 특성 공학이 필요한 이유

선형 모델은 선형 관계만 학습할 수 있다. 만약 타깃 변수 y가 입력 x의 제곱에 비례한다면, y = βx 형태로는 이 관계를 포착할 수 없다. 그러나 x² = z라는 새 특성을 만들면 y = βz는 선형이 되어 학습 가능하다. 트리 기반 모델은 축에 평행한 분할만 수행한다. x₁ + x₂ > 10이라는 경계는 여러 번의 분할로 근사해야 한다. x₃ = x₁ + x₂라는 특성을 만들면 x₃ > 10 한 번으로 경계를 포착한다. 신경망은 이론적으로 어떤 함수든 근사할 수 있지만, 적절한 특성이 주어지면 더 적은 층과 더 적은 데이터로 학습할 수 있다.

### 2.4.2 수치형 변수 변환

**로그 변환**은 오른쪽으로 치우친(right-skewed) 분포를 대칭에 가깝게 만든다. 소득, 인구, 가격 등 양의 값이면서 상한이 불명확한 변수에 효과적이다. 로그 변환은 곱셈 관계를 덧셈 관계로 바꾸어 선형 모델이 포착하기 쉽게 한다. 이 모형에서 education이 1 증가하면 income이 exp(β₁)배 증가한다는 해석이 가능하다.

```
log(income) = β₀ + β₁·education + β₂·experience
```

**제곱근, 박스-콕스 변환**도 분포를 정규화하는 데 사용된다. 박스-콕스는 최적의 변환 파라미터 λ를 데이터에서 추정한다. **비닝(Binning)**은 연속형 변수를 구간으로 나누어 범주화한다. 나이를 10대, 20대, 30대 등으로 구분하는 것이 예시다. 비선형 관계를 포착할 수 있지만, 정보 손실이 발생하고 구간 경계 설정이 임의적이다.

### 2.4.3 범주형 변수 인코딩

**원-핫 인코딩(One-Hot Encoding)**은 각 범주를 별도의 이진 변수로 변환한다. 3개 범주 {A, B, C}는 3개의 이진 변수 [1,0,0], [0,1,0], [0,0,1]이 된다. 범주 간 순서나 거리 관계를 가정하지 않으므로 명목형 변수에 적합하다. 다만, 원-핫 인코딩의 문제는 고차원이다. 범주가 1,000개면 1,000개 변수가 생성된다. 또한 범주 간 관계 정보가 손실된다. '서울'과 '인천'이 '서울'과 '부산'보다 가깝다는 정보가 반영되지 않는다. **레이블 인코딩(Label Encoding)**은 범주에 정수를 할당한다. {A, B, C} → {0, 1, 2}. 트리 기반 모델에서는 사용 가능하지만, 순서형이 아닌 범주에 사용하면 거짓 순서 관계가 도입된다. **타겟 인코딩(Target Encoding)**은 각 범주를 해당 범주의 타깃 평균으로 대체한다. 고객 직업별로 대출 부도율의 평균을 계산하여 직업 변수를 대체하는 것이 예시다. 타겟 인코딩은 차원을 늘리지 않고 타깃과의 관계를 직접 반영한다. 그러나 **데이터 누수(data leakage)** 위험이 있다. 훈련 데이터의 타깃 정보가 입력에 포함되어 과적합을 야기할 수 있다. 그래서 타겟 인코딩은 교차검증 절차 안에서 폴드별로 계산하여 누수를 줄이는 것이 기본이다. 또한 표본 수가 적은 범주의 평균은 분산이 크므로, 전체 평균 쪽으로 수축시키는 스무딩(smoothing)을 적용해 과적합을 완화한다. 운영 환경에서는 인코딩에 사용한 통계량을 훈련 데이터로만 추정하고 버전 관리하여, 학습·검증·배포 단계에서 동일한 파라미터가 적용되도록 관리해야 한다.

### 2.4.4 날짜/시간 변수 처리

날짜/시간 데이터는 다양한 특성으로 분해할 수 있다. 순환적 특성은 요일, 월, 시간처럼 주기적으로 반복되는 값을 표현한다. 23시와 0시가 가깝다는 것을 반영하려면 사인/코사인 변환을 사용한다. 예를 들어 시간을 hour_sin = sin(2π × hour / 24)과 hour_cos = cos(2π × hour / 24)로 변환하면 연속적인 순환 구조를 표현할 수 있다. 이진 특성은 특정 조건의 충족 여부를 나타낸다. 주말 여부, 공휴일 여부, 업무 시간 여부 등이 대표적이다. 상대적 특성은 특정 기준점까지의 거리를 측정하는데, 월초까지 남은 일수, 급여일까지 남은 일수 등이 예시다. 누적 특성은 기준 시점으로부터 경과한 시간을 나타내며, 서비스 가입 후 경과일이나 마지막 구매 후 경과일 등이 고객 행동 예측에 유용하다.

### 2.4.5 상호작용 특성

두 변수의 조합이 중요한 경우 상호작용 특성을 생성한다. 집값 예측에서 방 개수와 면적의 곱(방당 면적)이 유용할 수 있다. 광고 효과 분석에서 채널과 시간대의 조합이 중요할 수 있다. 상호작용 특성은 도메인 지식에 기반하여 선택적으로 생성해야 한다. 모든 변수 쌍의 상호작용을 만들면 차원이 p²로 폭발한다.

### 2.4.6 LLM을 활용한 특성 아이디어 생성

LLM은 도메인 지식을 바탕으로 특성 아이디어를 제안할 수 있다. 다음과 같은 프롬프트가 유용하다:

```
신용카드 사기 탐지 모델을 구축하려 합니다.
거래 데이터에는 금액, 시간, 위치, 가맹점 유형이 있습니다.
유용한 파생 특성을 제안해주세요.
```

LLM은 예를 들어 최근 N시간 내 거래 횟수, 평소 거래 금액 대비 비율, 이전 거래 위치와의 거리, 야간·새벽 거래 여부, 동일 가맹점 재방문 빈도 같은 파생 특성 후보를 제안할 수 있다. 다만 이러한 제안은 데이터 누수, 변수 정의의 일관성, 운영에서의 계산 가능성 같은 제약을 함께 점검한 뒤에야 실무 특성으로 채택할 수 있다. 이러한 아이디어는 분석가가 검증하고 구현해야 하지만, 도메인 전문가와의 브레인스토밍을 대체할 수 있다.

## 2.5 LLM으로 텍스트를 분석 가능한 수치로 변환

텍스트 데이터는 고객 리뷰, 뉴스 기사, 소셜 미디어 게시물 등 다양한 형태로 존재한다. 머신러닝 모델에 입력하려면 이 텍스트를 수치 벡터로 변환해야 한다. 전통적으로 TF-IDF가 사용되었지만, LLM 임베딩은 단어의 의미와 문맥을 반영하여 훨씬 풍부한 표현을 제공한다.

### 2.5.1 TF-IDF

TF-IDF(Term Frequency-Inverse Document Frequency)는 문서에서 단어의 중요도를 측정한다. 문서 내 출현 빈도(TF)가 높고, 전체 문서에서 희귀한(IDF) 단어에 높은 가중치를 부여한다. TF-IDF의 수식은 다음과 같다.

```
TF-IDF(t, d, D) = TF(t, d) × IDF(t, D)
IDF(t, D) = logₑ (N ÷ (1 + DF(t)))
TF-IDF(t, d, D) = TF(t, d) × logₑ (N ÷ (1 + DF(t)))
```

여기서
- TF(t, d): 특정 문서 d에서 단어 t의 등장 빈도
- IDF(t, D): 전체 문서 집합 D에서 단어 t의 역문서 빈도
- N: 전체 문서 수
- DF(t): 단어 t가 등장한 문서의 수

그러나 TF-IDF에는 근본적인 한계가 있다. 첫째, 단어의 의미를 무시하여 "좋다"와 "훌륭하다"처럼 의미가 유사한 단어도 완전히 다른 단어로 취급된다. 둘째, 문맥을 고려하지 않아 "은행에 돈을 맡겼다"와 "강 은행에 앉았다"에서 '은행'이 동일하게 처리된다. 셋째, 어순을 무시하므로 "개가 사람을 물었다"와 "사람이 개를 물었다"가 동일한 벡터로 표현된다. 넷째, 어휘 크기만큼의 차원이 필요하고 대부분의 값이 0인 희소 행렬이 생성되어 저장 공간과 계산 효율 측면에서 비효율적이다.

### 2.5.2 임베딩: 의미를 보존하는 벡터 변환

임베딩(Embedding)은 텍스트를 밀집 벡터(dense vector)로 변환한다. 일반적으로 384-1536 차원의 실수 벡터로 표현되며, 의미적으로 유사한 텍스트는 벡터 공간에서 가깝게 배치된다. 임베딩 모델은 대규모 텍스트 코퍼스에서 사전학습되어 언어의 의미적 관계를 학습한다. "King - Man + Woman = Queen"과 같은 유명한 예시는 임베딩이 의미적 관계를 벡터 연산으로 표현할 수 있음을 보여준다.

### 2.5.3 Sentence-BERT 임베딩

문장 수준 임베딩에는 **Sentence-BERT(SBERT)**(Reimers & Gurevych, 2019)가 널리 사용된다. BERT는 토큰 수준 임베딩을 생성하는데, 이를 문장 임베딩으로 집계하면 성능이 떨어진다. SBERT는 문장 쌍의 유사도 학습을 통해 문장 임베딩에 최적화되었다. 실무에서는 속도와 품질의 균형을 고려해 모델을 고른다. 예를 들어 `all-MiniLM-L6-v2`는 384차원으로 빠르고, `all-mpnet-base-v2`는 768차원으로 상대적으로 느리지만 성능이 좋은 편이다. 한국어 텍스트에는 다국어 모델이나 한국어 특화 모델을 사용하는 것이 일반적이며, 도메인과 데이터 규모에 따라 임베딩 모델의 선택이 달라진다.

### 2.5.4 OpenAI/Claude/HyperCLOVA 임베딩

클라우드 API를 통해 고품질 임베딩을 얻을 수 있다. 대표적으로 OpenAI의 `text-embedding-3-small`(1536차원), Claude 호환 임베딩 서비스, 그리고 네이버의 HyperCLOVA 임베딩 API가 있다. HyperCLOVA 임베딩은 한국어와 다국어 데이터에 특화된 대규모 언어 모델 기반 임베딩을 제공하며, 네이버 클라우드 플랫폼(NCP)에서 API 형태로 사용할 수 있다. HyperCLOVA 임베딩은 문장, 문서, 쿼리 등 다양한 입력에 대해 의미 기반 벡터를 생성하며, 검색, 추천, 분류 등 여러 태스크에 활용된다. 주요 장점은 한국어 표현력과 최신 LLM 기반 품질, 그리고 네이버 클라우드 인프라와의 통합이다. 사용 시에는 API 키 발급 및 과금 정책, 데이터 보안(외부 전송) 등을 고려해야 한다.

- OpenAI 임베딩: https://platform.openai.com/docs/guides/embeddings
- Claude 임베딩: https://docs.anthropic.com/claude/docs/embeddings
- HyperCLOVA 임베딩: https://guide.ncloud-docs.com/docs/ai-hyperclova-embedding-overview

API 임베딩의 장점은 최신 모델이 적용되어 품질이 높고, 별도의 GPU나 인프라 관리가 필요 없으며, 다국어 지원이 우수하다는 점이다. 반면 토큰당 과금되어 대규모 처리 시 비용이 발생하고, 인터넷 연결이 필수이며, 민감한 데이터를 외부 서버로 전송해야 하므로 프라이버시 고려가 필요하다는 단점이 있다.

### 2.5.5 임베딩의 활용

임베딩은 다양한 방식으로 활용된다. 먼저, 문서 임베딩 간의 코사인 유사도를 계산하여 유사한 문서를 찾는 유사도 검색에 사용할 수 있다. 예를 들어 고객 문의가 들어왔을 때, 임베딩을 통해 과거의 유사한 문의와 그 해결책을 빠르게 검색할 수 있다. 또한, 임베딩된 문서들을 K-Means나 HDBSCAN과 같은 군집화 알고리즘에 적용하면 주제별로 문서 그룹을 자동으로 발견할 수 있으며, 이에 대한 자세한 내용은 4장에서 다룬다. 임베딩 벡터는 분류 모델(로지스틱 회귀, XGBoost 등)의 입력 특성으로도 활용되는데, 전통적인 TF-IDF 방식보다 적은 데이터로도 더 높은 분류 성능을 얻을 수 있다는 장점이 있다. 마지막으로, RAG(Retrieval-Augmented Generation)와 같이 질문과 문서를 임베딩하여 유사도가 높은 문서를 검색하고, 이를 LLM의 컨텍스트로 제공하는 방식도 널리 쓰인다. 이와 관련된 심화 내용은 8장에서 자세히 설명한다.

### 2.5.6 실습: 텍스트 임베딩과 분류

실습에서는 TF-IDF와 문장 임베딩(SBERT)을 각각 특성으로 만들어 동일한 분류기(예: 로지스틱 회귀)로 학습시키고, 정확도와 처리 시간, 벡터 차원을 같은 기준으로 비교한다. 비교 결과는 데이터셋 구성과 샘플 수, 모델 선택, 실행 환경에 따라 달라지므로, 본문에는 특정 숫자를 고정해 제시하지 않고 실행으로 확인한다.

## 2.6 임베딩: 범주형 변수의 "관계"를 학습하는 방법
범주형 변수를 처리할 때 가장 널리 쓰이는 방법은 원-핫 인코딩이다. 하지만 범주 수가 많아질수록 변수 차원이 급격히 증가하고, 각 범주 간의 관계나 유사성이 완전히 무시된다는 한계가 있다. 예를 들어 도시 변수를 원-핫 인코딩하면 서울은 [1,0,0,...], 부산은 [0,1,0,...], 인천은 [0,0,1,...]처럼 표현된다. 이 방식에서는 서울과 부산, 서울과 뉴욕의 거리가 모두 동일하게 취급되며, 인천이 서울과 지리적으로 가깝다는 정보나 인구·경제 규모 등 유사성은 전혀 반영되지 않는다. 따라서 모델이 서울에서 학습한 패턴을 인천에 적용하려 해도, 두 도시가 유사하다는 사실을 표현할 방법이 없다.
이러한 한계를 극복하기 위해 Entity Embedding이 등장했다. Entity Embedding은 각 범주를 학습 가능한 저차원 벡터로 표현하며, 신경망 학습 과정에서 타깃 예측에 유용하도록 벡터 값이 최적화된다. 구체적으로는 각 범주에 임의의 벡터(예: 4차원)를 할당하고, 이 벡터를 신경망 입력에 포함시킨 뒤 역전파로 값을 업데이트한다. 학습이 완료되면, 매출 예측에서 매출 패턴이 유사한 가게들(예: 같은 체인, 유사한 입지)이 벡터 공간에서 가깝게 위치하는 식으로, 범주 간 의미 있는 구조가 자연스럽게 형성된다.
Entity Embedding의 가장 큰 장점은 차원 축소 효과와 범주 간 유사성 학습이다. 예를 들어 1,000개 범주를 4차원 임베딩으로 표현하면 변수 수가 250배 줄어들어 메모리와 계산 비용이 크게 감소한다. 또한 도메인 지식 없이도 타깃 예측에 유용한 범주 간 유사성을 자동으로 학습할 수 있다. 희귀 범주도 유사한 범주들과 벡터 공간을 공유하므로 정보가 전이되고, 완전히 새로운 범주가 등장할 때는 기존 임베딩의 평균값 등으로 대체할 수 있다. 학습된 임베딩은 다른 태스크에 재사용(전이 학습)할 수도 있어, 예를 들어 매출 예측에서 학습한 가게 임베딩을 재고 예측 등 다른 문제에 활용하는 것도 가능하다.

### 2.6.1 Kaggle 성공 사례: Rossmann Store Sales

2015년 Kaggle Rossmann Store Sales 경진대회에서는 3위를 차지한 팀이 Entity Embedding을 도입해 큰 주목을 받았다(Guo & Berkhahn, 2016). 이 팀은 1,115개에 달하는 가게 ID와 요일, 주(state), 프로모션 등 주요 범주형 변수를 모두 임베딩으로 변환하여 신경망에 입력했다. 학습이 완료된 임베딩 벡터를 시각화하면, 지리적으로 인접한 주(state)들이 벡터 공간에서도 가까이 배치되고, 매출 패턴이 유사한 가게들이 자연스럽게 군집을 이루는 등 데이터의 의미 구조가 드러난다. 요일 임베딩에서는 주중과 주말이 명확히 구분되는 패턴이 나타나기도 한다. 이처럼 Entity Embedding은 범주 간의 관계와 유사성을 효과적으로 학습할 수 있음을 실증적으로 보여주었고, 이후 정형 데이터 딥러닝 분야에서 표준적인 기법으로 자리잡았다.
Entity Embedding의 구현은 범주형 변수의 ID를 임베딩 테이블에서 밀집 벡터로 변환한 뒤, 연속형 변수와 함께 신경망에 입력하는 방식으로 이루어진다. 실무에서는 임베딩 차원이 지나치게 크면 과적합과 계산 비용이 증가하므로, 범주 수에 따라 점진적으로 차원을 늘리되 상한을 두는 경험적 규칙이 널리 쓰인다. 예를 들어 `emb_dim = min(50, (cat_dim + 1) // 2)`와 같이 범주 수가 많아질수록 임베딩 차원을 늘리되 50을 넘지 않도록 제한하는 방식이 일반적이다.
또한, 처음부터 임베딩을 학습하지 않고 사전학습된 임베딩을 활용하는 경우도 있다. Word2Vec이나 GloVe와 같은 단어 임베딩은 제품명이나 직업명처럼 텍스트 형태의 범주에 적용할 수 있고, Uber의 H3 임베딩처럼 위도·경도 정보를 반영한 지리 임베딩, 사용자-아이템 상호작용 그래프에서 학습한 그래프 임베딩 등도 활용된다. 사전학습 임베딩은 데이터가 적을 때는 고정(freeze)하여 과적합을 방지하고, 데이터가 충분할 때는 미세조정(fine-tune)하여 타깃 태스크에 최적화하는 전략이 효과적이다. 다만 사전학습 임베딩은 학습 코퍼스의 의미와 편향을 그대로 가져오므로, 도메인이 다르거나 범주의 라벨이 연구 맥락의 의미를 충분히 담지 못하면 효과가 제한되거나 성능이 떨어질 수도 있다. 따라서 가능하면 도메인 데이터로 추가 학습·미세조정을 하고, 검증 데이터로 실제 개선 여부를 확인해야 한다.

### 2.6.2 Entity Embedding vs 원-핫 인코딩 선택 기준

| 상황                   | 권장 방법                   |
| ---------------------- | --------------------------- |
| 범주 < 10개            | 원-핫 (또는 레이블)         |
| 범주 10-100개          | 원-핫 또는 Entity Embedding |
| 범주 > 100개           | Entity Embedding            |
| 트리 기반 모델         | 레이블 인코딩               |
| 신경망 모델            | Entity Embedding            |
| 범주 간 관계 활용 필요 | Entity Embedding            |
