# 제1장: 데이터 사이언스의 진화 - 통계에서 AI까지

본 장에서는 데이터 사이언스가 통계학에서 출발해 데이터마이닝, 머신러닝, 딥러닝, 그리고 대규모 언어 모델(LLM)로 확장되는 과정을 학습한다. 이어서 지도학습·비지도학습·강화학습·자기지도학습을 구분하고, 문제 구조와 데이터 조건에 따라 어떤 학습 유형을 선택해야 하는지 정리한다. 또한 전통적 CRISP-DM과 AI 증강 워크플로우를 비교하여 실무에서의 역할 분담과 검증 원칙을 이해하도록 하겠다. 마지막으로 LLM 기반 탐색적 데이터 분석의 활용 가능성과 한계를 점검하고, 데이터 특성에 따라 전통 ML과 딥러닝 중 적절한 접근을 선택하는 기준을 익힌다.

## 1.1 데이터 사이언스의 역사와 패러다임 변화

데이터 사이언스(Data Science)라는 용어는 비교적 최근에 등장했지만, 데이터에서 지식을 추출하려는 시도는 오랜 역사를 가지고 있다. 통계학에서 시작하여 데이터마이닝, 머신러닝, 딥러닝을 거쳐 현재의 대규모 언어 모델(LLM)에 이르기까지, 각 시대는 고유한 방법론과 도구를 발전시켜왔다. 이 절에서는 데이터 사이언스의 역사적 발전 과정을 살펴보고, 각 패러다임이 이전 시대의 한계를 어떻게 극복했는지 이해한다.

### 1.1.1 데이터 사이언스 시장의 성장

데이터 사이언스 분야는 지속적인 성장세를 보이고 있다. Grand View Research(2023)에 따르면 글로벌 머신러닝 시장 규모는 2022년 214억 달러에서 2027년 약 1,000억 달러로 연평균 36.2% 성장할 것으로 전망된다. 이 성장을 주도하는 요인은 세 가지다.

첫째, **데이터 폭증**이다. 전 세계에서 생성되는 데이터는 기하급수적으로 증가하고 있으며, 2010년대 후반에 제시된 전망에서는 2025년에 연간 175 제타바이트에 도달할 것으로 예상되었다(IDC, 2018). 이 방대한 데이터에서 가치를 추출하려는 수요가 데이터 사이언스 성장의 근본 동력이다.

둘째, **컴퓨팅 자원의 민주화**다. 클라우드 서비스(AWS, GCP, Azure)와 오픈소스 도구(scikit-learn, PyTorch, Hugging Face)의 발전으로 진입 장벽이 낮아졌다. 과거에는 대기업만 접근할 수 있었던 고성능 컴퓨팅과 고급 알고리즘이 이제 누구에게나 열려 있다.

셋째, **생성형 AI의 등장**이다. 2022년 ChatGPT의 공개 이후 AI에 대한 기업의 관심과 투자가 급증했다. McKinsey(2023)에 따르면 생성형 AI는 전 세계적으로 연간 2.6조~4.4조 달러의 경제적 가치를 창출할 잠재력이 있다.

### 1.1.2 데이터 사이언스의 역사적 발전

**통계학의 시대 (1900s~1970s)**: 현대 데이터 분석의 뿌리는 통계학에 있다. 20세기 초 칼 피어슨(Karl Pearson)과 로널드 피셔(Ronald Fisher)가 정립한 통계적 방법론은 데이터에서 패턴을 발견하고 가설을 검증하는 체계적인 프레임워크를 제공했다. 회귀분석, 분산분석, 가설검정 등 이 시기에 개발된 기법들은 오늘날에도 데이터 분석의 기초로 사용된다. 통계학 시대의 특징은 **가설 주도(hypothesis-driven)** 접근이다. 연구자가 먼저 가설을 설정하고, 데이터를 수집하여 가설을 검증한다. 이 접근은 과학적 엄밀성을 보장하지만, 데이터에 숨겨진 예상치 못한 패턴을 발견하기 어렵다는 한계가 있다.

**데이터마이닝의 시대 (1980s~1990s)**: 컴퓨터의 발전과 함께 대량의 데이터를 자동으로 탐색하는 **데이터마이닝(Data Mining)**이 등장했다. 통계학이 가설 검증에 초점을 맞췄다면, 데이터마이닝은 **데이터 주도(data-driven)** 패턴 발견을 추구한다. 연관규칙 분석, 의사결정나무, 군집 분석 등이 이 시기에 발전했다. 1996년 Fayyad 등이 제안한 KDD(Knowledge Discovery in Databases) 프로세스는 데이터마이닝을 체계화했다. 이 프로세스는 데이터 선택, 전처리, 변환, 마이닝, 해석의 5단계로 구성되며, 이후 CRISP-DM의 기초가 되었다.

**머신러닝의 시대 (2000s~2010s)**: 2000년대에 들어서면서 **머신러닝(Machine Learning)**이 데이터 사이언스의 중심으로 부상했다. 서포트 벡터 머신(SVM), 랜덤 포레스트, 그래디언트 부스팅 등 강력한 알고리즘이 개발되었고, 특히 Kaggle과 같은 데이터 과학 경진대회에서 앙상블 기법의 위력이 입증되었다. 이 시기의 핵심 변화는 **특성 공학(Feature Engineering)**의 중요성이다. 원시 데이터를 모델이 학습하기 좋은 형태로 변환하는 것이 성능의 핵심 결정 요인이 되었다. 숙련된 데이터 과학자는 도메인 지식을 활용하여 효과적인 특성을 설계할 수 있었지만, 이는 높은 전문성을 요구하는 병목점이기도 했다.

**딥러닝의 시대 (2012~2020)**: 2012년 ImageNet 대회에서 AlexNet이 기존 방법론을 압도하면서 **딥러닝(Deep Learning)** 시대가 본격적으로 시작되었다. 딥러닝의 혁신은 **표현 학습(Representation Learning)**에 있다. 사람이 특성을 설계하는 대신, 신경망이 원시 데이터에서 유용한 특성을 스스로 학습한다. 이미지, 음성, 텍스트 등 비정형 데이터에서 딥러닝은 기존 방법론을 크게 앞섰다. 그러나 정형 데이터(tabular data)에서는 여전히 XGBoost와 같은 그래디언트 부스팅 기법이 경쟁력을 유지하고 있다. Kaggle 대회에서도 정형 데이터 문제에서는 트리 기반 앙상블이 여전히 주류를 이룬다.

**LLM의 시대 (2020s~현재)**: 2017년 Transformer 아키텍처의 등장(Vaswani et al., 2017)과 2020년 GPT-3의 공개(Brown et al., 2020) 이후, **대규모 언어 모델(Large Language Model, LLM)**이 AI의 새로운 패러다임을 열었다. LLM은 기존 머신러닝과 근본적으로 다른 접근을 제시한다.

첫째, **사전학습-미세조정(Pre-training and Fine-tuning)** 패러다임이다. 대규모 코퍼스로 사전학습된 모델을 특정 태스크에 미세조정하면, 적은 데이터로도 높은 성능을 달성할 수 있다. 둘째, **프롬프트 기반(Prompt-based)** 접근이다. 자연어 지시를 통해 모델의 행동을 제어하며, 코드 없이도 복잡한 태스크를 수행할 수 있다. 셋째, **멀티모달(Multimodal)** 확장이다. 최근 대부분의 llm 모델은 텍스트와 이미지를 동시에 이해하고 생성한다.

### 1.1.3 AI가 데이터 분석가를 대체하는가?

LLM의 등장으로 "AI가 데이터 분석가를 대체할 것인가?"라는 질문이 자주 제기된다. 이에 대한 균형 잡힌 시각이 필요하다.

LLM이 **대체할 수 있는** 영역은 반복적이고 패턴화된 작업이다. 기초 EDA, 코드 생성, 문서화, 데이터 정제 등은 LLM이 효율적으로 처리할 수 있다. 그러나 LLM이 **대체하기 어려운** 영역도 명확하다. 비즈니스 맥락을 고려한 문제 정의, 이해관계자와의 커뮤니케이션, 윤리적 판단, 예상치 못한 상황에서의 창의적 문제 해결은 여전히 인간 분석가의 역할이다. 가장 현실적인 관점은 LLM을 **"대체재"가 아닌 "증강 도구"**로 보는 것이다. LLM은 분석가의 생산성을 높이고, 반복 작업에서 해방시켜 더 가치 있는 활동에 집중하게 한다. 이 책 전체에서 우리는 전통적 기법과 AI 도구를 어떻게 효과적으로 결합할 수 있는지 탐구한다.

## 1.2 머신러닝의 주요 유형

머신러닝은 학습 방식에 따라 크게 네 가지 유형으로 분류된다. 각 유형은 서로 다른 문제 상황에 적합하며, 데이터의 특성과 가용한 레이블에 따라 적절한 접근법을 선택해야 한다.

### 1.2.1 지도학습: 정답이 있는 학습

**지도학습(Supervised Learning)**은 입력 데이터 X와 정답 레이블 Y가 쌍으로 주어질 때 X에서 Y를 예측하는 함수 f를 학습한다. 수식으로 표현하면:

Y = f(X) + ε

여기서 f는 학습할 함수, ε는 예측 불가능한 오차다. 지도학습의 목표는 이 함수 f를 데이터로부터 추정하는 것이다. 지도학습은 출력 변수의 유형에 따라 **분류(Classification)**와 **회귀(Regression)**로 나뉜다. 분류는 출력이 범주형(예: 스팸 여부, 고객 이탈 여부)인 경우, 회귀는 출력이 연속형(예: 주택 가격, 매출액)인 경우다. 지도학습의 강점은 명확한 평가 기준이 있다는 것이다. 예측값과 실제값을 비교하여 모델의 성능을 객관적으로 측정할 수 있다. 대표적인 알고리즘으로는 선형 회귀, 로지스틱 회귀, 의사결정나무, 랜덤 포레스트, 그래디언트 부스팅, 서포트 벡터 머신, 신경망 등이 있다.

지도학습은 금융, 마케팅, 의료, 제조 등에서 가장 흔히 쓰이는 접근이다. 금융에서는 신용 점수나 부도 위험을 예측하고 사기 거래를 분류 문제로 다루는 경우가 많다. 마케팅에서는 고객 이탈과 캠페인 반응을 예측하며, 의료와 제조에서는 진단·재입원 위험 예측이나 품질 불량·잔여 수명 추정 같은 과제가 대표적이다.

### 1.2.2 비지도학습: 정답 없이 구조 발견

**비지도학습(Unsupervised Learning)**은 레이블 없이 데이터 X만 주어질 때 데이터의 내재된 구조를 발견한다. 비지도학습은 크게 세 가지 태스크로 나뉜다. **군집 분석(Clustering)**은 유사한 데이터 포인트를 그룹으로 묶는다. 고객 세분화, 문서 그룹화, 이상 패턴 탐지 등에 활용된다. K-Means, DBSCAN, 계층적 군집화 등이 대표적이다. **차원 축소(Dimensionality Reduction)**는 고차원 데이터를 저차원으로 변환하면서 핵심 정보를 보존한다. PCA(주성분 분석), t-SNE, UMAP 등이 사용된다. 시각화, 노이즈 제거, 계산 효율성 향상에 활용된다. **밀도 추정(Density Estimation)**은 데이터의 확률 분포를 추정한다. 이상치 탐지, 생성 모델의 기초가 된다. 비지도학습의 어려움은 평가 기준이 명확하지 않다는 점이다. 군집이 "좋은" 군집인지 판단하는 것은 종종 도메인 전문가의 주관적 평가에 의존한다.

비지도학습은 레이블이 없거나 데이터에 숨겨진 구조를 먼저 파악해야 할 때 유용하다. 예를 들어 구매 패턴으로 고객을 세분화하거나, 네트워크 트래픽과 제조 센서 로그에서 비정상 패턴을 찾아 이상 징후를 조기에 감지할 수 있다. 대량 문서의 토픽을 자동으로 발견해 분류에 활용하거나, 유사 사용자·상품을 군집화해 추천 품질을 높이는 것도 전형적인 활용 사례다.

### 1.2.3 강화학습: 시행착오를 통한 학습

**강화학습(Reinforcement Learning)**은 에이전트가 환경과 상호작용하며 보상을 최대화하는 행동을 학습한다. 지도학습처럼 정답이 주어지지 않고, 행동의 결과로 받는 보상 신호를 통해 학습한다. 강화학습을 이해하려면 상태(state), 행동(action), 보상(reward), 정책(policy)의 네 가지 개념이 필요하다. 에이전트는 특정 상태에서 행동을 선택하고 그 결과로 보상을 관측하며, 정책은 상태가 주어졌을 때 어떤 행동을 취할지를 결정하는 규칙 또는 함수다. 강화학습은 게임 AI(AlphaGo), 로봇 제어, 추천 시스템, 자율주행 등에 적용된다. 그러나 데이터 과학의 일반적인 분류/회귀/군집 문제에서는 지도학습이나 비지도학습이 더 적합한 경우가 많다. 강화학습은 동적 가격 책정이나 재고 관리처럼 의사결정이 시간에 걸쳐 누적되고, 행동의 효과가 지연되어 나타나는 문제에서 특히 유용하다. 또한 사용자의 장기 만족도를 목표로 추천 정책을 학습하거나, 다수의 실험 변형 중 최적안을 자동으로 선택하는 밴딧(bandit) 문제로 A/B 테스트를 자동화하는 데에도 적용된다. 강화학습의 도입은 환경 시뮬레이션이 가능하고, 행동의 결과가 지연되어 나타나고 탐색-활용 트레이드오프가 중요한 문제에서 고려한다.

### 1.2.4 자기지도학습: LLM의 핵심 원리

**자기지도학습(Self-Supervised Learning)**은 레이블 없이 데이터 자체에서 학습 신호를 생성한다. 대규모 언어 모델(LLM)의 핵심 학습 방식이다. GPT 계열 모델은 **다음 토큰 예측(Next Token Prediction)** 태스크로 학습한다. "오늘 날씨가"라는 입력이 주어지면, 다음에 올 가능성이 높은 토큰("좋다", "흐리다" 등)을 예측한다. 이 과정에서 문법, 의미, 세계 지식 등을 학습한다. BERT 계열 모델은 **마스크 언어 모델링(Masked Language Modeling)**을 사용한다. 입력 문장의 일부 토큰을 마스킹하고, 마스킹된 토큰을 복원하도록 학습한다. 자기지도학습의 혁신적인 점은 레이블링 비용 없이 대규모 데이터에서 학습할 수 있다는 것이다. 인터넷의 방대한 텍스트 데이터를 활용하여 언어의 패턴을 학습하고, 이를 다양한 다운스트림 태스크에 전이(transfer)할 수 있다.

자기지도학습은 데이터 사이언스 실무에도 직접 연결된다. 텍스트에서는 BERT(Devlin et al., 2019)나 Sentence-BERT 임베딩으로 문서를 벡터로 바꿔 유사도 검색과 군집화에 활용하며, 이미지에서는 SimCLR이나 CLIP처럼 레이블 없이 학습한 표현을 분류기의 입력으로 사용할 수 있다. 시계열 사전학습 모델(TimeGPT 등)이나 코드 생성 도구(Codex, GitHub Copilot)도 같은 계열의 아이디어가 확장된 결과물이다. 자기지도학습은 데이터 라벨링 비용을 획기적으로 줄이면서, 전이학습을 통해 적은 데이터로도 높은 성능을 달성할 수 있게 한다. LLM 시대 데이터 사이언스의 핵심 패러다임이다.

<표 1-2> 머신러닝 유형별 특징비교

| 유형         | 학습 데이터 | 학습 신호   | 대표 태스크     | 대표 알고리즘                   |
| ------------ | ----------- | ----------- | --------------- | ------------------------------- |
| 지도학습     | X, Y 쌍     | 정답 레이블 | 분류, 회귀      | XGBoost, Random Forest, 신경망  |
| 비지도학습   | X만         | 없음        | 군집, 차원축소  | K-Means, PCA, UMAP              |
| 강화학습     | 상태-행동   | 보상 신호   | 게임, 로봇 제어 | Q-Learning, PPO                 |
| 자기지도학습 | X만         | 자기생성    | 사전학습        | GPT, BERT, Contrastive Learning |

## 1.3 AI 시대 데이터 분석 워크플로우

데이터 분석 프로젝트를 체계적으로 수행하기 위해서는 표준화된 프로세스가 필요하다. 이 절에서는 전통적인 CRISP-DM 프로세스와 AI 시대에 진화한 워크플로우를 비교하고, 실무에 적용할 수 있는 통합 접근법을 제시한다.

### 1.3.1 CRISP-DM: 전통적 데이터 분석 프로세스

**CRISP-DM(Cross-Industry Standard Process for Data Mining)**은 1990년대 후반에 개발되어 2000년에 공식 발표된 데이터 마이닝 프로세스 모델로(Wirth & Hipp, 2000), 20년 이상 업계 표준으로 사용되어왔다. 이 프로세스는 비즈니스 이해, 데이터 이해, 데이터 준비, 모델링, 평가, 배포의 여섯 단계로 구성된다. 비즈니스 이해에서는 목표를 분석 가능한 문제로 재정의하고, 데이터 이해와 준비 단계에서 데이터 품질을 점검하며 분석에 적합하게 가공한다. 이어지는 모델링과 평가 단계에서 알고리즘 선택과 성능·비즈니스 성과 검증을 수행하고, 배포 단계에서 운영 적용과 모니터링·유지보수 계획을 수립한다. CRISP-DM의 강점은 반복적(iterative) 특성이다. 어느 단계에서든 이전 단계로 돌아갈 수 있으며, 새로운 인사이트에 따라 방향을 수정할 수 있다.

### 1.3.2 AI 증강 워크플로우: 진화한 프로세스

LLM과 AutoML의 등장으로 데이터 분석 워크플로우도 진화하고 있다. 이 변화의 핵심인 'AI 증강 워크플로우'는 각 단계마다 LLM, AutoML, MLOps 같은 도구를 활용해 생산성을 극대화한다. 우선 **비즈니스 이해** 단계에서는 LLM이 업계 보고서나 유사 사례를 요약해 문제 구조화를 돕지만, 최종 문제 정의는 여전히 이해관계자의 책임이다. **데이터 이해와 준비** 단계에서는 대화형 분석 도구나 코드 생성 도구를 활용해 초기 EDA와 프로파일링을 가속화한다. **모델링** 단계에서는 AutoML이 알고리즘 선택과 튜닝을 자동화해 빠르게 베이스라인 모델을 제시한다. 마지막으로 **평가와 배포** 단계에서는 실험 추적과 버전 관리, 모니터링 자동화를 통해 재현성과 운영 안정성을 강화한다.

**CRISP-DM vs AI 증강 워크플로우 비교**: 두 워크플로우의 핵심 차이를 표로 정리하면 다음과 같다.

<표 1-3> 데이터 분석 워크플로우 단계별 비교
| 단계 | CRISP-DM (전통) | AI 증강 워크플로우 |
|------|----------------|------------------|
| 비즈니스 이해 | 수동 자료 조사 | LLM으로 도메인 지식 수집 가속화 |
| 데이터 이해 | 수동 EDA 코드 작성 | ChatGPT/Claude로 자동 EDA |
| 데이터 준비 | 60-80% 시간 소요 | 자동화 도구로 시간 단축 |
| 모델링 | 수동 알고리즘 선택/튜닝 | AutoML로 베이스라인 자동 구축 |
| 평가 | 수동 성능 측정 | 자동 실험 추적, 비교 대시보드 |
| 배포 | 수동 배포 스크립트 | MLOps 파이프라인 자동화 |

핵심적인 변화는 **시간 배분**이다. 전통적 워크플로우에서 데이터 준비에 60-80%의 시간이 소요되었다면, AI 증강 워크플로우에서는 이 비중이 30-40%로 줄어들고, 분석가는 비즈니스 문제 정의, 모델 해석, 의사결정 지원에 더 많은 시간을 할애할 수 있다.

### 1.3.3 AI 시대 워크플로우의 원칙

AI 도구를 적극적으로 활용하되, 분석의 책임과 통제권은 사람에게 남겨두는 것이 중요하다. LLM이 제안하는 인사이트는 언제든 잘못될 수 있으므로, 데이터와 도메인 지식으로 검증하는 절차가 필요하다. 또한 빠른 반복을 통해 초안을 만들고 개선하되, 핵심 의사결정에 사용된 프롬프트와 결과는 문서화해 재현성을 확보해야 한다. 마지막으로 API 호출에는 비용이 수반되므로, 개발 단계와 최종 산출물 단계에서 모델 선택과 호출량을 분리해 관리하는 편이 효율적이다.

## 1.4 LLM 기반 탐색적 데이터 분석

탐색적 데이터 분석(EDA)은 데이터 분석의 첫 단계이자 가장 중요한 단계다. 데이터의 구조, 패턴, 이상치를 파악하고, 후속 분석 방향을 결정한다. 이 절에서는 LLM을 활용하여 EDA를 자동화하고 가속화하는 방법을 살펴본다.

### 1.4.1 전통적 EDA의 한계

전통적인 EDA는 분석가가 직접 코드를 작성하고 결과를 해석하는 과정이다. 숙련된 분석가에게는 익숙한 작업이지만, 새로운 데이터셋마다 기본 통계, 분포 확인, 결측치 파악 같은 표준 코드를 반복 작성해야 한다는 점에서 시간이 많이 든다. 또한 어떤 패턴을 어디까지 볼지, 어떤 시각화를 만들지의 선택이 경험에 좌우되기 때문에 중요한 신호를 놓치거나, 반복 작업에서 실수가 누적될 가능성이 있다.

### 1.4.2 LLM 기반 EDA의 접근

LLM 기반 EDA는 두 가지 방식으로 구현할 수 있다.

**대화형 접근(ChatGPT Code Interpreter, Claude Artifacts)**: 대화형 접근은 데이터 파일을 업로드하고 자연어로 분석을 요청하는 방식이다. 기본 통계나 상관관계 시각화처럼 반복적인 작업을 빠르게 수행할 수 있지만, 세션 기반 작업은 분석 과정의 재현성과 커스터마이징 측면에서 제약이 생길 수 있다.

**API 기반 접근**: API 기반 접근은 데이터 프로파일을 생성해 LLM API에 전달하고, 해석과 권장사항을 받아 자동화 파이프라인에 연결하는 방식이다. 코드로 구현되기 때문에 실행 로그, 프롬프트, 결과를 함께 관리할 수 있어 재현성과 운영 자동화에 유리하다.

### 1.4.3 주요 LLM EDA 도구 비교

현재 사용 가능한 주요 LLM 기반 EDA 도구를 비교하면 다음과 같다.

<표 1-4> 주요 LLM 기반 EDA 도구 비교
| 도구 | 특징 | 장점 | 단점 |
|------|------|------|------|
| ChatGPT Code Interpreter | 파일 업로드 후 대화형 분석 | 직관적, 코드 자동 생성 | 파일 크기 제한, 세션 만료 |

각 도구의 선택은 분석 목적에 따라 달라진다. 빠른 탐색에는 ChatGPT나 Claude가 효과적이고, 재현 가능한 파이프라인 구축에는 API 기반 접근이 적합하다. 혼합형으로 사용하는 경우도 많다. 아이디어 발굴·가설 설정·다음 분석 단계 추천·결과 해석은 대화형(인간-루프)으로 빠르게 돌리고, 실제 산출물(표/수치/그림/리포트)을 자동으로 만들거나 반복 실행해야하면 API 기반(스크립트/파이프라인)으로 실행한다.

### 1.4.4 LLM EDA의 한계와 주의점

LLM 기반 EDA를 활용할 때는 환각, 보안, 비용의 세 가지 축을 중심으로 통제 전략을 세워야 한다. LLM은 데이터에 없는 패턴을 그럴듯하게 “발견”했다고 말할 수 있으므로, 상관관계와 인과관계를 구분하고 통계적 검증 절차를 반드시 포함해야 한다. 또한 민감 데이터를 외부 API로 전송하는 것은 보안 위험이므로, 기업 환경에서는 데이터 마스킹이나 온프레미스 모델을 검토하는 편이 안전하다. 마지막으로 API 호출은 토큰 단위로 비용이 발생하므로, 대규모 데이터셋에서는 프로파일을 요약하거나 샘플링을 활용해 입력을 설계해야 한다.

### 1.4.5 실습: LLM 기반 자동 EDA

이 장의 실습은 Iris 같은 공개 데이터셋으로 데이터 프로파일을 만들고, 그 요약 정보를 LLM에 전달해 분석 결과를 받는 과정을 다룬다. 핵심은 LLM의 출력을 그대로 “정답”처럼 받아들이는 것이 아니라, 입력 프로파일과 프롬프트, 결과를 함께 관리해 재현 가능한 분석 흐름으로 만드는 것이다. LLM의 출력은 모델과 프롬프트, 시점에 따라 달라질 수 있으므로, 문서에는 실제 실행으로 얻은 결과만 포함해야 한다.

|------|-----------|-------------|
| 소요 시간 | 수 시간 ~ 수일 | 수 분 |
| 초기 인사이트 | 분석가 역량 의존 | 자동 생성 |
| 도메인 지식 | 분석가가 제공 | 프롬프트로 주입 |
| 비용 | 인건비 | API 비용 |
| 확장성 | 선형 증가 | 병렬 처리 가능 |
| 깊이 있는 분석 | 우수 | 제한적 |
| 커스터마이징 | 완전 가능 | 프롬프트 의존 |

**권장 활용 전략**: 실무에서는 LLM을 "첫 탐색 단계의 가속기"로 두고, 핵심 의사결정은 전통적 EDA와 검증 절차로 마무리하는 전략이 유용하다. LLM이 제안한 인사이트는 가설 후보로 취급하고, 통계적 검정이나 추가 시각화로 확인하는 과정이 뒤따라야 한다. 반복 업무가 많은 조직이라면 입력 프로파일 생성과 결과 저장을 템플릿화해, 동일한 기준으로 여러 데이터셋을 빠르게 비교할 수 있도록 만드는 편이 효율적이다.

## 1.5 전통 ML vs 딥러닝: 선택 기준

데이터 분석 프로젝트에서 가장 자주 마주치는 질문 중 하나는 "전통적인 머신러닝을 사용할 것인가, 딥러닝을 사용할 것인가?"이다. 이 절에서는 두 접근법의 특성을 비교하고, 상황에 따른 선택 기준을 제시한다.

### 1.5.1 정형 데이터에서 트리 기반 모델의 우위

흥미롭게도, 정형 데이터(행과 열로 구성된 테이블 형태의 데이터)에서는 전형적인 머신러닝인 XGBoost(Chen & Guestrin, 2016), LightGBM, CatBoost와 같은 그래디언트 부스팅 모델이 딥러닝보다 일반적으로 우수한 성능을 보인다. 2022년 NeurIPS에 발표된 연구(Grinsztajn et al.)에 따르면, 중소 규모의 정형 데이터에서 트리 기반 앙상블이 다양한 딥러닝 모델을 상회했다. 트리 기반 모델이 정형 데이터에서 강한 이유는 입력 변수의 이질성을 그대로 다루기 쉽기 때문이다. 분할 규칙으로 비선형 관계와 상호작용을 자연스럽게 포착하고, 극단값이나 일부 결측에 대한 민감도가 상대적으로 낮다. 또한 변수 중요도나 SHAP 같은 도구로 예측 근거를 설명하기 쉬워, 규제 산업에서 요구되는 해석 가능성 측면에서도 장점이 있다.

### 1.5.2 딥러닝이 유리한 상황

딥러닝은 이미지·텍스트·음성처럼 비정형 데이터가 핵심 신호를 담고 있을 때 강점을 보인다. 또한 샘플이 충분히 크거나, 순서와 문맥 같은 장기 의존성이 중요한 문제, 혹은 텍스트와 이미지처럼 여러 모달리티를 함께 다뤄야 하는 문제에서 유연한 표현 학습이 가능하다. 사전학습된 모델을 전이학습으로 활용할 수 있다면, 적은 데이터로도 성능을 끌어올릴 여지도 커진다. 적어도 지금 시점에서는 딥러닝이 만능은 아니라는 것이다.

### 1.5.3 의사결정 가이드

접근법 선택은 데이터 유형, 데이터 규모, 해석 가능성 요구, 프로젝트 제약을 순서대로 점검하는 방식이 실무적으로 안전하다. 이 네 가지 축은 서로 영향을 주기 때문에, 단일 기준으로 결론을 내리기보다 우선순위를 정해 조합으로 판단해야 한다. 표 1.6은 이러한 판단을 빠르게 정리할 수 있도록 전통 ML과 딥러닝의 유리한 조건을 비교한 가이드다.

<표 1-6> 전통 ML vs 딥러닝 선택 가이드

| 고려 요소   | 전통 ML 유리       | 딥러닝 유리            |
| ----------- | ------------------ | ---------------------- |
| 데이터 유형 | 정형 데이터        | 이미지, 텍스트, 시계열 |
| 데이터 규모 | 중소규모 (< 100만) | 대규모 (> 100만)       |
| 해석 필요성 | 높음               | 낮음                   |
| 개발 속도   | 빠름               | 상대적으로 느림        |
| 컴퓨팅 자원 | CPU 충분           | GPU 권장/필수          |
| 특성 공학   | 수동 또는 경량     | 자동 학습              |
| 전이학습    | 제한적             | 활용 가능              |

**실무에서의 접근**: 실무에서는 트리 모델로 빠르게 베이스라인을 구축한 뒤, 추가 성능이 필요하거나 비정형 데이터를 활용해야 할 때 딥러닝을 확장하는 순서가 효율적이다. 예를 들어 AutoGluon이나 LightGBM으로 기준선을 만들고, 요구 수준을 충족하면 복잡도를 불필요하게 늘리지 않는 것이 합리적이다. 반대로 성능 한계가 분명하다면 딥러닝을 도입하거나, LLM으로 텍스트 특성을 추출해 XGBoost 입력으로 쓰는 하이브리드 접근처럼 두 계열을 결합하는 전략도 고려할 수 있다. 무엇보다 데이터가 축적되고 요구사항이 변하는 만큼, 선택은 일회성이 아니라 주기적으로 재평가해야 한다.

### 1.5.4 실무 사례: 언제 무엇을 선택하는가

은행 신용 평가처럼 규제 준수와 설명 가능성이 중요한 문제에서는, 소득·부채·신용 이력 같은 정형 데이터로 XGBoost를 학습하고 SHAP으로 예측 근거를 제시하는 조합이 실무적으로 설득력이 높다. 이커머스 추천처럼 사용자-상품 상호작용이 매우 크고 이미지·설명 텍스트까지 함께 쓰는 상황에서는, 임베딩과 이미지 특성을 함께 학습하는 딥러닝이 성능 면에서 유리한 경우가 많다. 제조 설비 이상 탐지는 비지도학습(Isolation Forest)으로 이상 징후를 먼저 포착하고, 시계열 패턴을 학습하는 LSTM을 결합해 신호를 강화하는 식의 단계적 설계가 자주 쓰인다. 콜센터 문의 자동 분류처럼 텍스트 레이블 데이터가 어느 정도 확보되어 있다면 BERT 미세조정을 우선 검토하고, 레이블이 부족할 때는 GPT 기반의 zero-shot 분류를 후보로 두고 비교하는 편이 현실적이다. 이처럼 최적의 선택은 데이터 유형, 규모, 비즈니스 요구사항의 조합에 따라 결정된다. 중요한 것은 단일 기준이 아닌, 종합적인 판단이다.

이 장에서는 데이터 사이언스가 통계학에서 출발해 데이터마이닝, 머신러닝, 딥러닝을 거쳐 LLM 시대로 확장되는 흐름을 정리했다. 학습 방식의 관점에서는 지도학습·비지도학습·강화학습·자기지도학습이 서로 다른 신호를 사용하며, 문제의 구조와 레이블 가용성에 따라 선택이 달라진다는 점이 핵심이다. 또한 CRISP-DM은 여전히 유효한 뼈대이지만, LLM·AutoML·MLOps를 적절히 결합하면 반복 작업을 줄이고 분석가가 문제 정의와 검증, 의사결정 지원에 더 집중할 수 있다. 마지막으로 정형 데이터에서는 트리 기반 모델이 강력한 경우가 많고, 비정형·대규모·전이학습 환경에서는 딥러닝이 유리해지는 경향이 있으므로, 데이터 유형과 규모, 해석 요구, 제약 조건을 함께 고려해 판단해야 한다.

## 1.6 본 도서에서 다루는 분석 기법 총람

### 제2장: 데이터 전처리와 특성 공학

| 방법 | 기능/역할 | 특징 |

| Mean/Median/KNN 대체 | 결측치를 통계량 또는 유사 샘플로 채움 | 단순 대체는 빠르나 분산 손실, KNN은 관계 보존 |

| UMAP | 전역+지역 구조 모두 보존하는 비선형 축소 | t-SNE보다 빠르고 새 데이터 투영 가능 |

| Sentence-BERT | 문장 단위 의미 임베딩 생성 | 밀집 벡터, 의미 유사도 계산에 적합 |
| Entity Embedding | 범주형 변수의 저차원 표현 학습 | 유사 범주 자동 발견, 고카디널리티 처리 |

| TabNet | 어텐션 기반 정형 데이터 모델 | 순차적 특성 선택, 해석 가능 마스크 |
| FT-Transformer | 정형 데이터용 트랜스포머 | 특성 토큰화, 상호작용 학습 |
| 오토인코더 | 비선형 차원 축소 및 표현 학습 | 재구성 손실, 잠재 표현 추출 |

### 제8장: 생성 모델

| 방법 | 기능/역할 | 특징 |
|------|----------|------|
| GAN | 생성자-판별자 적대적 학습 | 고품질 합성 데이터, 모드 붕괴 주의 |
| CTGAN | 정형 데이터 전용 GAN | 모드별 정규화, 조건 벡터로 희귀 범주 처리 |
| VAE | 확률적 잠재 표현 학습 | 연속 잠재공간, 샘플링으로 새 데이터 생성 |
| VAE 데이터 증강 | 소수 클래스 합성 샘플 생성 | 불균형 데이터 보정, SMOTE 대안 |
| VAE 이상탐지 | 재구성 오차 기반 이상 점수 | 비지도 학습, 정상 패턴 학습 후 탐지 |
| Diffusion Model | 역확산 과정으로 분포 샘플링 | 안정적 학습, 불확실성 정량화 |
| 팬 차트 | 분위수 기반 예측 구간 시각화 | 5%/50%/95% 시나리오, 리스크 커뮤니케이션 |

### 제9장: 시계열 분석

| 방법 | 기능/역할 | 특징 |
|------|----------|------|
| STL 분해 | 추세/계절/잔차 분리 | 이상치 강건, 시계열 구조 파악 |
| ARIMA/SARIMA | 자기회귀 이동평균 예측 | Box-Jenkins 방법론, 계절성 지원 |
| Prophet | 추세+계절+휴일 가법 모형 | 비즈니스 시계열 특화, 변화점 자동 탐지 |
| LSTM | 장기 의존성 학습 신경망 | 게이트 메커니즘, 시퀀스 패턴 포착 |
| GRU | LSTM 경량화 버전 | 게이트 수 감소, 빠른 학습 |
| Transformer | 어텐션 기반 시계열 예측 | 병렬화 가능, 장거리 의존성 직접 참조 |
| Informer | 장기 시계열 특화 트랜스포머 | 희소 어텐션, 계산 효율성 향상 |

### 제10장: 생존 분석

| 방법 | 기능/역할 | 특징 |
|------|----------|------|
| Kaplan-Meier | 비모수적 생존 함수 추정 | 중도절단 처리, 생존 곡선 시각화 |
| 로그순위 검정 | 두 그룹 생존 곡선 비교 | 비모수 검정, 전체 기간 통합 비교 |
| Cox 비례위험 | 공변량 효과 추정 회귀 모형 | 반모수, 위험비(HR) 해석 |
| AFT | 생존 시간 직접 모형화 | 비례위험 가정 불필요, 시간 스케일 해석 |
| XGBoost AFT | 부스팅 기반 생존 분석 | 비선형 관계 학습, 중도절단 처리 |
| Random Survival Forest | 트리 기반 생존 분석 | 앙상블 효과, 변수 중요도 제공 |
| DeepSurv | Cox 손실 + 신경망 | 비선형 위험 함수 학습, 고차원 입력 |

### 제11장: 그래프 분석

| 방법 | 기능/역할 | 특징 |
|------|----------|------|
| 연결 중심성 | 직접 연결 수로 영향력 측정 | 인플루언서 식별, 계산 단순 |
| 매개 중심성 | 최단 경로 통과 빈도로 브리지 탐지 | 정보 병목점 발견, 계산 비용 높음 |
| 근접 중심성 | 다른 노드까지 평균 거리 측정 | 정보 전파 효율성 평가 |
| PageRank | 재귀적 중요도 계산 | 중요 노드 연결 가중치, 스팸 강건 |
| Louvain | 모듈성 최적화 커뮤니티 탐지 | 빠른 속도, 대규모 네트워크 적합 |
| 링크 예측 | 미래 관계 형성 확률 추정 | 공통 이웃, Jaccard, Adamic-Adar |

### 제12장: 그래프 신경망

| 방법 | 기능/역할 | 특징 |
|------|----------|------|
| GCN | 이웃 특성 정규화 평균 집계 | 스펙트럴 기반, 단순 구현 |
| GAT | 어텐션으로 이웃 가중치 학습 | 이웃 중요도 차별화, 해석 가능 |
| GraphSAGE | 샘플링 기반 귀납적 학습 | 새 노드 임베딩 가능, 대규모 확장 |
| GNN 노드 분류 | 그래프 구조 활용 노드 속성 예측 | 구조+속성 결합, 반라벨 학습 |
| GNN 링크 예측 | 노드 임베딩으로 관계 확률 산출 | 내적/거리 기반 점수, 휴리스틱 대체 |
