### 6.1 토픽 모델링이 해결하는 문제

토픽 모델링(Topic Modeling)은 **레이블 없이 대량의 문서에서 잠재된 주제를 자동으로 발견**하는 비지도학습 기법이다. 핵심 질문은 "수천 개의 문서에서 사람들이 주로 이야기하는 주제는 무엇인가?"이다.

#### 산출물 두 가지

| 산출물 | 설명 | 예시 |
| ------ | ---- | ---- |
| 토픽별 키워드 분포 | 각 토픽이 어떤 단어들로 구성되는지 | "배송, 택배, 도착, 늦음" → 배송 관련 토픽 |
| 문서별 토픽 분포 | 각 문서가 어떤 토픽들의 조합인지 | "배송 60% + 품질 40%" |

#### 군집화 vs 토픽 모델링

비유하면, 군집화는 "이 학생은 A반"처럼 **하나의 그룹에 배정**하는 것이고, 토픽 모델링은 "이 학생은 수학 60% + 영어 40% 관심"처럼 **여러 주제의 혼합으로 표현**하는 것이다.

| 구분 | 군집화 | 토픽 모델링 |
| ---- | ------ | ----------- |
| 할당 방식 | 하드 할당 (문서 → 하나의 군집) | 소프트 할당 (문서 → 여러 토픽의 혼합) |
| 핵심 산출물 | 군집 레이블 | 토픽-단어 분포, 문서-토픽 분포 |
| 대표 알고리즘 | K-Means, HDBSCAN | LDA, BERTopic |

#### 비즈니스 응용

- **고객 VOC 분석**: 수천 건의 리뷰에서 "배송 불만", "품질 문제", "가격 만족" 등 주요 이슈를 자동 발견
- **뉴스 트렌드**: 매일 쏟아지는 뉴스에서 주요 이슈를 파악하고 시간별 변화를 추적
- **연구 동향**: 학술 논문 초록에서 급부상/쇠퇴하는 연구 주제를 파악

#### 두 가지 패러다임

| 구분 | LDA | BERTopic |
| ---- | --- | -------- |
| 입력 표현 | Bag-of-Words (단어 빈도) | 임베딩 (의미 벡터) |
| 문맥 이해 | 불가 | 가능 |
| 토픽 수 결정 | 사전 지정 필수 | 자동 결정 |
| 짧은 텍스트 | 취약 | 강건 |

| 전통 방식 | AI 시대 진화 | 개선 효과 |
| --------- | ------------ | --------- |
| LDA (단어 빈도 기반) | BERTopic (임베딩 기반) | 문맥 이해, 동의어 처리 |
| 수동 토픽 명명 | LLM 자동 레이블링 | 일관성, 확장성 |
| 정적 토픽 분석 | 동적 토픽 모델링 | 시간별 트렌드 추적 |

### 6.2 LDA: 단어 빈도로 주제를 추론하는 원리

LDA(Latent Dirichlet Allocation)는 **"문서가 어떻게 만들어졌는가?"를 역추론**하는 생성 모델이다.

#### 핵심 가정 두 가지

1. 각 문서는 **여러 토픽의 혼합**으로 구성된다 (예: 리뷰 = "배송 60% + 품질 40%")
2. 각 토픽은 **단어들의 확률 분포**로 정의된다 (예: "배송" 토픽 → "택배", "도착", "늦음" 높은 확률)

비유하면, 여러 색 페인트(토픽)를 섞어 그림(문서)을 그린다고 생각하면 된다. 파라미터 alpha는 팔레트에 미리 담아 놓은 각 색의 양이다. alpha가 작으면 한두 색이 압도적인 선명한 그림(소수 토픽 집중), alpha가 크면 여러 색이 골고루 어우러진 복합적 그림이 나온다.

#### 문서 생성 과정 (역추론 대상)

```
1. 문서 d의 토픽 분포 θ_d를 디리클레 분포에서 샘플링
2. 각 단어 위치 n에 대해:
   a. 토픽 z_n을 θ_d에서 샘플링 (이 단어가 어느 토픽에 속하는지 결정)
   b. 선택된 토픽 z_n의 단어 분포 φ_z에서 단어 w_n을 샘플링
```

#### LDA의 한계

- **문맥 무시**: Bag-of-Words 가정으로 "은행(금융)" vs "은행(강가)"를 구분하지 못함
- **짧은 텍스트에 취약**: 트윗, 짧은 리뷰는 동시 출현 정보가 부족
- **토픽 수 K 사전 지정 필수**: 여러 K를 시도하며 일관성 점수를 비교해야 함
- **동의어 분산**: "배송", "택배", "배달"을 별개로 처리

#### 토픽 평가지표

| 지표 | 의미 | 기준 |
| ---- | ---- | ---- |
| C_v (일관성) | 상위 단어 쌍의 의미적 유사도 평균 | 0.4 이상 수용, 0.5 이상 양호 |
| UMass (일관성) | 단어 쌍의 공동 발생 빈도 | 0에 가까울수록 좋음 |
| NPMI (일관성) | 정규화된 점별 상호정보량 | -1~1, 높을수록 좋음 |
| 토픽 다양성 | 상위 N개 단어 중 고유 단어 비율 | 1에 가까울수록 좋음 (중복 적음) |

### 6.3 BERTopic: 의미적 유사성으로 주제 발견

BERTopic은 **"의미적으로 유사한 문서를 먼저 군집화하고, 그 후에 각 군집을 대표하는 키워드를 추출한다"**는 접근이다. LDA가 단어 빈도에 의존하는 것과 달리, 문맥을 반영한 임베딩을 사용한다.

#### BERTopic 5단계 파이프라인

비유하면, 도서관의 책을 정리하는 과정이다:

1. **임베딩** = 각 책의 내용을 요약 카드로 만든다 (SentenceTransformer → 384차원 벡터)
2. **UMAP** = 요약 카드를 큰 테이블 위에 비슷한 것끼리 가까이 놓는다 (384차원 → 5차원)
3. **HDBSCAN** = 가까이 모인 카드들을 그룹으로 묶는다 (밀도 기반 군집화)
4. **토큰화** = 각 그룹의 카드에서 공통 키워드를 추출한다 (CountVectorizer)
5. **c-TF-IDF** = "이 그룹에서만 특별히 자주 등장하는 단어"를 찾아 토픽 대표 키워드로 선정한다

#### UMAP 주요 파라미터

| 파라미터 | 의미 | BERTopic 기본값 |
| -------- | ---- | --------------- |
| n_neighbors | 지역 구조 보존 정도 | 15 |
| n_components | 출력 차원 | 5 |
| min_dist | 점들 간 최소 거리 | 0.0 (밀집 군집 허용) |
| metric | 거리 측정 방식 | cosine |

#### HDBSCAN 주요 파라미터

| 파라미터 | 의미 | 조정 가이드 |
| -------- | ---- | ----------- |
| min_cluster_size | 최소 군집 크기 | 크면 → 큰 토픽 적게, 작으면 → 작은 토픽 많이 |
| min_samples | 코어 포인트 밀도 | 노이즈 허용 수준 조절 |

#### LDA vs BERTopic 상세 비교

| 비교 항목 | LDA | BERTopic |
| --------- | --- | -------- |
| 입력 표현 | Bag-of-Words | 임베딩 (밀집 벡터) |
| 문맥 이해 | 불가 (단어 독립 가정) | 가능 (문맥 반영) |
| 토픽 할당 | 소프트 (확률 분포) | 하드 (단일 토픽) |
| 토픽 수 결정 | 사전 지정 필수 | 자동 결정 (HDBSCAN) |
| 짧은 텍스트 | 취약 (통계 부족) | 강건 (임베딩 활용) |
| 동의어 처리 | 불가 | 가능 (의미적 유사성) |
| 계산 비용 | 낮음 | 높음 (임베딩 생성) |
| 동적 토픽 | 별도 확장 필요 | 내장 지원 |

#### 노이즈 문서와 토픽 축소

- HDBSCAN은 **노이즈(-1 토픽)**를 자동으로 분리한다. `reduce_outliers()`로 가장 유사한 토픽에 재할당할 수 있다
- `reduce_topics()`로 유사한 토픽을 병합하여 해석 가능성을 높일 수 있다

### 6.4 LLM 기반 토픽 라벨링

토픽 모델링의 산출물은 **토픽별 키워드 리스트**다. ["배송", "택배", "도착", "늦음"]이라는 키워드를 보고 적절한 이름을 붙이는 것은 전통적으로 도메인 전문가의 몫이었다. 그러나 이 방식은 주관적이고, 토픽이 수십 개로 늘어나면 비현실적이다.

#### BERTopic + LLM 통합

BERTopic 0.15+에서는 `representation_model` 파라미터로 OpenAI GPT 등을 연동하여 자동 레이블링이 가능하다.

```python
from bertopic.representation import OpenAI

representation_model = OpenAI(
    model="gpt-3.5-turbo",
    chat=True,
    prompt="다음 키워드들을 바탕으로 토픽 이름을 5단어 이내로 지어주세요: [KEYWORDS]"
)
```

#### 프롬프트 엔지니어링 핵심

| 기법 | 설명 | 예시 |
| ---- | ---- | ---- |
| 도메인 컨텍스트 | 역할과 분석 영역을 명시 | "당신은 이커머스 고객 리뷰 분석 전문가입니다" |
| 출력 형식 지정 | 구조화된 응답 요청 | JSON 형식: {"label": "...", "confidence": 0.9} |
| Few-shot 예시 | 원하는 레이블 스타일 전달 | ["배송", "택배", "도착"] → "배송 서비스" |
| 제약 조건 | 길이, 언어 등 제한 | "5단어 이내, 한국어로 작성" |

#### 레이블링 품질 평가

- **일관성**: 유사한 키워드 집합에 일관된 레이블이 생성되는가?
- **도메인 적합성**: 실무자가 이해할 수 있는 용어인가?
- **샘플 검증**: 대표 문서를 읽어보고 레이블이 적절한지 수동 확인

### 6.5 동적 토픽: 시간에 따른 토픽 변화 추적

동적 토픽 모델링은 **시간에 따른 토픽의 등장, 증가, 감소를 추적**하는 방법론이다. "뉴스에서 코로나 관련 토픽이 언제 급증했는가?", "백신 토픽은 언제 나타났는가?"와 같은 질문에 답한다.

#### 추세 유형

| 패턴 | 설명 | 비즈니스 의미 |
| ---- | ---- | ------------- |
| 급격한 상승 | 짧은 기간에 빈도 급증 | 새로운 이슈 발생, 위기 상황 |
| 점진적 상승 | 완만하게 증가 | 장기적 관심 증가 |
| 급격한 하락 | 짧은 기간에 빈도 급감 | 이슈 해결, 관심 이동 |
| 주기적 변동 | 규칙적 등락 | 계절성, 정기 이벤트 |
| 안정적 패턴 | 일정 빈도 유지 | 지속적 관심 주제 |

#### BERTopic의 동적 분석

```python
# 문서별 타임스탬프를 제공하면 시간에 따른 토픽 변화를 계산
topics_over_time = topic_model.topics_over_time(texts, timestamps, nr_bins=12)
```

`nr_bins`는 시간을 몇 개 구간으로 나눌지 결정한다. 3년치 데이터에서 `nr_bins=36`이면 월별, `nr_bins=12`이면 분기별 분석이 된다.

#### 토픽 생명 주기 분석

각 토픽의 **출현(emergence), 최고점(peak), 소멸(decline)** 시점을 자동으로 식별하여, 이슈의 흥망성쇠를 정량적으로 파악한다.

### 6.6 멀티모달 토픽 모델링: 텍스트와 이미지 통합 분석

멀티모달 토픽 모델링은 **텍스트와 이미지를 동일한 벡터 공간에 임베딩**하여 더 풍부한 토픽을 발견하는 기법이다. CLIP(Contrastive Language-Image Pre-training)이 텍스트와 이미지를 512차원의 동일한 공간에 투영하여, 의미적으로 유사한 텍스트와 이미지가 가까운 위치에 배치된다.

#### 활용 사례

| 분야 | 텍스트 | 이미지 | 통합 효과 |
| ---- | ------ | ------ | --------- |
| 소셜 미디어 | 캡션 | 게시물 사진 | 시각적 트렌드 포착 |
| 전자상거래 | 상품 설명 | 제품 사진 | 정확한 카테고리 분류 |
| 뉴스 모니터링 | 기사 본문 | 관련 이미지 | 텍스트-이미지 불일치 감지 |

#### BERTopic 멀티모달 지원

```python
from bertopic.backend import MultiModalBackend
from bertopic.representation import VisualRepresentation

# CLIP 기반 결합 임베딩
embedding_model = MultiModalBackend('clip-ViT-B-32', batch_size=32)

# 문서와 이미지를 함께 학습
topics, probs = topic_model.fit_transform(documents=captions, images=image_paths)
```

텍스트 임베딩과 이미지 임베딩을 평균화하여 결합 임베딩을 만들고, 이를 기반으로 UMAP + HDBSCAN으로 토픽을 발견한다.

```text
1. 토픽 모델링은 비지도학습으로 문서의 잠재 주제를 자동 발견한다. 군집화와 달리 소프트 할당이 가능하다.
2. LDA는 단어 빈도 기반의 생성 모델로 해석이 쉽지만, 문맥을 무시하고 짧은 텍스트에 취약하다.
3. BERTopic은 임베딩 기반으로 문맥을 반영하며, 5단계 파이프라인의 각 모듈을 교체할 수 있다.
4. LLM 레이블링은 프롬프트 엔지니어링으로 품질을 조절한다. 도메인 컨텍스트와 Few-shot이 핵심이다.
5. 동적 토픽 모델링은 시간에 따른 토픽 변화를 추적한다. 생명 주기 분석으로 이슈의 흥망성쇠를 파악한다.
6. 멀티모달 토픽 모델링은 텍스트와 이미지를 결합하여 단일 모달리티보다 정확한 토픽을 발견한다.
```

---

### 제출 기준

- ✓ 5개 제공 코드를 모두 실행하고 결과 수치를 기록했다
- ✓ Step 3의 프롬프트 중 최소 4개 이상을 AI 도구로 코드를 작성하고 실행했다
- ✓ 직접 작성한 코드와 실행 결과를 포함했다
- ✓ 각 실습의 결과를 왜 그런 결과가 나왔는지 1~2문장으로 해석했다

### 바이브 코딩 팁

- AI 도구에 프롬프트를 줄 때, **데이터와 목표를 구체적으로** 적을수록 좋은 코드가 나온다
- 생성된 코드를 그대로 실행하지 말고, **코드를 읽고 이해한 뒤** 실행한다
- 에러가 나면 에러 메시지를 AI에 다시 붙여넣어 수정을 요청한다
- 결과가 예상과 다르면 왜 다른지 AI에게 물어본다
- **AI가 생성한 코드의 결과도 반드시 검증**한다 — 이것이 이론에서 배운 원칙의 실천이다
