### 8.1 생성 모델이 데이터 분석에 기여하는 방식

전통적인 판별 모델(Discriminative Model)은 "이 입력이 어떤 클래스인가?"를 예측한다. 즉 조건부 확률 p(Y|X)를 학습한다. 반면 생성 모델(Generative Model)은 "이 데이터는 어떻게 만들어졌는가?"를 학습한다. 데이터의 분포 p(X) 자체를 학습하므로, 관측되지 않은 새로운 데이터를 생성할 수 있다.

비유하면, 판별 모델은 "이 그림이 고양이인지 개인지" 판별하는 감정사이고, 생성 모델은 "고양이 그림을 직접 그릴 수 있는" 화가다. 화가가 되려면 고양이의 구조를 더 깊이 이해해야 한다.

#### 생성 모델의 세 가지 기여

| 기여 방식 | 핵심 아이디어 | 실무 예시 |
| --------- | ------------- | --------- |
| 합성 데이터 생성 | 원본의 통계적 특성을 보존하면서 새 데이터 생성 | 개인정보 보호, 데이터 공유 |
| 데이터 증강 + 이상치 탐지 | 소수 클래스 분포를 학습하여 합성 샘플 생성 / 재구성 오차로 이상 탐지 | 사기 탐지, 불량품 검출 |
| 불확실성 정량화 | 점 추정이 아닌 분포 전체를 생성 | 매출 예측, 리스크 관리 |

#### 대표 생성 모델 비교

| 모델 | 핵심 원리 | 비유 | 강점 | 약점 |
| ---- | --------- | ---- | ---- | ---- |
| GAN | Generator와 Discriminator의 경쟁 | 위조범 vs 감정사 | 고품질 합성 데이터 | 학습 불안정, Mode Collapse |
| VAE | 잠재공간으로 압축 후 복원 | 데이터를 "요약"했다가 "복원" | 구조화된 잠재공간, 이상치 탐지 | 생성 품질이 GAN보다 낮음 |
| Diffusion | 노이즈를 점진적으로 추가/제거 | 깨끗한 사진에 눈보라를 뿌렸다가 천천히 복원 | 다양성과 안정성 | 높은 계산 비용 |

### 8.2 합성 데이터 생성: GAN과 CTGAN

#### GAN의 기본 원리

GAN(Generative Adversarial Network)은 두 신경망의 경쟁으로 학습한다.

- **Generator(생성자)**: 무작위 노이즈 z에서 가짜 데이터를 생성한다. 목표는 Discriminator를 속이는 것이다.
- **Discriminator(판별자)**: 진짜 데이터와 가짜 데이터를 구별한다. 목표는 속지 않는 것이다.

이 과정은 위조지폐범과 경찰의 관계와 같다. 위조범(Generator)이 점점 정교한 위조지폐를 만들수록, 경찰(Discriminator)도 감별 능력을 높인다. 학습이 충분히 진행되면, Generator가 만든 데이터는 진짜와 구별할 수 없게 된다.

수학적으로 GAN은 미니맥스 게임이다:

```
min_G max_D V(D, G) = E[log D(x)] + E[log(1 - D(G(z)))]
```

이상적으로 학습이 완료되면 Discriminator의 정확도는 50%(랜덤 추측)에 수렴한다.

#### 정형 데이터의 문제: 왜 기본 GAN으로는 부족한가

이미지와 달리 정형 데이터(테이블)는 세 가지 특수한 문제가 있다:

1. **혼합 데이터 타입**: 나이(연속형)와 직업(범주형)이 한 테이블에 공존
2. **다양한 분포**: 소득은 편포(right-skewed), 나이는 다봉(multimodal) 등 변수마다 분포가 다름
3. **클래스 불균형**: 사기 거래 1%, 정상 99%처럼 범주 빈도가 극단적

#### CTGAN: 정형 데이터 전용 GAN

CTGAN(Conditional Tabular GAN)은 위 문제를 해결하기 위해 세 가지 기법을 도입했다:

| 기법 | 해결하는 문제 | 비유 |
| ---- | ------------- | ---- |
| Mode-specific Normalization | 다봉 분포 보존 | 산이 여러 개인 지형을 봉우리별로 따로 정규화 |
| Conditional Vector | 클래스 불균형 | 소수 범주를 의도적으로 균등하게 뽑아 학습 |
| Training-by-Sampling | 희귀 조합 학습 | 매 학습마다 다른 조건을 골고루 경험 |

**Wasserstein Loss와 학습 안정성**: 기본 GAN은 학습이 불안정하다. CTGAN은 Wasserstein Distance를 사용하여 이를 해결한다. 비유하면, "흙더미 옮기기" 문제다. 실제 분포(산)를 생성 분포(골짜기)로 바꾸는 데 필요한 최소 이동 비용이 Wasserstein Distance다. 이 지표는 두 분포가 멀리 떨어져 있어도 "어느 방향으로 가면 가까워지는지" 안정적인 학습 신호를 제공한다.

#### 합성 데이터 품질 평가

합성 데이터는 "만들었다"가 끝이 아니라, 세 가지 차원에서 평가해야 한다:

| 평가 차원 | 질문 | 대표 지표 | 권장 기준 |
| --------- | ---- | --------- | --------- |
| 통계적 유사도 | 분포가 비슷한가? | KS 검정, 상관구조 보존도 | p >= 0.05, >= 95% |
| ML 효용성 | 합성 데이터로 학습한 모델이 실제에서도 잘 작동하는가? | TSTR Ratio | >= 90% |
| 프라이버시 | 원본 개인을 역추적할 수 없는가? | DCR, MIA 성공률 | DCR 충분히 큼, MIA <= 55% |

**TSTR(Train on Synthetic, Test on Real)**: 합성 데이터로 모델을 학습하고 실제 데이터로 테스트한다. 실제 데이터로 학습한 모델(TRTR)과 비교하여, TSTR Ratio = (TSTR 정확도 / TRTR 정확도) x 100이 90% 이상이면 실무 대체 가능으로 판단한다.

#### 정형 데이터 생성 모델 비교

| 모델 | 아키텍처 | 범주형 처리 | 적합한 상황 |
| ---- | -------- | ----------- | ----------- |
| CTGAN | GAN + VGM | Mode-specific | 다봉 분포, 불균형 범주 |
| ARF | Random Forest | 자연스러움 | 빠른 프로토타이핑 |
| TT-GAN | Transformer | 임베딩 | 변수 간 복잡한 상호작용 |
| TVAE | VAE | 유사 CTGAN | 잠재공간 활용 필요 시 |

### 8.3 VAE: 데이터 증강과 이상치 탐지

#### VAE의 핵심 아이디어

VAE(Variational Autoencoder)는 데이터를 저차원 잠재공간(Latent Space)으로 "요약"했다가 다시 "복원"하는 모델이다.

- **인코더**: 고차원 입력 x를 저차원 잠재벡터 z로 압축한다. 이때 z는 확률 분포(평균 mu, 분산 sigma)로 표현된다.
- **재매개변수화(Reparameterization)**: z = mu + sigma * epsilon (epsilon ~ N(0,1)). 이 트릭으로 랜덤 샘플링 과정에서도 역전파가 가능하다.
- **디코더**: 잠재벡터 z를 원래 차원으로 복원한다.

비유하면, 인코더는 긴 소설을 "줄거리 요약"으로 압축하는 과정이고, 디코더는 줄거리에서 소설을 다시 써내는 과정이다. 좋은 요약(잠재공간)은 핵심 정보를 보존하면서 불필요한 세부를 제거한다.

#### 데이터 증강: 소수 클래스 생성

클래스 불균형은 실무에서 매우 흔하다. 사기 거래는 전체의 0.1% 미만, 희귀질환 환자는 1% 미만이다. 이때 모델은 "모두 정상"으로 예측해도 99% 정확도를 달성하지만, 실제로는 쓸모가 없다.

**VAE 증강 절차**:
1. 소수 클래스 샘플로 VAE를 학습한다
2. 소수 클래스를 인코딩하여 잠재공간의 분포(mu, sigma)를 파악한다
3. 해당 분포에서 새로운 잠재벡터를 샘플링한다
4. 디코딩하여 합성 소수 클래스 샘플을 생성한다

SMOTE와의 차이: SMOTE는 기존 샘플 사이를 선형 보간하여 새 샘플을 만든다. VAE는 잠재공간에서 데이터의 구조를 학습하므로, 더 현실적인 변형을 생성할 수 있다. 예를 들어 "심야 고액 결제" 패턴을 학습하면, "새벽 해외 결제"처럼 원본에 없던 현실적인 변형도 생성 가능하다.

#### 이상치 탐지: 재구성 오차 활용

VAE는 정상 데이터의 패턴을 학습한다. 이상치는 학습하지 않은 패턴이므로 재구성이 부정확하여 **재구성 오차가 크다**.

| 단계 | 설명 |
| ---- | ---- |
| 1. 정상 데이터만 학습 | VAE가 "정상이란 이런 것"을 학습 |
| 2. 새 데이터 입력 | 인코딩 → 디코딩으로 복원 |
| 3. 재구성 오차 계산 | 원본과 복원의 차이 (MSE) |
| 4. 임계값 비교 | 오차가 95% 백분위수 초과 → 이상치 판정 |

비유하면, 정상 제품만 보고 학습한 검수 로봇이 불량품을 보면 "이건 내가 아는 패턴과 다르다"고 판단하는 것이다.

#### VAE 사용 권장 영역

| 용도 | 조건 | 주의사항 |
| ---- | ---- | -------- |
| 데이터 증강 | 소수 클래스 수백 개 이상 | 합성 데이터는 학습용, 실제 관측을 대체하지 않음 |
| 이상치 탐지 | 정상 데이터 충분, 이상치 라벨 없음 | 이상치 유형 분류는 추가 분석 필요 |

```python
# VAE 손실 = 재구성 오차 + KL Divergence
def vae_loss(recon_x, x, mu, logvar):
    recon_loss = F.mse_loss(recon_x, x, reduction='sum')
    kld = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return recon_loss + kld
```

- 학습용 사기 샘플 30개만으로도 효과가 있는 이유: VAE가 30개의 잠재 분포를 학습하여 그 주변에서 다양한 변형을 생성

### 8.4 Diffusion Model: 불확실성 정량화

#### Diffusion의 핵심 아이디어

Diffusion Model은 두 단계로 작동한다:

1. **Forward Process (노이즈 추가)**: 깨끗한 데이터에 점진적으로 노이즈를 추가하여 순수 노이즈로 만든다 (T 단계)
2. **Reverse Process (노이즈 제거)**: 순수 노이즈에서 시작하여 점진적으로 노이즈를 제거하여 데이터를 복원한다

비유하면, 깨끗한 사진에 눈보라를 점점 강하게 뿌려 하얀 화면으로 만든 뒤, 그 과정을 역으로 학습하여 하얀 화면에서 사진을 복원하는 것이다. 복원할 때마다 조금씩 다른 결과가 나올 수 있으므로, 여러 번 복원하면 "가능한 미래의 분포"를 얻는다.

#### 왜 불확실성 정량화에 유용한가

전통적 회귀 모델은 "다음 달 매출은 100억원"이라는 점 추정만 제공한다. 하지만 CFO는 "최악의 경우 얼마까지 떨어질 수 있는가?"를 알아야 비상 자금을 준비할 수 있다.

Diffusion Model은 100번 샘플링하면 100개의 서로 다른 "가능한 미래"를 생성한다. 이를 분위수로 정리하면:

| 시나리오 | 분위수 | 리스크 대응 |
| -------- | ------ | ----------- |
| 최악 | 5%ile | 긴급 비용 절감 |
| 보수적 | 25%ile | 보수적 예산 |
| 중립 | 50%ile (중앙값) | 계획대로 진행 |
| 낙관적 | 75%ile | 공격적 투자 |
| 최선 | 95%ile | 확장 계획 |

#### Diffusion의 적합/부적합 상황

| 적합한 경우 | 부적합한 경우 |
| ----------- | ------------- |
| 매출/수요 예측, 리스크 시나리오 | 실시간 예측 (계산 비용이 높음) |
| 초기 설계 단계의 조건부 탐색 | 구조적 변화 (팬데믹 등 과거에 없던 사건) |
| 비대칭/다봉 불확실성 표현 | |

핵심 한계: Diffusion은 **과거 패턴이 반복된다**는 가정 하에서만 유효하다. 과거에 없던 극단적 사건은 예측할 수 없다.

```python
# Reverse Diffusion: 노이즈 → 데이터 복원
def reverse_diffusion(self, condition, n_samples=100):
    x_t = torch.randn(n_samples, 1)  # 순수 노이즈로 시작
    condition = condition.unsqueeze(0).repeat(n_samples, 1)

    for t in reversed(range(self.T)):
        noise_pred = self.predict_noise(x_t, t_tensor, condition)
        x_t = x_t - beta_t * noise_pred  # 노이즈 제거
        if t > 0:
            x_t = x_t + sqrt(beta_t) * randn  # 확률적 변동 추가
    return x_t
```

- n_samples=100은 100개의 서로 다른 "가능한 미래"를 생성한다는 뜻이다
- 매 역방향 단계에서 랜덤 노이즈를 추가하는 이유: 확률적 변동으로 다양한 경로를 탐색하여 분포를 형성
- 상대적 불확실성이 40% 이상이면 리스크 관리를 강화해야 한다
- condition은 과거 12개월 매출 패턴으로, "이런 패턴 이후 과거에 어떤 일이 일어났는가"를 조건으로 사용
