데이터 엔지니어링(상류)은 원시 데이터를 수집하고 변환(ETL/ELT)하여 모델링에 적합한 형태로 준비한다. 데이터 품질 관리를 통해 결측치, 이상치, 불일치를 처리하며, 이 단계의 품질이 후속 모델 성능에 결정적 영향을 미친다.

AutoML(중류)은 전처리된 데이터를 받아 모델링 단계를 자동화한다. 여러 알고리즘 중 최적 모델을 선택하고, 하이퍼파라미터를 자동으로 튜닝하며, 필요시 앙상블을 구성하여 예측 성능을 최대화한다. 이 단계는 데이터 과학자의 반복적 실험 작업을 대체한다.

MLOps(하류)는 학습된 모델을 프로덕션 환경에 배포하고 운영한다. 실시간 모니터링을 통해 모델 성능 저하(drift)를 감지하고, 필요시 자동 재학습을 수행하여 모델을 최신 상태로 유지한다.
💡 AutoML은 "모델링 자동화"에 특화되어 있으며, 상류의 데이터 품질과 하류의 운영 관리는 여전히 인간의 전문성이 필요하다. AutoML을 도입해도 데이터 엔지니어와 MLOps 엔지니어의 역할은 계속 중요하다.
